# 1. 게임 핵심 컨셉

## 한 줄 요약
> **같은 두 사람이 매번 다른 사건으로 다시 찾아오는 AI 심문 미스터리 게임**

## 장르 & 타겟
- **장르**: LLM Interrogation Mystery / Narrative Simulation
- **타겟**: 20~30대 남성, 스토리 기반 게임 팬
- **세션 길이**: 10~20분 (1사건 기준)
- **플랫폼**: 웹 브라우저 (모바일 우선)

## 핵심 판타지
플레이어는 **재판관**이다. 두 AI 당사자(원고/피고)가 사건을 들고 찾아오면, 심문하고 증거를 분석하고 진실을 밝혀 판결을 내린다. 단, 양쪽 모두 거짓말을 하고, 감정이 있고, 각자의 사정이 있다. 정답은 하나가 아니다.

## 3-Layer 데이터 아키텍처
```
Layer 1 (고정) : DuoSeed — 캐릭터 쌍 (성격, 말투, 관계사)
Layer 2 (변동) : ContextSeed — 배경 압력 (명절, 계약 만료 등)
Layer 3 (생성) : CaseSeed — 이번 사건 (쟁점, 증거, 반전)
```
→ 같은 커플이 매번 다른 사건으로 재등장하는 "반복 캐릭터 + 다른 사건" 모델

## 진실 3-Layer 구조
```
실제 진실 (Actual Truth) — 사건의 객관적 사실
  ↓
각자의 이해 (Perceived Truth) — 같은 사건을 다르게 기억/해석
  ↓
법정 진술 (Testified Truth) — 전략적으로 재구성한 말
```
→ 플레이어는 "말" 뒤에 숨겨진 "이해"를 뚫고 "실제 진실"에 도달해야 한다

## 7가지 USP (Unique Selling Points)

### USP 1: 같은 사람, 다른 사건
한지석 & 오세린은 spouse-01에서 비밀 송금 건으로, spouse-02에서 육아 갈등으로, spouse-03에서 재산 분할로 찾아온다. 캐릭터를 알수록 거짓말 패턴을 읽기 쉬워진다.

### USP 2: 진실 3-Layer
단순히 "누가 범인인가?"가 아니다. 양쪽 모두 자기가 피해자라 믿고, 그 믿음 위에 전략적 거짓말을 쌓는다.

### USP 3: AI 논쟁 중재
두 AI가 실시간으로 서로 반박하고, 끼어들고, 감정적으로 반응한다. 플레이어는 이 논쟁을 통제하는 재판관이다.

### USP 4: 압박 vs 신뢰, 두 갈래 길
- **압박 루트**: 증거를 들이밀어 거짓말을 부순다 → 빠르지만 셧다운 위험
- **신뢰 루트**: 공감으로 마음을 열게 한다 → 느리지만 자발적 고백 가능

### USP 5: 판결은 지혜
"누가 잘못했나?"보다 "어떻게 해결할 것인가?"가 중요하다. 책임 비율을 정하고, 조정안을 제시하고, 모호한 쟁점은 보류할 줄 아는 것이 높은 점수다.

### USP 6: 편향 역전
시스템이 플레이어의 패턴을 읽는다. 항상 한쪽만 몰아세우면 시스템이 반전을 넣고, 항상 같은 질문만 하면 효율 패널티를 준다.

### USP 7: Discovery 시스템
Phase 3~5에서 작동하는 4가지 능동 발견 메커닉:
1. **진실 공방** — A vs B 충돌 진술을 플레이어가 판단
2. **증거 감별** — 증거의 신뢰도를 3단계로 평가
3. **숨겨진 쟁점 발현** — 심문 중 새로운 쟁점이 드러남
4. **감정 전략** — 감정 상태를 이용한 실수 유도

## 7가지 설계 원칙

1. **신뢰할 수 없는 화자** — 양쪽 모두 거짓말한다. 플레이어가 유일한 객관자다.
2. **사건은 반드시 구조적** — 랜덤이 아니라 설계된 쟁점·증거·반전으로 구성된다.
3. **판결은 3단계 + 조정** — 사실 인정 → 책임 비율 → 해결 방안 → (조정)
4. **캐릭터 우선, 사건 차순** — 사건이 아니라 사람이 재미의 중심이다.
5. **증거는 사실이자 적법성** — 결정적 증거라도 위법이면 쓸 수 없다.
6. **압박과 신뢰, 두 루트** — 어느 쪽이든 정답이 될 수 있다.
7. **시스템도 플레이어를 읽는다** — 편향, 반복, 효율을 추적하여 피드백한다.
