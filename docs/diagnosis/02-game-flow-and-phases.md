# 2. 게임 플로우 & 페이즈 상세

> **V3 업데이트 (2026-04-02)**: Phase 3/4/5 통합, 사건카드 도입, 미터 시스템
> **V2 스크립트 전환 (2026-04-02)**: Phase 3 LLM→BeatScript V2 전환 완료. spouse-01/family-01 파일럿 활성. 상세는 `11-v2-script-transition-spec.md` 참조

## 전체 플로우
```
타이틀 → 챕터 선택 → 스테이지 선택
  ↓
Phase 0: 사건 개요 (읽기 전용)
  ↓
Phase 1: 초기 진술 (스크립트 자동 재생)
  ↓
Phase 2: 반박 (스크립트 자동 재생)
  ↓
Phase 3: 통합 심문 — 12~16턴 (V3 게임 루프)
  ↓
Phase 6: 조정 (AI 종합)
  ↓
Phase 7: 최종 판결 (플레이어 판단 입력)
  ↓
결과 화면 (점수, 진실 공개, 칭호, 후일담, 공유)
```

Phase 4(증거 심리)와 Phase 5(재심문)는 Phase 3에 **통합**되었습니다. PHASE_ORDER에서 제외되어 있으며, 코드에는 enum만 남아 하위 호환을 유지합니다.

## 스크립트 vs AI 구분

| 구간 | 현재 방식 | 전환 방향 |
|------|----------|----------|

| 결과/후일담 | AI | AI 유지 (판결 결과 기반 서사) |

## 각 페이즈 상세

### Phase 0: 사건 개요
- 사건번호, 사건명, 당사자 소개(이름/나이/직업/성격)
- **핵심 쟁점 최대 2개만 노출** (high weight + neither_knows/shared_misconception 제외)
- 나머지 쟁점은 "심문 중 추가 쟁점이 드러날 수 있습니다"로 표시
- AI 프롬프트/에이전트 백그라운드 로딩 시작

### Phase 1: 초기 진술 (스크립트 자동 재생)
- A와 B가 각자 입장을 밝히는 오프닝 (JSON 사전 작성 10개 내외)
- 스킵 가능
- 여기서 플레이어는 관찰만 — 쟁점 파악의 시작점

### Phase 2: 반박 (스크립트 자동 재생)
- A의 진술에 대한 B의 반박, B의 진술에 대한 A의 반박
- "그건 사실이 아닙니다" 같은 직접 충돌이 드러남
- 여전히 관찰 모드

### Phase 3: 통합 심문 (V3 핵심 게임플레이, 12~16턴)

Phase 3 진입 시 **RuntimeStartBridge** 적용: Phase 1~2에서 인정된 사실이 lieState에 반영됨 (S0→S1 등).

**플레이어 행동 체계:**
```
심문 대상 선택 (A / B)
  ├── 심문 (3종 질문 카드) — 1턴부터
  │   ├── 📌 사실추궁 (fact_pursuit) → 모순토큰 ↑
  │   ├── 🎯 동기탐색 (motive_search) → 누설미터 ↑
  │   └── 🤝 공감접근 (empathy_approach) → 신뢰창구 ↑
  ├── 📋 사건카드 (턴 4 이후 해금) — 증거 묶음 카드
  │   └── 대상별 맞춤 challenge 질문 3개
  ├── 📄 증거 제시 — 개별 증거
  │   └── DossierCard 맞춤 질문 or 유형별 fallback
  └── ⚡ 스킬
      ├── 이의 제기 (⚡1) — 전체 쟁점 압박
      ├── 분리 심문 (🔍1) — 3턴간 1:1
      └── 즉답 요구 (⚡1) — 특정 쟁점 즉시 붕괴
```

**V3 미터 시스템 (질문 효과):**
| 미터 | 올리는 질문 | 임계 효과 |
|------|------------|----------|
| 모순토큰 (0~5) | 사실추궁 | 3개: 모순 이벤트, 4개: flat_denial 봉쇄 |
| 누설미터 (0~100%) | 동기탐색 | 50%: 숨긴 동기 누설, 80%: 감정폭발 |
| 신뢰창구 (0~100%) | 공감접근 | 60%: 자백 가능성 개방 (3턴간) |

미터 간 상호작용: 공감하면 모순토큰 -1, 사실추궁만 하면 누설/신뢰 자연 감소.

**V3 게임 루프 (매 턴):**
```
1. 행동 선택 (질문 3종 / 사건카드 / 증거 / 스킬)
2. NPC 응답 (현재: LLM → 전환 예정: BeatScript ×5 랜덤)
3. 즉시 결과 표시
   - 미터 변화 (모순토큰/누설미터/신뢰창구)
   - 상태 전이 라벨 (균열/궁지/개방/시인)
   - 증거 토스트 (버팀/균열/붕괴)
4. 이벤트 발동 (조건 충족 시, 우선순위: 모순 > 감정폭발 > 새쟁점 > 끼어들기)
   - 모순 감지 → 지적/넘어가기
   - 끼어들기 → 허용/제지
   - 감정 폭발 → 압박/진정
   - 새 쟁점 출현 → 추적/보류
5. Readiness 갱신 → 판결 가능 여부 체크
```

**판결 가능 조건 (Readiness Engine):**
```
일반 판결: 턴 8+ AND readinessScore 5+ AND 진행 쟁점 2+ AND 돌파 1+
조기 종료: 턴 6+ AND (고백 1+ OR resolved 2+ OR resolved 1 + collapse 1 + 조사 2)
강제 판결: 턴 16 초과 → forced_incomplete (감점 적용)
```

### Phase 6: 조정
- AI가 양측 진술 + 판결 기록을 종합하여 조정안 제시
- 플레이어는 즉시 판결 / 조건부 조정 / 추가 조사 / 사실 우선 해결 중 선택

### Phase 7: 최종 판결
1. **사실 인정** — 각 쟁점이 사실인지 거짓인지 판단
2. **책임 비율** — 각 쟁점에서 A:B 비율 설정 (0~100)
3. **증거 적법성** — 각 증거의 사용 가부 판단
4. **해결 방안** — 조정안 중 선택

### 결과
- 3축 점수 (통찰력/권위/지혜)
- 진실 공개 + 놓친 진실
- 칭호 + 후일담 (AI 생성)
