# 2. 게임 플로우 & 페이즈 상세

## 전체 플로우
```
타이틀 → 챕터 선택 → 스테이지 선택
  ↓
Phase 0: 사건 개요 (읽기 전용)
  ↓
Phase 1: 초기 진술 (사전 생성 스크립트, 자동 재생)
  ↓
Phase 2: 반박 (사전 생성 스크립트, 자동 재생)
  ↓
Phase 3: 심문 — 교차 검증 (LLM 실시간 대화)
  ↓
Phase 4: 증거 심리 (LLM 실시간 대화)
  ↓
Phase 5: 재심문 (LLM 실시간 대화)
  ↓
Phase 6: 조정 (LLM 조정안 제시)
  ↓
Phase 7: 최종 판결 (플레이어 판단 입력)
  ↓
결과 화면 (점수, 진실 공개, 칭호, 후일담, 공유)
```

## 사전 생성 vs 실시간 LLM 구간

| 구간 | 방식 | 설명 |
|------|------|------|
| Phase 0 | 정적 JSON | 사건 개요 텍스트 표시 |
| Phase 1~2 | 사전 생성 스크립트 | GPT Pro로 84개 사건 × 2 페이즈 = 168개 스크립트 미리 생성. 세션 중 LLM 호출 없음 |
| Phase 3~5 | 실시간 LLM | GPT-4o-mini가 플레이어 질문에 실시간 응답. 핵심 게임플레이 구간 |
| Phase 6 | 실시간 LLM | 조정안 생성 |
| Phase 7 | 룰 엔진 | 판결 입력 → 채점 (LLM 불필요) |
| 결과 | 룰 엔진 + 정적 | 점수 계산 + 후일담/칭호 표시 |

## 각 페이즈 상세

### Phase 0: 사건 개요
- 사건번호, 사건명, 당사자 소개, 상황 배경 표시
- 플레이어에게 읽을 시간을 준 뒤 "심리 개시" 버튼

### Phase 1: 초기 진술 (자동 재생)
- A와 B가 각자 입장을 밝히는 오프닝 독백
- 대사가 순서대로 표시 (스킵 가능)
- 여기서 플레이어는 관찰만 함 — 쟁점 파악의 시작점

### Phase 2: 반박 (자동 재생)
- A의 진술에 대한 B의 반박, B의 진술에 대한 A의 반박
- "그건 사실이 아닙니다" 같은 직접 충돌이 드러남
- 여전히 관찰 모드 — 어디를 파고들지 판단하는 시간

### Phase 3~5: 심문 (핵심 게임플레이)
플레이어가 직접 행동을 선택하는 구간. 세 페이즈가 연속이며, 페이즈 전환 조건이 있다.

**플레이어 행동 옵션:**
- 🔵 **사실 추궁** (fact_pursuit) — "그 돈은 어디로 갔습니까?"
- 🟣 **동기 탐색** (motive_search) — "왜 그런 선택을 하셨습니까?"
- 🩷 **공감 접근** (empathy_approach) — "많이 힘드셨겠습니다" (+12 신뢰)
- 📎 **증거 제시** (evidence_present) — 증거를 보여주며 압박
- ❓ **자유 질문** (free_question) — AI에게 자유롭게 질문 (조사 토큰 1 소모)
- 🗣️ **증인 소환** (witness_call) — 제3자 증인의 증언 듣기 (조사 토큰 1 소모)

**스킬 시스템:**
- 🔍 회피 감지 (Phase 4, ⚡1) — 답변의 회피도를 수치로 표시
- 🔒 비공개 보호 (Phase 5, 무료) — "이 증거는 판결에 안 씁니다"
- ❗ 이의제기 (⚡1) — 모든 쟁점 강제 전이
- 🚪 분리 심문 (🔍1) — 3턴간 끼어들기 차단
- ⚡ 즉답요구 (⚡1) — 특정 쟁점 S5 강제

**끼어들기(Interjection) 시스템:**
- 한 쪽을 심문하면 다른 쪽이 25% 확률로 끼어듦
- 플레이어는 **허용** (-3 권위 패널티) 또는 **제지** (패널티 없음) 선택
- 법정 통제력의 핵심 요소

### Phase 전환 조건
```
Phase 3 → 4: MIN_TURNS(10) OR (거짓말 전이 2회 이상) OR (10턴 경과)
Phase 4 → 5: MIN_TURNS(10) OR (거짓말 전이 2회 이상) OR (10턴 경과)
Phase 5 → 6: MAX_TURNS(20) 도달 시 자동 전환
판결 가능 조건: 총 30턴 이상 사용
```

### Phase 6: 조정
- AI가 양측의 진술을 종합하여 조정안 제시
- 플레이어는 조정을 수락하거나 직접 판결로 넘어감

### Phase 7: 최종 판결
플레이어가 입력하는 것들:
1. **사실 인정** — 각 쟁점이 사실인지 거짓인지 판단 (또는 보류)
2. **책임 비율** — 각 쟁점에서 A:B 비율 설정 (0~100)
3. **증거 적법성** — 각 증거의 사용 가부 판단
4. **해결 방안** — 조정안 중 선택
