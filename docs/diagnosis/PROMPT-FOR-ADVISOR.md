# Solomon 게임 설계 논의 — V3 스크립트 전환 + 게임성 강화

## 당신에게 부탁하는 역할

당신은 **게임 디자인 컨설턴트**입니다. Solomon이라는 AI 심문 미스터리 게임의 **V3 리뉴얼을 완료**했고, 이제 두 가지 큰 전환을 앞두고 있습니다. 첨부한 진단 자료(01~10번)와 논의 문서를 읽고, **구체적이고 실행 가능한 설계 제안**을 해주세요.

## 현재 상황 한 줄 요약

> V3 엔진(5대+3대)과 게임 루프는 완성. 이제 "LLM 비용을 없애면서 게임이 더 재미있어지는" 방향을 찾아야 한다.

## 논의할 핵심 주제 2가지

### 주제 1: LLM → 스크립트 전환
- 심문 NPC 응답을 사전 작성 BeatScript(×5 변형)로 교체
- AI는 조정(Phase 6)과 후일담(Result)에서만 사용
- 자유 심문은 S3+ 상태에서만 AI 허용 (검토 중)
- **질문**: 변형 수, 질문 유형별 분기, 조건부 대사 등

### 주제 2: 게임 패턴화 방지
- 미터가 보이면 정답이 뻔해지는 문제
- 6가지 대응안: NPC 확률 반응, 질문 피로도, 상대 간섭, 감정 변동, 증거 타이밍, 자유 심문
- **질문**: 우선순위, 빠진 문제점, 레퍼런스 메커닉
