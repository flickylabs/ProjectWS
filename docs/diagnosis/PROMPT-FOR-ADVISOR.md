# Solomon 게임 진단 요청 — 재미 강화 조언

## 당신에게 부탁하는 역할

당신은 **게임 디자인 컨설턴트**입니다. 저는 "Solomon"이라는 AI 심문 미스터리 게임을 만들고 있고, 핵심적인 재미 문제에 봉착해 있습니다. 첨부한 진단 자료(00~10번 파일)를 읽고, **구체적이고 실행 가능한 기획 제안**을 해주세요.

---

## 게임 한 줄 요약

> 두 AI 당사자가 분쟁을 들고 찾아오면, 플레이어(재판관)가 심문하고 증거를 분석하여 진실을 밝히고 판결을 내리는 게임.

---

## 현재 상황: 재미가 없다

시스템은 거의 다 만들어졌습니다. 84개 사건, 420개 쟁점, 500개 증거, 거짓말 상태머신, Discovery 메커닉, 채점 시스템 — 구조는 갖췄습니다. **그런데 플레이하면 재미가 없습니다.**

### 핵심 문제 4가지

#### 문제 1: AI가 거짓말을 제대로 못 한다
- 6단계 거짓말 상태머신(S0~S5)을 설계했는데, **실제 LLM 출력은 상태 구분이 무의미할 정도로 비슷하다.**
- S0(완전 부정)인데 죄책감을 내비치고, S2(부분 인정)인데 여전히 완전 부정처럼 말한다.
- "거짓말을 간파하는 쾌감"이 게임의 핵심 재미여야 하는데, **AI의 거짓말이 거짓말처럼 느껴지지 않는다.**
- 프롬프트에 "S0에서는 절대 인정하지 마세요"라고 써도, GPT-4o-mini가 한국어로 미묘한 뉘앙스를 컨트롤하지 못한다.

#### 문제 2: AI가 캐릭터 연기를 못 한다
- 각 캐릭터에 archetype(avoidant, confrontational, victim_cosplay, cold_logic), speechStyle, verbalTell을 상세히 정의해뒀다.
- **그런데 누구랑 대화해도 비슷하다.** "회피형"이 돌연 공격적이 되고, "대립형"이 갑자기 순해진다.
- verbalTell(거짓말 시 숫자가 정밀해지는 버릇, 되묻는 버릇 등)이 거의 발현되지 않는다.
- 사전 생성된 Phase 1~2 스크립트는 캐릭터성이 살아있는데, Phase 3~5 실시간 LLM 구간에서 캐릭터성이 사라진다.

#### 문제 3: 증거가 의미 없다
- 증거마다 6가지 조사 결과(원본 확인, 메타데이터, 맥락 복원, 출처 검증, 편집 탐지, 취득 경위)를 정교하게 설계해뒀다.
- **그런데 증거를 제시해도 AI가 증거의 의미를 이해하는 것 같지 않다.**
- "이 증거가 당신의 진술과 모순됩니다"라는 논리적 압박이 형성되지 않는다.
- 플레이어가 증거를 파고드는 재미, 증거 조합으로 반전을 만드는 재미가 없다.

#### 문제 4: 어디서부터 손대야 할지 모르겠다
- 문제가 LLM의 한계인지, 프롬프트 설계인지, 게임 메커닉인지, 데이터 구조인지 구분이 안 된다.
- 시스템을 더 복잡하게 만들면 해결될 문제인지, 아니면 근본적으로 접근이 잘못된 건지 판단이 필요하다.

## 당신에게 요청하는 것

### 1. 진단 (Diagnosis)
- 위 4가지 문제의 **근본 원인**이 뭐라고 보는지
- LLM 한계인지, 프롬프트 설계 문제인지, 게임 메커닉 문제인지, 데이터 구조 문제인지
- "이 부분은 GPT-4o-mini로는 원래 안 된다"는 한계와 "이렇게 바꾸면 된다"는 영역을 구분해주세요

### 2. 재미 강화 전략 (Fun Strategy)
- **거짓말 간파의 쾌감**을 살리려면 어떤 접근이 필요한지
- **캐릭터 연기**를 강화하려면 프롬프트를 어떻게 바꿔야 하는지, 아니면 프롬프트가 아닌 다른 방법이 있는지
- **증거 활용의 재미**를 만들려면 현재 구조에서 뭘 바꿔야 하는지
- 현재 시스템에서 **제거하거나 단순화**해야 할 것이 있는지 (과도한 복잡성이 오히려 재미를 해치는 부분)

### 3. 구체적 제안 (Actionable Proposals)
- "이런 방향이 좋겠다" 수준이 아니라, **"이 데이터 구조를 이렇게 바꾸고, 프롬프트를 이렇게 수정하고, 이 메커닉을 이렇게 변경하라"** 수준의 제안
- 우선순위를 매겨주세요: 가장 적은 노력으로 가장 큰 재미 개선을 가져올 것부터
- 필요하다면 현재 시스템과 다른 접근도 제안해주세요 (예: LLM 의존도를 줄이고 사전 생성 비중을 높이기 등)

### 4. 참고 사례 (Reference)
- 비슷한 장르(심문, 추리, 재판)의 게임 중 "이건 잘했다"는 사례가 있다면 어떤 점을 배울 수 있는지
- LLM 기반 게임에서 "캐릭터 일관성"이나 "거짓말 시뮬레이션"을 잘 구현한 사례가 있는지

## 제약 조건

- **LLM**: 현재 GPT-4o-mini 사용 중. 비용 때문에 GPT-4o로 올리기 어렵지만, 모델 변경이 근본적 해결책이라면 그것도 고려 가능.
- **플랫폼**: 웹 브라우저 (모바일 우선). 무거운 로컬 모델은 불가.
- **세션 길이**: 10~20분. 너무 길어지면 안 됨.
- **언어**: 한국어 전용. 한국어 경어법/반말 컨트롤이 LLM의 큰 과제.
- **콘텐츠**: 84개 사건은 이미 만들어져 있음. 데이터 구조 변경은 가능하지만 전면 재생성은 부담.
- **개발 리소스**: 1인 개발. 대규모 리팩토링보다 핵심 포인트 수정 선호.

## 마지막으로

이 게임의 컨셉 자체 — "AI 당사자를 심문해서 거짓말을 간파하고 진실을 밝히는 재판관 판타지" — 는 매력적이라고 믿습니다. 문제는 실행입니다. 잘 설계된 시스템이 실제 플레이에서 재미로 전환되지 못하고 있습니다.

**"재미있게 만들 수 있는 길이 있는지, 있다면 어디를 어떻게 바꿔야 하는지"** — 솔직하고 구체적인 조언을 부탁드립니다. 컨셉 자체가 문제라면 그것도 말해주세요.
