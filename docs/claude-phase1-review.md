# Phase 1 대사 검수/보정 가이드 (Claude Code용)

이 프로젝트(d:\ProjectWS)의 "솔로몬" 게임에서 GPT가 Phase 1(초기 진술) 대사를 생성했습니다.
생성된 파일을 검수하고 보정하는 작업을 해주세요.

---

## 배경

- 게임: AI 갈등 해결 시뮬레이션. 두 캐릭터(A, B)가 다투고, 플레이어가 심문·증거로 진실을 파헤쳐 판결
- Phase 1 = 초기 진술. A와 B가 서로 끼어들며 감정이 격화되는 8~10개 대사
- GPT가 사건별로 JSON 배열을 생성했고, 이걸 검수/보정해야 함

---

## 파일 위치

- **GPT 출력물 (검수 대상):** `src/data/dialogues/phase1/*.json`
- **사건 원본 데이터:** `src/data/cases/generated/*.json`
- **사건 요약 정보:** `src/data/cases/phase1-input/*.json`
- **GPT 프롬프트 (규칙 확인용):** `docs/gpt-prompts/prompt-*.md`

## 검수 항목

### 1. 구조 체크
- [ ] JSON 파싱 에러 없는지
- [ ] 총 대사 수 8~10개인지
- [ ] speaker가 소문자 `"a"`, `"b"`, `"system"` 중 하나인지 (대문자 "A", 이름 사용 시 수정)
- [ ] behaviorHint가 text 안 괄호 내용과 일치하는지
- [ ] relatedDisputes의 ID가 해당 사건의 disputes에 존재하는지

### 2. 호칭 규칙 (관계 유형별)

**캐릭터 대사에서 상대 이름을 직접 사용하면 안 됩니다.**

| 관계 | 상대에게 직접 | 재판관에게 상대 언급 |
|------|-------------|-------------------|
| spouse (부부) | "자기" (고정) | "제 남편이", "제 아내가" |
| family (가족) | 관계 호칭 (형/누나/동생) | "제 누나가", "제 동생이" |
| friend (친구) | "야", "너" | "제 친구가", "이 사람이" |
| neighbor (이웃) | "~씨", "저분" | "옆집 분이", "저 사람이" |
| workplace (직장) | 직급 호칭 | "제 팀장이", "제 부하직원이" |
| partnership (동업) | "~씨", 직함 | "제 동업자가", "이 사람이" |
| tenant (임대) | "집주인", "세입자" | "집주인이", "세입자가" |

**확인 방법:** `phase1-input/*.json`에서 `relationshipType`을 확인하고, 대사 내 호칭이 맞는지 체크.

### 3. 말투 규칙
- [ ] 반말 관계(spouse, family, friend): 상대에게 반말, 재판관에게 존댓말
- [ ] 존댓말 관계(neighbor, workplace, partnership, tenant): 기본 존댓말, 격해지면 반말 가능
- [ ] 한 대사 안에서 반말/존댓말 혼합 없는지
- [ ] "~냐?" 사용 없는지 (→ "~야?"로 수정)

### 4. 이름 플레이스홀더
- [ ] system 대사에서 이름이 직접 사용된 경우 → `{A}`, `{B}` 플레이스홀더로 변환
- [ ] behaviorHint에서 이름이 직접 사용된 경우 → `{A}`, `{B}` 플레이스홀더로 변환
- [ ] 조사가 필요한 경우: `{A:이}`, `{B:은}`, `{A:을}`, `{B:과}`, `{A:로}` 사용
  - 이/가, 은/는, 을/를, 과/와, 으로/로 → 런타임에 받침 기반 자동 치환

**예시:**
```
수정 전: "한지석이 참지 못하고 끼어든다"
수정 후: "{A:이} 참지 못하고 끼어든다"

수정 전: "오세린은 감정을 누르며 정리한다"
수정 후: "{B:은} 감정을 누르며 정리한다"
```

### 5. 내용 확인
- [ ] 대화 흐름: system→a진술→b반박→a↔b격화→(system개입)→마무리
- [ ] 모든 쟁점이 최소 1번은 언급되는지
- [ ] archetype(회피형/대면형 등)에 맞는 행동인지
- [ ] speechStyle/verbalTells의 특징이 반영되었는지

1. 해당 사건의 `phase1-input/*.json`을 읽어서 관계 유형·캐릭터 정보 확인

3. 문제 있는 항목은 직접 수정
4. 수정 후 JSON 유효성 확인

## 이름 플레이스홀더 참고

`src/utils/korean.ts`의 `resolveNameTemplate()` 함수가 런타임에 치환:

```
{A}    → nameA                    (예: "한지석")
{B}    → nameB                    (예: "오세린")
{A:이} → nameA + 이/가            (예: "한지석이")
{A:은} → nameA + 은/는            (예: "한지석은")
{A:을} → nameA + 을/를            (예: "한지석을")
{A:과} → nameA + 과/와            (예: "한지석과")
{A:로} → nameA + 으로/로          (예: "한지석으로")
{B:이}, {B:은}, {B:을} 등 동일
```
