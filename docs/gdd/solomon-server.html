2.2 테이블 상세

players — 플레이어

컬럼타입제약설명
idTEXTPKUUID (클라이언트 생성)
nameTEXTNOT NULL플레이어 닉네임 (기본: '재판관')
regionTEXTDEFAULT 'KR'지역 코드
created_atTEXTNOT NULLISO 8601 생성 시각
last_activeTEXTNOT NULL마지막 활동 시각

game_history — 게임 기록

컬럼타입제약설명
idINTEGERPK AUTO기록 ID
player_idTEXTFK → players플레이어 참조
case_idTEXTNOT NULL사건 ID (e.g. case-spouse-01)
scoreINTEGERDEFAULT 0총점
insightINTEGERDEFAULT 0통찰력 점수
authorityINTEGERDEFAULT 0권위 점수
wisdomINTEGERDEFAULT 0지혜 점수
relationship_typeTEXT—관계 유형
name_a / name_bTEXT—당사자 이름
season_idTEXT—시즌 ID
titlesTEXTJSON획득 칭호 배열
played_atTEXTNOT NULL플레이 시각

notices — 공지사항

컬럼타입설명
idINTEGER PK공지 ID
titleTEXT NOT NULL공지 제목
contentTEXT NOT NULL공지 본문 (줄바꿈 포함)
typeTEXTinfo / event / update / maintenance
priorityINTEGER0=일반, 1=중요, 2=긴급
is_pinnedINTEGER1=상단 고정
is_activeINTEGER1=활성, 0=비활성
start_date / end_dateTEXT노출 기간 (null=무기한)

mail — 우편

컬럼타입설명
idINTEGER PK우편 ID
titleTEXT NOT NULL우편 제목
contentTEXT NOT NULL우편 본문
senderTEXT발신자 (기본: '솔로몬 운영팀')
reward_typeTEXTnull / coin / item / title
reward_amountINTEGER보상 수량
reward_dataTEXT JSON아이템 상세 (JSON)
target_typeTEXTall / specific
target_playersTEXT JSON대상 플레이어 ID 배열
expires_atTEXT만료일 (null=무기한)

ai_prompts — AI 프롬프트

컬럼타입설명
idINTEGER PK프롬프트 ID
keyTEXT UNIQUE고유 키 (e.g. dialogue_system)
nameTEXT표시 이름
descriptionTEXT설명
categoryTEXTdialogue / analysis / generation / system
contentTEXT프롬프트 본문 ({변수} 포함)
variablesTEXT JSON사용 가능 변수 목록
is_activeINTEGER1=활성
versionINTEGER현재 버전 (자동 증가)

프롬프트 버전 관리: 프롬프트 내용 변경 시 이전 버전이 ai_prompt_history에 자동 저장됩니다. WebAdmin에서 원클릭 롤백이 가능합니다.

4. AI 모듈화 시스템

🤖
핵심 설계 원칙: AI 프롬프트를 코드에서 분리하여 DB로 관리합니다. WebAdmin에서 프롬프트를 수정하면 60초 이내에 게임에 실시간 반영됩니다. 코드 배포 없이 AI 행동을 튜닝할 수 있습니다.

4.1 프롬프트 카테고리

카테고리설명프롬프트 예시
dialogue심문 대화 생성dialogue_system, free_question, emotion_speech_guide
analysis증언/판결 분석testimony_analysis, verdict_evaluation
generation콘텐츠 자동 생성phase1_script
system시스템 레벨 프롬프트(확장 예정)

4.2 변수 치환 시스템

프롬프트 내 {변수명} 형식의 플레이스홀더는 런타임에 실제 값으로 치환됩니다.

// 예시: dialogue_system 프롬프트
const prompt = getPrompt('dialogue_system', {
  party: 'A',
  name: '민준',
  archetype: 'avoidant',
  speechStyle: '말끝을 흐리며 회피하는 스타일',
  emotionalState: 'shaken',
  lieState: 'S2',
  otherName: '서연',
  relationship: '제 아내',
});

4.3 캐시 & 폴링 메커니즘

PromptManager 동작 흐름
1. 앱 시작 시 → loadPrompts(force=true)
   ├─ 서버 연결 성공 → DB 프롬프트 로드 → promptCache 갱신
   └─ 서버 연결 실패 → DEFAULT_PROMPTS 사용 (오프라인 모드)

2. startPromptPolling(60000) → 60초마다 자동 갱신
   ├─ 서버 응답 OK → 캐시 갱신 (WebAdmin 변경사항 반영)
   └─ 서버 오류 → 기존 캐시 유지 (게임 중단 없음)

3. getPrompt(key, vars) 호출 시
   ├─ 캐시에서 프롬프트 조회
   ├─ {변수} 치환
   └─ 최종 프롬프트 반환

충전 규칙: 보유량이 FREE_CAP(10) 이상이면 자동 충전 정지. 하나라도 소비하여 10 미만이 되면 충전 타이머 재가동. 유료 구입은 PAID_CAP(99)까지 가능.
