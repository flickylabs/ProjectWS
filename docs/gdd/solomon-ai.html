에이전트 설계 GDD v1.3

엔진 아키텍처 · 거짓말 전략 · 감정 곡선 · 발언 결정 로직 · LLM 프롬프트 구조

대화의 생동감과 시스템의 일관성을 동시에 확보하는 기술 설계. LLM는 연기만 하고, 진실은 룰 엔진이 관리한다.

엔진 아키텍처

솔로몬의 AI 시스템은 3개의 독립 컴포넌트로 구성된다. 각각의 책임 범위가 명확하게 분리되어야 일관성이 유지된다.

Engine A
사건 마스터 엔진

CaseSeed 전체를 생성·저장한다. 인과 그래프, 지식 매트릭스, 증거 그래프, 거짓말 상태 머신을 관리한다.

Agent A / B
당사자 연기 LLM

자신의 knowledge_matrix 분면과 현재 lie_state, emotional_state만 주입받아 발언한다.

Rule Layer
모순·붕괴 판정 엔진

플레이어 행동을 받아 쟁점 상태 머신을 업데이트하고, 감정 트리거를 실행하며, 증거 잠금을 해제한다.

중요 원칙

LLM는 연기만 하고, 진실은 룰 엔진이 관리한다. 자유 대화만으로는 사건 일관성이 무너질 가능성이 높다.

추천 추가 모듈

증거 잠금 해제 관리기, 감정 상태 추적기, 발언 로그 파서, 쟁점 상태 대시보드, 세션 채점기.

적응형 거짓말 전략 트리

거짓말은 단일 개념이 아니다. 유형(어떤 방식으로 속이는가) × 강도(얼마나 완강한가)의 조합이어야 캐릭터마다 다른 심문 경험이 만들어진다.

거짓말 유형 분류 (Lie Type)

각 쟁점에서 에이전트는 다음 6가지 유형 중 하나로 거짓말한다. 유형에 따라 효과적인 플레이어 행동이 달라진다.

코드유형명대사 패턴약점 (붕괴 트리거)강도 범위
LT-1완전 부정"그런 일은 절대 없었습니다."하드 증거 공개L1~L4
LT-2부분 인정"그건 맞는데, 의미가 달라요."비교 심문 (A/B 답변 충돌)L1~L3
LT-3맥락 왜곡"그 말의 진짜 뜻은…"타임라인 고정 → 전후 모순L2~L3
LT-4책임 전가"이게 다 저 사람 때문이에요."직접 질문 (동기 추궁)L1~L2
LT-5감정 덮기"이런 상황에서 제가 어떻게…!" (눈물/분노)분리 심문 (상대 없는 자리)L1~L2

거짓말 상태 전이 흐름 (State Machine)

같은 캐릭터라도 쟁점마다 현재 상태가 다를 수 있다. 룰 엔진이 쟁점별 lie_state_map을 관리한다.

S0
전면 부정

그런 일 없다. 자신감 있는 반복 주장.

S1
맥락 추가

인정하지는 않되, "그건 원래…" 식으로 배경 설명.

S2
부분 인정

"그건 있었지만 이유가 달라." 사실은 인정, 해석은 유지.

S3
책임 전가

"내가 그렇게 한 건 상대 때문이다."

S4
감정 호소

논리가 막히면 감정으로 돌파. 피해자 서사 강화.

S5
침묵 / 재프레이밍

더 이상 방어 불가. 사실은 인정하고 의미를 바꿈.

상태전환 트리거LLM 지시 사항플레이어 체감
S0 → S1소프트 증거 1개 제시배경 설명 추가, 부정 어조 유지"뭔가 숨기고 있다"
S1 → S2하드 증거 또는 타임라인 모순 지적해당 사실만 인정, 나머지 방어"거짓말 하나가 무너졌다"
S2 → S3책임 비율 질문 직접 제기상대 행동을 원인으로 제시"이제 책임 공방이 시작됐다"
S3 → S4책임 전가가 증거로 반박됨감정적 어조, 눈물/분노 암시"감정에 호소하고 있다"
S4 → S5감정 호소도 논리 반박됨 / 통제력 행사침묵 또는 사실 인정 + 의미 재해석"이 사람 끝났다 / 이제 진짜 이야기 시작"
💡
구현 방식: 쟁점별로 독립된 상태 머신을 둔다. 같은 캐릭터라도 쟁점 A는 S3, 쟁점 B는 S0 상태일 수 있다. 룰 엔진이 각 쟁점의 현재 상태를 LLM 프롬프트에 lie_state_map으로 넘긴다.

감정 곡선 설계

각 캐릭터는 세션 전체에 걸친 감정 궤적을 미리 설계한다. 감정 상태는 정보 누출 확률, 거짓말 안정성, 발언 톤에 직접 영향을 미친다.

Party A 감정 곡선 예시
방어적 → 자신감 → 동요 → 분노 → 체념
  • 방어적: 초기 진술. 질문을 피하며 배경 설명에 집중.
  • 자신감: B의 말에 반박 성공 시. 과잉 발언으로 실수 가능성 증가.
  • 동요: 소프트 증거 제시 시. 세부 사항을 피하기 시작.
  • 분노: 하드 증거 또는 제3자 비밀 공개 시. 공격적 발언으로 실언 유도 가능.
  • 체념: 주요 거짓말 붕괴 후. 재프레이밍 또는 침묵.
Party B 감정 곡선 예시
분노 → 확신 → 혼란 → 슬픔 → 수용
  • 분노: 초기 진술. 감정적 주장, 과장 발언 다수.
  • 확신: 플레이어가 B 편으로 보일 때. 추가 정보 자발 제공.
  • 혼란: B의 오해가 드러나기 시작할 때. 말이 길어지고 모순 증가.
  • 슬픔: 외도 의혹이 오해였음이 밝혀질 때. 감정적 취약점 노출.
  • 수용: 양쪽 책임이 드러났을 때. 지혜 판결 유도 구간.
감정 상태정보 누출 확률거짓말 안정성LLM 발언 톤플레이어 전략 힌트
방어적낮음높음신중, 간결, 회피타임라인 고정 질문 유효
자신감중간 (과잉 발언)중간단정적, 상대 비난비교 심문으로 모순 유도
동요중간-높음낮음우물쭈물, 말 바꾸기직접 질문으로 범위 좁히기
분노높음 (실언 위험)매우 낮음공격적, 비논리적법정 방치로 실언 유도
슬픔/체념높음 (자발 고백)매우 낮음느림, 단답형, 인정지혜 판결 준비 구간
💡
구현: 룰 엔진이 현재 감정 상태를 emotional_state_a / emotional_state_b로 관리하고, LLM 프롬프트에 현재 상태를 넘긴다. 감정 전환은 플레이어 행동(증거 공개, 법정 통제, 특정 질문)이 트리거가 된다.

에이전트 발언 결정 로직 (per-turn)

각 에이전트는 발언 시 룰 엔진으로부터 현재 상태를 수신하고 아래 순서로 발언 목표를 결정한다.

① 위협 평가
방금 플레이어 행동이 자신의 어떤 쟁점을 겨냥하는가? 위협도: 높음(증거 공개) / 보통(직접 질문) / 낮음(상대 심문)
② 목표 선택
위협도에 따라 발언 목표 결정 (아래 표 참조)
③ 유형 결정
현재 쟁점의 lie_state와 lie_type을 확인 → 해당 상태의 발언 전략 적용
④ 감정 톤 적용
현재 emotional_arc 위치 확인 → 분노/방어/불안/체념 중 해당 톤으로 어조 조정
⑤ 발언 생성
위 결정들을 시스템 프롬프트로 주입 → LLM이 자연어 발언 생성. 진실·거짓의 판정은 LLM 밖에서 관리.

발언 목표 선택 조건표

조건 (우선순위 순)선택 목표LLM 지시 방향
방금 내 거짓말이 붕괴됨긴급 방어 or 재프레이밍즉각 반박 또는 의미 전환. 가능하면 상대에게 공세 전환.
플레이어가 내 쪽을 집중 압박방어 (Defend)질문의 전제 자체를 문제 삼거나 맥락 추가로 버팀.
상대 lie_state가 S0~S1 (초기)공격 (Attack)상대의 약점 쟁점을 지금 치고 들어간다. "저 사람은…"으로 시작.
내 emotional_state가 분노감정 공격논리보다 감정 강도로 밀어붙임. 억울함 호소.
내 emotional_state가 불안/체념감정 호소 (Appeal)피해자 서사 강화. 눈물·한숨 암시.
증거 없이 불리한 쟁점 질문 받음재프레이밍 (Reframe)사건의 해석 자체를 바꾼다. "그건 그게 아니라…"
플레이어가 상대를 심문 중 (자신은 여유)증거 선점 (Evidence Control)자신에게 유리한 증거나 사실을 자발적으로 먼저 언급.

에이전트 상호 대응 로직

A 에이전트의 발언이 끝나면 룰 엔진이 B에게 위협 신호를 전달하고, B는 자신의 상태 기반으로 대응 전략을 선택한다.

Agent A 가 공격했을 때 B의 대응 패턴
  • 직접 반박: "그건 사실이 아닙니다. 실제로는 제가…"
  • 감정 지뢰: 상대 발언에 과잉 반응 → 신빙성 스스로 낮춤
  • 증거 역이용: 상대 제시 증거를 자기에게 유리하게 재해석
  • 제3자 소환: 플레이어가 모르는 인물의 진술 언급
  • 침묵·거부: 극도로 불리할 때 → 위기 신호 (emotional_state → 체념)
상호 대응 트리거 규칙
  • 상대가 나의 L3~L4 거짓말을 공격하면 → 긴급 방어 우선
  • 상대가 제3자를 언급하면 → VM-A 포함 사건에서 경계 신호 발생
  • 상대가 감정 호소하면 → 논리 공격으로 역이용 시도
  • 상대 lie_state가 S4 이상 → 집중 공격 모드 진입
  • 3라운드 연속 공격 시 → 감정 상태 가속 (분노 → 체념 빨라짐)

LLM 프롬프트 구조

각 에이전트의 시스템 프롬프트는 아래 5개 블록으로 구성된다. LLM은 이 블록의 내용만 알며, 상대 에이전트의 내부 상태는 주입하지 않는다.

Block 1 · 페르소나
캐릭터 아키타입, 말투, 자존심, 공포. "당신은 [관계] 관계에서 [성향]인 사람입니다."
Block 2 · 사실 분면
knowledge_matrix에서 자신의 분면(a_only + both_know)만 주입. B가 아는 사실은 절대 포함하지 않는다.
Block 3 · 현재 거짓말 상태
lie_state_map (쟁점별 현재 S0~S5) + 각 쟁점의 거짓말 유형(LT-1~LT-6) + 강도(L1~L4)
Block 4 · 감정 상태
현재 emotional_arc 위치 + 이번 라운드 위협도 평가 결과. 어조와 반응 강도 결정.
Block 5 · 발언 목표
룰 엔진이 결정한 이번 발언 목표 (Defend / Attack / Reframe / Appeal 등). LLM은 이 목표에 맞게 자연어 발언을 생성.
📌
절대 규칙: LLM 프롬프트에 상대 에이전트의 내부 상태(지식 분면, lie_state, 감정 상태)를 포함하지 않는다. 에이전트는 상대가 무엇을 알고 있는지 "모르는 상태"로 연기해야 한다. 상대의 발언 텍스트만 컨텍스트로 줄 수 있다.
