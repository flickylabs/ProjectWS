에이전트 설계 GDD v4.1

엔진 아키텍처 · 거짓말 6단계 상태머신 · 감정 & 행동 징후 · 신뢰/보호 로직 · 발언 결정 · 증인 시스템 · 교차 반응 · 심문 이력 · AI 3계층 파이프라인 (Agent+Block+DataField) · Context Flags · Phase Transcript

대화의 생동감과 시스템의 일관성을 동시에 확보하는 기술 설계. LLM은 연기만 하고, 진실은 룰 엔진이 관리한다. v4.0에서 253명 증인 시스템, 교차 당사자 자동반응, 심문 이력 추적, callTerms LLM 주입 추가. v4.1에서 프롬프트 전면 압축, "자기"→"자기야" 강제 보정, 혼합 발화 규칙, 안티패턴 차단, 재판관 질문 이름 자동 제거.

엔진 아키텍처

솔로몬의 AI 시스템은 3개의 독립 컴포넌트 + LLM 분업 구조로 구성된다.

사전 생성
GPT Pro → 사건 데이터 + Phase 1 대사

84개 사건 JSON(DuoSeed·CaseSeed·증거·거짓말 전략)과 Phase 1 초기 진술 스크립트를 GPT로 사전 생성. Claude Code가 텍스트 정제·보정.

런타임 LLM
OpenAI GPT-4o-mini — Phase 2~5 대화

Phase 2(반박) LLM 생성, Phase 3~5 심문 시 재판관 질문+NPC 응답 동시 생성, 자유질문 처리, 증거 대질, 진술 분석. 턴마다 호출.

클라이언트 로직
룰 엔진 (판정) — Zustand + TypeScript

거짓말 상태 머신 전이, 감정 트리거, 증거 잠금 해제, 심문 중 증거 발견, 과정 점수 추적, 판결 채점. LLM 없이 구조화된 로직으로 처리.

LLM vs 룰 엔진 책임 분리 (v2.0)

클라우드 LLM이 하는 것룰 엔진이 하는 것
자연어 발언 생성 (대사, 감정 표현)발언이 사실인지 거짓인지 판정
Verbal Tell(거짓말 버릇) 자연스럽게 발현lie_state_map + lie_motive 전이 판정
캐릭터 일관성 유지 (DuoSeed 성격 연기)감정 상태(emotional_arc) 수치 업데이트
희생 진술(LT-7) 타이밍에 맞는 자연어 표현희생 진술 후 신뢰도 변화 계산
신뢰/보호 루트 질문에 대한 감정적 반응trust_toward_judge 수치 업데이트
상대 발언에 즉각 반응하는 대화증거 잠금 해제 조건 체크

거짓말 2축 전략: 유형 x 동기

거짓말은 "어떻게 속이는가"(유형)와 "왜 속이는가"(동기) 두 축의 조합이다. 같은 "전면 부정"이라도 자기 보호인지, 아이 보호인지, 체면인지에 따라 심문 반응이 완전히 달라진다.

축 1: 거짓말 유형 (Lie Type) — v2.0 확장

코드유형명대사 패턴약점 (붕괴 트리거)강도 범위
LT-1완전 부정"그런 일은 절대 없었습니다."하드 증거 공개L1~L4
LT-2부분 인정"그건 맞는데, 의미가 달라요."비교 심문 (A/B 답변 충돌)L1~L3
LT-3맥락 왜곡"그 말의 진짜 뜻은..."타임라인 고정 → 전후 모순L2~L3
LT-4책임 전가"이게 다 저 사람 때문이에요."직접 질문 (동기 추궁)L1~L2
LT-5감정 덮기"이런 상황에서 제가 어떻게...!"분리 심문 (상대 없는 자리)L1~L2
LT-6질문 회피"지금 그게 중요한 게 아니잖아요."타임라인 고정 3회 반복 → 자동 붕괴 신호L1
LT-7희생 진술 (신규)"솔직히 말씀드리면, 제가 [작은 잘못]을 했습니다."희생 진술의 전략적 의도 파악. 인정한 잘못 뒤에 숨긴 큰 잘못 추적.L2~L3
💡
LT-7 희생 진술: 중요하지 않은 잘못 하나를 먼저 인정해서 전체 신뢰도를 높이려는 전략. 현실에서 매우 자주 쓰이는 기술이다. 플레이어가 "이 사람은 솔직하네"라고 착각하면 진짜 핵심을 놓친다. 감정 미끼(드라마 비트 Beat 2)와 시너지가 크다.

축 2: 거짓말 동기 (Lie Motive) — 신규

같은 유형의 거짓말이라도 동기가 다르면 붕괴 조건과 심문 반응이 달라진다.

동기 코드동기명특징적 반응효과적 심문 루트
LM-1자기 보호논리적으로 버팀. 증거에 약함.하드 증거 제시 + 직접 질문
LM-2체면 보호작은 사실 먼저 인정, 큰 책임을 숨김단계적 질문으로 범위 좁히기. 희생 진술(LT-7) 경계.
LM-3제3자 보호특정 질문에만 완강. 나머지는 협조적.보호 대상 안전 보장 → 신뢰 루트로 무너짐
LM-4복수압박에 오히려 공격적. 상대 비난 강화.감정 방치로 실언 유도 / 동기 질문으로 직접 추궁
LM-5수치심 회피질문 자체를 피하려 함. 감정 호소 빈번.신뢰 루트 핵심 — 비공개 진술 보호 + 공감 질문
LM-6직장/평판 보존공식적이고 사무적인 어조로 방어조직 내부 증거(로그, 이메일)로 입증
LM-7관계 유지 욕구사실을 알면서도 관계를 지키려 숨김관계 유지가 오히려 해가 됨을 보여주는 증거 + 공감

거짓말 상태 전이 흐름 (v2.0)

쟁점별 독립 상태 머신. 동기에 따라 전이 조건이 달라진다.

S0
전면 부정

그런 일 없다. 자신감 있는 반복 주장.

S1
맥락 추가

"그건 원래..." 식으로 배경 설명.

S2
부분 인정

"그건 있었지만 이유가 달라."

S3
책임 전가

"내가 그렇게 한 건 상대 때문이다."

S4
감정 호소

논리가 막히면 감정으로 돌파.

S5
침묵 / 재프레이밍

더 이상 방어 불가. 의미를 바꿈.

전이기본 트리거동기별 변형
S0 → S1소프트 증거 1개LM-5(수치심): 소프트 증거 없이도 공감 질문으로 전이 가능
S1 → S2하드 증거 또는 타임라인 모순LM-2(체면): 희생 진술(LT-7) 발동 가능. S1에서 곧바로 S2로 넘어가며 작은 것을 인정.
S2 → S3책임 비율 질문 직접 제기LM-4(복수): S3을 건너뛰고 공격적 S4로 바로 점프
S3 → S4책임 전가가 증거로 반박됨LM-3(제3자 보호): 보호 대상 안전 보장 시 S3에서 곧바로 S5(인정)로 점프
S4 → S5감정 호소도 반박됨 / 통제력 행사LM-7(관계 유지): 비공개 진술 보호 시 S4에서 자발적 S5 전이

감정 곡선 & 행동 징후

감정 상태는 내부적으로 수치 관리하되, 플레이어에게는 숫자가 아닌 행동 징후로만 노출한다. "분노 78"이 아니라 "말을 끊고 반격하려 든다".

감정 상태 → 행동 징후 매핑

내부 상태행동 징후 (플레이어에게 노출)정보 누출 확률거짓말 안정성전략 힌트
방어적"답변 전 잠시 생각하고, 신중하게 단어를 고른다."낮음높음타임라인 고정 질문 유효
자신감"단정적으로 말하고, 상대를 직접 가리킨다."중간 (과잉 발언)중간비교 심문으로 모순 유도
동요"답변이 길어지고, 같은 표현을 반복하기 시작한다."중간-높음낮음직접 질문으로 범위 좁히기
분노"말을 끊고 반격하려 든다. 목소리가 높아진다."높음 (실언)매우 낮음법정 방치로 실언 유도
슬픔/체념"답변이 짧아지고, 시선을 피한다. 한숨이 늘어난다."높음 (자발 고백)매우 낮음지혜 판결 준비 구간. 신뢰 루트 최적 타이밍.
💡
구현: 룰 엔진은 emotional_state_a / emotional_state_b를 수치로 관리한다. 플레이어에게는 "회피 신호 판독" 스킬 사용 시 행동 징후 텍스트만 보여준다. 스킬 없이도 LLM 발화의 어조 변화로 플레이어가 간접적으로 감지할 수 있도록 한다.

감정 곡선 예시 (DuoSeed 연동)

Party A · 회피형 / 체면 중시
방어적 → 자신감 → 동요 → 분노 → 체념

Lie Motive: LM-2(체면) + LM-5(수치심). 초반에 희생 진술(LT-7)로 작은 잘못을 인정해 신뢰를 쌓으려 함. 핵심이 드러나면 분노로 전환.

Party B · 정면돌파형 / 의심 빠름
분노 → 확신 → 혼란 → 슬픔 → 수용

Lie Motive: LM-1(자기보호) + LM-4(복수). 초반에 과장 발언이 많고, B의 오해가 드러나면 혼란 → 슬픔으로 전환. 신뢰 루트가 효과적인 후반부.

신뢰/보호 에이전트 로직

모든 진실이 압박으로 나오는 것은 아니다. 에이전트는 플레이어(판사)에 대한 신뢰 상태를 별도로 관리하며, 이것이 특정 진실의 공개 여부를 결정한다.

에이전트별 신뢰 상태

상태 변수범위영향변경 트리거
trust_toward_judge0~100 높을수록 자발적 정보 제공 확률 상승. 비공개 진술 수용 확률 상승. 공감 질문 +10~15 / 비공개 보호 +20 / 과도한 압박 -15 / 상대 편들기 -20
fear_of_exposure0~100 높을수록 방어적. 질문 회피(LT-6) 빈도 증가. 하드 증거 공개 +20 / 관련 쟁점 질문 +10 / 비공개 보호 -15 / 보복 우려 확인 -20
retaliation_worry0~100 높을수록 제3자 보호 동기(LM-3) 강화. 특정 정보 완강히 숨김. 상대의 공격적 발언 +15 / 보복 우려 확인 -25 / 끼어들기 금지 선언 -10

신뢰 루트로만 나오는 진실

📌
핵심 규칙: CaseSeed의 일부 TruthItem에는 collapse_via_trust: true 플래그가 있다. 이 진실은 하드 증거로는 붕괴되지 않고, trust_toward_judge가 일정 수준 이상이면서 비공개 진술 보호 또는 공감 질문이 사용되었을 때만 에이전트가 자발적으로 공개한다.
수치심 기반 비밀

LM-5(수치심 회피) 동기. 도박 중독, 빚, 건강 문제 등. 법정에서 공개적으로 인정하는 것 자체가 고통. 비공개 진술 보호 + 공감 질문에서만 나옴.

보복 두려움 기반 비밀

LM-3(제3자 보호) 또는 직장 내 권력 관계. 말하면 보복당할까 두려움. 보복 우려 확인 + 감정 안정화에서만 나옴.

관계 보호 기반 비밀

LM-7(관계 유지). 진실을 말하면 관계가 끝남을 알기에 숨김. 관계 유지가 오히려 해임을 보여주는 증거 제시 + 신뢰에서 나옴.

💡
중요: 비공개 진술은 플레이어의 내부 단서가 될 수는 있지만, 그 자체가 바로 공개 판결 근거가 되면 안 된다. 즉, 플레이어는 진실을 더 잘 알게 되지만, 공적으로 확인 가능한 증거를 따로 찾아야 한다. 이것이 비공개 루트의 밸런스 핵심.

에이전트 발언 결정 로직 (per-turn)

각 에이전트는 발언 시 룰 엔진으로부터 현재 상태를 수신하고 아래 순서로 발언 목표를 결정한다.

① 위협 평가
플레이어 행동이 자신의 어떤 쟁점을 겨냥하는가? + 신뢰 상태는? trust_toward_judge가 높으면 위협 인식이 약화.
② 목표 선택
위협도 + 거짓말 동기(Lie Motive) + 신뢰 상태에 따라 발언 목표 결정.
③ 유형 결정
현재 쟁점의 lie_state + lie_type + lie_motive → 해당 상태의 발언 전략 적용.
④ Verbal Tell 체크
현재 쟁점에 매핑된 Verbal Tell이 있으면 발현 조건 확인 → LLM 프롬프트에 버릇 지시 추가.
⑤ 감정 톤 적용
현재 emotional_arc 위치 확인 → 행동 징후에 해당하는 어조로 조정.
⑥ 발언 생성
위 결정들을 시스템 프롬프트로 주입 → 클라우드 LLM이 자연어 발언 생성.

발언 목표 선택 조건표 (v2.0)

조건 (우선순위 순)선택 목표LLM 지시 방향
방금 내 거짓말이 붕괴됨긴급 방어 or 재프레이밍즉각 반박 또는 의미 전환. LM에 따라 감정 호소 or 논리적 재구성.
trust_toward_judge가 높고, collapse_via_trust 쟁점 질문자발적 부분 공개조심스럽게 사실의 일부를 인정. 비공개 요청과 함께.
플레이어가 내 쪽을 집중 압박방어 (Defend)LM-1/2: 논리적 방어. LM-5: 감정 회피. LM-4: 역공.

심문 이력 추적 시스템 (v4.0 신규)

당사자별 x 쟁점별 심문 이력을 추적하여 LLM 프롬프트에 컨텍스트로 주입한다.

추적 항목

  • 첫 질문 감지: 특정 쟁점에 대해 처음 질문하는지 감지 → "이미 말했다" 방지
  • 이전 질문 유형과 횟수: fact_pursuit 3회, empathy_approach 1회 등
  • 상대방의 폭로된 비밀: A가 S4에서 폭로한 정보를 B의 프롬프트에 주입
  • 교차 쟁점 일관성: 다른 쟁점에서 한 말과 모순되지 않도록 강제

질문 순서 의존성
