플레이 메커닉 GDD v4.1

진실 3층 · 쟁점 & 붕괴 · Phase 0-7 + 결과 · 질문 3종 + 자유질문 · 토글/액티브 스킬 · 증거 대질 · 증인 소환 · 리소스(게임 내+글로벌) · 미니게임 · 보상 · 판결 & 과정 점수

플레이어가 실제로 경험하는 모든 메커닉을 다룬다. v4.0 핵심 변경: Phase 1/2 사전생성 JSON(NO LLM), 증인 소환, 글로벌 리소스(상점/광고), 기억 퍼즐 미니게임, 보상 시스템, 교차 당사자 자동반응. v4.1 변경: 스크립트 로더 안정화 (caseId 매핑 수정, dialogues prop 우선), 재판관 질문 이름 자동 제거, 리소스 밸런스 확정.

진실 3층 구조

Truth Layer 1 · 진실 1층
실제 진실

실제로 무슨 일이 일어났는지에 대한 절대 데이터. 48h 타임라인에 기록. 채점 기준.

Truth Layer 2 · 진실 2층
각자의 인지와 오해

당사자가 진짜 알고 있는 것, 착각하고 있는 것, 듣기만 한 것. Relationship Ledger의 왜곡된 과거도 여기에 포함.

Truth Layer 3 · 진실 3층
법정에서의 발언

같은 사실이라도 숨기거나, 과장하거나, 왜곡. 거짓말 동기(Lie Motive)에 따라 왜곡 방식이 달라진다.

ℹ️
진실 3층(Truth Layers)과 인과 4층(Causal Layers)은 별개 체계다. 진실 3층은 "정보가 어떻게 왜곡되는가"(사실→인지→발언)를 다루고, 인과 4층은 "사건이 왜 일어났는가"(동기→촉발→갈등→프레이밍)를 다룬다. 사건 구조 페이지의 인과 그래프를 참고할 것.

쟁점 항목 & 거짓말 붕괴 규칙

거짓말은 사건 전체에 대해 한 번에 무너지지 않고, 쟁점 단위로 붕괴한다. 붕괴 경로는 압박 루트와 신뢰 루트 두 가지.

상태설명압박 루트 AI 행동신뢰 루트 AI 행동
숨김아직 쟁점이 공개되지 않음관련 질문 피하기/화제 돌리기신뢰가 높으면 간접적 힌트 제공
주장사실처럼 단정하거나 부인자신감 있는 반복 주장공감 질문에 살짝 흔들림
충돌증거/진술로 흔들림말 바꾸기, 감정적 반발비공개 보장 시 부분 인정 가능
붕괴더는 거짓말 유지 불가인정 또는 침묵자발적 고백 + 재프레이밍
재프레이밍사실은 인정, 해석을 바꿈"맞지만 이유가 있었다""사실 말씀드리고 싶었는데..."

핵심 플레이 루프 (v3.0)

한 사건은 약 10~20분. v3.0에서 Phase별 액션 해금 구조가 적용되어, 단계가 올라갈수록 새로운 도구가 열리며 같은 질문에도 더 깊은 답변이 나온다.

Phase 0

사건 개요

인물 소개 + 배경. 진실은 숨겨진 채 시작.

Phase 1

초기 진술

사전 생성 JSON 스크립트 (NO LLM). 84개 사건별 Phase 1 스크립트. import.meta.glob + 로버스트 폴백. dialogues prop 우선, LLM prefetch는 폴백. [ScriptLoader] 디버그 로그.

Phase 2

즉각 반박

사전 생성 JSON 스크립트 (NO LLM). 84개 사건별 Phase 2 스크립트. 감정 격화 + 첫 모순 힌트. caseId 매핑 수정 완료 (partnership→case-partner, workplace→case-work).

Phase 3

심문 (LLM, min 3턴)

🔓 질문 3종(fact_pursuit, motive_search, empathy_approach) + 자유질문(AI)

🔓 이의제기(Phase 3+), 분리심문(Phase 3+)

🔒 증거 잠김 / 즉답요구·회피판독·비공개보호 잠김

NPC 태도: 경계심 높음, 표면적 답변. 심문으로 거짓말 동요 시 증거 자동 발견 가능.

교차 반응: 25% 기본 끼어들기 확률. S3 책임전가 시 100% 강제 반박.

Phase 4

증거 심리 (LLM, min 1턴)

🔓 증거 제시·대질 해금 + 즉답요구(Phase 4+) · 회피판독(토글, Phase 4+) 해금

🔓 증인 소환 해금 (조사 토큰 1개 소모)

NPC 태도: 증거 앞에 흔들림, 새 정보 흘림. 증거 조합 격상.

Phase 5

최종 심문 (LLM, min 2턴)

🔓 비공개보호(토글, Phase 5+) 해금 + AI 진술분석 해금

NPC 태도: 방어 느슨, 깊은 고백 가능. "사실은..." "솔직히 말하면..."

S4/S5 비밀 폭로 시 100% 충격 반응.

Phase 6

중재안 (선택적)

즉시 판결 / 조건부 조정안 / 추가 조사 보류 / 사실+해결책 우선.

Phase 7

최종 판단

사실 인정 → 책임 배분 → 해결책 선택 → 증거 정당성 판단.

결과

판결 리뷰

숨겨진 진실 비교 + 과정 점수 공개. 홈/다음 재판/다시 판결.

Phase별 해금 요약

Phase심문증거스킬 (액티브)스킬 (토글)AI 특수
3 · 심문✅ 질문 3종 + 자유질문🔒이의제기, 분리심문🔒—
4 · 증거 심리✅✅ 제시 + 대질+ 즉답요구회피판독—
5 · 최종 심문✅✅전부+ 비공개보호AI 진술분석

질문 시스템 (3종 + 자유질문)

기존 6종을 3종으로 통합. 각 유형의 의도와 효과가 명확히 차별화된다. AI 모드에서는 자유 텍스트 입력도 가능.

🔵 사실 추궁 (fact_pursuit)

사실 여부를 직접 추궁한다. "정확히 언제", "얼마를", "어디서" — 거짓말 상태 전이의 핵심 트리거. 날짜·금액·행위를 확정.

🟣 동기 탐색 (motive_search)

"왜 그랬는지"를 파고든다. 동기와 이유를 설명하거나 회피하게 만든다. 감정적 배경을 드러내는 질문.

🩷 공감 접근 (empathy_approach)

부드럽게 마음을 여는 질문. trust_toward_judge +12. 경계를 낮추고 솔직한 마음/본심을 유도. 신뢰 루트의 핵심. (v4.1: "동정" 표현 → "솔직한 마음/본심" 표현으로 톤 변경)

자유 질문 (AI 모드 전용)

LLM 연결 시 플레이어가 직접 텍스트를 입력할 수 있다. LLM이 질문 유형을 자동 분류하고, 해당 쟁점의 NPC가 반응. 조사 토큰 1개 소모.

재판관 질문 이름 자동 제거 (v4.1)

쟁점명에 대상 당사자의 이름이 포함되어 있을 때 자동으로 제거하여 자연스러운 질문을 생성한다.

상황변환 전변환 후
대상 이름 포함"세린의 새벽 휴대폰 열람" (세린에게 질문 시)"새벽 휴대폰 열람"
상대방 이름 포함"민준의 계좌 인출" (세린에게 질문 시)"상대방의 계좌 인출"

buildFallbackJudgeQuestion()에도 동일한 이름 제거 로직이 적용된다.

✍️
자유 질문이 쟁점에 적중하면 과정 점수 보너스가 적용된다. AI를 적극 활용할수록 높은 점수 가능.

Phase별 NPC 응답 깊이

같은 질문이라도 Phase에 따라 NPC의 답변 깊이가 달라진다. LLM 시스템 프롬프트에 Phase별 지침이 주입된다.

PhaseNPC 태도같은 "사실추궁"에 대한 차이
3 · 심문경계심 높음, 준비된 답변"그건 오해입니다" — 표면적 부정
4 · 증거 심리증거 앞에 흔들림"사실 그때..." — 새 정보 흘림
5 · 최종 심문방어 느슨, 지침"솔직히 말하면..." — 깊은 고백

주장 그래프 (Claim Graph)

누가 / 어떤 쟁점에 대해 / 무엇을 말했는지를 자동 요약하는 UI. 텍스트 피로를 줄이는 핵심 장치.

기능설명
발언 자동 등록에이전트 발언이 관련 쟁점에 자동 태깅·요약된다.
충돌 표시같은 쟁점에 대한 A/B 발언이 모순되면 빨간색 표시.
변경 추적이전 발언과 달라진 내용은 노란색 표시.
붕괴 표시거짓말 붕괴된 쟁점은 취소선 + 실제 사실 병기.

스킬 시스템 (토글 + 액티브)

스킬은 두 종류로 나뉜다. 토글 스킬은 심문 시 ON/OFF로 부가 효과를 걸고, 액티브 스킬은 독립적으로 턴을 소모한다.

토글 스킬 (심문 강화 — 턴 소모 없음)

심문 패널 안에 토글 버튼으로 존재. ON 상태에서 질문하면 해당 효과가 자동 적용되고 OFF로 돌아간다.

증거 시스템

증거는 정답 버튼이 아니라, 진술을 흔들고 해석 싸움을 재구성하는 장치.

Hard

계좌 내역 / 원본 계약서

기관 발급 원본. 1개만으로도 거짓말 붕괴 가능.

Soft

채팅 캡처 / 목격 진술

완전성에 의문 가능. 2개 이상 조합 시 Hard로 격상.

정당성 의심

몰래 열람 / 불법 녹취

사실일 수 있지만 판결 근거로 쓰면 권위 점수 하락.

증거 조사 (3단계 프로그레시브)

각 증거에 대해 3단계 조사를 진행할 수 있다. 각 단계마다 조사 토큰 1개 소모.

단계조사 내용얻는 정보
1. 원본 확인증거의 원본 상태를 확인원본 여부, 캡처/발췌 여부
2. 맥락 복원잘린 맥락을 복원전후 대화, 추가 배경
3. 조작 여부편집/조작 흔적 검사편집 유무, 삭제 흔적

심문 중 증거 발견 (v3.0 신규)

Phase 3에서 심문으로 거짓말 상태가 전이되면, NPC의 진술에서 새로운 증거가 자동 발견될 수 있다. 발견 확률은 거짓말 상태가 깊을수록 높다.

거짓말 상태발견 확률발견 맥락 예시
S0 (완강 부정)15%"말하다 실수로 금전 거래를 언급했다"
S1 (동요)30%"흔들리면서 메시지 기록을 언급했다"
S2 (일부 인정)50%"인정하다가 제3자를 언급했다"
S3+ (책임 전가~)65~100%"상대 탓을 하다 기록의 존재를 언급했다"

증거 대질 (v3.0 신규 — AI 모드)

증거를 제시할 때, 단순 제시 외에 자유 텍스트로 추궁할 수 있다. 자동 제안 질문 또는 직접 입력.

💬 자동 제안

증거 유형과 조사 결과에 기반한 추궁 질문 2~3개가 자동 생성. 탭하면 바로 실행.

✍️ 직접 입력 (고득점)

플레이어가 직접 추궁 포인트를 입력. LLM이 맞춤 NPC 반응 생성. 과정 점수 보너스 적용.

📤 질문 없이 제시

기존처럼 증거만 제시. 기본 효과만 적용.

증거 조합 격상

특정 Soft 증거 조합이 모두 제시되면 Hard로 자동 격상. 사건 데이터의 evidenceCombinations에 정의.

중재안 Phase

솔로몬의 진짜 클라이맥스. "누가 더 잘못했는가"만이 아니라 "어떻게 정리하는 것이 가장 현명한가".

⚖️
진입 방식: Phase 5(재심문) 종료 후, 플레이어는 아래 4가지 중 하나를 선택한다. 즉시 판결을 원하면 중재안을 건너뛸 수 있다.
즉시 판결

중재안 없이 바로 Phase 7로. 확신이 있을 때.

조건부 조정안 제시

양측에 조건을 제시하고 반응을 본다. 에이전트가 수용/거부/수정 요청으로 반응.

추가 조사 후 보류

부분 해결 사건에서 유효. 확정 가능한 것만 확정하고 나머지는 보류. 지혜 점수 유지 또는 소폭 상승.

사실 확정 + 해결책 우선

사실 인정을 먼저 하고, 해결책 논의를 별도로 진행. 복잡한 공동 책임 사건에 적합.
