2. Dashboard (대시보드)

대시보드는 WebAdmin의 랜딩 페이지로, 게임 운영 상태를 한눈에 파악할 수 있는 8개의 통계 카드와 2개의 요약 테이블로 구성됩니다.

2.1 통계 카드 (Stats Cards)

8개 통계 카드 구성

카드는 4열 × 2행 그리드(grid-template-columns: repeat(4, 1fr))로 배치됩니다. 각 카드는 흰색 배경에 미세한 그림자가 적용되고, 아이콘 + 수치 + 라벨 구조입니다.

#카드 이름아이콘데이터 소스설명
1Total Players👥GET /api/stats → totalPlayers전체 등록 플레이어 수
2Total Games🎮GET /api/stats → totalGames누적 게임 플레이 횟수
3Today Games📅GET /api/stats → todayGames오늘 플레이된 게임 수
4Active (7d)🔥GET /api/stats → activePlayers7d최근 7일 내 접속한 플레이어 수
5Avg Score⭐GET /api/stats → avgScore전체 게임 평균 점수
6Active Notices📢GET /api/stats → activeNotices현재 활성화된 공지 수
7Active Mail📬GET /api/stats → activeMail현재 발송 중인 우편 수
8AI Prompts🤖GET /api/stats → totalPrompts등록된 AI 프롬프트 수

2.2 Recent Games 테이블

최근 게임 기록

대시보드 하단 좌측에 위치하며, 가장 최근에 완료된 게임을 시간순으로 표시합니다.

컬럼설명예시
Player플레이어 이름김판관
Case사건 IDcase_family_001
Score획득 점수85
Date플레이 일시2026-03-28 14:30

2.3 Games by Relationship 테이블

관계 유형별 게임 통계

대시보드 하단 우측에 위치하며, 관계 유형(가족, 연인, 친구, 직장, 이웃)별 게임 플레이 횟수와 평균 점수를 집계합니다.

컬럼설명예시
Relationship관계 유형 (rel_type)family
Games해당 유형의 총 게임 수42
Avg Score해당 유형의 평균 점수78.5
API 엔드포인트 — 대시보드 통계는 단일 GET /api/stats 호출로 모든 데이터를 수신합니다. Recent Games와 Relationship 통계는 응답 내 recentGames, gamesByRelationship 배열로 포함됩니다.

5. AI 프롬프트 관리 (AI Prompts)

게임에서 사용하는 모든 LLM 프롬프트를 실시간으로 편집하고 관리하는 페이지입니다. 프롬프트 변경 시 게임 클라이언트가 자동으로 새로운 프롬프트를 로드합니다.

5.3 변수 관리 (Variables)

프롬프트 변수 시스템

각 프롬프트에는 variables 필드가 JSON 배열로 저장되어 있으며, 프롬프트 본문 내에서 {{변수명}} 형태로 참조됩니다. 게임 엔진이 실행 시점에 실제 값으로 치환합니다.

변수명설명사용 예시
{{character_name}}현재 NPC 캐릭터 이름당신은 {{character_name}}입니다.
{{personality}}캐릭터 성격 설정성격: {{personality}}
{{emotion_state}}현재 감정 상태감정: {{emotion_state}}
{{case_summary}}사건 요약 정보사건 개요: {{case_summary}}
{{evidence_list}}수집된 증거 목록증거: {{evidence_list}}
{{conversation_history}}대화 히스토리이전 대화: {{conversation_history}}

5.4 버전 히스토리 & 원클릭 롤백

버전 관리 시스템

프롬프트를 수정하면 이전 내용이 ai_prompt_history 테이블에 자동 저장되고, 현재 프롬프트의 version이 1 증가합니다.

기능설명
자동 히스토리 저장프롬프트 수정 시 서버가 자동으로 이전 버전을 ai_prompt_history에 백업
버전 번호각 프롬프트의 version 필드가 수정할 때마다 +1 증가
히스토리 조회GET /api/ai-prompts/:id/history로 특정 프롬프트의 전체 변경 이력 확인
원클릭 롤백POST /api/ai-prompts/:id/rollback/:version 호출로 특정 버전으로 즉시 복원
롤백 주의사항 — 롤백 실행 시 현재 버전이 히스토리에 저장된 후 지정 버전의 내용으로 덮어씁니다. 따라서 롤백 자체도 히스토리에 기록되어 추적이 가능합니다.

5.5 API 엔드포인트

메서드경로설명
