UX / 화면 구조

메인 법정 화면
  • 좌측 A, 우측 B의 초상 + 행동 징후 아이콘 (숫자 아닌 표정/동작)
  • 중앙 실시간 대화 로그
  • 하단 질문 6종 선택 패널 + 증거 공개 버튼 + 신뢰/보호 행동 패널
  • 상단 법정 통제력 / 조사 토큰 / 스킬 포인트 / 명성 영향 예고
보조 정보 화면
  • 주장 그래프 (Claim Graph) — 자동 요약, 충돌 표시, 변경 추적, 정식 기록 하이라이트
  • 관계별 미니 보드 — 부부: 돈 흐름 / 이웃: 건물 배치 / 직장: 문서 비교
  • 확보 증거 보드 (잠금/해제 + 4축 속성 표시)
  • 쟁점 체크리스트 (상태별 색상: 숨김/주장/충돌/붕괴/재프레이밍)
  • 48h 타임라인 (확인된 이벤트만 점진적 공개)

화면 구성 원칙

정보 계층 분리

즉시 필요한 정보(대화 로그, 리소스)와 참조 정보(주장 그래프, 증거 보드)를 레이어로 분리. 한 번에 모두 보여주지 않는다.

행동 징후 가시화

캐릭터 초상 옆에 행동 징후 아이콘/표현 변화. 숫자가 아닌 "답변 전 호흡이 길어진다", "말을 끊으려 한다" 등 텍스트 힌트.

주장 그래프 상시 접근

텍스트 피로 해소의 핵심. 누가 어떤 쟁점에 무엇을 주장했는지 한눈에 보이는 요약 보드.

질문 의도 선명화

6종 질문 유형을 아이콘+짧은 설명으로 표시. 플레이어가 "어떤 질문이 어떤 효과를 내는지" 직관적으로 파악.

기술 스택 (LLM 분업 구조)

컴포넌트권장 기술역할비용 구조
프론트엔드React / Vue + TypeScript법정 UI, 주장 그래프, 관계별 보드, 판결/중재 화면고정 개발비
룰 엔진Node.js 또는 Python FastAPI거짓말 상태 머신, 감정 곡선, 증거 잠금 해제, 모순 판정, 채점서버 운영비
사건 생성 LLM로컬 LLM (Llama 3, Mistral 등)DuoSeed/ContextSeed/CaseSeed 전체 생성, 48h 타임라인, 드라마 비트 검증로컬 처리, API 비용 없음
실시간 연기 LLMOpenAI API (GPT-4o 이상)당사자 발화 생성, 감정 표현, Verbal Tell, 신뢰 루트 반응턴당 API 호출
데이터 저장JSON + 세션 DB (PostgreSQL/SQLite)DuoSeed 영구 저장, CaseSeed 세션 저장, 플레이 이력 저장DB 운영비
반복 방지 + 편향 추적Redis 또는 로컬 스토리지최근 N판 태그, 편향 카운터 빠르게 읽기/쓰기최소
💡
LLM 분업의 핵심: 로컬 LLM은 사건 시작 전에 한 번 무겁게 돌리고 끝. 플레이 중에는 클라우드 API만 턴당 호출. 로컬에서 풍부한 데이터(DuoSeed, 48h 타임라인, 증거 그래프)를 만들고, 클라우드 API는 그 데이터 위에서 자연스러운 연기만 담당. 비용 효율과 품질을 동시에 확보.

주요 리스크와 대응

리스크: 사건 반복감

대응: DuoSeed/ContextSeed/CaseSeed 3층 분리 + 반복 방지 메커니즘(8종 태그) + 편향 디렉터(카운터 기반). 같은 인물이라도 맥락과 사건이 항상 다르다.

리스크: LLM 할루시네이션

대응: 룰 엔진이 lie_state_map + lie_motive로 발언 허용 범위 관리. LLM은 연기만, 진실 판정은 룰 엔진. Verbal Tell도 구조화된 지시로만 발현.

리스크: 풀 수 없는 사건

대응: 드라마 비트 검증(앵커 진실 필수), 해결성 검증(입증 루트 최소 1개), 초기 인상 균형 점수. 부분 해결 사건도 앵커 진실 1개는 보장.

리스크: 텍스트 피로

대응: 주장 그래프 자동 요약, 관계별 미니 보드, 행동 징후 아이콘, 질문 6종 패널로 정보 접근 최적화. 대화만 읽지 않아도 상태 파악 가능.

리스크: 감정 조작에 취약한 판결

대응: 증거 기반 판단에 높은 가중치. 감정 편향 판결은 지혜 점수 감소. 편향 디렉터가 감정 과신 패턴을 역이용.

리스크: 압박 일변도 플레이

대응: collapse_via_trust 진실이 모든 사건에 1개 이상. 신뢰 루트를 안 쓰면 최고 점수에 도달 불가. 두 루트 모두 유효한 밸런스.

리스크: 로컬 LLM 생성 품질

대응: 11단계 파이프라인의 Step 10~11 검증이 안전망. 드라마 비트, 해결성, 초기 인상 균형, 동선 정합성 모두 검증 후 통과된 사건만 플레이에 투입. 실패 시 자동 재생성.

리스크: 증거 정당성 시스템의 복잡도

대응: Stage 6부터 단계적 도입. 초반 Stage에서는 모든 증거가 적법. 후반으로 갈수록 정당성 쟁점 비중 상승. 캠페인이 자연스러운 튜토리얼 역할.

MVP 범위

재미 요소는 모두 갖추고, 디자인만 조금 아쉬운 수준. 시스템은 전부 작동하되, 콘텐츠 볼륨과 비주얼 폴리시가 v2에서 확장.

필수 포함 (전체 시스템)
  • 3층 데이터 아키텍처: DuoSeed / ContextSeed / CaseSeed
  • DuoSeed 5쌍 (관계 5종: 부부, 이웃, 직장, 동업, 가족)
  • Relationship Ledger (DuoSeed당 3~5개 이력)
  • Social Graph (DuoSeed당 3~5명 제3자)
  • Context Pack 9종
  • 사건 모듈 17종: CS 6종 + VM 5종 + TW 6종(TW-6 포함)
  • 48h 이벤트 타임라인 (블록 기반)
  • 증거 4축 시스템 + 조사 하위 액션 6종
  • 거짓말 2축: 유형 7종(LT-7 포함) x 동기 7종
  • Verbal Tell (캐릭터당 3~4개)
  • 질문 6종 시스템
  • 신뢰/보호 행동 5종
  • 스킬 7종 (리테마이징 완료)
  • 주장 그래프 UI
  • 중재안 Phase + 관계별 해결 프레임
  • 드라마 비트 템플릿 + 초기 인상 균형
  • 편향 디렉터 (카운터 기반)
  • 반복 방지 메커니즘 (8종 태그)
  • 명성 3축 (통찰/권위/지혜)
  • 캠페인 10 Stage + 무한 사건 모드
  • 부분 해결 사건 (Stage 8~10)
  • 11단계 생성 파이프라인
  • LLM 분업: 로컬(생성) + 클라우드 API(연기) + 룰 엔진(판정)
  • 관계별 미니 보드 (범용 컴포넌트 + 탭 구성 차이)
후순위 (v2+)
  • 음성 합성 (캐릭터별 목소리)
  • 완전한 애니메이션 초상화
  • 온라인 랭킹 시스템
  • 플레이어 커스텀 법정 테마
  • 커뮤니티 공유 씨드 (사건 코드 공유)
  • 멀티엔딩 시네마틱
  • 관계별 미니 보드 전용 아트 리소스
  • DuoSeed 추가 확장 (관계 7종 → 10종+)
  • 데일리 챌린지 사건
  • 시즌 이벤트 (명절 특별 사건 등)

MVP 검증 질문:

  2. 같은 DuoSeed의 2번째·3번째 사건에서 "이 사람은 알겠는데 이번 사건은 처음이다"를 느끼는가?
  3. 압박 루트와 신뢰 루트를 둘 다 시도해보고 싶은 동기가 생기는가?
  4. 판결 후 "이 증거를 쓰는 게 맞았나?", "중재안이 적절했나?"를 되돌아보게 되는가?
결론

솔로몬 v2.0의 핵심은 반복 등장하는 인물쌍(DuoSeed)과 그들의 생활 루틴·과거 이력·사회관계망·현재 맥락(ContextSeed) 위에서 사건(CaseSeed)이 발생하며, 증거와 거짓말은 그 사건의 결과가 아니라 그 생활의 부산물로 생성된다는 구조에 있다.

그 위에서 플레이어는 사실뿐 아니라, 오해와 체면, 수치심과 보호 본능, 정당한 증거와 부당한 증거, 판결과 중재 사이를 가르며 '지혜'를 증명한다. 이 다층적 판단이 솔로몬의 정체성이자 장기 플레이의 동력이다.
