UX / 화면 구조

메인 게임 화면 (모바일 퍼스트)
  • 상단: TopBar (← 나가기 | PhaseIndicator | 📋 정보 | ⚙️ 설정)
  • 상단바: PartyStatusBar (😐A이름 🔍3 ⚡5 B이름😤 — 감정 이모지 + 리소스)
  • 중앙: 채팅 UI (좌=A 말풍선, 우=B 말풍선, 중앙=시스템 메시지)
  • 하단 고정: ActionPanel (대상 선택 → [❓심문 | 📄증거 | ⚡스킬] 3탭)
  • 토스트: 액션 상세 패널이 하단 위에 absolute로 팝업
  • max-width: lg (모바일 최적화)
보조 정보 (📋 버튼으로 토글)
  • 쟁점 체크리스트 — 쟁점별 거짓말 상태 + 감정 표시
  • 주장 그래프 — 발언 요약, 충돌 표시
  • 증거 보드 — 잠금/해제/제시 상태 + 조사 진행도
  • 진술 분석 팝업 (Phase 5) — AI 분석 결과 모달

화면 구성 원칙

정보 계층 분리

즉시 필요한 정보(대화 로그, 리소스)와 참조 정보(주장 그래프, 증거 보드)를 레이어로 분리. 한 번에 모두 보여주지 않는다.

행동 징후 가시화

캐릭터 초상 옆에 행동 징후 아이콘/표현 변화. 숫자가 아닌 "답변 전 호흡이 길어진다", "말을 끊으려 한다" 등 텍스트 힌트.

주장 그래프 상시 접근

텍스트 피로 해소의 핵심. 누가 어떤 쟁점에 무엇을 주장했는지 한눈에 보이는 요약 보드.

질문 의도 선명화

6종 질문 유형을 아이콘+짧은 설명으로 표시. 플레이어가 "어떤 질문이 어떤 효과를 내는지" 직관적으로 파악.

주요 리스크와 대응

리스크: 사건 반복감

대응: DuoSeed/ContextSeed/CaseSeed 3층 분리 + 반복 방지 메커니즘(8종 태그) + 편향 디렉터(카운터 기반). 같은 인물이라도 맥락과 사건이 항상 다르다.

리스크: LLM 할루시네이션

대응: 룰 엔진이 lie_state_map + lie_motive로 발언 허용 범위 관리. LLM은 연기만, 진실 판정은 룰 엔진. Verbal Tell도 구조화된 지시로만 발현.

리스크: 풀 수 없는 사건

대응: 드라마 비트 검증(앵커 진실 필수), 해결성 검증(입증 루트 최소 1개), 초기 인상 균형 점수. 부분 해결 사건도 앵커 진실 1개는 보장.

리스크: 텍스트 피로

대응: 주장 그래프 자동 요약, 관계별 미니 보드, 행동 징후 아이콘, 질문 6종 패널로 정보 접근 최적화. 대화만 읽지 않아도 상태 파악 가능.

리스크: 감정 조작에 취약한 판결

대응: 증거 기반 판단에 높은 가중치. 감정 편향 판결은 지혜 점수 감소. 편향 디렉터가 감정 과신 패턴을 역이용.

리스크: 압박 일변도 플레이

대응: collapse_via_trust 진실이 모든 사건에 1개 이상. 신뢰 루트를 안 쓰면 최고 점수에 도달 불가. 두 루트 모두 유효한 밸런스.

리스크: 로컬 LLM 생성 품질

대응: 11단계 파이프라인의 Step 10~11 검증이 안전망. 드라마 비트, 해결성, 초기 인상 균형, 동선 정합성 모두 검증 후 통과된 사건만 플레이에 투입. 실패 시 자동 재생성.

리스크: 증거 정당성 시스템의 복잡도

대응: Stage 6부터 단계적 도입. 초반 Stage에서는 모든 증거가 적법. 후반으로 갈수록 정당성 쟁점 비중 상승. 캠페인이 자연스러운 튜토리얼 역할.

MVP 범위

재미 요소는 모두 갖추고, 디자인만 조금 아쉬운 수준. 시스템은 전부 작동하되, 콘텐츠 볼륨과 비주얼 폴리시가 v2에서 확장.

구현 완료 (v3.0 현재)
  • 84개 사건 데이터: 7 관계 유형 × 12개 (spouse, neighbor, workplace, family, friend, partnership, tenant)
  • Phase별 해금 구조: Phase 3(심문) → 4(증거+스킬) → 5(토글+AI분석)
  • 질문 3종 + 자유질문(AI)
  • 스킬 2분류: 토글(회피판독, 비공개보호) + 액티브(이의제기, 분리심문, 즉답요구)
  • 증거 시스템: 3단계 조사, 조합 격상, 심문 중 자동 발견, 증거 대질(AI)
  • 거짓말 상태 머신: S0~S5 × 동기 7종
  • 감정 엔진: defensive → confident → shaken → angry → resigned
  • 과정 점수: 결과 점수 + 심문 효율/증거 활용/AI 활용 보너스
  • Phase별 LLM 응답 깊이 (Phase 3 표면 → Phase 5 고백)
  • AI 진술 분석 (Phase 5, 전체 대화 요약 + 모순 탐지)
  • CaseMap (7챕터 × 12스테이지, 별 시스템)
  • 리더보드 + 시즌제 + 프로필 + 명성
  • 한국어 조사 자동 처리: korean.ts (이/가, 은/는, 을/를 등)
  • 이름 플레이스홀더: {A}, {B}, {A:이} → resolveNameTemplate()
  • 시스템 메시지 톤: 관찰자 톤 (~한다/~했다), 생동감

결론

솔로몬 v2.0의 핵심은 반복 등장하는 인물쌍(DuoSeed)과 그들의 생활 루틴·과거 이력·사회관계망·현재 맥락(ContextSeed) 위에서 사건(CaseSeed)이 발생하며, 증거와 거짓말은 그 사건의 결과가 아니라 그 생활의 부산물로 생성된다는 구조에 있다.

그 위에서 플레이어는 사실뿐 아니라, 오해와 체면, 수치심과 보호 본능, 정당한 증거와 부당한 증거, 판결과 중재 사이를 가르며 '지혜'를 증명한다. 이 다층적 판단이 솔로몬의 정체성이자 장기 플레이의 동력이다.
