UX / 화면 구조

메인 게임 화면 (모바일 퍼스트)
  • 상단: TopBar (← 나가기 | PhaseIndicator | 정보 | 설정)
  • 상단바: PartyStatusBar (A이름 감정이모지 + 리소스 + B이름 감정이모지)
  • 중앙: 채팅 UI (좌=A 말풍선, 우=B 말풍선, 중앙=시스템 메시지)
  • 하단 고정: ActionPanel (대상 선택 → [심문 | 증거 | 스킬] 3탭)
  • 토스트: 액션 상세 패널이 하단 위에 absolute로 팝업
  • max-width: lg (모바일 최적화)
보조 정보 (정보 버튼으로 토글)
  • HistoryPanel — 쟁점별 거짓말 상태 + 감정 표시
  • 증거 보드 — 잠금/해제/제시 상태 + 조사 진행도
  • 증인 목록 — 소환 가능 증인 + 편향 정보 (Phase 4+)
  • TestimonyModal — 증인 증언 결과 표시 (Phase 4+)
  • 진술 분석 팝업 — AI 분석 결과 모달 (Phase 5)
  • ResourcePopup — 글로벌 리소스 상태, 타이머, 광고 버튼

IntroSlides (v4.0)

첫 접속 시 게임 소개 슬라이드. 게임 컨셉, 조작 방법, Phase 설명을 3-4장 슬라이드로 안내.

Phase별 전환 화면

PhaseTransition 컴포넌트: Phase 전환 시 전체화면 오버레이로 다음 Phase 이름과 설명을 표시. 자동 또는 탭으로 진행.

주요 리스크와 대응

리스크: 사건 반복감

대응: DuoSeed/ContextSeed/CaseSeed 3층 분리 + 반복 방지 메커니즘(8종 태그) + 편향 디렉터. 같은 인물이라도 맥락과 사건이 항상 다르다.

리스크: LLM 할루시네이션

대응: 룰 엔진이 lie_state_map + lie_motive로 발언 허용 범위 관리. LLM은 연기만, 진실 판정은 룰 엔진. Verbal Tell도 구조화된 지시로만 발현. callTerms로 호칭 일관성 보장.

리스크: LLM 반복 패턴

대응: 심문 이력 추적 (첫 질문 감지, "이미 말했다" 방지), 교차 쟁점 일관성 강제, "매번 호칭으로 시작 금지" 안티-반복 규칙. v4.1: 프롬프트 압축 + 혼합 발화 규칙("재판관에게 1문장 + 상대에게 1문장") + 안티패턴("자기 불리한 정보 먼저 꺼내지 마라", "같은 내용 반복 금지").

리스크: Phase 1/2 LLM 비용

대응: Phase 1/2를 사전생성 JSON으로 완전 대체. 168개 스크립트로 LLM 호출 없이 초기 진술/반박 처리. LLM은 Phase 3부터만 사용. v4.1에서 import.meta.glob 로버스트 폴백 + caseId 매핑 수정으로 안정화 완료 (0 mismatch).

리스크: 텍스트 피로

대응: 관계별 미니 보드, 행동 징후 아이콘, 질문 3종 패널로 정보 접근 최적화. MS Fluent Emoji로 시각적 피로 감소. 증인 소환으로 변주.

리스크: 증인 활용도

대응: 253명 증인에 다양한 편향/왜곡 부여. 직접 증인 81%. LLM이 편향/왜곡 기반으로 생동감 있는 증언 생성. 거짓말 상태 전이 유발 가능.

결론

솔로몬 v4.1은 v4.0 기반에서 스크립트 로더 안정화 (caseId 매핑 수정, 168개 0 mismatch), LLM 프롬프트 전면 압축 ("자기"→"자기야" 강제 보정, 혼합 발화 규칙, 안티패턴 차단), 재판관 질문 이름 자동 제거, 글로벌 리소스 밸런스 확정 (무료 캡 10, 유료 99, 판당 조사 6/스킬 5/통제 3)을 완료한 상태이다. 비주얼 에셋 확보와 온라인 리더보드가 다음 단계의 핵심 과제다.
