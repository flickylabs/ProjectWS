제작 계획 GDD v1.3

UX / 화면 구조 · 주요 리스크 대응 · MVP 범위 제안

플레이어가 경험하는 화면 구조, 생성형 게임의 주요 리스크와 대응 방안, 최소 기능 제품의 범위를 정의한다.

UX / 화면 구조

정보가 많아도 플레이어가 압도되지 않도록, 핵심 정보는 말싸움과 증거 보드 두 축으로 보여준다.

메인 법정 화면
  • 좌측 A, 우측 B의 초상/감정 상태 아이콘
  • 중앙 실시간 대화 로그
  • 하단 질문 버튼 및 증거 공개 버튼
  • 상단 법정 통제력 / 조사 토큰 / 명성 영향 예고
보조 정보 화면
  • 고정된 타임라인
  • 확보 증거 보드 (잠금/해제 상태 표시)
  • 쟁점 체크리스트 (상태별 색상 표시)
  • 붕괴한 거짓말과 남은 의혹 구분 표시
💡
UI는 복잡한 조사 게임처럼 보이기보다, "두 사람이 내 앞에서 싸우고 있다"는 현장감을 중심에 두는 편이 좋다.

화면 구성 원칙

정보 계층 분리

즉시 필요한 정보(대화 로그, 리소스)와 참조 정보(타임라인, 증거 보드)를 레이어로 분리한다. 한 번에 모두 보여주지 않는다.

감정 상태 가시화

캐릭터 초상 옆에 감정 아이콘 또는 색상 변화를 적용한다. 플레이어가 언제 압박하면 실언이 나오는지 직관적으로 파악할 수 있어야 한다.

쟁점 상태 표시

각 쟁점의 현재 상태(숨김/주장/충돌/붕괴)를 색상으로 구분하는 체크리스트를 제공한다. 무엇이 확정되었고 무엇이 남아 있는지 한눈에 보인다.

리소스 가시성

조사 토큰, 스킬 포인트, 법정 통제력을 상시 노출한다. 자원 부족 경고와 획득 피드백을 즉각적으로 제공해야 한다.

판결 화면

단계입력 내용UX 설계 원칙

주요 리스크와 대응

생성형 게임에서 가장 흔한 문제를 사전에 구조적으로 막는다.

리스크: 사건 반복감

자주 같은 패턴처럼 느껴질 수 있다.

대응: 반복 방지 메커니즘으로 패턴 다양성 보장. 최근 3판의 태그를 추적해 가중치 패널티 부여.

리스크: LLM 할루시네이션

이미 정해진 사실과 다른 내용을 즉흥적으로 말할 수 있다.

대응: 룰 엔진이 lie_state_map으로 발언 허용 범위 관리. LLM은 연기, 진실 판정은 룰 엔진.

리스크: 풀 수 없는 사건

증거가 부족해 플레이어가 억울함을 느낄 수 있다.

대응: Step 8 해결성 검증으로 입증 루트 최소 1개 보장. 패널티 합산 임계값 초과 시 재생성.

리스크: 지나친 텍스트 피로

말이 많아지면 플레이 템포가 느려질 수 있다.

대응: 짧은 라운드 구조, 자동 요약, 쟁점 하이라이트, 감정 아이콘 제공.

리스크: 감정 조작에 취약한 판결

LLM의 설득력 있는 발언에 플레이어가 편향될 수 있다.

대응: 채점 시 증거 기반 판단에 높은 가중치. 감정적 발언에 휘둘린 판결은 지혜 점수 감소.

리스크: 조합 불균형

일부 모듈 조합이 너무 어렵거나 너무 쉬울 수 있다.

대응: Stage별 허용 모듈 풀을 단계적으로 개방. 난이도 곡선을 캠페인에서 먼저 검증.

MVP 범위 제안

최소 기능 제품은 "재미의 핵심"만 검증해야 한다. 동적 생성과 판결의 손맛이 우선이다.

필수 포함
  • 관계 3종 (부부, 이웃, 직장)
  • 사건 모듈 15종 (CS 6종 + VM 5종 + TW 5종 — 없음 포함)
  • 증거 타입 8종 (bank, chat, cctv, contract, testimony, log + 조합 2종)
  • 캠페인 10 Stage
  • 무한 사건 생성 모드
  • 기본 명성 시스템 (통찰/권위/지혜 3축)
  • 인과 그래프 + 거짓말 상태 머신 + 증거 의존성 그래프
  • Party A / B 연기 LLM
  • 반복 방지 메커니즘
  • 기본 특수 스킬 4종 (진실의 서약, 교차 대질, 기억 소환, 진술 봉인)
후순위 (v2+)
  • 음성 합성 (캐릭터별 목소리)
  • 완전한 애니메이션 초상화
  • 온라인 랭킹 시스템
  • 플레이어 커스텀 법정 테마
  • 커뮤니티 공유 씨드 (사건 코드 공유)
  • 멀티엔딩 시네마틱
  • 감정 곡선 고도화 (7단계 이상)
  • 제3자 풀 소환 (VM-A 특화 확장)
  • 데일리 챌린지 사건

결론

솔로몬의 성공 포인트는 자유로운 말싸움 그 자체가 아니라, 인과 그래프가 결정하는 사건 논리, 지식 비대칭이 만드는 정보 긴장감, 적응형 거짓말 전략이 연출하는 드라마, 증거 의존성이 만드는 추리 루트가 하나의 구조로 맞물려 돌아가는 데 있다. 이 구조 위에서 같은 관계와 같은 캐릭터를 반복해서 만나도, 사건·증거·오해·트위스트가 달라지면 플레이는 매번 새로워진다. 이 반복성이 솔로몬의 가장 큰 상품 가치이자 장르적 정체성이다.
