⚖️ 솔로몬 GDD v2.0

LLM 심문 추리 게임 · Procedural Case Drama — 반복 등장하는 인물쌍의 생활 위에서 사건이 발생하고, 플레이어는 사실과 오해, 압박과 신뢰, 판결과 중재 사이를 가르며 '지혜'를 증명하는 리플레이형 추리 게임

⚖️
게임 개요
프로젝트 핵심 구조, USP 5종, 디자인 원칙 7종, 3층 데이터 아키텍처(DuoSeed/ContextSeed/CaseSeed), LLM 분업 구조를 정리합니다.
USP3층 아키텍처디자인 원칙LLM 분업
🧱
사건 구조
DuoSeed 스키마, Relationship Ledger, Social Graph, 상황 모듈 시스템, 인과 그래프, 지식 비대칭 매트릭스, 공동 책임 스펙트럼, TW-6 공유 오해 반전을 설계합니다.
DuoSeed관계 이력Social Graph인과 그래프TW-6
⚙️
생성 엔진
동적 사건 생성 시스템, 48h 이벤트 타임라인, 증거 4축 시스템, 드라마 비트 템플릿, 편향 디렉터, 반복 방지, 확장 파이프라인, CaseSeed v2 데이터 모델을 정의합니다.
48h 타임라인증거 4축드라마 비트편향 디렉터
🤖
에이전트 설계
거짓말 2축(유형 x 동기), LT-7 희생 진술, Verbal Tell, 감정 행동 징후, 신뢰/보호 에이전트 로직, 발언 결정 로직, LLM 프롬프트 구조를 고도화합니다.
Lie MotiveVerbal Tell행동 징후신뢰/보호
🎮
플레이 메커닉
진실 3층, 질문 6종 시스템, 신뢰/보호 행동 5종, 리테마이징된 스킬, 증거 조사 하위 액션, 중재안 Phase, 주장 그래프, 부분 해결 사건을 다룹니다.
질문 6종신뢰 루트중재안주장 그래프
📈
콘텐츠 & 밸런스
ContextSeed 맥락 팩, 캠페인 Stage 갱신, 관계별 미니 보드, 반복 플레이 예시, DuoSeed 기반 샘플 사건, 밸런스 원칙을 정리합니다.
Context Pack관계별 보드샘플 사건밸런스
🔧
제작 계획
LLM 분업 기술 스택(로컬 + 클라우드 API), UX 갱신(주장 그래프, 행동 징후, 관계별 보드, 중재 UI), 리스크 대응, MVP 범위를 다룹니다.
LLM 분업 스택UX리스크MVP
⚖️
핵심 포지셔닝
반복 등장하는 인물쌍(DuoSeed)의 생활 위에서 사건이 발생하고, 플레이어는 사실과 오해, 압박과 신뢰, 판결과 중재 사이를 가르며 '지혜'를 증명한다. 같은 인물, 매번 다른 사건으로 수백 가지 경우의 수를 만드는 Procedural Case Drama.
