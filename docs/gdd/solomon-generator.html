생성 엔진 GDD v2.0

동적 생성 · 48h 타임라인 · 증거 4축 · 드라마 비트 · 편향 디렉터 · 반복 방지 · 파이프라인 · 데이터 모델

반복 플레이를 만드는 핵심 엔진. DuoSeed(인물쌍)의 생활 위에서 사건이 파생되고, 증거는 그 생활의 부산물로 생성되며, 드라마 리듬이 보장되는 완전한 생성 시스템.

동적 사건 생성 시스템

단순 범주 조합이 아니라, 인물쌍의 생활·이력·관계망·현재 맥락 위에서 인과 구조·지식 비대칭·감정 곡선·증거 의존성이 맞물려 매번 다른 드라마를 만든다.

레이어고정 요소가변 요소반복 플레이 기여
DuoSeed (인물쌍) 이름, 성격, 말투, 취약점, 생활 루틴, 디지털 습관, Verbal Tell 없음 (게임 전체 고정) 관계 학습 축적, 캐릭터 애착과 예측 재미
Relationship Ledger 과거 이력 3~5개 (확정/왜곡/침묵) 활성화 이력 1~2개 (사건마다 다름) 감정 온도와 동기의 깊이. "왜 저렇게 예민한지" 납득
Social Graph 제3자 3~5명 (가족, 지인, 제도적 인물) 활성화 제3자 0~2명 (사건마다 다름) 같은 인물쌍이라도 제3자가 다르면 사건이 다름
ContextSeed (배경 압박) 없음 명절, 평가, 갱신, 병원비 등 (매 판 선택) 적은 비용으로 감정 톤과 동기를 크게 변화
CaseSeed (사건) 없음 전체 (매 판 새로 생성) 인과 그래프, 증거, 거짓말, 반전 모두 새로움
💡
핵심 원칙: 동적 생성은 "아무거나 섞는 랜덤"이 아니라, DuoSeed의 생활 구조 + ContextSeed의 상황 압박 + 인과 그래프의 사건 논리 + 48h 타임라인의 증거 파생 + 드라마 비트의 리듬 보장의 다중 결합이다.

48시간 이벤트 타임라인

증거가 "정답 버튼처럼 떨어지는 느낌"을 없애는 핵심 시스템. 사건 발생 전 48~72시간의 이벤트 기반 타임라인을 먼저 생성하고, 증거는 이 타임라인에서 파생한다.

📌
설계 방식: 풀 분 단위 시뮬레이션이 아니라 시간대 블록 기반 이벤트 배치. 48시간 = 12블록 (4블록/일 x 3일 또는 2일). 블록당 이벤트 0~2개. 총 이벤트 8~16개. 검증은 블록 단위로만 수행해서 재생성 확률을 낮춘다.

이벤트 → 증거 파생 규칙

돈 이동 이벤트

계좌 내역, 영수증, ATM 인출 기록 파생. DuoSeed의 소득대와 소비 패턴에 맞아야 한다.

위치 이벤트

CCTV, 출입 기록, 주차 기록 파생. DuoSeed의 생활 루틴과 일치하거나 의미 있게 벗어나야 한다.

커뮤니케이션 이벤트

채팅 캡처, 통화 기록, 이메일 파생. DuoSeed의 디지털 습관에 따라 증거 유형이 달라진다.

문서/시스템 이벤트

문서 수정 로그, 접근 권한 기록, 계약서 파생. 직장/동업 관계에서 주로 발생.

목격 이벤트

Social Graph의 제3자가 같은 블록·같은 장소에 있었으면 목격 진술 파생 가능. 편향과 왜곡 속성이 적용된다.

블록 단위 검증 규칙

동선 일관성

같은 블록에 한 인물이 두 곳에 있을 수 없다. 위반 시 해당 이벤트 재배치.

접근 권한

권한이 없는 시스템/공간의 증거는 생성되지 않거나, "위법 취득" 태그가 붙는다.

증거 간 시간 정합성

한 증거의 타임스탬프가 다른 증거와 모순되면 사건 의미가 달라질 수 있다. 의도적 모순은 함정 증거로, 비의도적 모순은 재생성.

증거 4축 시스템

증거는 "하드/소프트" 이분법을 넘어, 4개 축으로 평가된다. "진실은 맞지만 써도 되는 증거인가?"라는 지혜의 문제가 여기서 발생한다.

축 1
신뢰도 (Reliability)

기존 하드/소프트 분류를 유지. 원본 데이터(하드)인지, 간접·해석 가능 자료(소프트)인지.

축 2
완전성 (Completeness)

원본 / 편집됨 / 부분 캡처 / 맥락 결여. 같은 채팅이라도 전문이냐 캡처냐에 따라 해석이 달라진다.

축 3
출처성 (Provenance)

본인 소지 / 제3자 전달 / 익명 제보 / 기관 제공. 누가 어떤 경로로 가져왔는지가 신뢰도를 좌우한다.

축 4
정당성 (Legitimacy)

적법 취득 / 사생활 침해 소지 / 위법 취득. 휴대폰 몰래 열람, 몰래 녹취, 사적 공간 촬영 등. 정당하지 않은 증거는 사실이더라도 판결 근거로 쓰면 권위 점수가 떨어진다.

⚠️
함정 증거 설계: 사건마다 1~2개의 오해 유도 증거를 포함한다. 4축 중 완전성이 낮거나 정당성이 의심스러운 증거가 함정 역할을 한다. 단독으로 보면 A가 범인처럼 보이지만, 다른 증거와 결합하거나 맥락을 복원하면 B의 오해였음이 드러난다.

드라마 비트 템플릿

LLM 기반 분쟁 게임은 구조를 잘못 잡으면 말만 길고 사건은 밋밋해진다. 모든 사건은 최소한의 드라마 리듬을 보장해야 한다.

📌
강제 규칙: 모든 사건은 아래 4개 비트 중 최소 3개를 포함해야 한다. 파이프라인 검증 단계에서 확인.
Beat 1
앵커 진실

반드시 확정 가능한 핵심 사실 1개. 플레이어가 아무리 못해도 이것 하나는 파악할 수 있어야 좌절감이 줄어든다. 부분 해결 사건에서도 앵커 진실은 반드시 존재.

Beat 2
감정 미끼

플레이어가 처음엔 이게 핵심이라 착각하기 쉬운 표면 이슈. 감정적으로 강렬하지만 실제 사건의 본질은 아닌 것. 이것에 휘둘리면 지혜 점수가 떨어진다.

Beat 3
구조 반전

중반 이후 이해관계가 뒤집히는 순간. TW-1~6 반전 모듈이 이 비트를 담당하거나, 반전 모듈 없이도 지식 비대칭 매트릭스의 비밀이 드러나는 것으로 대체 가능.

Beat 4
해결 딜레마

사실은 밝혀졌지만 처분이 어려운 지점. "둘 다 잘못이 있다", "증거는 맞지만 정당하지 않다", "관계를 끊는 게 맞는가" 같은 지혜의 문제. 중재안 Phase의 핵심 재료.

초기 인상 균형 점수

A/B 초기 진술에서 한쪽이 너무 뻔히 수상하게 보이면 추리의 재미가 없다. 생성 시 초기 인상 균형을 검증한다.

평가 항목기준위반 시 조치
초기 진술 설득력A/B 모두 초기에 그럴듯해야 한다한쪽이 너무 약하면 프레이밍 재조정
감정 균형둘 다 분노/억울함의 강도가 비슷해야 한다한쪽만 과잉 감정이면 감정 곡선 재조정
증거 노출 균형초기에 한쪽에만 불리한 증거가 집중되지 않아야 한다증거 공개 순서 재배치

플레이어 편향 디렉터

플레이어의 판단 습관을 추적하고, 다음 사건에서 한 요소만 그 편향을 역이용한다. "내가 AI를 읽는 줄 알았는데, 시스템이 나를 읽고 있었다."

💡
절제 규칙: 한 판에 편향 역이용은 딱 1개 요소만. 너무 많으면 부당하게 느껴진다. 캠페인 Stage 5 이후부터 활성화 (최소 5판의 판단 데이터 축적 후).

추적 카운터

편향 유형측정 방식역이용 방법
감정 과신감정적 당사자를 3판 연속 더 신뢰다음 판에 감정적 측이 거짓말쟁이인 사건
하드 증거 맹신하드 증거만 3판 연속 의존다음 판에 하드 증거가 함정(편집/위조)
한쪽 편향3판 연속 A 또는 B에 과도한 책임 부과공동 책임(VM-D) 사건 비중 상승
성급한 판단Phase 3~4에서 이미 결론을 내는 패턴후반 반전이 있는 사건(TW 포함) 비중 상승
증거 정당성 무시위법 취득 증거를 그대로 판결 근거로 사용다음 판에 정당성이 핵심 쟁점인 증거 배치

반복 방지 메커니즘

동적 생성이라도 비슷한 패턴이 연속되면 플레이어는 금방 지루함을 느낀다. 플레이어 이력을 추적해 패턴 다양성을 보장한다.

추적 항목 (v2.0 확장)
  • 최근 3판의 핵심 증거 타입
  • 최근 3판의 트위스트 유형
  • 최근 3판의 주 책임 당사자 (A / B / 공동)
  • 최근 2판의 관계 유형
  • 최근 2판의 거짓말 주요 전략
  • 최근 2판의 ContextSeed 유형
  • 최근 3판의 Ledger 활성화 이력
  • 최근 3판의 Social Graph 활성 제3자
가중치 패널티 방식

씨드 생성 시 이력과 겹치는 조합에 점수 패널티를 부여하고, 가장 낮은 점수 조합을 선택한다. 완전 배제가 아니라 확률 억제 방식.

보장 다양성 규칙
  • 3판 연속 같은 관계 유형 금지
  • 2판 연속 같은 트위스트 유형 금지
  • 3판 연속 같은 당사자가 주 책임자인 사건 금지
  • 핵심 증거가 2판 연속 같은 타입이면 다음 판 강제 변경
  • 2판 연속 같은 ContextSeed 유형 금지

사건 생성 파이프라인 (v2.0)

DuoSeed·ContextSeed·CaseSeed 3층 구조와 모든 신규 시스템을 결합한 확장 파이프라인. 전체 생성은 로컬 LLM이 담당한다.

Step 1

DuoSeed 확인 또는 생성

인물쌍이 이미 있으면 로드, 없으면 새로 생성. 성격, 말투, Verbal Tell, 생활 루틴, 디지털 습관, Relationship Ledger(3~5개), Social Graph(3~5명).

Step 2

ContextSeed 선택

배경 압박을 선택한다. 반복 방지 태그 검사 포함. 이 맥락이 인과 그래프의 촉발 사건(Layer 1)을 결정한다.

Step 3

모듈 조합 + 인과 그래프 생성

갈등 씨앗(CS) + 변수 모듈(VM) + 반전 모듈(TW) 선택. 근본 원인 → 촉발 사건 → 표면 갈등 → 프레이밍 4층 체인 구성. Ledger 활성화 이력 1~2개 선택.

Step 4

캐릭터 감정 곡선 설계

DuoSeed의 성격 아키타입 + ContextSeed 압박 + Ledger 감정 잔여에 기반해 이번 판의 감정 궤적을 설계한다.

Step 5

지식 비대칭 매트릭스 구성

각 진실 항목을 4분면에 배치. "둘 다 모르는 비밀" 최소 1개. TW-6 사용 시 "공유 오해" 항목 추가.

Step 6

48h 이벤트 타임라인 생성

DuoSeed의 생활 루틴 + ContextSeed 상황 위에 12블록 이벤트를 배치. 블록 단위 동선 검증. Social Graph 제3자 목격 기회 배치.

Step 7

증거 의존성 그래프 생성 (4축)

48h 타임라인에서 증거를 파생. 4축 속성 부여. 잠금 해제 관계, 조합 격상 규칙, 함정 증거 1~2개 삽입.

Step 8

적응형 거짓말 전략 배정

쟁점별 거짓말 유형(LT-1~7) x 강도(L1~L4) x 동기(Lie Motive) 배정. Verbal Tell 활성화 매핑. 희생 진술(LT-7) 배치 여부 결정.

Step 9

공동 책임 스펙트럼 + 중재안 프레임 설정

쟁점별 A/B 책임 비율, 모호성, 가중치, 중재 연결 태그 설정. 관계별 해결 프레임 선택.

Step 10

드라마 비트 검증 + 초기 인상 균형 검사

4비트 중 3개 이상 포함 확인. A/B 초기 진술 균형 점수 검사. 위반 시 Step 3으로 돌아가 재조정.

Step 11

해결성 검증 + 반복 방지 태그 등록 + 편향 디렉터 적용

핵심 쟁점 입증 루트 최소 1개 보장. 함정 증거 해소 루트 존재 확인. 감정 곡선 최소 2회 전환 확인. 반복 방지 태그 이력 기록. 편향 디렉터 역이용 요소 1개 반영 여부 확인.

⚠️
검증 기준 (Step 10~11): ① 드라마 비트 4개 중 3개 이상 포함, ② 핵심 쟁점마다 입증 루트 ≥ 1개, ③ 함정 증거가 해소 가능, ④ 감정 곡선이 최소 2회 상태 전환 포함, ⑤ 초기 인상 균형 점수 통과, ⑥ 반복 방지 패널티 합산 ≤ -50점이면 재생성, ⑦ 48h 타임라인 블록 단위 동선 정합성 통과.

데이터 모델 v2.0

3층 구조(DuoSeed/ContextSeed/CaseSeed)와 모든 신규 시스템을 통합한 확장 데이터 모델.

/* ════════════════════════════════════════════
   DuoSeed — 인물쌍 (게임 전체 유지)
   ════════════════════════════════════════════ */
DuoSeed {
  duo_id: string,
  relationship_type: "spouse"|"neighbor"|"boss_employee"|"partnership"|"family"|"tenant_landlord"|"friend",

  party_a: CharacterProfile,
  party_b: CharacterProfile,

  relationship_ledger: LedgerEntry[],   // 3~5개
  social_graph: ThirdParty[]            // 3~5명
}

CharacterProfile {
  name: string,
  age: number,
  occupation: string,
  income_bracket: "low"|"mid"|"high",
  residence_type: "apartment"|"house"|"officetel"|"shared",
  archetype: "avoidant"|"confrontational"|"victim_cosplay"|"cold_logic",
  speech_style: string,
  pride: number,              // 1-10
  fear: string,
  risk_appetite: number,      // 1-10
  digital_habit: "sns_active"|"messenger_main"|"minimal",
  daily_routine: string,      // 자연어 설명
  sensitive_points: string[], // 반복적 민감 주제
  verbal_tells: VerbalTell[]  // 3~4개
}

VerbalTell {
  type: "over_precision"|"moral_leap"|"repetition"|"topic_switch"|string,
  trigger: "lying"|"cornered"|"emotional"|"avoiding",
  pattern: string             // 발현 패턴 자연어 설명
}

LedgerEntry {
  id: string,
  category: "confirmed"|"distorted"|"silenced",
  description: string,
  is_real: boolean,
  who_remembers_accurately: "a"|"b"|"both"|"neither",
  who_distorts: "a"|"b"|"both"|"none",
  distortion_direction: string,
  currently_resolved: "resolved"|"unresolved"|"surface_only",
  emotional_residue: "none"|"mild"|"strong",
  connection_to_current: "direct"|"indirect"|"none"
}

ThirdParty {
  id: string,
  slot: "family_1"|"family_2"|"acquaintance_1"|"acquaintance_2"|"institutional",
  name: string,
  relation_to: "a"|"b"|"both",
  knowledge_scope: string,
  witnessed_directly: boolean,
  bias: "pro_a"|"pro_b"|"neutral"|"self_interest",
  distortion_risk: "intentional"|"unconscious"|"accurate",
  contact_frequency: "frequent"|"occasional"|"rare"
}


/* ════════════════════════════════════════════
   ContextSeed — 배경 압박 (한 판에만 적용)
   ════════════════════════════════════════════ */
ContextSeed {
  context_type: "holiday"|"review"|"renewal"|"medical"|"exam"|"rainy"|"move"|"audit"|string,
  description: string,
  emotional_pressure: number,    // 1-10
  affects: "a"|"b"|"both",
  trigger_amplifier: string      // 인과 그래프 Layer 1 촉발 조건 강화
}


/* ════════════════════════════════════════════
   CaseSeed — 이번 사건 (매 판 새로 생성)
   ════════════════════════════════════════════ */
CaseSeed {
  case_id: string,
  duo_id: string,                       // DuoSeed 참조
  context_seed: ContextSeed,
  anti_pattern_tags: string[],
  bias_director_target: string|null,    // 이번 판 편향 역이용 타겟 (1개)

  /* ── 모듈 조합 ─────────────────────────── */
  conflict_seed: "CS-1"|"CS-2"|"CS-3"|"CS-4"|"CS-5"|"CS-6",
  variable_modules: ("VM-A"|"VM-B"|"VM-C"|"VM-D"|"VM-E")[],
  twist_module: "TW-1"|"TW-2"|"TW-3"|"TW-4"|"TW-5"|"TW-6"|null,

  /* ── 인과 그래프 ──────────────────────── */
  causal_chain: {
    root_cause:       { type: string, hidden: true },
    trigger_event:    { description: string, context_link: string },
    surface_conflict: { description: string },
    framing: {
      party_a: string,
      party_b: string
    }
  },

  /* ── Ledger 활성화 ────────────────────── */
  active_ledger_entries: string[],      // LedgerEntry.id 1~2개

  /* ── Social Graph 활성화 ──────────────── */
  active_third_parties: string[],       // ThirdParty.id 0~2개

  /* ── 48h 이벤트 타임라인 ──────────────── */
  timeline: {
    blocks: TimeBlock[]                 // 12개
  },

  /* ── 지식 비대칭 매트릭스 ─────────────── */
  knowledge_matrix: {
    both_know:            TruthItem[],
    a_only:               TruthItem[],
    b_only:               TruthItem[],
    neither_knows:        TruthItem[],
    shared_misconception: TruthItem[]   // TW-6 전용
  },

  /* ── 적응형 거짓말 전략 ───────────────── */
  lie_strategies: {
    party_a: LieStateMachine[],
    party_b: LieStateMachine[]
  },

  /* ── 증거 의존성 그래프 (4축) ─────────── */
  evidence_graph: {
    nodes: EvidenceNode[],
    combinations: EvidenceCombination[]
  },

  /* ── 공동 책임 스펙트럼 ───────────────── */
  responsibility_spectrum: ResponsibilityItem[],

  /* ── 드라마 비트 ──────────────────────── */
  drama_beats: {
    anchor_truth:      string,          // 반드시 확정 가능한 사실
    emotional_bait:    string|null,     // 감정 미끼
    structural_reversal: string|null,   // 구조 반전
    resolution_dilemma:  string|null    // 해결 딜레마
  },

  /* ── 초기 인상 균형 ───────────────────── */
  initial_balance: {
    a_persuasiveness: number,           // 1-10
    b_persuasiveness: number,           // 1-10
    balanced: boolean
  },

  /* ── 해결성 보장 ──────────────────────── */
  solvability: {
    must_be_provable:       string[],
    optional_ambiguity:     string[],
    contradiction_paths:    string[],
    third_party_secret_id:  string|null,
    partial_resolution:     boolean     // 부분 해결 사건 여부
  },

  /* ── 중재안 프레임 ────────────────────── */
  mediation_frame: {
    relationship_type_options: string[],  // 관계별 해결 옵션
    wisdom_factors: string[]              // 지혜 평가 요소
  }
}


/* ── 보조 타입 ─────────────────────────────── */
TimeBlock {
  day: number,                          // -2, -1, 0
  period: "morning"|"afternoon"|"evening"|"night",
  events: TimeEvent[]                   // 0~2개
}

TimeEvent {
  id: string,
  actor: "a"|"b"|"third_party",
  actor_id: string|null,                // ThirdParty.id (제3자인 경우)
  location: string,
  action: string,
  generates_evidence: string|null       // EvidenceNode.id
}

TruthItem {
  id: string,
  fact: string,
  true: boolean,
  weight: number
}

LieStateMachine {
  issue_id: string,
  lie_type: "LT-1"|"LT-2"|"LT-3"|"LT-4"|"LT-5"|"LT-6"|"LT-7",
  lie_intensity: "L1"|"L2"|"L3"|"L4",
  lie_motive: "self_protection"|"face_saving"|"third_party_protection"|"revenge"|"shame_avoidance"|"career_preservation"|"relationship_maintenance",
  initial_state: "S0"|"S1"|"S2"|"S3"|"S4"|"S5",
  collapse_via_trust: boolean,          // 신뢰 루트로 붕괴 가능 여부
  transitions: { from: string, to: string, trigger: string }[]
}

EvidenceNode {
  id: string,
  type: "bank"|"chat"|"cctv"|"contract"|"testimony"|"log"|"device"|"sns",
  reliability: "hard"|"soft",
  completeness: "original"|"edited"|"partial"|"context_missing",
  provenance: "self_possessed"|"third_party"|"anonymous"|"institutional",
  legitimacy: "lawful"|"privacy_concern"|"unlawful",
  proves: string[],
  is_trap: boolean,
  requires: string[],
  timeline_block: string                // 파생된 TimeBlock 참조
}

EvidenceCombination {
  requires: string[],
  upgrades_to: "hard",
  proves: string[]
}

ResponsibilityItem {
  issue: string,
  party_a_pct: number,
  party_b_pct: number,
  ambiguity: "none"|"low"|"high",
  weight: "high"|"medium"|"low",
  mediation_link: string,               // 연결되는 해결책 태그
  legitimacy_issue: boolean             // 증거 정당성 쟁점 여부
}
