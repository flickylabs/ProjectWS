반복 플레이를 만드는 핵심 엔진. 이 페이지는 사건이 어떻게 생성되고, 증거가 어떻게 연결되며, 중복 없는 다양성을 어떻게 보장하는지 다룬다.

동적 사건 생성 시스템

반복 플레이를 만드는 핵심 엔진. 단순 범주 조합이 아니라, 인과 구조·지식 비대칭·감정 곡선·증거 의존성이 맞물려 매번 다른 드라마를 만든다.

핵심 정의
"같은 인물, 같은 관계이지만 다른 사건"

캐릭터 아키타입과 관계의 감정선은 유지하되, 사건의 원인·전개·증거·트위스트는 매번 새롭게 조합한다. 플레이어는 익숙한 관계를 상대하면서도 사건 자체는 처음 접하는 느낌을 받는다.

설계 의도
기억과 추리의 균형

캐릭터 성향이 반복되면 플레이어는 메타 감각을 얻고, 사건이 계속 바뀌면 그 메타 지식만으로는 결론을 내릴 수 없게 된다. 이 균형이 장기 플레이의 핵심이다.

레이어고정 요소가변 요소반복 플레이 기여
관계 템플릿 부부 / 이웃 / 상사-부하 / 동업자 / 형제자매 갈등의 초점(돈, 신뢰, 명예, 안전, 가족) 익숙한 긴장 구조 제공
캐릭터 아키타입 회피형, 정면 돌파형, 피해자 코스프레형, 냉정 논리형 감정 상태, 말투, 자존심, 취약점 캐릭터 기억과 예측 재미 형성
인과 그래프 근본 원인 유형 (재정 불안, 소통 단절, 권력 욕구 등) 촉발 사건, 파급 범위, 제3자 개입 여부 LLM 발언의 동기 일관성 보장
증거 의존성 그래프 증거 타입 분류 (하드/소프트/논리 모순) 잠금 해제 순서, 조합 시 신뢰도 변화 심문 루트 다양화
거짓말 전략 트리 전략 유형 (부정, 전가, 감정 호소, 재프레이밍) 전환 트리거, 버티는 쟁점, 붕괴 속도 같은 사건군도 다른 템포 유도
💡
중요 원칙: 동적 생성은 "아무거나 섞는 랜덤"이 아니라, 관계가 유지하는 감정 구조 + 인과 그래프가 결정하는 사건 논리 + 증거 의존성이 만드는 추리 루트의 삼중 결합이어야 한다.

증거 의존성 그래프

증거가 독립적으로 존재하면 "증거 버튼 누르기" 게임이 된다. 증거끼리 연결되어야 추리다. 플레이어가 어떤 순서로 증거를 얻느냐에 따라 심문 루트가 달라진다.

잠금 해제형
E1이 있어야 E2를 요청 가능

계좌 내역(E1)을 확보해야 베팅 기록(E2)을 조사 토큰으로 요청할 수 있다. E1 없이 E2를 보면 맥락이 없어 의미 없다.

조합 격상형
E1 + E2 = 신뢰도 Hard로 상승

카톡 캡처(소프트) + 출입 기록(소프트) = 동선 일치 → Hard 증거로 격상. 단독으로는 "아닐 수도 있지만", 결합 시 "이건 확정"이 된다.

증거 ID단독 신뢰도조합 조건조합 후 신뢰도잠금 해제드라마 역할
E1 계좌 내역 Hard — Hard E2, E3 요청 가능 A의 전면 부정 붕괴
E2 베팅 기록 Hard E1 필요 Hard — 도박 사실 확정
E3 채팅 캡처 Soft E4와 조합 Hard — 외도 의혹 해소 or 확정
E4 출입 기록 Soft E3와 조합 Hard — 알리바이 검증
E5 제3자 진술 Soft E1 또는 E4와 조합 Soft+ E6 잠금 해제 제3자 비밀 진입점
E6 내부 로그 Hard E5 필요 Hard — 제3자 비밀 폭로 (최대 반전)
⚠️
함정 증거 설계: 사건마다 1~2개의 오해 유도 증거를 포함한다. 단독으로 보면 A가 범인처럼 보이지만, 다른 증거와 결합하면 B의 오해였음이 드러나는 증거. 이것이 없으면 게임이 너무 단순해진다.

반복 방지 메커니즘

동적 생성이라도 비슷한 패턴이 연속되면 플레이어는 금방 지루함을 느낀다. 플레이어 이력을 추적해 패턴 다양성을 보장한다.

추적 항목
  • 최근 3판의 핵심 증거 타입
  • 최근 3판의 트위스트 유형
  • 최근 3판의 주 책임 당사자 (A / B / 공동)
  • 최근 2판의 관계 유형
  • 최근 2판의 거짓말 주요 전략
가중치 패널티 방식

씨드 생성 시 이력과 겹치는 조합에 점수 패널티를 부여하고, 가장 낮은 점수 조합을 선택한다. 완전 배제가 아니라 확률 억제 방식 — 어쩔 수 없이 비슷한 구조가 나와도 세부 내용은 달라진다.

보장 다양성 규칙
  • 3판 연속 같은 관계 유형 금지
  • 2판 연속 같은 트위스트 유형 금지
  • 3판 연속 같은 당사자(A 또는 B)가 주 책임자인 사건 금지
  • 핵심 증거가 2판 연속 같은 타입이면 다음 판에 강제 변경
태그 유형예시 값패널티 범위
관계 유형spouse / neighbor / workplace / partnership / family직전 2판 일치 시 -30점
트위스트 유형misunderstanding / concealment / third_party / fabrication직전 1판 일치 시 -40점
주 책임 당사자party_a / party_b / shared직전 2판 일치 시 -20점
핵심 증거 타입bank / chat / cctv / contract / testimony / log직전 2판 일치 시 -25점
거짓말 주요 전략denial / deflection / emotional / reframing직전 1판 일치 시 -15점

사건 생성 파이프라인

앞서 설계한 모든 구조를 결합한 완전한 생성 흐름. 8단계를 거쳐 플레이 가능한 사건이 완성된다.

Step 1

관계 템플릿 선택

관계 유형, 감정 축, 이해관계 초점을 정한다. 반복 방지 태그 검사 포함.

Step 2

인과 그래프 생성

근본 원인 → 촉발 사건 → 표면 갈등 → 각자의 프레이밍 4층 체인을 구성한다.

Step 3

캐릭터 배치 + 감정 곡선

아키타입, 자존심, 취약점을 배정하고 세션 감정 궤적을 설계한다.

Step 4

지식 비대칭 매트릭스 구성

각 진실 항목을 4분면에 배치한다. "둘 다 모르는 비밀" 최소 1개 삽입.

Step 5

증거 의존성 그래프 생성

증거 노드 배치, 잠금 해제 관계, 조합 신뢰도 격상 규칙, 함정 증거 1~2개 삽입.

Step 6

적응형 거짓말 전략 배정

쟁점별 초기 상태 설정. 전환 트리거와 최종 붕괴 조건 정의.

Step 7

공동 책임 스펙트럼 계산

쟁점별 A/B 책임 비율, 모호성, 가중치 설정. 채점 기준 확정.

Step 8

해결성 검증 + 반복 방지 태그 등록

핵심 쟁점 입증 루트 최소 1개 보장. 이번 사건의 태그를 이력에 기록.

⚠️
검증 기준 (Step 8): ① 핵심 쟁점마다 입증 루트 ≥ 1개, ② 함정 증거가 오해를 유발하지만 결국 해소될 수 있어야 함, ③ 감정 곡선이 6단계 심문 중 최소 2회 상태 전환을 포함할 것, ④ 반복 방지 태그 패널티 합산 ≤ -50점 이면 재생성.

데이터 모델 (CaseSeed)

앞서 설계한 7가지 구조를 통합한 확장 CaseSeed. 모든 LLM 호출은 이 구조에서 필요한 필드만 추출해 프롬프트를 구성한다.

CaseSeed {
  case_id: string,
  anti_pattern_tags: string[],          // 반복 방지 이력 등록용

  /* ── 관계 & 기본 정보 ──────────────────────── */
  relationship_type: "spouse" | "neighbor" | "boss_employee" | "partnership" | "family",
  stakes: ("money" | "trust" | "reputation" | "safety" | "power")[],

  /* ── 인과 그래프 ────────────────────────────── */
  causal_chain: {
    root_cause:      { type: string, hidden: true },   // Layer 0
    trigger_event:   { description: string },           // Layer 1
    surface_conflict:{ description: string },           // Layer 2
    framing: {
      party_a: string,  // Layer 3 — A의 서사
      party_b: string   // Layer 3 — B의 서사
    }
  },

  /* ── 캐릭터 ─────────────────────────────────── */
  recurring_cast: {
    party_a: {
      archetype: "avoidant" | "confrontational" | "victim_cosplay" | "cold_logic",
      speech_style: string,
      pride: number,       // 1-10
      fear: string,
      moral_line: string,
      emotional_arc: ("defensive"|"confident"|"shaken"|"angry"|"resigned")[],
      emotion_triggers: { from: string, to: string, condition: string }[]
    },
    party_b: { /* 동일 구조 */ }
  },

  /* ── 지식 비대칭 매트릭스 ───────────────────── */
  knowledge_matrix: {
    both_know:     TruthItem[],   // 공통 사실
    a_only:        TruthItem[],   // A만 아는 것
    b_only:        TruthItem[],   // B만 아는 것
    neither_knows: TruthItem[]    // 제3자 비밀 (최대 반전 재료)
  },

  /* ── 적응형 거짓말 전략 ─────────────────────── */
  lie_strategies: {
    party_a: LieStateMachine[],   // 쟁점별 상태 머신
    party_b: LieStateMachine[]
  },

  /* ── 증거 의존성 그래프 ─────────────────────── */
  evidence_graph: {
    nodes: {
      id: string,
      type: "bank"|"chat"|"cctv"|"contract"|"testimony"|"log",
      reliability: "hard"|"soft",
      proves: string[],         // truth_item id 목록
      is_trap: boolean,         // 오해 유도 증거 여부
      requires: string[]        // 선행 증거 ID (잠금 해제 조건)
    }[],
    combinations: {
      requires: string[],       // 조합에 필요한 증거 ID
      upgrades_to: "hard",      // 조합 후 신뢰도
      proves: string[]
    }[]
  },

  /* ── 공동 책임 스펙트럼 ─────────────────────── */
  responsibility_spectrum: {
    issue: string,
    party_a_pct: number,        // 0-100
    party_b_pct: number,
    ambiguity: "none"|"low"|"high",
    weight: "high"|"medium"|"low"
  }[],

  /* ── 해결성 보장 ─────────────────────────────── */
  solvability: {
    must_be_provable:      string[],   // 반드시 입증 가능한 쟁점
    optional_ambiguity:    string[],   // 회색지대 허용 쟁점
    contradiction_paths:   string[],   // 논리 모순 루트
    third_party_secret_id: string      // 제3자 비밀 TruthItem ID
  }
}

/* ── 보조 타입 ─────────────────────────────────── */
TruthItem { id: string, fact: string, true: boolean, weight: number }

LieStateMachine {
  issue_id:     string,
  initial_state: "S0"|"S1"|"S2"|"S3"|"S4"|"S5",
  transitions: {
    from:      string,
    to:        string,
    trigger:   string   // "hard_evidence" | "contradiction_pointed" | "emotion_threshold" | ...
  }[]
}
