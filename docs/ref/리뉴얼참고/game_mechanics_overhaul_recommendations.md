# Solomon 게임 메커닉/피드백 재설계 권고안

## 핵심 진단

V2 atom 시스템으로 **응답 품질은 안정화**되었지만, 플레이 감각은 아직 “좋은 대화”에 머물러 있다. 지금 부족한 것은 문장 품질이 아니라 **행동의 결과가 눈에 보이고, 다음 선택이 달라지는 루프**다.

핵심 원인은 하나로 묶인다.

> **내부 상태 변화가 플레이어에게 “새로운 행동 가능성”으로 번역되지 않는다.**

즉 lie state, blocked vector, trust, hidden dispute는 엔진 안에서 움직이지만,
플레이어는 여전히 “질문 → 2문장 응답 → 다시 질문”으로 체감한다.

따라서 다음 단계는 LLM 품질 미세조정이 아니라,
**매 턴 결과를 플레이어가 읽을 수 있는 게임 단위로 표면화**하는 것이다.

---

## 0. 권장 코어 루프 (V3)

권장 턴 구조는 아래 6단계다.

1. **쟁점 선택**
2. **행동 선택**
   - 질문 3종 중 하나
   - 또는 증거 제시
3. **NPC 응답**
4. **즉시 결과 표시**
   - 균열
   - 방어선 봉쇄
   - 모순 토큰 생성
   - 신뢰 상승
   - 숨겨진 단서 획득
5. **이벤트/반응 선택** (조건 충족 시)
   - 모순 지적
   - 끼어들기 허용/차단
   - 감정 폭발 대응
   - 새 쟁점 추적/보류
6. **Board 갱신 + 다음 유효 수 선택지 강조**

핵심 원칙은 **매 턴 적어도 하나의 게임 결과를 남기는 것**이다.

## 1. 증거 시스템 재설계

### 설계안

현재 6개 증거 × 6개 조사 액션은 모바일 12~16턴 구조에 과밀하다. 다만 원본 데이터를 버릴 필요는 없다. **플레이어가 다루는 단위를 3~4개의 플레이어블 도시에(dossier)로 축소**하고, 기존 6개 증거는 그 도시어 내부의 업그레이드/보강 자료로 재배치하는 편이 가장 실용적이다.

### 1-A. 증거를 “개별 파일”이 아니라 “도시어 카드”로 묶기

spouse-01 기준 권장 묶음:

- **자금 흐름 카드**: e-1(기본) + e-2(업그레이드)
- **잘린 대화/장소 카드**: e-3(기본) + e-4(업그레이드)
- **우회 송금 카드**: e-5(기본) + e-6(업그레이드)
- (선택) **공동 약속 카드**: d-5 전용 메타 카드. 없으면 위 1번+3번 조합으로 처리

즉 데이터는 6개를 유지하되, **플레이어 인터페이스에서는 3~4개 카드**만 보이게 한다.

### 1-B. 증거 제시는 반드시 “질문 선택”과 결합

증거 제시만으로 큰 진전이 일어나면 전략이 약해진다. 반대로 증거를 내도 아무 일 없으면 손맛이 없다.

가장 좋은 구조는 **기본 효과 + 적중 효과** 2단이다.

- **기본 효과**: 증거를 내는 순간 관련 방어선과 쟁점이 드러난다.
  - 예: 맥락/취득경위/정체/진위 중 현재 남아 있는 방어 아이콘 공개
- **적중 효과**: 플레이어가 그 증거에 맞는 질문을 고르면 실제로 벡터가 봉쇄된다.
  - 예: `수취인이 누구입니까?` → `identity`
  - 예: `잘린 부분은 무엇입니까?` → `context`
  - 예: `이 자료는 어떻게 얻었습니까?` → `legality`

즉,

- `증거만 제시` = 정보 공개 + 약한 압박
- `증거 + 적절한 질문` = 벡터 봉쇄 + lie state 압박
- `모든 벡터 봉쇄` = 방어선 붕괴(자동 효과)

### 1-C. 증거별 특화 질문을 사전 정의

이건 효과적이다. 일반 문구 대신 **증거별 2~3개 challenge prompt**를 카드 자체에 붙여라.

예:

#### 자금 흐름 카드 (A에게)
- `수취인 정체를 지금도 숨기시겠습니까?`
- `사전 상의 없이 보낸 사실 자체는 인정하십니까?`
- `이 송금 직전 통화 상대가 누구였습니까?`

#### 잘린 대화/장소 카드 (A에게)
- `이 문장의 앞뒤 맥락을 설명하시겠습니까?`
- `만난 장소가 정말 사적인 만남 장소였습니까?`
- `최민정이 어떤 사람인지 분명히 말씀하시겠습니까?`

#### 같은 카드 (B에게)
- `이 캡처가 잘린 자료라는 점을 알고도 결론을 내리셨습니까?`
- `이 자료를 확보한 방식이 정당했다고 보십니까?`
- `이 문장 하나로 외도를 단정한 이유가 무엇입니까?`

즉 **같은 증거라도 대상별 질문 세트가 다르게** 붙어야 한다.

### 1-D. both 증거는 “대상별 1회 사용” 허용

`subjectParty: both` 또는 shared dispute 관련 증거는 **A에게 1회, B에게 1회** 별도 사용 가능하게 두는 것이 좋다.

이때 같은 카드라도:
- A에게 쓰면 `d-3` 해명 압박
- B에게 쓰면 `d-2` 정당화 압박
처럼 효과가 달라져야 한다.

권장 규칙:
- 증거 카드는 **대상별 1회 주요 제시** 가능
- 같은 대상에게 재제시는 조사 업그레이드가 있을 때만 허용

### 1-E. 효과는 반드시 시각화

증거를 내면 아래 셋 중 하나는 반드시 보여야 한다.

1. **방어선 아이콘 파손**
   - 정체 / 맥락 / 취득경위 / 진위
2. **봉쇄된 주장 칩**
   - 예: `외도 자금 주장 봉쇄`
   - 예: `조작 주장 봉쇄`
3. **강제 반응 라벨**
   - `다음 답변은 양보/침묵 위주`

증거가 먹힌 턴의 목표는 “좋은 대사”가 아니라
**상대가 이제 무엇으로 못 버티는지 보이게 하는 것**이다.

### 근거
- 현재 테스트에서는 증거를 제시해도 NPC 반응이 크게 달라지지 않고, 조사 성공이 플레이어에게 보이지 않는다는 문제가 직접 보고되었다.
- 기존 EvidenceChallenge 구조는 이미 attackVectors / resolvedBy / blockedClaims / full collapse를 갖고 있어, 시스템 갈아엎기보다 **플레이어 표면 단위만 재구성**하면 된다.

### 주의점
- 자동 lie state 전이는 **full collapse**나 hard proof 수준에서만 허용하고, 기본적으로는 `증거 + 맞는 질문` 조합으로 진전되게 해야 전략이 산다.
- 증거 카드를 너무 많이 남기면 여전히 선택 피로가 생긴다. “원본 6개 유지, 플레이 단위 3~4개”를 지켜라.
- 질문 선택지를 4개 이상 주면 다시 UI가 붐빈다. 카드당 2~3개가 적절하다.

## 2. 심문 하위 액션 재검토

### 결론

**3종은 유지**하는 쪽을 권한다. 다만 지금처럼 “톤 차이”가 아니라 **서로 다른 게임 효과를 가진 기술**로 바꿔야 한다.

Ace Attorney의 Press + Present가 강한 이유는 “누르면 무엇이 일어나는지”가 분명해서다. Solomon도 같은 원칙을 가져가되, 감정/관계라는 차별점을 버릴 필요는 없다.

### 설계안

하위 액션 3종을 아래처럼 재정의한다.

#### 2-A. 사실추궁 (`fact_pursuit`)
**목적**: 행위, 시점, 순서, 기준을 고정한다.

**주 효과**
- `act / timeline / rule` atom 우선 추출
- `contradiction token` 생성 확률 상승
- `timeline hole` 또는 `admission gap` 발견 시 crack 보너스
- deny/hedge 상대로 강함

**보이는 결과**
- `모순 후보 발견`
- `행위 인정 확보`
- `시점 고정`

#### 2-B. 동기탐색 (`motive_search`)
**목적**: 왜 숨겼는지, 왜 미뤘는지, 어떤 합리화를 하는지 꺼낸다.

**주 효과**
- `motive / self_justification / responsibility` atom 우선
- `emotionalLeak meter` 상승
- 숨겨진 쟁점 hook 생성
- partial/blame 상대로 강함

**보이는 결과**
- `속내 단서 확보`
- `새 쟁점 단서`
- `누설 위험 상승`

#### 2-C. 공감접근 (`empathy_approach`)
**목적**: 두려움, 수치심, 억울함을 건드려 자발적 열림을 유도한다.

**주 효과**
- `emotion / fear / shame / relationship` atom 우선
- trust 상승
- `confession window` 개방
- explosive/shutdown 또는 고수치 shame 캐릭터에 강함

**보이는 결과**
- `신뢰 상승`
- `감정 개방`
- `고백 경로 열림`

### 2-D. 각 액션의 약점도 분명히 둔다

효과만 강하면 아무거나 누르는 게임이 된다. 약점도 보여줘야 전략이 생긴다.

- 사실추궁은 감정적으로 무너지는 상대에겐 효율이 낮다.
- 동기탐색은 trust가 낮을 때 counterattack을 유발하기 쉽다.
- 공감접근은 hard proof 없이 차가운 캐릭터에게 거의 먹히지 않는다.

즉 플레이어는 “무슨 상황에서 이걸 골라야 하는가”를 읽어야 한다.

### 2-E. UI는 효과 중심으로 바꾼다

버튼 이름만으로는 부족하다. 버튼 아래 **예상 결과 아이콘**을 붙여라.

예:
- 사실추궁 → `모순` `시점 고정`
- 동기탐색 → `속내` `누설 위험`
- 공감접근 → `신뢰` `고백 창구`

그리고 현재 상태 기준으로 `효과 높음 / 보통 / 낮음` 힌트를 주면 더 좋다.

예:
- 상대가 hedge일 때: 사실추궁 `효과 높음`
- shame가 높고 S3일 때: 공감접근 `효과 높음`
- evidence collapse 직후: 동기탐색 또는 공감접근 `효과 높음`

### 2-F. 그래도 복잡하면 “표면 2개 + 조건부 1개”로 축소

현재 구조를 최대한 살리려면 3개 유지가 맞지만,
플레이어 테스트에서 여전히 혼란이 크면 표면적으로는:
- 기본 질문 2개 표시
- 현재 맥락에서 유효한 1개만 추가 노출
으로 축소할 수 있다.

즉 내부 엔진은 3종을 유지하고,
UI만 상황 맞춤형으로 좁혀 보여주는 방식이다.

### 근거
- 기존 V2 구조는 이미 subAction별 atom 선택 규칙과 slot 선택 규칙을 갖고 있다. 따라서 완전 폐기보다 **효과 정의와 피드백을 강화하는 편이 구현 대비 효율이 높다.**
- 테스트 문서에서도 문제는 “3종이 존재해서”가 아니라 “결과 차이가 안 보인다”는 점으로 정리돼 있다.

### 주의점
- 동일한 progress를 3종 모두에서 똑같이 얻으면 다시 무의미해진다.
- button copy가 추상적이면 전략이 안 보인다. 효과 중심 라벨/아이콘을 반드시 붙여라.
- `evidence_present`는 별도 행동으로 유지해야 한다. 질문 3종에 흡수하면 증거의 손맛이 약해진다.

## 3. 돌파 순간 연출

### 핵심 원칙

**연출은 짧아야 하고, 변화는 길게 남아야 한다.**

즉 화면 흔들림 자체보다 중요한 건,
그 이후에 “이제 상대가 전처럼 못 버틴다”가 플레이어에게 보이는 것이다.

### 설계안

lie state 자체는 내부값으로 두되, 플레이어에게는 아래 5개 의미 상태로 번역해 보여라.

- `경계 중` = S0~S1
- `균열 발생` = S2
- `궁지` = S3
- `개방` = S4
- `시인` = S5

### 3-A. 전이별 연출 패키지

#### S1 → S2 (`균열`)
- 짧은 화면 노크/금 가는 효과
- 쟁점 카드에 crack 아이콘
- 배너: `방어 균열` 또는 `핵심 사실 일부 인정`
- **기계적 변화 표시**: 새 질문 루트 1개 열림 / 모순 토큰 생성 / 다음 증거 효과 상승

#### S2 → S3 (`궁지`)
- 방어선 아이콘 1개 이상 붕괴
- NPC hint가 더 불안정하게 바뀜
- 배너: `버티기 어려워짐`
- **기계적 변화 표시**: 다음 적중 행동 +1 압박 / counter-call 가능

#### S3 → S4 (`개방`)
- 짧은 정적/호흡 연출
- 공감/동기 버튼 강조
- 배너: `속내가 드러나기 시작함`
- **기계적 변화 표시**: confession window open

#### S4 → S5 (`시인`)
- BeatScript 기반 고백 컷인
- 쟁점 카드 resolved 전환
- 배너: `핵심 사실 공개`
- **기계적 변화 표시**: readiness 크게 상승, 판결 단서 갱신

### 3-B. 방어선 붕괴는 lie state와 별개로 연출

증거의 full collapse는 별도 쾌감이다. 이건 꼭 분리해서 보여야 한다.

- 배너: `방어선 붕괴`
- 어떤 벡터가 사라졌는지 표시
  - `맥락 방어 붕괴`
  - `정체 숨기기 불가`
  - `취득경위 공격 무력화`
- 다음 턴 효과까지 같이 띄움
  - `다음 응답: 양보/침묵 위주`

### 3-C. 연출보다 더 중요한 것은 “다음 행동 추천 변화”

돌파 뒤에는 꼭 버튼 강조가 바뀌어야 한다.

예:
- 균열 직후 → `동기탐색 효과 높음`
- full collapse 직후 → `증거 재제시보다 공감/추궁이 유효`
- 개방 직후 → `지금 공감접근하면 고백 가능`

이게 없으면 연출만 화려하고 손맛은 없다.

### 근거
- 현재 테스트 문제는 state transition 자체가 없는 게 아니라, **플레이어가 그 전이를 인지하지 못한다**는 데 있다.
- 엔진은 이미 S1→S5 진행을 만들 수 있으므로, 다음 단계는 상태를 플레이어가 읽을 수 있는 언어로 번역하는 것이다.

### 주의점
- 연출은 0.5~0.8초 이내, 스킵 가능하게 두는 것이 좋다.
- state 전이를 숫자(S2, S3)로 그대로 보여주면 지나치게 시스템적으로 느껴질 수 있다.
- 같은 연출을 너무 자주 쓰면 금방 무감각해진다. crack / collapse / confession의 3단계 감도를 구분하라.

## 4. 턴 간 이벤트 부활

### 설계 방향

이벤트는 랜덤 장식이 아니라, **플레이어가 개입할 수 있는 반응 창구**여야 한다.

권장하는 이벤트 4종은 아래다.

### 4-A. 모순 지적 (`contradiction_callout`)

#### 트리거
- 현재 답변의 atom/slot이 과거 답변과 충돌
- 이미 인정한 atom과 새 답변이 충돌
- timeline 또는 beneficiary가 바뀜

#### 흐름
1. 시스템이 `모순 토큰` 생성
2. 다음 턴 시작 시 `모순 지적` 선택지 등장
3. 성공 시:
   - lie state +1 또는
   - defenseMode 강등 또는
   - blockedClaim 1개 추가
4. 실패 시:
   - 턴만 소비하거나
   - trust 소폭 하락

즉 모순은 자동 처리하지 말고,
**플레이어가 잡아야 하는 득점 기회**로 만들어야 한다.

### 4-B. 끼어들기 (`interruption`)

#### 트리거
- 비심문 당사자가 현재 쟁점과 높은 관련성을 갖고 있음
- 감정 tier가 agitated/explosive
- hidden dispute hook를 쥐고 있음

#### 선택지
- `허용한다`
- `제지한다`

#### 결과
- 허용:
  - 새 정보 / 새 쟁점 hook 획득
  - 그러나 흐름 혼잡, focus 분산
- 제지:
  - 재판 권위 유지
  - 현재 쟁점 압박 보너스
  - 다만 새 hook는 늦게 얻음

즉 끼어들기는 “방해”가 아니라 **정보와 통제의 교환**이 되어야 한다.

### 4-C. 감정 폭발 (`emotional_surge`)

#### 트리거
- emotionalLeak meter 임계치 도달
- evidence collapse 직후
- shame/fear 기반 캐릭터가 S3 이상

#### 선택지
- `더 압박한다`
- `진정시킨다`

#### 결과
- 더 압박:
  - 즉시 누설/고백 확률 증가
  - trust 하락
  - 역공 가능성도 존재
- 진정:
  - trust 상승
  - S4 진입 / confession window 개방 확률 상승

### 4-D. 새 쟁점 발현 (`hidden_dispute_reveal`)

#### 트리거
- blockedClaims가 unopened dispute와 연결
- motive_search에서 hidden hook 누적
- contradiction + evidence hit 조합 성립

#### 선택지
- `지금 추적`
- `핀으로 보류`

#### 결과
- 지금 추적: focus 전환, 즉시 큰 진전 가능
- 보류: 현재 쟁점 유지, 나중에 확정 보너스

### 4-E. 발생 빈도 규칙

권장 세션당 이벤트 리듬:
- **보장 이벤트 2개**
  - 중반 crack 이후 1개
  - 후반 collapse/open 이후 1개
- **조건부 이벤트 1~2개**
  - contradiction / interruption / hidden reveal 중 성립 시

하드 규칙:
- 일반적으로 **연속 턴 major event 금지**
- 단, 클라이맥스 구간(S4~S5 또는 full collapse 직후)만 예외

### 4-F. 이벤트 우선순위

한 턴에 여러 조건이 겹치면 아래 순으로 처리:

1. `contradiction_callout`
2. `evidence_collapse_reaction`
3. `hidden_dispute_reveal`
4. `interruption`
5. `emotional_surge`

이 순서가 좋은 이유는,
플레이어가 직접 잡아낸 성과를 가장 먼저 보상하기 때문이다.

### 근거
- 현재 단조로움은 “내용이 반복된다”기보다 “모든 턴의 리듬이 같다”는 문제다.
- 기존 시스템에도 모순 감지와 끼어들기 같은 재미 요소가 있었으므로, 완전 신규보다는 **V2 atom/slot/trust 구조에 다시 꽂아 넣는 방식**이 리스크가 낮다.

### 주의점
- 이벤트는 랜덤보다 **조건 기반**으로 발생해야 억울하지 않다.
- major event는 세션당 2~4개 정도가 적절하다. 그 이상은 시끄럽다.
- 끼어들기를 너무 자주 띄우면 focus dispute가 흐려진다.

## 5. 품질 안정 — 새 정보 해금과 반복 심문 깊이 강제

### 핵심 결론

**상태 전이 = 새 정보 해금**이 되어야 한다.

지금 V2 atom이 품질을 올렸다면, 다음 단계는 각 state가 “무슨 새 사실을 열어 주는지”를 플레이어와 시스템 모두에게 분명히 만드는 것이다.

### 설계안

### 5-A. state를 정보 계층으로 나눈다

각 dispute를 최소 4계층으로 본다.

- **Surface**: S0~S1
  - 사실 부정/유보/기본 방어
- **Structural**: S2~S3
  - 행위 인정, 동기 일부, 책임 전가 구조
- **Core**: S4
  - 숨긴 이유, 감정, 실질 의미
- **Confession**: S5
  - 핵심 사실, 자기식 해석, 관계 감정

즉 새 state에 가면 **그 state에서만 열리는 atom**이 반드시 있어야 한다.

이미 state별 claimAtoms를 만들어 두었으므로,
이를 단순 풀셋으로 쓰지 말고 **새로 열린 atom**을 우선 소진하게 만들어라.

### 5-B. 전이 순간에는 BeatScript를 강제 삽입

pivot turn은 LLM 자유발화보다 사전 생성 beat가 안정적이다.

권장 매핑:
- S1→S2: `evidence_hit` 또는 `partial` beat
- S2→S3: `blame` 또는 `evidence_hit` beat
- S3→S4: `emotional` 계열 전이용 beat
- S4→S5: `confession` beat
- contradiction_callout 성공 시: `partial/blame` 전용 beat

즉 BeatScript는 fallback이 아니라 **전환 앵커**로 써야 한다.

### 5-C. 반복 심문에는 “새 정보 없음”을 정직하게 보여준다

같은 쟁점에 같은 접근을 새 leverage 없이 반복하면,
엔진이 억지로 새로운 말을 만들지 않게 해야 한다.

권장 규칙:
- 같은 dispute + 같은 subAction + 새 증거/이벤트/전이 없음
  - 첫 반복: 답변 짧아짐 + `새 정보 거의 없음`
  - 두 번째 반복: `교착` 판정 + 다른 접근 유도

플레이어에게는
- `이 경로는 더 캐낼 정보가 없습니다`
- `증거나 다른 접근이 필요합니다`
같은 메시지를 짧게 준다.

이건 재미를 해치는 게 아니라,
**불필요한 반복을 끊고 전략 전환을 유도하는 품질 장치**다.

### 5-D. freshness budget를 둔다

각 dispute마다 현재 state에서 새롭게 제공할 수 있는 atom 수를 1~2개로 제한하고,
이를 소모하면 다음 단계로 가거나 다른 루트를 써야 한다.

예:
- S1에서 fact_pursuit로 1개
- S1에서 motive_search로 1개
- 그 뒤에는 evidence / empathy / contradiction 없이는 더 깊은 새 정보 없음

즉 “무한 변주 응답”이 아니라
**유한한 깊이를 가진 조사 층**처럼 느껴져야 한다.

### 5-E. 엔진 차원에서 중복 억제

이미 V2 draft에 있는 `recentlyUsedAtomIds`, `recentlyUsedSlotPaths`, `recentlyUsedOpeners`를 더 공격적으로 활용해라.

추가 권장:
- 같은 atom 2턴 내 재사용 금지
- 같은 slot family exact surface 연속 사용 금지 (unless mustUseTell)
- 같은 dispute에서 같은 opener 1회차 이후 금지
- same-state repeat 시 `bannedAtomIds += exhaustedAtoms`

### 근거
- 테스트 문서가 지적한 품질 문제는 더 이상 “문장이 이상하다”가 아니라, **상태가 진행돼도 깊이 차이가 체감되지 않는다**는 점이다.
- V2 atom 시스템은 이미 상태별 데이터 분리가 가능하므로, 새 설계는 기존 구조 위에 `unlock / exhaustion / transition beat`만 얹으면 된다.

### 주의점
- 새 정보 해금이 너무 많으면 12~16턴 안에 사건이 과잉 노출된다.
- 반대로 너무 적으면 다시 단조롭다. state당 체감 신정보 1~2개가 적당하다.
- 교착 피드백은 솔직해야 하지만 처벌처럼 느껴지면 안 된다. “다른 길을 시도하라”는 힌트와 함께 줘라.

## 최종 우선순위

지금 가장 먼저 손대야 할 순서는 아래다.

1. **증거를 3~4개 플레이 카드로 재구성하고, 카드마다 대상별 challenge 질문 2~3개를 붙인다.**
2. **질문 3종에 각기 다른 게임 효과(모순/누설/신뢰)를 부여하고 UI에 효과 아이콘을 붙인다.**
3. **state 전이와 evidence collapse를 player-facing 이벤트로 번역한다.**
4. **모순 지적 / 끼어들기 / 감정 폭발 / 새 쟁점 발현을 조건 기반 이벤트로 되살린다.**
5. **state 전이 = 새 정보 해금 규칙을 강제하고, 반복 심문에는 교착 피드백을 준다.**

한 문장으로 요약하면,

> **지금 Solomon에 필요한 것은 “더 좋은 대사”가 아니라, 매 턴 결과가 눈에 보이고 다음 수가 달라지는 심문 게임 구조”다.**
