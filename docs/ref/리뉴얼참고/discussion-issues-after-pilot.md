# 파일럿 테스트 후 발견된 구조적 문제 — GPT 논의 요청

## 배경

Phase A(엔진 구조 전환) + Session 1~2(spouse-01 데이터) 통합이 완료되어 실제 플레이 테스트를 진행했습니다. Blueprint 경로가 활성화된 상태에서 아래 문제들이 발견됐습니다.

**현재 작동 흐름**:
1. 케이스 로드 → ClaimPolicy/Bridge/EvidenceChallenge/Tell 데이터 등록 ✅
2. Phase 3 진입 → 브리지 적용 (d-1은 S1로 시작 등) ✅
3. 심문 시 → `tryBlueprintPath` 활성 → `generateBlueprint` → 경량 프롬프트로 LLM 호출 ✅

**하지만 응답 품질에 여전히 심각한 문제가 있습니다.**

## 문제 1: stance와 defenseMode의 의미 충돌

### 현상
한지석에게 d-1(280만원 송금)에 대해 사실추궁 → 응답: **"그런 적 없습니다. 송금한 것은 맞지만, 그 돈의 용도에 대해서는 설명할 순서가 있습니다."**

같은 응답 안에서 "그런 적 없습니다"(완전 부정) + "송금한 것은 맞지만"(부분 인정)이 공존.

### 원인 분석
브리지에 의해 d-1은 S1로 시작. Blueprint 매트릭스에서:
- `baseStance('S1', trust)` = **'hedge'** (얼버무리기)
- `defenseMode` = **'flat_denial'** (그런 적 없습니다)

→ **stance는 '얼버무리기'인데 defenseMode는 '완전 부정'** — 모순되는 지시를 받은 LLM이 둘 다 시도하면서 자기모순 응답 생성.

### 해결 필요
**hedge 상태에서 defenseMode가 flat_denial이면 안 됩니다.** hedge에 맞는 defenseMode가 필요합니다.

제안: hedge의 기본 defenseMode를 `deflection`(화제 전환) 또는 `vague_denial`(애매한 부정)로 바꾸거나, 기존 모드 중 `silence`(최소 답변)로 매핑?

## 문제 2: 재판관 질문이 하위 액션(사실추궁/동기탐색/공감접근)을 반영하지 않음

### 현상
"동기 탐색"을 선택했는데 재판관 질문: "새벽 휴대폰 열람 건에서 그런 판단을 내린 이유가 무엇입니까?"
→ 이건 사실 확인에 가까움. 동기 탐색이라면 "새벽에 한지석 씨의 휴대폰을 열람하게 된 이유가 뭡니까?"처럼 왜 그런 행동을 했는지를 물어야 함.

### 원인 분석
현재 재판관 질문 생성은 `buildJudgeQuestion()` 함수가 담당하는데, 하위 액션별 차이가 미미합니다. Blueprint 프롬프트에서도 재판관 질문 생성을 LLM에게 맡기고 있어서, LLM이 하위 액션 의도를 제대로 반영 못 함.

### 해결 필요
**재판관 질문을 LLM이 생성하는 것 자체가 문제인가?**
- A안: 재판관 질문도 엔진이 템플릿으로 생성 (가장 안정적)
- B안: LLM에게 맡기되, 하위 액션별 질문 구조를 더 강하게 강제
- C안: 하위 액션 자체를 재설계 (아래 문제 3과 연결)

## 문제 3: 심문 하위 액션(사실추궁/동기탐색/공감접근)의 실효성

### 현상
세 가지 하위 액션을 골라도 NPC 응답이 거의 동일하게 나옴. 플레이어 입장에서:
- "사실추궁"으로 물어도 감정적 응답이 옴
- "동기탐색"으로 물어도 사실 부정부터 함
- "공감접근"으로 물어도 차이를 못 느낌

### 원인 분석
Blueprint 매트릭스에서 `questionType`은 stance를 보정하는 데만 사용:
- fact_pursuit: 변화 없음
- motive_search: soften (한 단계 완화)
- empathy_approach: trust ≥ 50이면 soften

→ **stance 보정은 미묘한 차이일 뿐, NPC가 "무엇에 대해 대답하는지"를 바꾸지 않습니다.** 질문 유형이 달라지면 NPC가 답하는 내용의 초점이 달라져야 하는데, 현재는 같은 publicClaim을 동일하게 사용.

### 해결 필요 — 구조적 질문
1. **하위 액션을 유지할 것인가?**
   - 유지한다면: 각 액션이 NPC 응답의 어떤 차원을 바꾸는지 명확히 정의 필요
   - 제거한다면: 플레이어는 "질문하기" + 쟁점 선택만. 엔진이 심문 깊이에 따라 자동 조절

2. **유지한다면 어떻게 차별화할 것인가?**
   - A안: 각 액션이 publicClaim의 다른 부분을 꺼내게 함 (사실추궁→행위 관련 claim, 동기탐색→동기 관련 claim, 공감→감정 관련 claim)
   - B안: 각 액션이 다른 Beat를 트리거하도록 매핑
   - C안: 각 액션이 stance가 아니라 **NPC의 응답 초점**을 바꿈 (사실추궁→"무엇을 했는지", 동기탐색→"왜 했는지", 공감→"어떤 심정이었는지")

3. **제거한다면 대체 구조는?**
   - 플레이어는 쟁점만 선택 → 엔진이 심문 횟수/이전 질문/증거 상태를 보고 자동으로 최적 접근 결정
   - 이러면 플레이어의 전략적 선택이 줄어드는 단점

## 문제 4: Phase 1에서 이미 나온 사실을 Phase 3에서 재부정하는 모순

### 현상
Phase 1에서 한지석: "공동통장에서 돈이 나간 건 사실입니다" (수정 후에도 인정)
Phase 3에서 같은 쟁점 심문 → "그런 적 없습니다"

오세린도 마찬가지: Phase 1에서 "정황이 있었으니까 확인한 거지" (열람 사실 함축)
Phase 3에서 → 열람 자체를 부정할 가능성

### 원인 분석
Bridge가 S1로 올려놓지만, S1의 ClaimPolicy.publicClaim이 "이미 인정한 사실"과 충돌:
- S1 publicClaim: "공동계좌에서 2,800,000원을 보낸 건 제가 맞습니다" (인정)
- 그런데 defenseMode가 flat_denial: "그런 적 없습니다" (부정)
- → 문제 1과 동일한 원인

### 해결 필요
Bridge의 `alreadyPubliclyAdmitted`를 프롬프트에 명시적으로 전달하여, LLM이 이미 인정한 사실을 재부정하지 못하게 해야 합니다. 현재 blueprintPromptBuilder에 이 필드가 포함되어 있지만, LLM이 무시하고 있을 가능성.

추가로: **S1 상태에서 "행위 자체는 인정, 의도/용도만 숨기기"가 기본 자세여야 하는데**, 현재 defenseMode가 flat_denial이라 전면 부정을 하고 있음.

## 문제 5: 사건 개요에서 모든 쟁점이 노출됨

### 현상
사건 시작 전 Phase 0에서 "확인된 쟁점 4개"가 전부 보여짐:
1. 지석의 비밀 송금 280만원 [핵심]
2. 세린의 새벽 휴대폰 열람 [핵심]
3. 세린의 우회 송금 150만원 [핵심]
4. 100만원 사전 상의 약속 위반 [중요]

### 문제
- 플레이어가 심문 시작 전에 이미 전체 구조를 파악 → 추리/발견의 재미 없음
- Discovery 시스템(숨겨진 쟁점 발현)이 무의미해짐
- "심문을 통해 진실을 밝힌다"는 핵심 판타지 훼손

### 해결 필요
- 초기에 1~2개 핵심 쟁점만 노출하고, 나머지는 심문 과정에서 발견되게?
- 쟁점 이름 자체도 "비밀 송금 280만원"처럼 답을 주는 게 아니라 "불분명한 송금 내역"처럼 중립적으로?

---

## 정리: GPT 의견이 필요한 5가지

1. **stance-defenseMode 매핑 보정**: hedge + flat_denial 조합이 모순. hedge의 기본 defenseMode를 무엇으로 바꿀 것인가?

2. **재판관 질문 생성 방식**: LLM 생성 vs 엔진 템플릿 vs 하이브리드?

3. **심문 하위 액션(사실추궁/동기탐색/공감접근) 재설계**: 유지/제거/재정의 중 어느 방향? 유지한다면 차별화 방법은?

4. **Phase 1에서 인정한 사실의 Phase 3 연결 강화**: 현재 Bridge + alreadyPubliclyAdmitted 구조가 충분한가? 추가 장치가 필요한가?

5. **사건 개요 쟁점 노출 범위**: 초기 노출 쟁점 수와 이름 표현 방식?
