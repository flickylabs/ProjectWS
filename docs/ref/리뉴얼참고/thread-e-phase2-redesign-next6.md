## spouse-05 — 잠긴 방의 31분

### 사건 훅
대형 광고주 최종 PT 30분 전, 잠겨 있던 서재 문이 열리고 회의용 SSD가 사라진다.  
곧 재생된 녹취에는 "회의 망쳐버릴까"라는 아내 목소리가 들리고, 남편은 배우자가 일부러 일을 날리려 서재에 침입했다고 주장한다.  
하지만 추궁이 깊어질수록, 남편은 서로 다른 날짜의 통화를 이어붙여 범행 서사를 만들었고 아내 역시 구 관리자 코드를 지우지 않고 서재에 몰래 들어가 자료를 확보하려 했다는 사실이 드러난다.

### 양측 프로필
- A: 강민재, 프리랜서 광고영상 감독, 숨긴 것=`서재를 사실상 개인 금고처럼 전용하고 소담의 번역 자산을 무단 참고했으며, 편집 녹취로 배우자를 업무방해범처럼 몰아간 점`, 거짓말 전략=`cold_logic`
- B: 이소담, 특허번역사, 숨긴 것=`구 관리자 코드로 서재에 들어가 자료를 확인하고 SSD를 잠시 옮겨 민재를 압박하려 한 점`, 거짓말 전략=`confrontational`

### 쟁점 구조
- d-1: 소담의 서재 무단 출입은 단순 확인이었나, 실제 방해였나
- d-2: 민재의 잠금권한 일방 변경과 서재 전용화는 정당했나
- d-3: 회의 직전 사라진 SSD는 누가 왜 옮겼나
- d-4: 민재가 제출한 녹취는 실제 위협 발언을 증명하는가
- d-5: 부부의 공동 공간과 직업 자산 경계는 어디서 무너졌나

### 증거 설계
- 초기 공개: e-1 스마트락 출입기록, e-2 31초 녹취 파일, e-3 광고주 최종 PT 일정표
- 1차 해금(S1+): e-4 녹취 원본 파형 비교
  - requires: `e-2`
- 2차 해금(S2+): e-5 공유 드라이브 버전 이력 + 소담 번역 용어집 접근 흔적
  - requires: `e-1`
- 3차 해금(S2+): e-6 SSD 복구 로그와 이동 시각 기록
  - requires: `e-3`
- 최종 해금(S4+): e-7 "먼저 겁을 줘야 말이 통한다" 부부 메신저 대화
  - requires: `e-4`, `e-6`
- 증거 조합:
  - `e-2 + e-4` → 녹취가 두 날짜 음성을 잇댄 편집본임이 드러난다
  - `e-1 + e-6` → 소담이 실제로 SSD를 건드린 사실이 열린다
  - `e-5 + e-7` → 민재의 통제와 소담의 잠입이 같은 불신 구조에서 나왔음이 드러난다

### 거짓말 상태 전이 설계
- A / d-4:
  - S0→S1: `direct_question`으로 "강한 말은 있었지만 취지는 같았다" 인정
  - S1→S2: `e-4` 제시 시 "앞뒤 정리만 한 것"이라고 핑계
  - S2→S3: `responsibility_question`으로 "먼저 들어온 쪽이 소담"이라 책임 전가
  - S3→S5: `e-7` 제시 시 녹취 연출과 서재 통제를 자백
- B / d-1:
  - S0→S1: `direct_question`으로 "들어간 적은 있다" 인정
  - S1→S2: `e-1` 제시 시 "내 자료가 쓰인 것 같아 확인한 것" 핑계
  - S2→S4: `emotion_threshold`로 억눌린 분노 폭발
  - S4→S5: `e-6`, `e-7` 제시 시 SSD 이동과 압박 의도를 자백

### 핵심 모순 쌍
- A는 "위협 발언 증거"라고 하지만 `e-2 + e-4`는 편집 서사를 보여준다
- B는 "확인만 했다"고 하지만 `e-1 + e-6`은 실제 물리적 개입을 보여준다
- A는 "보안상 어쩔 수 없었다"고 하지만 `e-5`는 이미 공동 자산을 사적으로 잠갔음을 보여준다

### 판결 딜레마
`가정 내 보안 명분으로 배우자를 배제한 통제`와 `신뢰 붕괴 뒤 구 코드로 침입해 자료를 옮긴 침범` 중 무엇을 더 무겁게 볼지 갈린다.  
과정 중시 재판관은 편집 녹취와 잠금권한 남용을 더 크게 볼 수 있고, 해결 중시 재판관은 부부의 작업공간과 비용 자산을 어떻게 다시 분리할지에 무게를 둘 수 있다.

## family-05 — 엄마 카드로 막고

### 사건 훅
늦은 밤 딸 방에서 이삿짐 상자가 나오고, "엄마 카드로 막고 그냥 나갈게"라는 31초 음성 클립이 재생된다.  
어머니는 이미 임대인에게 "딸은 계약 능력이 부족하다"고 말해 둔 상태였고, 딸은 공유주거 계약을 단순 집구경 정도로 축소 설명해 왔다.  
하지만 후반에 드러나는 건, 31초 클립은 세 통화를 잘라붙인 위조본이고 딸도 실제 보증 구조와 카드 결제 문제를 숨긴 채 독립을 밀어붙였다는 사실이다.

### 양측 프로필
- A: 박은주, 동네 미용실 원장, 숨긴 것=`현관 비밀번호와 임대인 연락망을 이용해 딸의 계약을 무력화하고, 위조 녹취로 여론을 자기 편으로 만든 점`, 거짓말 전략=`victim_cosplay`
- B: 박소현, 웹툰 배경 채색 어시스턴트, 숨긴 것=`비환불 홀드비를 어머니 카드 정보로 결제하려 했고 공유주거 계약 리스크를 끝까지 축소 설명한 점`, 거짓말 전략=`avoidant`

### 쟁점 구조
- d-1: 은주의 딸 방 출입과 임대인 개입은 어디까지 계획적이었나
- d-2: 소현은 공유주거 계약 조건을 얼마나 숨겼나
- d-3: 31초 녹취는 실제 자백인가, 편집된 위협 프레임인가
- d-4: 사전 상담 후 계약하기로 한 약속은 누가 먼저 깼나
- d-5: 독립을 막는 보호와 생활 통제의 경계는 어디인가

### 증거 설계
- 초기 공개: e-1 31초 음성 클립, e-2 공유주거 계약서 초안, e-3 임대인 통화내역
- 1차 해금(S1+): e-4 음성 편집 파형과 원통화 3건
  - requires: `e-1`
- 2차 해금(S2+): e-5 홀드비 카드결제 승인내역
  - requires: `e-2`
- 3차 해금(S2+): e-6 하우스메이트 단톡과 보증구조 설명 캡처
  - requires: `e-5`
- 최종 해금(S4+): e-7 은주-임대인 사전 차단 통화 녹취
  - requires: `e-3`, `d-1` S3 이상
- 증거 조합:
  - `e-1 + e-4` → 자백처럼 들리는 음성이 위조였음이 드러난다
  - `e-2 + e-5` → 소현이 계약 단계를 축소 설명했음이 드러난다
  - `e-3 + e-7` → 은주가 미리 계약을 막는 방향으로 움직였음이 확정된다

### 거짓말 상태 전이 설계
- A / d-3:
  - S0→S1: `direct_question`으로 "취지 전달용으로 정리했다" 인정
  - S1→S2: `e-4` 제시 시 "딸이 나쁜 선택할까 봐 급했다" 핑계
  - S2→S3: `responsibility_question`으로 "먼저 속인 건 소현"이라 책임 전가
  - S3→S5: `e-7` 제시 시 임대인 차단과 녹취 위조를 자백
- B / d-2:
  - S0→S1: `direct_question`으로 "계약이 거의 된 건 맞다" 인정
  - S1→S2: `e-5` 제시 시 "홀드비라 곧 돌려막을 수 있었다" 핑계
  - S2→S4: `emotion_threshold`로 독립 좌절 분노 폭발
  - S4→S5: `e-6` 제시 시 카드 사용 계획과 보증 리스크 축소를 자백

### 핵심 모순 쌍
- A는 "딸을 보호하려 했다"고 하지만 `e-3 + e-7`은 생활 통제를 보여준다
- B는 "계약 전 단계였다"고 하지만 `e-2 + e-5`는 이미 돈이 움직였음을 보여준다
- A는 "실제 발언을 요약한 것"이라 말하지만 `e-1 + e-4`는 감정 조작 편집을 보여준다

### 판결 딜레마
`전세사기 트라우마를 이유로 성인 자녀의 계약을 막는 보호`와 `가족 카드와 신뢰를 앞세워 리스크를 숨긴 독립 강행` 중 무엇이 더 무거운지 갈린다.  
논리형 재판관은 녹취 위조와 카드 사용 계획을 각자 따로 책임화할 수 있고, 해결 중시 재판관은 독립 비용과 보호 경계를 다시 어떻게 설계할지에 무게를 둘 수 있다.

## friend-07 — 운영진의 비밀

### 사건 훅
동호회 리더 선발 전날 밤, "임원 연애는 숨겨야 이득"이라는 캡처가 운영진방 밖으로 터진다.  
리더 후보였던 성호는 순식간에 연애를 숨긴 채 자리를 얻으려 한 인물로 찍히고, 진아는 자신이 공정성을 위해 폭로했다고 주장한다.  
하지만 파고들수록 그 캡처는 서로 다른 시점 메시지를 이어붙인 편집본이고, 성호 역시 작년 소문 사태 뒤 둘이 직접 만든 제한 공개 합의를 먼저 어겼다는 사실이 드러난다.

### 양측 프로필
- A: 서진아, 공연기획 어시스턴트, 숨긴 것=`연애 캡처를 편집해 내부 고발처럼 보이게 만들고 여러 채널로 확산시킨 점`, 거짓말 전략=`cold_logic`
- B: 박성호, 영어학원 강사, 숨긴 것=`부회장과의 연애를 숨긴 채 리더 선발에 들어가고 진아에게 알리바이 요청까지 한 점`, 거짓말 전략=`victim_cosplay`

### 쟁점 구조
- d-1: 성호의 비밀 연애 은닉과 알리바이 요청은 선발 공정성을 해쳤나
- d-2: 진아의 메시지 편집과 다중 채널 확산은 내부고발인가 여론전인가
- d-3: 성호는 실제로 연애 관계를 이용해 리더 선발을 유리하게 만들었나
- d-4: 작년 비밀 유지 합의는 누가 먼저 깼나
- d-5: 공동체 규칙과 개인 사생활의 경계는 어디인가

### 증거 설계
- 초기 공개: e-1 유출된 캡처 2장, e-2 리더 선발 일정표, e-3 작년 제한 공개 합의 메모
- 1차 해금(S1+): e-4 원본 채팅 export
  - requires: `e-1`
- 2차 해금(S2+): e-5 성호-진아 알리바이 요청 DM
  - requires: `e-3`
- 3차 해금(S2+): e-6 운영진 1인 고지 여부를 보여주는 캘린더/메모 기록
  - requires: `e-2`
- 최종 해금(S4+): e-7 진아의 다중 채널 배포 로그
  - requires: `e-4`, `d-2` S3 이상
- 증거 조합:
  - `e-1 + e-4` → 폭로 핵심 캡처가 편집본임이 드러난다
  - `e-3 + e-5` → 성호가 제한 공개 합의를 먼저 어기고 진아에게 알리바이를 요청했음이 드러난다
  - `e-6 + e-7` → 진아의 폭로가 규칙 보정이 아니라 여론 선점이었음이 강화된다

### 거짓말 상태 전이 설계
- A / d-2:
  - S0→S1: `direct_question`으로 "문맥을 정리한 것" 인정
  - S1→S2: `e-4` 제시 시 "핵심 의미는 같았다" 핑계
  - S2→S3: `responsibility_question`으로 "먼저 규칙을 깬 건 성호" 책임 전가
  - S3→S5: `e-7` 제시 시 여론 선점 목적의 확산을 자백
- B / d-1:
  - S0→S1: `direct_question`으로 "연애를 숨긴 건 맞다" 인정
  - S1→S2: `e-3` 제시 시 "제한 공개 규칙 안에서 해결하려 했다" 핑계
  - S2→S4: `emotion_threshold`로 사생활 침해 분노 폭발
  - S4→S5: `e-5`, `e-6` 제시 시 알리바이 요청과 선제 합의 파기를 자백

### 핵심 모순 쌍
- A는 "공정성을 위한 폭로"라 말하지만 `e-1 + e-4 + e-7`은 편집 확산을 보여준다
- B는 "사적인 일이라 숨겼다"고 하지만 `e-3 + e-5`는 규칙 위반과 알리바이 요청을 보여준다
- B는 "운영진에게 이미 알렸다"고 하지만 `e-6`은 고지가 불완전했음을 드러낸다

### 판결 딜레마
`이해충돌을 숨기고 리더 선발에 들어간 행위`와 `공정성 명분으로 편집 증거를 퍼뜨린 행위` 중 무엇이 더 공동체를 망가뜨렸는지 갈린다.  
직관형 재판관은 연애 은닉의 배신감을 더 크게 볼 수 있고, 과정 중시 재판관은 편집 증거 유포를 더 엄격히 볼 수 있다.

## neighbor-03 — 19초짜리 물림

### 사건 훅
피 묻은 귀, 소형견의 비명, 커뮤니티에 퍼진 19초 클립 하나로 현우의 대형견은 순식간에 "물어뜯는 개"가 된다.  
서아는 치료비와 공개 사과를 요구하고, 현우는 자신의 개가 살인견 취급을 받았다며 맞선다.  
하지만 원본을 열면 상처는 물림이 아니라 자동줄이 철망에 감기며 생긴 마찰 열상이고, 서아는 예약 없이 입실해 직원 PIN까지 무단 사용했으며 현우 역시 앞 8초를 잘라 공포가 더 큰 클립을 유포했다는 사실이 드러난다.

### 양측 프로필
- A: 강현우, 헬스장 야간매니저, 숨긴 것=`원본 CCTV를 자르고 속도를 바꿔 자신의 개가 덜 위험해 보이도록 반대편 프레임을 만든 점`, 거짓말 전략=`cold_logic`
- B: 진서아, 반려견 유치원 콘텐츠 판매자, 숨긴 것=`무예약 입실과 자동줄 방치를 숨기고, 다친 개를 앞세워 현우를 일방 가해자로 만든 점`, 거짓말 전략=`victim_cosplay`

### 쟁점 구조
- d-1: 서아의 무예약 입실과 직원 PIN 무단 사용은 어느 정도였나
- d-2: 자동줄 방치가 상처 원인과 얼마나 연결되나
- d-3: 19초 CCTV 클립은 원본인가, 편집본인가
- d-4: 상처 원인은 물림인가, 마찰 열상인가
- d-5: 다친 동물에 대한 공감과 증거 조작 책임을 어떻게 분리할 것인가

### 증거 설계
- 초기 공개: e-1 19초 클립, e-2 동물병원 초진 소견서, e-3 출입 예약표
- 1차 해금(S1+): e-4 원본 CCTV 45초
  - requires: `e-1`
- 2차 해금(S2+): e-5 직원 PIN 출입로그
  - requires: `e-3`
- 3차 해금(S2+): e-6 자동줄 파손 사진 + 철망 위치도
  - requires: `e-2`
- 최종 해금(S4+): e-7 현우의 속도조절 편집 파일 + 서아의 커뮤니티 업로드 초안
  - requires: `e-4`, `e-5`
- 증거 조합:
  - `e-1 + e-4` → 19초 클립이 앞부분을 삭제한 편집본임이 드러난다
  - `e-2 + e-6` → 상처가 물림보다 마찰 열상 가능성이 높다는 점이 강화된다
  - `e-5 + e-7` → 서아의 규칙 위반과 현우의 편집 대응이 각각 별도 책임으로 열린다

### 거짓말 상태 전이 설계
- A / d-3:
  - S0→S1: `direct_question`으로 "짧게 잘라 보낸 건 맞다" 인정
  - S1→S2: `e-4` 제시 시 "핵심 장면만 보낸 것" 핑계
  - S2→S3: `responsibility_question`으로 "먼저 무단 입실한 건 서아" 책임 전가
  - S3→S5: `e-7` 제시 시 공포 프레임을 의도한 편집을 자백
- B / d-1:
  - S0→S1: `direct_question`으로 "급해서 예약 없이 들어간 적은 있다" 인정
  - S1→S2: `e-5` 제시 시 "직원이 늘 그렇게 하라 했다" 핑계
  - S2→S4: `emotion_threshold`로 반려견 상처 분노 폭발
  - S4→S5: `e-6`, `e-7` 제시 시 자동줄 방치와 커뮤니티 선제 확산을 자백

### 핵심 모순 쌍
- B는 "현우 개가 물었다"고 하지만 `e-2 + e-6`은 마찰 열상 가능성을 보여준다
- A는 "원본 그대로 보냈다"고 하지만 `e-1 + e-4 + e-7`은 편집을 보여준다
- B는 "절차는 지켰다"고 하지만 `e-3 + e-5`는 무예약/PIN 무단 사용을 보여준다

### 판결 딜레마
`다친 동물을 앞세운 공감의 힘`을 얼마나 참작할지와 `그 공감을 위해 사실을 밀어붙인 왜곡`을 얼마나 제재할지가 갈린다.  
공감형 재판관은 서아 쪽 절박함에 흔들릴 수 있지만, 논리형 재판관은 원본 편집과 규칙 위반을 별도로 볼 가능성이 크다.

## partnership-04 — 같은 배신, 다른 증거

### 사건 훅
토요일 새벽 2시, 은행 도장처럼 보이는 PDF 세 장이 공동대표 협상방에 올라오고 "역시 또 네가 해먹었지"라는 문장이 붙는다.  
민재는 현재 문제의 주말 이체는 해지 정산을 위한 보증금·충당금 이동이었다고 주장하고, 라희는 과거 리베이트 은폐 전력을 근거로 이번에도 똑같은 배신이 반복됐다고 몰아붙인다.  
추궁이 깊어질수록, 이번 증거는 라희가 과거 상처를 이용해 더 나쁘게 보이도록 편집한 것이고 민재 역시 실제 이동 사유를 고의로 늦게 공유해 의심을 키웠다는 사실이 드러난다.

### 양측 프로필
- A: 오민재, 모바일 커머스 개발사 공동대표 겸 개발운영총괄, 숨긴 것=`정산성 자금 이동을 사전에 설명하지 않고 위기 때마다 단독 처리해 불신을 키운 점`, 거짓말 전략=`cold_logic`
- B: 김라희, 모바일 커머스 개발사 공동대표 겸 영업재무총괄, 숨긴 것=`편집 거래내역으로 현 상황을 과거 배신의 반복처럼 연출해 협상 우위를 잡으려 한 점`, 거짓말 전략=`victim_cosplay`

### 쟁점 구조
- d-1: 라희가 제출한 편집 거래내역은 어디까지 왜곡됐나
- d-2: 민재의 과거 외주 리베이트 은폐는 현재 판단에 얼마나 반영돼야 하나
- d-3: 주말 세 줄 이체는 횡령인가, 정산성 이동인가
- d-4: 라희는 과거 배신 사건을 협상용 무기로 다시 소환했나
- d-5: 반복된 불신 속에서 누가 현재 위기를 더 크게 키웠나

### 증거 설계
- 초기 공개: e-1 편집 거래내역 PDF 3장, e-2 주말 이체 알림 캡처, e-3 과거 리베이트 사건 메모
- 1차 해금(S1+): e-4 은행 원본 거래내역 export
  - requires: `e-1`
- 2차 해금(S2+): e-5 보증금·충당금 정산 계획서
  - requires: `e-2`
- 3차 해금(S2+): e-6 buyout 협상 초안과 라희 수정본
  - requires: `e-3`
- 최종 해금(S4+): e-7 "이번엔 네가 설명 못 하게 만들겠다" 대화 기록
  - requires: `e-4`, `e-6`
- 증거 조합:
  - `e-1 + e-4` → PDF가 금액 흐름과 시점을 왜곡한 편집본임이 드러난다
  - `e-2 + e-5` → 주말 이체가 정산성 이동이었다는 사실이 열린다
  - `e-3 + e-7` → 라희가 과거 배신 기억을 현재 협상용으로 활용했음이 강화된다

### 거짓말 상태 전이 설계
- A / d-3:
  - S0→S1: `direct_question`으로 "돈을 옮긴 건 맞다" 인정
  - S1→S2: `e-5` 제시 시 "설명할 틈이 없었다" 핑계
  - S2→S3: `responsibility_question`으로 "라희가 먼저 PDF로 몰아갔다" 책임 전가
  - S3→S5: `e-7` 제시 시 사전 미공유와 단독 처리 습관을 자백
- B / d-1:
  - S0→S1: `direct_question`으로 "보기 쉽게 정리한 것" 인정
  - S1→S2: `e-4` 제시 시 "의미는 같았다" 핑계
  - S2→S4: `emotion_threshold`로 과거 배신 기억 폭발
  - S4→S5: `e-6`, `e-7` 제시 시 협상 우위 확보용 편집을 자백

### 핵심 모순 쌍
- B는 "현재도 같은 횡령"이라 말하지만 `e-2 + e-5`는 다른 성격의 이동이었음을 보여준다
- A는 "오해일 뿐"이라 말하지만 `e-7`은 스스로도 설명 지연을 계산했음을 보여준다
- B는 "원본 그대로 정리"라 말하지만 `e-1 + e-4`는 조작을 보여준다

### 판결 딜레마
`현재의 편집 증거 사용`과 `과거의 실제 배신 전력` 중 무엇을 더 무겁게 현재 책임에 반영할지 갈린다.  
과정 중시 재판관은 이번 증거 조작을 더 크게 볼 수 있고, 해결 중시 재판관은 반복 배신 구조 자체를 끊는 정산과 결별에 더 무게를 둘 수 있다.

## workplace-12 — 켜 놓은 얼굴

### 사건 훅
지훈이 만든 PDF에는 수민의 유휴시간이 빨간색으로 칠해져 있고, 수민이 제출한 민원에는 거실 일부와 얼굴까지 요구한 웹캠 지시가 박제돼 있다.  
둘 중 누구든 먼저 보면 상대가 더 악질처럼 보인다.  
그러나 심문과 증거가 진행될수록, 지훈은 회사 기준을 넘는 감시와 편집 PDF로 압박했고 수민은 마우스 지글러와 상태창 편집으로 실제 근무 공백을 숨겼다는 사실이 함께 드러난다.

### 양측 프로필
- A: 백지훈, 고객지원운영 매니저, 숨긴 것=`정책 범위를 넘는 웹캠·화면 캡처 지시와 특정 시간대만 강조한 모니터링 PDF 편집`, 거짓말 전략=`cold_logic`
- B: 오수민, 재택 고객지원 사원, 숨긴 것=`마우스 지글러 사용과 상태창 편집으로 근무 공백을 숨기고 동시에 관리자 감시를 역으로 민원 카드로만 사용한 점`, 거짓말 전략=`confrontational`

### 쟁점 구조
- d-1: 과도한 모니터링과 사생활 침범은 실제 정책 범위를 넘었나
- d-2: 수민의 근무 태만과 지글러 사용은 어느 정도였나
- d-3: 지훈의 모니터링 PDF는 얼마나 과장 편집됐나
- d-4: 상태창·캘린더 편집 은폐는 반복적이었나
- d-5: 재택근무에서 성과 관리와 사생활 경계는 어디서 무너졌나

### 증거 설계
- 초기 공개: e-1 유휴시간 표시 PDF, e-2 웹캠 상시 지시 메일, e-3 수민의 민원 접수서
- 1차 해금(S1+): e-4 HR 재택 모니터링 기준 문서
  - requires: `e-2`
- 2차 해금(S2+): e-5 마우스 지글러 설치 흔적 + 실행 로그
  - requires: `d-2` S1 이상
- 3차 해금(S2+): e-6 원본 상태창 타임라인과 상담건 처리량 raw log
  - requires: `e-1`
- 최종 해금(S4+): e-7 지훈의 PDF 편집본 원레이어 파일
  - requires: `e-4`, `e-6`
- 증거 조합:
  - `e-2 + e-4` → 지훈의 지시가 회사 기준을 넘었다는 사실이 드러난다
  - `e-5 + e-6` → 수민이 실제 공백을 숨겼다는 사실이 드러난다
  - `e-1 + e-7` → 지훈이 특정 시간대만 강조해 압박용 자료를 만들었음이 드러난다

### 거짓말 상태 전이 설계
- A / d-1:
  - S0→S1: `direct_question`으로 "관리 강도를 높인 건 맞다" 인정
  - S1→S2: `e-4` 제시 시 "성과 압박 때문에 예외적으로 그랬다" 핑계
  - S2→S3: `responsibility_question`으로 "먼저 공백을 숨긴 건 수민" 책임 전가
  - S3→S5: `e-7` 제시 시 압박용 편집 자료 제작을 자백
- B / d-2:
  - S0→S1: `direct_question`으로 "상태창이 부정확할 때가 있었다" 인정
  - S1→S2: `e-5` 제시 시 "감시가 심해 방어적으로 켰다" 핑계
  - S2→S4: `emotion_threshold`로 감시 스트레스 폭발
  - S4→S5: `e-6` 제시 시 실제 근무 공백과 편집 은폐를 자백

### 핵심 모순 쌍
- A는 "정상 관리"라 말하지만 `e-2 + e-4`는 정책 초과를 보여준다
- B는 "감시 때문에 억울할 뿐"이라 말하지만 `e-5 + e-6`은 실제 은폐를 보여준다
- A는 "원본 수치"라 말하지만 `e-1 + e-7`은 압박용 편집을 보여준다

### 판결 딜레마
`사생활을 침범하는 과잉 감시`와 `그 감시를 명분으로 삼은 편법 은폐` 중 무엇이 더 조직을 망가뜨렸는지 갈린다.  
논리형 재판관은 양쪽 데이터를 따로 분리해 책임화할 수 있고, 직관형 재판관은 감시 환경이 은폐를 낳은 구조를 더 크게 볼 수 있다.
