# Solomon 리뉴얼 파일럿 테스트 — 테스트 스레드 브리핑

## 당신의 역할
Solomon 게임의 리뉴얼 파일럿 테스트를 반복 실행하고, 발견된 문제를 정리하여 리포트를 만드는 것입니다.

## 게임 개요
- LLM 기반 법정 심문 게임. 재판관(플레이어)이 양측 당사자(AI NPC)를 심문하여 진실을 밝힘
- 현재 spouse-01 사건으로 리뉴얼 파일럿 테스트 중
- 당사자 A: 한지석 (39세, avoidant 성격 — 회피형, 숫자/순서로 감정을 숨김)
- 당사자 B: 오세린 (36세, confrontational 성격 — 정면돌파형, 짧고 날카로움)

## 리뉴얼 핵심 구조
기존에는 LLM이 "무슨 말을 할지"와 "어떻게 말할지"를 모두 결정했으나, 리뉴얼 후에는:
1. **엔진이 의미를 결정** (ResponseBlueprint → stance/defenseMode/allowedClaims)
2. **LLM은 연기만 수행** (blueprint대로 1~2문장 생성)

### 핵심 개념
- **stance**: deny(부정) / hedge(회피) / partial(부분인정) / blame(전가) / emotional(감정호소) / confess(고백)
- **defenseMode**: flat_denial / silence / context_attack / legality_attack / authenticity_attack / counterattack / concession
- **하위 액션**: 사실추궁(fact_pursuit) / 동기탐색(motive_search) / 공감접근(empathy_approach)
  - 각각 NPC 응답의 초점을 바꿈: 행위 중심 / 이유 중심 / 심정 중심
- **Bridge**: Phase 1(사전 스크립트)에서 이미 인정한 사실을 Phase 3(심문)에 반영하여 lieState를 S0이 아닌 S1로 시작
- **tell (말버릇)**: over_precision(숫자 정밀), counter_question(되묻기), timeline_padding(동선 나열) 등. cadence 규칙으로 반복 방지

## 게임 실행 방법
1. 브라우저에서 `http://localhost:5173` 접속
2. 사건 선택 → spouse-01 (부부 — 한지석 vs 오세린)
3. Phase 0(사건 개요) → Phase 1(초기 진술) → Phase 2(반박) → Phase 3(심문) 순서로 자동 진행
4. Phase 3부터 하단의 심문/증거/스킬 탭으로 조작

## 심문 조작 방법
1. 하단에서 **대상 선택** (한지석 / 오세린)
2. **심문 탭** 클릭
3. **하위 액션 선택**: ❓사실추궁 / 🔍동기탐색 / 💛공감접근
4. **쟁점 선택**: 표시된 쟁점 목록에서 선택
5. 재판관 질문 → NPC 응답이 순서대로 표시됨

## 테스트 케이스

### TC-1: 하위 액션 차별화 (가장 중요)
같은 인물, 같은 쟁점에서 3가지 액션을 순서대로 실행.

| 순서 | 대상 | 액션 | 쟁점 |
|------|------|------|------|
| 1 | 한지석 | 사실추궁 | 지석의 비밀 송금 280만원 |
| 2 | 한지석 | 동기탐색 | 지석의 비밀 송금 280만원 |
| 3 | 한지석 | 공감접근 | 지석의 비밀 송금 280만원 |

**확인 포인트**:
- 3개 응답이 서로 다른지 (표현, 초점)
- 사실추궁 → "했는지 안 했는지" 중심
- 동기탐색 → "왜 그랬는지" 중심
- 공감접근 → "어떤 마음이었는지" 중심
- "그런 적 없습니다" + "맞지만" 같은 자기모순이 없는지

### TC-2: 캐릭터 차이 (archetype 검증)
같은 액션, 같은 쟁점에서 다른 인물.

| 순서 | 대상 | 액션 | 쟁점 |
|------|------|------|------|
| 1 | 한지석 | 사실추궁 | 100만원 사전 상의 약속 위반 |
| 2 | 오세린 | 사실추궁 | 100만원 사전 상의 약속 위반 |

**확인 포인트**:
- 한지석(avoidant): 길고 우회적, 순서/절차 앞세움
- 오세린(confrontational): 짧고 날카로움, 역공 시도

### TC-3: 연속 심문 (깊이 + 반복 검증)
같은 인물, 같은 쟁점 3회 연속.

| 순서 | 대상 | 액션 | 쟁점 |
|------|------|------|------|
| 1 | 오세린 | 사실추궁 | 세린의 새벽 휴대폰 열람 |
| 2 | 오세린 | 사실추궁 | 세린의 새벽 휴대폰 열람 |
| 3 | 오세린 | 동기탐색 | 세린의 새벽 휴대폰 열람 |

**확인 포인트**:
- 1턴과 2턴이 복붙이 아닌지 (다른 표현)
- 3턴에서 초점 전환이 되는지
- 재판관 질문도 깊이에 따라 달라지는지

### TC-4: 쟁점 전환 (맥락 분리)
쟁점을 바꿨을 때 이전 쟁점 내용이 섞이지 않는지.

| 순서 | 대상 | 액션 | 쟁점 |
|------|------|------|------|
| 1 | 한지석 | 사실추궁 | 지석의 비밀 송금 280만원 |
| 2 | 한지석 | 사실추궁 | 100만원 사전 상의 약속 위반 |

**확인 포인트**:
- 2번 응답에 280만원, 송금 등 d-1 내용이 안 나오는지

### TC-5: tell 반복 방지 (cadence)
over_precision tell이 매 턴 반복되지 않는지.

| 순서 | 대상 | 액션 | 쟁점 |
|------|------|------|------|
| 1 | 한지석 | 사실추궁 | 지석의 비밀 송금 280만원 |
| 2 | 한지석 | 동기탐색 | 지석의 비밀 송금 280만원 |
| 3 | 한지석 | 공감접근 | 지석의 비밀 송금 280만원 |

**확인 포인트**:
- "9월 20일", "2,800,000원" 같은 정밀 숫자가 3턴 연속 나오면 문제
- cadence: once_every_2_turns → 최소 1턴은 건너뛰어야

### TC-6: Phase 0 화면 검증
사건 개요 화면에서 확인.

**확인 포인트**:
- 쟁점 2개만 표시 (4개 아님)
- "심문 과정에서 추가 쟁점이 드러날 수 있습니다" 문구
- 최민정의 직업(재가돌봄센터 상담팀장)이 안 보이는지
- 관련 인물에 institutional 인물이 안 보이는지

### TC-7: 호칭 규칙
재판관 앞에서 호칭 오류 검증.

**확인 포인트**:
- 한지석 → 재판관에게 오세린 언급 시: "제 아내" (O) / "자기가" (X)
- 오세린 → 재판관에게 한지석 언급 시: "제 남편" (O) / "자기가" (X)

## 콘솔 로그 확인
브라우저 개발자 도구(F12 → Console)에서 다음 로그가 정상적으로 나오는지 먼저 확인:

```
[Renewal] spouse-01 리뉴얼 데이터 등록 시작
[Renewal] spouse-01 등록 완료: ClaimPolicy + Bridge + EvidenceChallenge + Tell
[Bridge] a/d-1: S0 → S1
[Blueprint] spouse-01/a/d-1/S1 — blueprint 경로 활성
```

위 로그가 안 나오면 리뉴얼 경로가 활성화되지 않은 것이므로, 기존 경로의 응답을 보고 있을 수 있습니다. 이 경우 먼저 보고해 주세요.

## 주의사항
- 매 TC 시작 시 **게임을 새로 시작**해야 합니다 (← 나가기 → spouse-01 재선택)
- TC-1, TC-3, TC-5는 **연속 턴**으로 진행해야 의미가 있습니다 (중간에 다른 캐릭터 심문 금지)
- 스크린샷이 있으면 가장 좋지만, 텍스트 복사도 괜찮습니다
