# Thread S v3 Case Design: Friend

### [friend] 손절한 절친

**훅**:
손절한 전 절친이 내 예비신랑에게 계속 연락한다.
결혼 3주 전, 그 장면만 보면 누구라도 친구가 선을 넘었다고 생각한다.

**A**: 송다은(31, 온라인 쇼핑몰 CS 직원) — B와 크게 틀어진 뒤 이미 관계가 끝났다고 단정했고, B가 예비신랑에게 연락한 사실을 공통 친구들에게 퍼뜨렸다 — `premature_summary`

**B**: 최수민(31, 필라테스 강사) — 예비신랑을 꼬시려 한 게 아니라 경고하려 했고, 그보다 오래전에는 A 아버지에게 당한 일을 말하지 못한 채 스스로 악역이 되었다 — `affect_flattening`

**callTerms**:
- A → B 기본: `수민아`
- B → A 기본: `다은아`
- A 격앙 시: `최수민!`
- B 격앙 시: `송다은!`
- A/B → 재판관: `제 친구`
- 규칙: 친구 사이 직접 호칭은 이름 중심, 재판관에게는 `제 친구`

## 1. 쟁점 재배치

**초기 표시 쟁점**:
- d-1: B의 연락은 예비신랑 집착인가, 뭔가를 전하려는 경고인가
  - 사건 접수 단계에서 가장 먼저 보이는 표면 쟁점
- d-4: 예전 손절 사건도 정말 B의 변심 때문이었는가
  - A가 `예전에도 이래서 손절했다`고 말하는 순간 자연스럽게 뜨는 과거 쟁점
- d-5: 이번 사건에서 B의 명예를 먼저 무너뜨린 건 누구인가
  - 단톡방 퍼뜨리기 때문에 바로 전면에 나오는 사회적 피해 쟁점

**숨은 쟁점**:
- h-d2: 예비신랑이 먼저 B에게 선을 넘었는가
  - 등장 조건:
  - `e-4`가 `Excerpt` 이상으로 올라가거나,
  - `e-1`과 B 발언 조합으로 `Contradiction` Lead가 생성되면 탭 생성
- h-d3: A 아버지의 돈 접근은 과거와 현재가 같은 패턴인가
  - 등장 조건:
  - `e-5`가 `Excerpt` 이상으로 올라가거나,
  - `e-5`와 `e-6` 조합으로 `Third-party` Lead가 생성되면 탭 생성

**쟁점별 앵커 진실**:
- d-1: B의 연락은 집착처럼 보였지만 실제 목적은 경고였다
- h-d2: 먼저 선을 넘은 쪽은 예비신랑이었다
- h-d3: A 아버지는 과거에도 B에게, 현재는 예비신랑에게 비슷한 방식으로 돈을 뜯어내려 했다
- d-4: 과거 손절의 진짜 이유도 B의 변심이 아니라 A 아버지의 사기였다
- d-5: A는 직접 확인 없이 B를 먼저 문제 인물로 만들었다

## 2. 증거 2축 설계

### e-1 [log] B→예비신랑 연락 기록
- 역할: 집착처럼 보이는 가장 강한 표면 단서
- 깊이:
  - Stub: `연락기록 1건` 존재만 보임
  - Excerpt: 최근 9일 동안 전화 6번, 문자 11번이라는 횟수만 보임
  - Original: 날짜, 시각, 미응답 여부, 짧은 답장 흐름까지 보임
  - Context: 결혼 3주 전이라는 시기, `e-2` 단톡방과 맞물린 공포가 붙음
  - Established: `B가 예비신랑에게 반복 연락한 사실`이 공식기록 채택
- 신뢰 상태:
  - 제출됨: A가 캡처 제출
  - 검증 중: 통신기록 원본 대조
  - 인증됨: 횟수와 시각 일치
  - 이의 제기됨: B가 "연락 자체와 의도는 별개"라고 주장
  - 조작/오독 판정: 연락 사실은 인증, `집착` 해석은 보류

### e-2 [chat] 공통 친구 단톡방 캡처
- 역할: B의 사회적 낙인을 만드는 공개 프레임
- 깊이:
  - Stub: `단톡방 캡처 1건` 존재만 보임
  - Excerpt: "수민이가 또 내 남자한테 연락한다", "예전에도 이래서 손절했다" 문장만 보임
  - Original: 친구들의 반응, A의 반복 표현, B를 배제하는 분위기까지 확인
  - Context: 누가 먼저 말했고, 누가 맞장구쳤고, B에겐 해명 기회가 없었다는 흐름 복원
  - Established: `A가 먼저 공통 친구들에게 B를 문제 인물로 퍼뜨렸다`로 공식기록 채택 가능
- 신뢰 상태:
  - 제출됨: B 또는 제3자가 제출
  - 검증 중: 단톡방 원본과 발화 순서 대조
  - 인증됨: 원본 대화방과 일치
  - 이의 제기됨: A가 "화가 나서 그랬다"고 감정 이의
  - 조작/오독 판정: 편집은 없고, 오히려 맥락 전체가 A에게 불리하게 확장됨

### e-3 [chat] 과거 손절 직전 카톡
- 역할: `예전에도 이랬다`는 A의 믿음을 받쳐 주는 듯한 과거 단서
- 깊이:
  - Stub: `과거 카톡 1건` 존재 표시
  - Excerpt: B가 "돈 문제는 내가 알아서 정리할게"라고만 하고 빠지는 장면
  - Original: A가 `넌 돈만 엮이면 사람이 달라진다`고 단정하는 흐름까지 확인
  - Context: B가 직접 진실을 말하지 못해 오해를 굳힌 구조가 복원
  - Established: `과거 손절은 돈 문제를 둘러싼 침묵과 오해 위에서 벌어졌다`로 공식기록 채택 가능
- 신뢰 상태:
  - 제출됨: A가 일부 또는 B가 전체 제출
  - 검증 중: 원본 채팅방 대조
  - 인증됨: 날짜, 발신자, 문장 일치
  - 이의 제기됨: A가 "그때도 설명하지 않았다"고 이의
  - 조작/오독 판정: 발췌만 보면 B에게 불리하지만, Context로 갈수록 오독 가능성 드러남

### e-4 [chat] 예비신랑의 선넘는 메시지 + B의 거절 답장
- 역할: 2차 반전. 예비신랑이 먼저 선을 넘었다는 증거
- 민감정보 봉인 대상:
  - 결혼 날짜·장소 관련 표현
  - 성적 뉘앙스가 강한 문장 일부
- 깊이:
  - Stub: `예비신랑과 B의 카톡 스레드` 존재만 보임
  - Excerpt: "커피 한 번 보자", "너 같은 스타일이 내 이상형"과 B의 "친구 남자친구니까 이러지 마"만 보임
  - Original: 몇 달간 반복된 접근과 B의 선 긋는 답장 전체 확보
  - Context: B가 왜 예비신랑 연락을 끊지 않고 `경고하려는 채널`로 남겨 뒀는지까지 읽힘
  - Established: `먼저 선을 넘은 쪽은 예비신랑`이라는 공식기록 후보 형성
- 신뢰 상태:
  - 제출됨: B가 발췌 제출하거나 재판관이 원본 요구
  - 검증 중: 휴대폰 원본 대화와 알림 기록 대조
  - 인증됨: 메시지 원본 일치
  - 이의 제기됨: A가 "그래도 왜 답장을 이어 갔느냐"고 이의
  - 조작/오독 판정: 편집 의심은 해소, `B 집착` 프레임이 크게 흔들림

### e-5 [chat] A 아버지와 예비신랑의 문자
- 역할: 현재 사건의 진짜 목적을 드러내는 제3자 증거
- 민감정보 봉인 대상:
  - 정확한 돈 액수
  - 결혼 일정·예식장 정보
  - 계좌번호
- 깊이:
  - Stub: `A 아버지와 예비신랑 문자` 존재만 보임
  - Excerpt: "결혼 전에 잠깐만 도와주면 금방 돌려준다" 수준만 보임
  - Original: 반복적인 돈 얘기, `사위 될 사람이면` 식의 압박 표현까지 보임
  - Context: B의 연락 시기와 정확히 겹치고, `지금 경고하지 않으면 또 당한다`는 서사가 붙음
  - Established: `A 아버지가 예비신랑에게 돈을 뜯어내려 접근했다`로 공식기록 채택 가능
- 신뢰 상태:
  - 제출됨: 예비신랑 또는 B가 캡처 제출
  - 검증 중: 예비신랑 폰 원본 대조
  - 인증됨: 문자 원본, 날짜 일치
  - 이의 제기됨: A가 "인사 차원이었을 뿐"이라고 이의
  - 조작/오독 판정: `돈 부탁` 자체는 인증, 악의 여부는 `Context` 필요

### e-6 [bank/chat] 과거 송금 영수증 + 아버지 문자
- 역할: 과거 손절 사건의 진짜 원인 확정
- 민감정보 봉인 대상:
  - 정확한 송금 액수
  - B의 계좌번호
- 깊이:
  - Stub: `과거 송금 내역 1건` 존재만 보임
  - Excerpt: B가 A 아버지에게 돈을 보낸 기록만 보임
  - Original: "급한 투자금", "한 달만 쓰고 갚겠다" 문자와 미루는 답장까지 한 세트로 보임
  - Context: B가 왜 A에게 직접 말하지 못하고 혼자 끊어냈는지까지 복원
  - Established: `과거 손절의 원인 중 핵심은 A 아버지의 사기성 돈 요구였다`로 공식기록 채택 가능
- 신뢰 상태:
  - 제출됨: B가 일부 내역 제출
  - 검증 중: 송금 영수증 원본과 문자 원본 대조
  - 인증됨: 금액, 날짜, 발신자 일치
  - 이의 제기됨: A가 "왜 그때 내게 말 안 했냐"고 반발
  - 조작/오독 판정: 데이터는 인증, `왜 침묵했는가`는 h-d3와 d-4에서 정리

### e-7 [timeline] 과거·현재 대조표
- 역할: 반복 패턴을 한 줄로 확정하는 최종 자료
- 깊이:
  - Stub: `과거·현재 대조표 생성 가능` 상태만 보임
  - Excerpt: 과거 B 피해 시점과 현재 예비신랑 접근 시점의 거친 겹침
  - Original: A 아버지의 비슷한 문구, B의 침묵, A의 오해, 이번 연락 시도가 날짜별로 정렬됨
  - Context: B가 두 번 다 A를 지키려다 악역이 된 감정 흐름까지 붙음
  - Established: `과거와 현재는 같은 패턴이었고, B의 연락은 그 반복을 막으려는 경고였다`로 공식기록 채택 가능
- 신뢰 상태:
  - 제출됨: 없음. 조합과 검증으로 생성
  - 검증 중: `e-4`~`e-6` 중 핵심 3개 이상 인증 시 생성
  - 인증됨: underlying evidence가 모두 인증되면 자동 인증
  - 이의 제기됨: A가 "그래도 연락 방식은 잘못"이라며 이의 가능
  - 조작/오독 판정: 종합표 자체는 조작 불가, 해석은 `Reframe` 선택에 좌우

## 3. 조합 → Lead → Dossier 설계

### L-1 Contradiction Lead: 집착처럼 보이는 그림
- 1차 조합 입력: `e-1 Excerpt` + `e-2 Excerpt`
- 생성 Lead: `Contradiction`
- 플레이어 해석 선택지:
  - 해석 A: B가 예비신랑에게 집착한다
  - 해석 B: 겉으로는 집착처럼 보여도 뭔가 설명 빠진 맥락이 있다
- 2차 조합: `L-1` + `e-4 Original`
- 생성 DossierCard: `dc-1 먼저 선을 넘은 사람`
- 성공 효과:
  - d-1이 `집착`에서 `왜 그런 모습으로 남았나` 쟁점으로 재해석
  - h-d2 생성

### L-2 Timeline Lead: 예전에도 이랬다
- 1차 조합 입력: `e-2 Original` + `e-3 Excerpt`
- 생성 Lead: `Timeline`
- 플레이어 해석 선택지:
  - 해석 A: B는 예전에도 비슷한 식으로 문제를 만들었다
  - 해석 B: 예전에도 설명되지 않은 공백이 있었다
- 2차 조합: `L-2` + `e-6 Original`
- 생성 DossierCard: `dc-2 손절의 시작점`
- 성공 효과:
  - d-4가 단순 과거사에서 현재 사건의 반복 패턴으로 전환
  - A의 `premature_summary`가 약해지기 시작함

### L-3 Third-party Lead: 이번에도 아버지인가
- 1차 조합 입력: `e-1 Original` + `e-5 Original`
- 생성 Lead: `Third-party`
- 플레이어 해석 선택지:
  - 해석 A: B는 또 친구 관계를 흔드는 핑계를 만든다
  - 해석 B: B는 A 아버지가 예비신랑에게 접근하는 걸 보고 경고하려 했다
- 2차 조합: `L-3` + B의 발언 "또 당하게 둘 수 없었다"
- 생성 DossierCard: `dc-3 이번에도 같은 패턴`
- 성공 효과:
  - h-d3 생성
  - d-1 연락 의도가 `집착`에서 `위험 경고`로 크게 기운다

### L-4 Context Lead: 왜 A에게 직접 말하지 못했나
- 1차 조합 입력: `e-6 Context` + B의 발언 "차라리 내가 나쁜 사람이 되는 게 낫다고 생각했다"
- 생성 Lead: `Context`
- 플레이어 해석 선택지:
  - 해석 A: B는 늘 말을 비틀어 더 큰 오해를 만든다
  - 해석 B: B는 A의 세계가 무너질까 봐 직접 진실을 못 말했다
- 2차 조합: `L-4` + `분리심문(B)` 또는 `w-3 증언`
- 생성 DossierCard: `dc-4 악역을 택한 이유`
- 성공 효과:
  - d-4와 h-d3의 감정선이 크게 열림
  - `affect_flattening`이 무너지기 시작함

### L-5 Reframe Lead: 두 번의 손절은 같은 패턴
- 1차 조합 입력: `e-5 Established` + `e-6 Established`
- 생성 Lead: `Reframe`
- 플레이어 해석 선택지:
  - 해석 A: A 아버지의 반복된 돈 접근이 두 사건을 만들었다
  - 해석 B: 그 패턴을 봐도 B는 항상 더 좋은 방식으로 말했어야 했다
- 2차 조합: `L-5` + `e-7 Original`
- 생성 DossierCard: `dc-5 두 번의 손절`
- 성공 효과:
  - 사건 전체가 `친구가 내 남자를 노렸다`에서 `친구가 두 번 다 경고했지만 두 번 다 악역이 됐다`로 재정의됨
  - 평의의 `왜 숨겼나`, `어떤 피해가 갔나` 칸을 크게 채움

## 4. DossierCard 상세

### dc-1 먼저 선을 넘은 사람

  - 목적: 아버지의 현재/과거 돈 접근을 모두 원본으로 확정

**정확히 답변하십시오**

- 효과: `premature_summary`를 깨고 시점과 사실을 분리하게 만듦

**분리심문**
- 1차 권장 시점: `e-4 Original` 직후 B
  - 이유: B는 A 앞에서는 더 닫히고, 혼자 두면 예비신랑 문제를 조금 더 풀 수 있음

  - 이유: A는 아버지 얘기가 나오면 즉각 방어적이 되므로 혼자 두어야 흔들림이 보임

**잠정 인정**
- 권장 인정 1: `B가 예비신랑에게 반복 연락한 사실`
- 권장 인정 2: `예비신랑이 먼저 B에게 선을 넘는 메시지를 보낸 사실`
- 권장 인정 3: `A가 단톡방에서 먼저 B를 문제 인물로 퍼뜨린 사실`
- 권장 인정 4: `A 아버지가 과거와 현재 모두 돈 접근을 시도한 사실`

**선처 창구**
- 권장 시점: `dc-3`, `dc-4` 직후
- 효과: B가 `또 내가 나쁜 사람 되는 쪽을 택했다`는 본심을 자발적으로 풀 가능성 큼

**발언 제지 / 기록 제외**
- 권장 시점: A가 계속 `내 남자` 프레임으로만 몰거나, B가 예비신랑 잘못만 강조하며 A의 충격을 무시할 때
- 효과: 프레임 싸움을 멈추고 사실-가설 분리

**민감정보 봉인 해제**

- 봉인 대상 3: 과거 송금 정확 액수와 계좌 정보
- 가장 효과적인 해제 시점:
  - h-d2에서 B 집착 프레임을 확실히 뒤집어야 할 때
  - h-d3가 열렸지만 A 아버지 패턴을 강하게 고정해야 할 때
