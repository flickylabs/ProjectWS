# V3 → 스크립트 전환 + 게임성 강화 논의 요청

## GPT에게: 이 문서의 목적

Solomon 게임의 V3 리뉴얼이 엔진/데이터 수준에서 완료되었습니다.
이제 **두 가지 큰 전환**을 논의하고 싶습니다:

1. **LLM 실시간 → 스크립트 기반으로 전환** (비용/안정성/품질)
2. **게임의 단순성/패턴화 방지** (재미/전략성/리플레이)

각 주제에 대해 당신의 의견, 보완 아이디어, 잠재적 문제점을 듣고 싶습니다.

## 1. 현재 게임 구조 (V3 완료 상태)

### Phase 흐름
```
Phase 0: 사건 개요 — UI 카드 (당사자/쟁점 소개)
Phase 1: 초기 진술 — JSON 스크립트 재생 (사전 작성)
Phase 2: 반박      — JSON 스크립트 재생 (사전 작성)
Phase 3: 통합 심문 — 12~16턴, 플레이어 주도 (현재 LLM 실시간)
Phase 6: 조정      — AI 종합 (양측 진술 기반 조정안 생성)
Phase 7: 판결      — 플레이어 직접 판단
Result:  결과/후일담 — AI 서사 생성
```

Phase 4(증거심리), Phase 5(재심문)는 Phase 3에 통합되어 PHASE_ORDER에서 제외됨.

### 플레이어 행동 체계 (Phase 3)
```
심문 대상 선택 (한지석 a / 오세린 b)
  ├── 심문 (3종 질문 카드)
  │   ├── 사실추궁 → 모순토큰 ↑
  │   ├── 동기탐색 → 누설미터 ↑
  │   └── 공감접근 → 신뢰창구 ↑
  ├── 사건카드 (4턴 이후 해금)
  │   └── 증거 묶음 카드 → 대상별 맞춤 질문 3개
  ├── 증거 제시
  │   └── 개별 증거 → DossierCard 맞춤 질문 or 유형별 fallback
  └── 스킬
      ├── 이의 제기 (전체 쟁점 압박)
      ├── 분리 심문 (3턴간 1:1)
      └── 즉답 요구 (특정 쟁점 즉시 붕괴)
```

### V3 게임 루프 (매 턴)
```
1. 쟁점 선택
2. 행동 선택 (질문 3종 / 사건카드 / 증거)
3. NPC 응답 (현재: LLM → 전환 예정: BeatScript)
4. 즉시 결과 표시
   - 미터 변화 (모순토큰/누설미터/신뢰창구)
   - 상태 전이 라벨 (균열/궁지/개방/시인)
   - 증거 결과 토스트 (버팀/균열/붕괴)
5. 이벤트 발동 (조건 충족 시)
   - 모순 감지 → 지적/넘어가기
   - 끼어들기 → 허용/제지
   - 감정 폭발 → 압박/진정
   - 새 쟁점 출현 → 추적/보류
6. Readiness 갱신 → 판결 가능 여부 체크
```

### 5대 엔진 구조
```
ResponseBlueprint   — 매 턴 stance/defenseMode/allowedAtoms 결정
ClaimPolicyV2       — atom 기반 의미 단위 (publicClaim 분해)
RuntimeStartBridge  — Phase 1→3 상태 연결 (인정 사실 반영)
EvidenceChallenge   — attackVector 봉쇄 + 방어선 붕괴 메커닉
ExecutableVerbalTell — 검증 가능한 말버릇 규칙
+ V3 추가:
  questionEffectEngine  — 질문 3종 미터 시스템
  gameEventTriggerEngine — 4종 이벤트 조건 평가
  v3GameLoopLoader      — DossierCard/UnlockAtom/EventText/TransitionBeat
```

### 데이터 현황

## 2. 전환 주제 A: LLM → 스크립트

### 현재 문제
1. **자유 심문(직접 입력)이 게임을 붕괴시킴** — lieState S0인데 "왜 돈 보냈어?" 하면 LLM이 Blueprint 무시하고 다 털어놓음
2. **LLM 응답 품질 불안정** — 같은 Blueprint를 줘도 때때로 호칭 오류, 반말, 자기모순 발생
3. **API 비용** — 게임당 ~20회 LLM 호출, 모바일 플레이 기준 비효율
4. **오프라인 불가** — 서버 없으면 게임 불가

### 전환 방안
```
Phase 1~2: 이미 스크립트 → 변경 없음
Phase 3 (심문): LLM → BeatScript 기반 (×5 변형 랜덤)
Phase 6 (조정): AI 유지 (조합이 무한, 스크립트화 불가)
Result (후일담): AI 유지 (판결 결과 기반 서사)
자유 심문: S3 이상 + 턴 6 이상에서만 AI 사용 허용
```

**BeatScript ×5 변형 구조**:
```
현재: d-1 / deny / S0~S1 → 대사 1개
확장: d-1 / deny / S0~S1 → [변형1, 변형2, 변형3, 변형4, 변형5]
       ↓ 엔진이 랜덤 선택 + 중복 방지
```

### 논의 요청
- ×5 변형이면 16턴 심문에서 충분한가? ×3이면 부족한가?
- 질문 유형(사실추궁/동기탐색/공감접근)별로 다른 BeatScript 풀이 필요한가?
  - 예: 같은 d-1/S1 이라도 사실추궁에 대한 응답 vs 공감접근에 대한 응답이 달라야?
- BeatScript에 **조건 분기**가 필요한가?
  - 예: "이전 턴에 증거를 제시했으면 이 대사" vs "아니면 이 대사"
- TransitionBeat (전이 순간 강제 삽입)와 일반 BeatScript의 우선순위는?
- 자유 심문(S3+) 허용 시, Blueprint 제약을 어떻게 강화해야 하는가?

## 3. 전환 주제 B: 게임성 — 족보화/패턴화 방지

### 현재 문제
```
미터가 보이면 정답이 보인다:
  모순 쌓고 싶으면 → 사실추궁
  신뢰 올리고 싶으면 → 공감접근
  → 유저가 archetype 한 번 파악하면 고정 루트 → 재미 없음
```

### 대응안 (검토 요청)

#### B-1. NPC 확률적 반응
```
같은 질문이라도 NPC가 순응/저항/역공 중 랜덤 반응:
  사실추궁 → 80% 순응 (모순토큰 +1)
            15% 저항 (효과 반감)
             5% 역공 (오히려 플레이어 미터 깎임)

archetype별 확률 조정:
  avoidant:       순응 높음, 역공 거의 없음
  confrontational: 역공 빈번, 순응 드뭄
  cold_logic:     저항 빈번, 감정 기반 접근에만 순응
```

**질문**: 확률을 몇 %로 잡아야 "운 게임"이 아니라 "불확실성 게임"이 되는가?

#### B-2. 질문 피로도
```
같은 쟁점에 같은 유형 질문 연속 시:
  2회: 효과 60%로 감소
  3회: "재판관님, 이미 말씀드렸습니다" → 효과 거의 0
  → 쟁점과 질문 유형을 바꿔가며 추궁해야 함
```

**질문**: 피로도가 쟁점별인가 전체인가? 리셋 조건은?

#### B-3. 상대방 간섭 (끼어들기 강화)
```
현재: 확률 기반 끼어들기 (감정 높으면 발동)
강화안:
  - 한쪽을 너무 오래 추궁하면 상대가 반드시 끼어듦
  - 끼어들기 허용 → 새 정보 획득 가능, 하지만 현재 추궁 흐름 리셋
  - 끼어들기 제지 → 추궁 흐름 유지, 하지만 권위 소모
  → "허용할 것인가 제지할 것인가"가 진짜 선택이 됨
```

**질문**: 끼어들기가 너무 자주 오면 짜증, 너무 드물면 의미 없음. 적정 빈도는?

#### B-4. 감정 변동성
```
현재: 감정이 선형적으로 오름 (질문마다 +5~15)
변동안:
  - NPC 감정이 비선형으로 움직임
  - 갑자기 shutdown (모든 질문 효과 감소, 공감만 유효)
  - 갑자기 explosive (감정 폭발 이벤트 강제)
  → "지금 밀어야 하나 빼야 하나" 실시간 판단
```

**질문**: shutdown 진입 확률과 탈출 조건은? 너무 잦으면 답답한 게임이 됨.

#### B-5. 증거 타이밍 보너스
```
특정 state에서 특정 증거를 제시하면 크리티컬 (2배 효과):
  예: d-1이 S2일 때 e-1(송금 내역) 제시 → "마침 부분 인정한 상태에서 증거 들이대기" → 대폭 효과
  예: d-1이 S0일 때 e-1 제시 → 완전 부정 중이라 "조작이다" 반격 → 효과 감소

→ 증거를 아무 때나 내밀지 말고, NPC 상태를 읽고 타이밍을 잡아야
```

**질문**: 타이밍 매트릭스를 어떻게 설계? state × evidence × questionType 조합이 방대.

#### B-6. 자유 심문 (S3+ 해금)
```
S3 이상이면 NPC가 숨길 게 별로 없음 → 자유 질문 허용
  - AI가 Blueprint 범위 안에서 답변
  - 플레이어가 직감으로 파고드는 재미
  - 구조화된 질문으로 못 얻는 정보를 캐낼 수 있음
```

**질문**: 자유 심문의 결과가 미터/이벤트에 반영되어야 하나? 아니면 순수 서사 탐색?

## 3-2. 전환 주제 C: 사건 구조의 깊이와 복잡성

### 현재 문제: 쟁점이 "넓기만 하고 깊지 않다"

spouse-01 예시:
```
쟁점 5개 — 전부 "돈을 왜 숨겼나" + "사생활 침해" 한 축
  d-1: 280만원 비밀 송금
  d-2: 새벽 휴대폰 열람
  d-3: 잘린 캡처 오해
  d-4: 150만원 우회 송금
  d-5: 사전 상의 약속 위반

→ 5개 쟁점이 있어도 깊이가 아니라 폭만 넓음
→ "같은 쟁점을 계속 두드리면 결국 털어놓는다"는 단순 구조
```

### C-1. 쟁점의 층위 구조 (Depth Layers)

현재 "하나의 쟁점 = S0→S5 직선 진행"인데, **쟁점 안에 층위**가 있으면 어떨까?

```
표면 쟁점: "280만원을 왜 숨겼나" (쉽게 밝혀짐, S2면 인정)
    ↓ 파고들면
중간 쟁점: "장모 간병이라면 왜 아내한테 말 못 했나" (동기, S3~S4)
    ↓ 더 파고들면
핵심 쟁점: "투자 실패 이후 가장 체면이 무너졌다" (진짜 이유, S4~S5)
    ↓ 최종
관계 쟁점: "서로를 보호한다면서 결국 서로를 배제했다" (관계 본질)
```

현재는 이게 atom의 S0→S5 텍스트 변화로만 표현됨. 플레이어가 "층이 깊어지고 있다"는 걸 **구조적으로 인지**할 수 있어야 함.

**질문**: 층위를 어떻게 시각화하면 좋을까? 쟁점 카드가 "뒤집히는" 느낌? Dispute Board에서 깊이 표시?

### C-2. 가짜 쟁점 (Red Herring)

추리 게임의 핵심 — **플레이어가 잘못된 방향에 시간을 쓰게 만드는 미끼**.

```
현재: 쟁점 5개가 전부 "진짜"
  → 어떤 쟁점을 파도 결국 진실에 도달
  → 잘못된 판단의 위험이 없음

개선안: 쟁점 7~8개 중 2~3개가 "가짜"
  예: "한지석이 최민정과 외도했다" ← 가짜 (실은 센터 담당자)
  예: "오세린이 이혼 변호사를 만났다" ← 가짜 (실은 상속 상담)

  → 가짜 쟁점에 시간을 쓰면 턴 낭비
  → 진짜 쟁점을 가려내는 것 자체가 게임
  → 판결에서 가짜 쟁점을 "사실"로 판단하면 감점
```

**이미 있는 토대**:
- `truth` 필드: 현재도 true/false가 있음 (d-3: "아버지 수첩 메모는 상속 예고인가" = false)
- `isTrap` 증거: 함정 증거도 이미 존재 (잘린 캡처, 재촬영 사진)
- `quadrant`: neither_knows, shared_misconception 쟁점이 존재

하지만 현재는 **가짜 쟁점을 파고들어도 패널티가 없고**, NPC가 가짜 쟁점에 대해서도 성실히 답변. 가짜 쟁점에서는 **NPC가 진심으로 모르거나 혼란스러워하는** 반응이 나와야 함.

### C-3. 쟁점 간 연결/의존 관계

현재 5개 쟁점이 **독립적**으로 S0→S5 진행. 하지만 실제로는:
```
d-1(비밀 송금)을 S3까지 밀면
  → d-5(사전 상의 약속)에서 "먼저 깬 건 당신" 공격이 가능해짐
  → d-3(캡처 오해)에서 "외도 아님" 근거가 생김

d-2(휴대폰 열람)를 S4까지 밀면
  → d-4(우회 송금)에서 "당신도 숨겼잖아" 반격이 약해짐
```

이런 **쟁점 간 영향력**이 게임에 반영되면, "어떤 쟁점을 먼저 파느냐"가 전략이 됨.

**질문**:
- 쟁점 간 의존 관계를 어떤 구조로 표현? 그래프? 선형 잠금?
- 너무 복잡하면 플레이어가 혼란. 적정 복잡도는?

### C-4. 사건 규모 확대

현재 1사건 = 쟁점 5개, 증거 6개. 12~16턴에 이걸 다 다루면 **각 쟁점당 2~3턴**밖에 없음.

확대 방안:
```
A안: 쟁점 8~10개 + 증거 8~10개 + 턴 20~24
  → 깊이는 유지하면서 가짜 쟁점/층위 추가 가능
  → 세션 시간 증가 (15분 → 25분)

B안: 쟁점 5개 유지 + 층위 3단계 (표면/중간/핵심)
  → 쟁점 수는 같지만 각 쟁점이 3배 깊어짐
  → 세션 시간 유지

C안: 메인 쟁점 3개 (깊이) + 서브 쟁점 4~5개 (폭) + 가짜 2개
  → 메인은 필수, 서브는 선택적, 가짜는 함정
  → "어디에 시간을 쓸 것인가" 자원 관리 게임
```

**질문**: 모바일 세션 기준 적정 시간은? 10~15분? 20~25분?
