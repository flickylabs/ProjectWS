## 스레드 역할

Thread R은 **기존 ScriptedText의 표현을 자연스러운 한국어 대화로 리뉴얼**하는 전담 스레드입니다.

GPT Pro가 생성한 스크립트의 **골격(쟁점, 증거, LieState 단계별 정보 수준)은 유지**하면서, **표현, 말투, 어체, 호칭을 실제 사람이 하는 말로 바꾸는 것**이 목표입니다.

### 너의 책임
1. 모든 대사를 **실제 사람이 법정에서 하는 말**로 리뉴얼
2. 쟁점명/증거명 복붙을 **자연스러운 대화 표현**으로 교체
3. 호칭/어체 규칙을 **빠짐없이** 적용
4. 내용/의미/정보 수준은 **변경하지 않음** — 표현만 바꿈

### 너의 책임이 아닌 것
- 스토리 변경, 쟁점 재설계, 증거 체인 수정 (→ Thread S)
- 파이프라인/코드 수정 (→ Thread G)
- UI (→ Thread U)

## 현재 스크립트의 문제점 (구체적 예시)

### 문제 1: 쟁점명/증거명을 대사에 그대로 복붙

**현재 (FAIL)**:
> "자기, 제가 예민했던 이유를 빼고 '오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제만 말하면 제 쪽만 과장한 사람처럼 보입니다."

사람은 이렇게 말하지 않습니다. '오피스텔 출입과 새벽 통화는 딴살림 흔적인가'는 기획 문서의 쟁점명입니다.

**교정 후 (PASS)**:
> "자기, 내가 왜 그렇게 예민했는지는 빼고, 매일 그 집에 드나든 것만 가지고 말하면 나만 이상한 사람 되잖아."

### 문제 2: 부부 사이에 합니다체 사용

**현재 (FAIL)**:
> "자기, 제 설명이 부족했다는 건 인정합니다. 그래도 '공동 적금 3,000만 원은 어디로 갔고, 배우자 동의 없이 움직인 것인가' 문제를 지워 버리면..."

"자기"라고 부르면서 "~합니다"는 모순입니다.

**교정 후 (PASS)**:
> "자기, 내가 제대로 설명 못 한 건 인정해. 그래도 우리 적금이 어디로 갔는지, 나한테 한마디도 안 하고 움직인 건 또 다른 문제잖아."

### 문제 3: 데이터 필드/기획 용어가 대사에 노출

**현재 (FAIL)**:
> "재판관님, 외도가 아니었는데도 자신이 먼저 2,000만 원을 움직였다는 실행 책임이 중심…"

"실행 책임"은 기획 용어입니다. 사람은 이렇게 말하지 않습니다.

**교정 후 (PASS)**:
> "재판관님, 바람이 아니었다는 건 이제 압니다. 그래도 제가 먼저 돈을 움직인 건 사실이고, 그건 따로 봐야 합니다."

### 문제 4: 한 대사에서 여러 쟁점을 나열식으로 언급

**현재 (FAIL)**:
> "자기, 셋째 갈래인 '아내의 2,000만 원 송금은 자기방어인가, 이혼 대비 은닉인가' 문제까지 같이 놓고 보면 제 한 말과 상대가 들린 해석 사이 간격이 더 분명해집니다."

"셋째 갈래인"은 발표 자료 문체입니다.

**교정 후 (PASS)**:
> "자기, 내가 그 돈을 왜 옮겼는지까지 같이 봐야 돼. 나는 나를 지키려고 한 거고, 자기는 형을 지키려고 한 거잖아. 그 차이를 무시하면 안 되지."

## 호칭 + 어체 규칙 (절대 규칙)

### 화자별 어체

| 화자 → 청자 | 어체 | 예시 |
|------------|------|------|
| 당사자 → 재판관 | **합니다체** | "재판관님, 그건 사실이 아닙니다." |
| 당사자 → 상대방 (부부) | **반말** | "자기, 그건 아니잖아." |
| 당사자 → 상대방 (형제) | **반말** | "형, 그건 아니야." / "정후야, 그건 아니지." |
| 당사자 → 상대방 (친구) | **반말** | "수민아, 그건 아니야." / "다은아, 그건 아니지." |
| 증인 → 재판관 | **합니다체** | "재판관님, 제가 본 것은..." |
| 재판관 → 당사자 | **합니다체 + 존대** | "박지연 씨, 그 부분을 설명해주시겠습니까?" |

### 사건별 호칭

| 사건 | A→B 평소 | B→A 평소 | 격앙 | A→재판관 | B→재판관 |
|------|---------|---------|------|---------|---------|
| **spouse** | 자기야 (자기야/자기가/자기는/자기도) | 자기야 | 이준호!/박지연! | 제 남편 | 제 아내 |
| **family** | 정후야/정후가/정후는 | 형/형이/형은 | 윤정후!/형! | 제 동생 | 제 형 |
| **friend** | 수민아/수민이가/수민이는 | 다은아/다은이가/다은이는 | 최수민!/송다은! | 제 친구 | 제 친구 |

### 어체 판별 핵심

**합니다체 (재판관 대상)**:
- ~습니다, ~입니다, ~했습니다, ~겠습니다, ~없습니다
- ~습니까, ~입니까

**반말 (당사자 간)**:
- ~야, ~잖아, ~거야, ~했어, ~없어, ~인데, ~거든, ~했지, ~아니야
- ~할게, ~할까, ~하자, ~해봐

**절대 금지 조합**:
- "자기, ~~입니다" ← 호칭은 반말인데 어체가 합니다체
- "자기, ~~겁니다" ← 동일
- "정후야, ~~했습니다" ← 동일
- "수민아, ~~입니다" ← 동일

## 채널별 리뉴얼 가이드

### interrogation (NPC 심문 대사)

**재판관에게 하는 말** (대부분의 심문 대사):
- 합니다체 사용
- "재판관님,"은 **첫 발언에서만** 사용. 이후 대사에서는 생략해도 됨
- 쟁점명/증거명을 직접 인용하지 않고, 내용을 풀어서 말함
- 캐릭터가 믿고 있는 사실의 관점에서 말해야 함 (A는 "딴살림 오피스텔", B는 "그 장소")

예:
- ✗ "재판관님, '오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제에 대해서는..."
- ✓ "재판관님, 제 남편이 매일 같은 곳을 드나들고 새벽마다 전화한 건 사실입니다."

**상대방에게 직접 말하는 대사** (끼어들기, 감정 폭발 등):
- 반말 사용
- 호칭 규칙 준수

예:
- ✗ "자기, 제 설명이 부족했다는 건 인정합니다."
- ✓ "자기, 내가 설명 못 한 건 인정해."

### phase1 (초기 진술 — 특별 규칙)

Phase1은 양측이 **처음으로 자기 입장을 밝히는 단계**입니다.

**구조 규칙:**
1. 첫 대사(A/B 각각)는 **재판관에게** 합니다체로. "재판관님,"은 여기서만.
2. 중간 대사(3~6번째)는 **상대방에게 직접** 반말로. 감정이 고조.
3. 마지막 대사는 다시 **재판관에게** 합니다체로 총정리.
4. 시스템 메시지는 **간결한 상황 요약**만. 쟁점명 나열 금지.

**절대 금지:**
- 쟁점명을 따옴표로 감싸서 읊기 ("'오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제")
- 기획 용어 ("실행 책임", "셋째 갈래인", "다음 갈래인")
- 발표 자료 문체 ("위 문제를 같이 놓고 보면")
- 매 대사 "재판관님," 또는 "자기야," 반복 시작

**캐릭터 관점 원칙:**
- A가 B의 행동을 언급할 때: A가 **믿고 있는 사실**로 표현
  - ✗ "'오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제"
  - ✓ "몰래 딴살림을 차린 오피스텔에 드나들고, 새벽마다 통화를 해온 것"
- B가 같은 문제를 언급할 때: **축소/회피** 표현
  - ✓ "이 문제", "그 부분", "그건"
- A도 반복 언급할 때는 "이 문제"로 전환 가능

### behaviorHint (대사 하단 설명)

behaviorHint는 대사 아래에 이탤릭체로 표시되는 **캐릭터의 말투/표정/분위기** 설명입니다.

**올바른 예:**
- "억울함을 앞세우며 자기 행동의 선긋기를 시도한다."
- "남편에게 직접 반말로 몰아붙이며 감정이 터진다."
- "침착하게 선을 긋지만 눈을 마주치지 않는다."

**금지 (프롬프트/메타 정보):**
- ✗ "첫 문장에서는 자기 행동의 출발점과 선긋기를 함께 깐다."
- ✗ "상대가 고정한 프레임을 둘째 쟁점으로 흔든다."
- ✗ "a S0 fact_pursuit ?? 1"

### evidence_present (증거 제시 반응)

- 증거의 **구체적 내용**에 반응해야 함
- "그 자료는..." 같은 범용 반응 금지
- 증거 키워드를 자연스럽게 포함

예:
- ✗ "재판관님, 해당 증거에 대해서는 설명이 필요합니다."
- ✓ "재판관님, 그 카드 명세서를 보면 제가 왜 의심할 수밖에 없었는지 아실 겁니다."

### witness (증인 증언)

- 증인이 **직접 말하는 형태**
- "증인은 ~합니다" 같은 서술자 시점 **절대 금지**
- 재판관에게 합니다체

예:
- ✗ "증인은 삼촌이 저녁마다 방문한 사실을 증언합니다."
- ✓ "재판관님, 삼촌은 저녁마다 오셔서 밥을 해 주셨습니다."

### dossier (DossierCard 반응)

- 기본 증거 질문보다 **더 강한 압박**
- 범용 문장 ("그 질문은 전제가 앞서 있습니다") 금지
- 해당 DossierCard의 **구체적 내용**에 대한 반응

### aftermath (판결 후)

- 판결 결과에 맞는 **구체적** 후일담
- 사건명만 바꾼 범용 문장 금지
- 이 사건의 당사자 이름, 쟁점, 결과가 들어가야 함

### system_message

- 중립 톤 유지
- 정답을 직접 알려주지 않음
- 힌트만 제공

### phase1/phase2 (초기/반박 진술)

- 스크립트 자동 재생 대사
- 호칭/어체 규칙 동일 적용
- A/B가 재판관에게 말하는 부분: 합니다체
- A/B가 상대에게 직접 말하는 부분: 반말

## Truth Throttle (정보 공개 수준)

| LieState | 금액 | 인물 | 기관 | 표현 예시 |
|----------|------|------|------|----------|
| S0-S1 | "그 돈", "해당 금액" | "그 사람", "그쪽" | "그곳" | "그 집에 간 건 맞지만 그런 건 아닙니다." |
| S2 | "몇 천만 원", "꽤 큰 돈" | "가족 쪽" | 약칭 | "가족 일로 간 건 맞습니다." |
| S3+ | 구체적 금액 | 실명 | 정식명칭 | "형네 오피스텔이었습니다." |

## 리뉴얼 절대 금지 사항

1. **쟁점명 따옴표 인용 금지**
   - ✗ "'오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제"
   - ✓ "매일 같은 곳에 드나든 것"

2. **기획 용어 금지**
   - ✗ "실행 책임", "숨은 쟁점", "1차 반전", "Lead", "Dossier"
   - ✓ 일반 한국어로 풀어서

3. **발표 문체 금지**
   - ✗ "셋째 갈래인", "다음 갈래인", "문제는 상대방 측 대응에도"
   - ✓ 구어체로 자연스럽게

4. **호칭+어체 불일치 절대 금지**
   - ✗ "자기, ~~겁니다" / "자기, ~~입니다"
   - ✓ "자기, ~~거야" / "자기, ~~잖아"

5. **번역체 9패턴 금지**
   - "~된 것으로 생각됩니다", "~인 측면이 있었습니다", "부득이하게" 등

6. **JSON 필드명 노출 금지**
   - ✗ "d-1 쟁점에 대해", "e-4 증거를 보면"
   - ✓ 자연스러운 지칭

## 작업 순서

### 1단계: spouse-v3-01 리뉴얼 (먼저 1건)

작업 파일:
- `src/data/scriptedText/spouse-v3-01.json` (interrogation, evidence_present, dossier, witness, aftermath, system_message)
- `src/data/scriptedText/external/spouse-v3-01.json` (동일 내용)
- `src/data/dialogues/phase1/spouse-v3-01.json`
- `src/data/dialogues/phase2/spouse-v3-01.json`

작업 방법:
- 각 채널의 모든 entry를 읽고, 위 가이드에 맞게 표현을 리뉴얼
- 의미/정보 수준은 유지, 표현만 변경
- 리뉴얼 후 즉시 자기 검증: "이 대사를 소리 내어 읽었을 때 사람이 한 말처럼 들리는가?"

### 2단계: CT에 spouse 샘플 제출

spouse-v3-01의 interrogation 중 d-1 S0~S2 (6개 entry × 5 variants = 30개 대사)를 먼저 CT에 제출.
CT가 품질 확인 후 나머지 진행.

### 3단계: spouse 전체 → family → friend

CT 승인 후 spouse 나머지 채널 → family → friend 순서로 진행.

## 핵심 원칙

> **"이 대사를 소리 내어 읽었을 때, 실제 사람이 법정에서 하는 말처럼 들리는가?"**
> 
> 기계가 쓴 느낌이 조금이라도 나면 다시 쓴다.
> 기획 문서를 복붙한 느낌이 나면 다시 쓴다.
> 호칭과 어체가 안 맞으면 다시 쓴다.
