# Session 3 notes — family-01

- `claimPolicies`는 양측 모두 `d-1`~`d-5` 전부를 포함하도록 구성했습니다.  
  - 서아(a): 주쟁점 `d-1/d-3/d-5`, 보조 `d-2/d-4`
  - 도현(b): 주쟁점 `d-2/d-4/d-5`, 보조 `d-1/d-3`
- publicClaim 문장은 재판관 앞 발화를 기준으로 써서 가족 호칭을 반영했습니다.  
  - 서아 쪽은 `제 동생`, `도현이`
  - 도현 쪽은 `제 누나`, `서아 누나`, `누나`
  - 부모 관련은 가능하면 `아버지`, `어머니`, `아버지 관리계좌`로 통일했습니다.
- `RuntimeStartBridge`는 Phase 1 발화 강도를 기준으로 보수적으로 잡았습니다.
  - 서아 `d-1/d-3`는 이미 핵심 해석과 선집행을 공개해 `S2`
  - 도현 `d-2/d-3`도 일정/수첩 해석을 일부 공개해 `S2`
  - 도현 `d-4`는 복지 절차를 아직 직접 언급하지 않아 `S0`
- `EvidenceChallenge`의 `e-4`는 요청대로 `isTrap` 성격을 반영해 `context`를 반드시 포함했고, `restore_context`로 context 방어가 봉쇄되게 설계했습니다.
- `d-4`는 원사건상 `neither_knows`라서, S0~S2의 privateKnowledge는 “모르고 있었던 상태”, S3~S5는 `e-6` 등으로 진실이 드러난 뒤의 상태를 반영했습니다.
