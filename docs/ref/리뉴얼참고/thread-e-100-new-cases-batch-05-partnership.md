# Thread-E 100 New Cases

## 배치 5: 동업 (10건)

### 5-01 장애보고서의 47분 — partnership (`partnership-new-01`)

**훅**: 예약 SaaS 서비스가 47분간 멈춘 날, 주요 고객 세 곳이 동시에 해지 통보를 보낸다. A는 "B가 해지 고객에게 몰래 평생 크레딧을 약속해 회사 장부를 비웠다"고 하고, B는 "A가 장애 원인을 축소해 투자자와 고객 모두를 속였다"고 맞선다.  
파고들수록 A는 서버 장애와 데이터 누락 범위를 의도적으로 줄여 incident report를 썼고, B는 회사 승인 없이 특정 고객에게 오프북 평생 이용권과 해지 위약금 면제를 약속해 자기 거래처를 붙잡고 있었다.

**A**: 송하준(35, 예약 SaaS 공동대표 겸 CTO) — 장애 범위와 데이터 누락 시간을 축소한 보고서를 만든 것 — 거짓말 전략: `cold_logic`  
**B**: 유다은(34, 공동대표 겸 세일즈총괄) — 해지 방지를 명분으로 평생 크레딧과 위약금 면제를 사적으로 약속한 것 — 거짓말 전략: `victim_cosplay`

**쟁점** (5개):
- d-1: 실제 장애 시간과 데이터 누락 범위는 어느 정도였는가
- d-2: B의 평생 크레딧 약속은 긴급 수습이었는가, 임의 계약이었는가
- d-3: A는 왜 incident report를 축소 작성했는가
- d-4: 주요 고객 이탈은 기술 결함 때문이었는가, 수습 실패 때문이었는가
- d-5: 고객 투명성과 회사 생존 중 무엇을 더 무겁게 볼 것인가

**증거 체인**:
- 초기: e-1 공식 장애공지, e-2 고객 해지 메일 3건, e-3 투자자 주간보고 초안
- 1차 해금: e-4 서버 raw 로그와 누락 쿼리 목록 (requires e-1, S1+)
- 2차 해금: e-5 평생 크레딧 약속 메일과 CRM 메모 (requires e-2, S2+)
- 3차 해금: e-6 incident report 수정 이력 PDF (requires e-3, S2+)
- 최종: e-7 위약금 면제 합의서 draft와 고객별 손실 추정표 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → 실제 장애가 공지보다 훨씬 컸다는 점이 드러난다
  - e-2+e-5 → B가 승인 없이 장기 혜택을 약속했다는 사실이 확인된다
  - e-3+e-6 → A가 투자자용 보고에서 핵심 수치를 축소했다는 점이 열린다

**중반 뒤집기**: 초반엔 B의 오프북 계약이 더 직접적 잘못처럼 보이지만, e-4와 e-6이 열리면 A의 축소 보고가 고객 불신의 출발점이었다는 점이 드러난다. 반대로 e-5와 e-7이 열리면 B도 회사 리스크를 떠안긴 사적 구제 합의를 했음이 확정된다.

**판결 딜레마**: `투명성 vs 생존`, `현재 오프북 계약 vs 과거 축소 보고`, `고객 신뢰 vs 회사 존속`이 충돌한다. A가 먼저 진실을 숨겼더라도 B의 임의 합의가 정당화되지는 않는다.

**난이도**: hard

### 5-02 몰드 넘버 M-12 — partnership (`partnership-new-02`)

**훅**: 향초 브랜드 리뉴얼 직전, OEM 공장에서 같은 몰드 번호가 붙은 제품이 다른 스토어에 먼저 풀린다. A는 "B가 우리 공동 몰드를 빼돌려 개인 라인을 찍었다"고 하고, B는 "A가 반품 재고를 불량 처리한 뒤 따로 팔아 차액을 챙겼다"고 맞선다.  
추궁해 보면 A는 반품 재고를 scrap 처리해 정산에서 빼고 있었고, B는 실제로 공동 금형으로 별도 소량 생산을 돌리며 배우자 스토어에서 선판매했다.

**A**: 윤민서(33, 향초 브랜드 디렉터) — 반품 재고 일부를 불량 scrap으로 처리해 별도 처분한 것 — 거짓말 전략: `avoidant`  
**B**: 강태훈(36, 생산·물류 공동대표) — 공동 금형으로 별도 소량 생산을 돌려 배우자 스토어에 넘긴 것 — 거짓말 전략: `confrontational`

**쟁점** (5개):
- d-1: 공동 몰드로 별도 생산이 실제로 이뤄졌는가
- d-2: A의 반품 재고 처리 방식은 정당한 손실 정리였는가
- d-3: B의 개인 라인 시도는 시장 테스트였는가, 공동 자산 침해였는가
- d-4: 반품 재고와 금형 사용권을 누가 더 먼저 사유화했는가
- d-5: 브랜드 자산과 재고 손실 보전 중 무엇을 더 무겁게 볼 것인가

**증거 체인**:
- 초기: e-1 OEM 생산 일정표, e-2 반품·불량 처리 보고서, e-3 배우자 스토어 판매 캡처
- 1차 해금: e-4 금형 사용 로그와 생산 배치표 (requires e-1, S1+)
- 2차 해금: e-5 scrap 재고 이동 전표와 창고 출고 CCTV 스냅샷 (requires e-2, S2+)
- 3차 해금: e-6 별도 스토어 정산표와 배송라벨 묶음 (requires e-3, S2+)
- 최종: e-7 OEM 카톡방 합의 메모와 향 조합 샘플 승인서 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → B가 공동 금형으로 별도 생산을 돌린 사실이 드러난다
  - e-2+e-5 → A가 불량 처리 명목으로 재고를 빼돌린 점이 확인된다
  - e-3+e-6 → B의 개인 판매가 테스트 수준이 아니라 실제 매출화였다는 점이 열린다

**중반 뒤집기**: 초반엔 B의 금형 사유화가 핵심처럼 보이지만, e-5가 열리면 A도 재고를 불량 처리로 돌려 별도 현금화했다는 점이 드러나 공방이 뒤집힌다.

**판결 딜레마**: `공동 자산 침해 vs 손실 회수`, `현재 개인 라인 선판매 vs 과거 재고 전용`, `브랜드 보호 vs 현금흐름`이 충돌한다. 누가 먼저 공동 자산을 자기 사업처럼 썼는지가 판결을 가른다.

**난이도**: hard

### 5-03 관세표의 두 줄 — partnership (`partnership-new-03`)

**훅**: 수입 유아용품 컨테이너가 통관 보류되며 과징금 예고가 뜬다. A는 "B가 AS 교환 재고를 몰래 가족 스토어에 팔아 창고 수량이 안 맞는다"고 하고, B는 "A가 관세 분류를 의도적으로 낮춰 적었다"고 맞선다.  
실제로는 A가 번들 부품을 따로 빼서 낮은 세율로 신고했고, B는 무상교환용 재고를 가족 명의 스토어에 소진 처리하며 현금화하고 있었다.

**A**: 김도윤(38, 해외소싱 공동대표) — 부품 분리 신고로 관세 분류를 낮춘 것 — 거짓말 전략: `premature_summary`  
**B**: 이수아(35, CS·운영 공동대표) — AS 교환 재고를 가족 스토어에 넘기고도 회수 불가 손실처럼 처리한 것 — 거짓말 전략: `affect_flattening`

**쟁점** (4개):
- d-1: 통관 신고 분류는 어디까지 의도적 축소였는가
- d-2: AS 교환 재고는 실제로 어디로 빠졌는가
- d-3: A의 신고는 생존형 절감이었는가, 명백한 허위 신고였는가
- d-4: B의 재고 유출은 손실 보전이었는가, 사적 유용이었는가

**증거 체인**:
- 초기: e-1 세관 보완 요구서, e-2 창고 재고표, e-3 가족 스토어 상품 캡처
- 1차 해금: e-4 수입신고 세부항목과 분리 포장 명세서 (requires e-1, S1+)
- 2차 해금: e-5 교환재고 출고 로그와 택배 송장 (requires e-2, S2+)
- 3차 해금: e-6 원산지·구성품 광고 문구와 CS 응대 매뉴얼 (requires e-3, S2+)
- 최종: e-7 과징금 계산표와 가족 스토어 정산 계좌흐름 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → A의 낮은 세율 신고가 의도적 설계였다는 점이 드러난다
  - e-2+e-5 → B가 교환 재고를 실제로 외부 판매로 돌렸다는 사실이 확인된다
  - e-3+e-6 → B가 교환용 재고를 판매하면서도 소비자 안내와 다른 문구를 썼다는 점이 열린다

**중반 뒤집기**: 초반엔 A의 허위 신고가 관세 리스크의 핵심으로 보이지만, e-5와 e-7이 열리면 B도 재고를 별도 현금화해 창고 차이를 키운 당사자였다는 사실이 드러난다.

**판결 딜레마**: `제도 준수 vs 사업 생존`, `통관 허위 vs 재고 유용`, `현재 과징금 리스크 vs 장기 AS 신뢰 붕괴`가 충돌한다. 어느 위반이 더 근본적으로 사업을 무너뜨렸는지가 갈린다.

**난이도**: medium

### 5-04 프리징 링크 14자리 — partnership (`partnership-new-04`)

**훅**: 필라테스 스튜디오 VIP 회원권이 대량 동결되며 현금흐름이 꼬인다. A는 "B가 개인 링크로 비공식 PT를 받아 회원을 빼돌린다"고 하고, B는 "A가 노쇼 패널티를 과다 부과해 장부상 매출을 부풀린다"고 맞선다.  
실제로는 A가 노쇼 기준을 임의로 강화해 선결제 환불을 막았고, B는 공식 예약을 우회하는 프리징 링크로 현금 PT와 사적 회원관리를 하고 있었다.

**A**: 박지해(34, 필라테스 공동대표) — 노쇼 패널티 규정을 강화해 환불을 막고 장부 매출을 유지한 것 — 거짓말 전략: `victim_cosplay`  
**B**: 윤서호(33, 공동대표 겸 메인강사) — 비공식 프리징 링크로 현금 PT와 회원 이동을 관리한 것 — 거짓말 전략: `cold_logic`

**쟁점** (5개):
- d-1: 프리징된 회원권은 왜 대량으로 발생했는가
- d-2: A의 노쇼 패널티 강화는 규정 관리였는가, 매출 방어였는가
- d-3: B의 프리징 링크는 편의 제공이었는가, 회원 빼돌리기였는가
- d-4: 환불·동결·현금 PT 중 무엇이 더 심각한 장부 왜곡인가
- d-5: 스튜디오 생존과 회원 신뢰 중 무엇을 더 우선해야 하는가

**증거 체인**:
- 초기: e-1 회원권 프리징 리스트, e-2 환불 요청 메일, e-3 비공식 예약 링크 스크린샷
- 1차 해금: e-4 노쇼 패널티 규정 수정 이력 (requires e-2, S1+)
- 2차 해금: e-5 현금 PT 정산 메모와 회원 개인 DM (requires e-3, S2+)
- 3차 해금: e-6 POS 매출표와 프리징 전후 예약캘린더 (requires e-1, S2+)
- 최종: e-7 강사별 커미션 산정표와 VIP 이관 리스트 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-2+e-4 → A가 노쇼 기준을 조용히 바꿨다는 점이 드러난다
  - e-3+e-5 → B의 비공식 링크가 실제 회원 이동 통로였음이 확인된다
  - e-1+e-6 → 프리징 증가와 환불 억제가 현금흐름 방어와 연결됐다는 구조가 열린다

**중반 뒤집기**: 초반엔 B의 사적 회원 관리가 더 문제처럼 보이지만, e-4와 e-6이 열리면 A도 환불을 막고 장부를 예쁘게 유지하려 규정을 바꿨다는 사실이 드러난다.

**판결 딜레마**: `회원 신뢰 vs 사업 생존`, `규정 남용 vs 사적 현금화`, `절차 위반 vs 실질적 이탈 유도`가 충돌한다. 누가 먼저 회원을 '자산'처럼 다뤘는지가 핵심이 된다.

**난이도**: hard

### 5-05 야간환불 단말기 — partnership (`partnership-new-05`)

**훅**: 백화점 팝업 종료 후 카드단말기엔 심야 부분취소가 연달아 찍혀 있고, 보증금은 예상보다 훨씬 적게 돌아온다. A는 "B가 백화점 정산 전에 케이터링 선금을 자기 계좌로 돌렸다"고 하고, B는 "A가 밤마다 환불단말기로 재고 부족을 덮었다"고 맞선다.  
실제로는 A가 품절 클레임을 조용히 막으려 수동 부분취소를 반복했고, B는 행사 외부 케이터링 계약금 일부를 개인 계좌로 빼서 인건비를 우선 정리했다.

**A**: 홍수인(31, 디저트 브랜드 공동대표) — 품절 클레임을 숨기기 위해 심야 부분취소를 반복한 것 — 거짓말 전략: `affect_flattening`  
**B**: 신우재(33, 오프라인 운영 공동대표) — 케이터링 선금 일부를 개인 계좌로 받아 인건비와 개인채무를 섞어 쓴 것 — 거짓말 전략: `avoidant`

**쟁점** (4개):
- d-1: 심야 부분취소는 고객 구제였는가, 품절 은폐였는가
- d-2: 케이터링 선금 개인수령은 긴급지급이었는가, 공금 혼용이었는가
- d-3: 팝업 보증금 부족의 주된 원인은 어느 쪽 행동인가
- d-4: 고객 신뢰와 직원·협력업체 지급 중 무엇을 더 우선해야 하는가

**증거 체인**:
- 초기: e-1 카드단말 부분취소 내역, e-2 케이터링 선금 송금 캡처, e-3 백화점 정산서
- 1차 해금: e-4 품절 클레임 DM와 재고표 (requires e-1, S1+)
- 2차 해금: e-5 개인계좌 입금 후 이체흐름 표 (requires e-2, S2+)
- 3차 해금: e-6 팝업 출고 누락 메모와 행사장 CCTV 스냅샷 (requires e-3, S2+)
- 최종: e-7 아르바이트 인건비 정산표와 환불 승인 규정 메모 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → A가 품절 사실을 조용히 덮으려 환불을 돌렸다는 점이 드러난다
  - e-2+e-5 → B가 선금을 개인 채무와 함께 흘려 쓴 사실이 확인된다

- d-5: 현재 무단 공유와 과거의 체크박스 허위 중 무엇을 더 무겁게 볼 것인가

**증거 체인**:
- 초기: e-1 시범사업 제안서 PDF, e-2 데이터룸 접근 안내 메일, e-3 비식별 체크 화면 캡처
- 1차 해금: e-4 CSV 샘플 원본과 식별필드 표 (requires e-1, S1+)
- 2차 해금: e-5 인수사 미팅 메모와 잔류보너스 draft (requires e-2, S2+)
- 3차 해금: e-6 제품팀 전달 슬랙·체크박스 클릭 로그 (requires e-3, S2+)
- 최종: e-7 churn 예측표와 병원별 pilot 동의 범위 비교표 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → A가 비식별됐다고 넘긴 파일에 실제 식별정보가 남아 있었음이 드러난다
  - e-2+e-5 → B가 투자 검토를 넘어 사전 조건 협상을 진행했다는 사실이 확인된다
  - e-3+e-6 → A의 체크박스 허위가 실수 수준이 아니었음을 보여준다

**중반 뒤집기**: 초반엔 B의 무단 데이터룸 공유가 더 위험하게 보이지만, e-4와 e-6이 열리면 A도 시작점에서 데이터 안전 기준을 허위로 통과시킨 당사자였다는 점이 드러난다.

**판결 딜레마**: `이용자 정보 보호 vs 회사 존속`, `현재 무단 공유 vs 과거 허위 체크`, `투자 유치 vs 동의 원칙`이 충돌한다. B의 사전 협상이 문제여도 A의 체크박스 허위는 별개로 무겁고, A의 허위가 있어도 B의 데이터룸 확장은 자동 정당화되지 않는다.
