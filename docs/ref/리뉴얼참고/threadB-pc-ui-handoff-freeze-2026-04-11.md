# Thread B PC UI Freeze Handoff

작성 시각: 2026-04-11  
작성 목적: 현재 PC UI 작업 상태를 동결하고, 다른 스레드가 이어서 수정할 수 있도록 실제 상태와 남은 문제를 정리

## 1. 현재 동결 대상

현재 PC UI는 아래 파일 기준으로 마지막 수정이 반영된 상태다.

- `D:/ProjectWS/src/app/pc.css`
- `D:/ProjectWS/src/components/pc/hotbar/PCBottomDock.tsx`
- `D:/ProjectWS/src/components/pc/layout/PCDialogueLog.tsx`
- `D:/ProjectWS/src/components/pc/layout/PCCourtLayout.tsx`
- `D:/ProjectWS/src/components/pc/panels/PCLeftPanel.tsx`
- `D:/ProjectWS/src/components/pc/panels/PCRightPanel.tsx`

이 중 가장 많이 꼬여 있는 파일은 `pc.css`다.  
핫바, 채팅, 좌우 패널 관련 override가 누적되어 있어, 의도한 비율과 실제 렌더가 자주 어긋난다.

## 2. 현재 빌드 상태

아래는 현재 기준 통과 상태다.

- `npx tsc -p tsconfig.app.json --noEmit` PASS
- `npm run build:pc` PASS

남아 있는 경고:

- Pretendard `@import` 순서 경고 1건
- `scriptedTextLoader` 관련 `INEFFECTIVE_DYNAMIC_IMPORT` 경고 다수
- chunk size 경고

즉, 현재 문제는 빌드/타입 에러가 아니라 **렌더 비율과 레이아웃 품질** 문제다.

## 3. 현재 가장 큰 미해결 문제

### 3-1. 하단 핫바 비율 불일치

사용자가 전달한 기준 이미지는:

- 좌우 캐릭터 카드가 더 좁다
- 중앙 핫바는 가로로 넓고 세로로 낮다
- 슬롯 내부 아이콘/글자는 유지하되, 슬롯의 위아래 여백만 줄어야 한다
- `F1/F2/F3`, `1~6`, `ESC`는 상단에 얹힌 얕은 키캡/배지 형태여야 한다
- 좌우 카드와 핫바 사이에 약간의 간격만 있어야 한다

현재 실제 화면은:

- 좌우 캐릭터 카드 폭이 여전히 크다
- 핫바 세로폭이 여전히 높다
- 슬롯 내부 비율이 레퍼런스와 다르다

### 3-2. `pc.css` 핫바 레이어 중복

`pc.css` 안에 `pc-play-hbar`, `slot`, `char` 관련 블록이 매우 많다.  
실제로 마지막에 먹는 블록은 파일 후반부의 최종 override인데, 이전 블록들과 합쳐져 디버깅이 어렵다.

핫바 관련 최종 블록은 대략 아래 구간이다.

- `D:/ProjectWS/src/app/pc.css`
- 약 `6932` 라인 이후

다음 담당 스레드는 핫바를 계속 미세조정하기 전에:

1. 이전 중복 블록을 정리하거나
2. 최종 레이어를 더 강한 selector로 단일화하거나
3. 아예 hotbar 전용 section을 마지막에 다시 재작성

하는 것이 안전하다.

### 3-3. 사용자 요구와 실제 반영의 반복 불일치

사용자는 “아이콘/폰트 크기를 줄이라는 게 아니라 여백만 줄이라”고 명시했다.  
하지만 실제 수정 과정에서 아이콘/폰트까지 같이 줄어 사용자가 크게 불만을 표했다.

즉, 다음 작업자는 **비율 조정 시 내부 콘텐츠 크기와 외곽 패딩을 분리해서 다뤄야 한다.**

## 4. 사용자가 마지막으로 명확히 요구한 하단 핫바 기준

아래 기준은 최신 사용자 의도다.

### 핫바

- 캐릭터 카드 가로폭은 훨씬 더 좁아야 한다
- 캐릭터 얼굴은 현재보다 더 크게 유지 또는 확대
- 핫바 전체 높이는 훨씬 더 낮아야 한다
- 슬롯 높이는 줄이되, 아이콘/폰트는 함부로 줄이지 않는다
- 조정 포인트는 “위아래 여백/패딩”
- 슬롯과 키캡은 핫바 영역 안에 자연스럽게 들어와야 한다
- 좌우 캐릭터 카드와 중앙 핫바는 약간의 간격만 두고 나란히 배치

### 채팅

- 채팅 전체 폭을 줄이는 게 아님
- 말풍선 폭만 줄이고, A는 좌측 끝, B는 우측 끝에서 시작
- 화자 이름은 말풍선 위가 아니라 캐릭터 이미지 아래

### 좌측 패널

- 타임라인은 좌측 패널 안의 고정 영역이 아니라
- 별도 북마크/토글 버튼을 눌렀을 때 바깥으로 나오는 토스트 패널 형태

## 5. 이번 턴까지 반영된 것 중 유지 가능한 것

완전히 버릴 필요는 없는 부분:

- `PCDialogueLog.tsx`는 깨진 문자열을 정리한 버전으로 재작성되어 있음
- A/B 말풍선 좌우 분리 구조 자체는 이미 있음
- `PCBottomDock.tsx` 마크업 구조는 레퍼런스와 크게 다르지 않음
- 빌드는 통과하는 상태

즉, 구조보다 **스타일 계층 재정리**가 더 중요하다.

## 6. 다음 담당 스레드 권장 작업 순서

1. `pc.css`의 핫바 관련 중복 섹션 확인
2. 핫바 최종 레이어를 단일 블록으로 재정의
3. 캐릭터 카드 폭, 핫바 높이, 슬롯 패딩만 먼저 잡기
4. 그 다음 채팅 말풍선 폭/정렬 조정
5. 마지막으로 좌측 타임라인 팝업 위치 조정

## 7. 참고 이미지 의미

사용자가 보낸 참조 이미지는 “색상”을 복제하라는 뜻이 아니라:

- 레이아웃
- 위치 배치
- 카드 비율
- 높이와 폭의 관계

를 맞추라는 뜻이다.

컬러 팔레트는 기존 프로젝트 톤을 유지해야 한다.

## 9. 결론

현재 상태는 **기능은 살아 있고 빌드도 되지만, 플레이 화면 특히 하단 핫바의 비율이 사용자가 원하는 레퍼런스와 다르다.**  
다음 담당자는 기능 수정이 아니라 **스타일 계층 정리와 비율 재설계**에 집중해야 한다.
