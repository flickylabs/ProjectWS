# 세션 2: family-01 V2 Atom + V3 Game Loop

## 프로젝트 배경

**Solomon**은 LLM 기반 법정 심문 게임입니다. 플레이어는 재판관으로서 양측 당사자(AI NPC)를 심문하여 진실을 밝힙니다.

### 핵심 아키텍처
- **엔진이 의미를 결정하고, LLM은 연기만 합니다.**
- NPC의 발화는 `ClaimAtom`(의미 단위) + `Slot`(표현 층) 조합으로 구성됩니다
- 매 턴: `BlueprintEngine`이 stance/defenseMode 결정 → `AtomSelectionEngine`이 atom 선택 → LLM이 연기
- NPC는 `lieState` 상태 머신(S0 완전부정 → S5 시인)을 따릅니다

### V2 Atom 시스템
기존 publicClaim(완성 문장)을 원자 단위(atom)로 분해한 구조입니다:
- `ClaimAtom`: 의미 단위 ("280만원을 송금한 사실", "추석 연휴 간병 공백")
- `Slot`: 표현 변형 (exact: "2,800,000원", neutral: "해당 금액", judgeRef: "장모님")
- `Tag`: 분류 (act, timeline, motive, emotion, rule, evidence 등 24종)
- `stanceHints`: 이 atom이 잘 어울리는 stance (deny, partial, confess 등)

### V3 게임 루프
- **DossierCard**: 증거 2개를 묶은 카드 → challenge 질문으로 벡터 공격
- **StateUnlockAtom**: state 진입 시 해금되는 새 사실 (S2~S5)
- **이벤트**: 모순 감지/끼어들기/감정 폭발 → 플레이어 선택지 2개
- **TransitionBeat**: lieState 전이 순간 강제 삽입 사전 작성 대사

## 산출물 1: family-01-v2-atoms.json

### V2 ClaimPolicy Atom 구조

기존 family-01-data.json의 publicClaim(완성 문장)을 atom으로 분해합니다.

**범위**: a(d-1, d-3, d-5) + b(d-2, d-4, d-5) × S0~S5
**목표**: 쟁점당 15~25개 atom → 총 약 120~150개

### atom 분해 규칙
1. publicClaim의 각 문장을 1~2개 atom으로 분해
2. 숫자/날짜/인명은 반드시 slot으로 분리
3. 태그는 내용에 맞게 복수 부여 (act, timeline, motive, emotion, rule 등)
4. S0~S1: deny/hedge용 atom (핵심 회피)
5. S2~S3: partial/blame용 atom (부분 인정 + 전가)
6. S4~S5: emotional/confess용 atom (감정 + 시인)
7. privateKnowledge → 고 state에서 해금되는 atom의 factText 소재
8. suppressions → 해당 state에서 forbiddenAtomIds로 처리 (atom 자체를 만들 필요 없음)
