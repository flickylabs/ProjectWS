## spouse-05 — 잠긴 서재의 31초

### 사건 훅
배우자가 잠긴 서재에 몰래 들어갔고, 그 뒤 31초짜리 녹취를 근거로 "상대가 먼저 거짓말했다"고 몰아붙인다.  
하지만 추궁이 깊어질수록, 서재 주인도 배우자에게 숨긴 문서와 돈의 흐름이 있었고, 31초 녹취는 여러 구간을 이어붙인 편집본이라는 사실이 드러난다.

### 양측 프로필
- A: 서재를 잠근 배우자, 숨긴 것=`서재 안 문서와 자금 관련 비밀`, archetype=`cold_logic`
- B: 무단 출입한 배우자, 숨긴 것=`31초 녹취 편집과 선제 프레이밍`, archetype=`victim_cosplay`

### 쟁점 구조
- d-1: 잠긴 서재는 정당한 사적 공간이었는가, 부부 공동 의사결정에서 숨긴 공간이었는가
- d-2: B의 무단 출입은 우발적 확인이었는가, 계획적 침범이었는가
- d-3: 31초 녹취는 원본 발언인가, 편집본인가
- d-4: A는 왜 서재 안 문서와 송금 정황을 배우자에게 숨겼는가
- d-5: B는 왜 원본 확인 전에 녹취를 결론 증거처럼 내밀었는가

### 증거 설계
- 초기 공개: e-1 31초 녹취 파일, e-2 서재 도어락 기록, e-3 문제의 문서 사진 2장
- 1차 해금(S1+): e-4 녹취 원본 파일 메타데이터
  - requires: `e-1`
- 2차 해금(S2+): e-5 서재 안 송금 메모와 계좌 스크랩
  - requires: `e-3`
- 3차 해금(S2+): e-6 도어락 비상번호 공유 흔적
  - requires: `e-2`
- 최종 해금(S4+): e-7 원본 대화 전체 녹음
  - requires: `e-4`, `e-5`
- 증거 조합:
  - `e-1 + e-4` → 31초 녹취가 편집본이라는 사실이 열린다
  - `e-3 + e-5` → A가 숨긴 비밀이 실제였음을 보여준다
  - `e-2 + e-6` → 완전한 외부 침입이 아니라, 관계 파탄 속 공유 권한 악용이었음이 드러난다

### 거짓말 상태 전이 설계
- A / d-4:
  - S0→S1: `direct_question`으로 "정리 중인 문서가 있었다" 일부 인정
  - S1→S2: `e-5` 제시 시 "괜히 불안을 줄까 봐 숨겼다" 핑계
  - S2→S3: `responsibility_question`으로 B의 무단 출입 탓으로 전가
  - S3→S5: `e-7` 제시 시 숨긴 자금/문서의 실체를 자백
- B / d-3:
  - S0→S1: `direct_question`으로 "핵심 부분만 들려준 것" 인정
  - S1→S2: `e-4` 제시 시 "맥락은 같았다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-7` 제시 시 편집과 선제 프레이밍을 자백

### 핵심 모순 쌍
- A는 "사생활 보안"이라 말하지만 `e-5`는 배우자에게 직접 영향이 가는 비밀을 보여준다
- B는 "원본 취지 그대로"라 말하지만 `e-1 + e-4`는 짜깁기를 입증한다
- B는 "침입이 아니다"라 말하지만 `e-2 + e-6`은 권한 악용을 드러낸다

### 판결 딜레마
`배우자에게 중대한 사실을 숨긴 통제`를 더 무겁게 볼지, `무단 출입과 편집 증거`를 더 무겁게 볼지 갈린다.  
과정 중시 재판관은 침범과 증거 조작을 크게 볼 수 있고, 직관형 재판관은 A의 은닉 동기를 더 의심할 수 있다.

## partnership-04 — 편집된 거래내역

### 사건 훅
동업 해지 협상장에서 A는 은행 거래내역 캡처를 내밀며 "B가 먼저 돈을 빼돌렸다"고 주장한다.  
하지만 그 캡처는 일부 행이 잘린 편집본이고, 그렇다고 해서 B의 과거 배신과 별도 거래 시도가 사라지지도 않는다.

### 양측 프로필
- A: 편집된 거래내역을 들고 나온 동업자, 숨긴 것=`현재 협상 우위를 위해 증거를 가공한 점`, archetype=`premature_summary`
- B: 과거 별도 거래와 신뢰 파탄을 만든 동업자, 숨긴 것=`예전의 실제 배신과 방어적 은폐`, archetype=`avoidant`

### 쟁점 구조
- d-1: 현재 제시된 거래내역은 원본인가
- d-2: B는 과거에 실제로 별도 거래나 자금 우회 시도를 했는가
- d-3: A는 왜 현재 협상에서 편집 증거를 썼는가
- d-4: 과거 배신은 현재 조작 증거를 정당화할 수 있는가
- d-5: 결별 비용과 정산 책임을 누가 더 크게 져야 하는가

### 증거 설계
- 초기 공개: e-1 거래내역 캡처본, e-2 동업 해지안, e-3 과거 거래처 메시지 일부
- 1차 해금(S1+): e-4 거래내역 CSV 원본
  - requires: `e-1`
- 2차 해금(S2+): e-5 과거 별도 견적서와 입금 예약 흔적
  - requires: `e-3`
- 3차 해금(S2+): e-6 정산 직전 내부 메신저
  - requires: `e-2`
- 최종 해금(S4+): e-7 "이번엔 이렇게 밀자" 증거 편집 합의 메모
  - requires: `e-4`, `e-6`
- 증거 조합:
  - `e-1 + e-4` → 현재 증거가 편집본이라는 사실이 드러난다
  - `e-3 + e-5` → B의 과거 배신이 실제였음이 열린다
  - `e-6 + e-7` → A가 현재 조작을 의식적으로 택했다는 사실이 굳어진다

### 거짓말 상태 전이 설계
- A / d-3:
  - S0→S1: `direct_question`으로 "보기 쉽게 정리한 것" 일부 인정
  - S1→S2: `e-4` 제시 시 "핵심 의미는 같았다" 핑계
  - S2→S3: `e-5` 제시 시 "상대가 먼저 배신했다" 책임 전가
  - S3→S5: `e-7` 제시 시 협상 우위 목적의 편집을 자백
- B / d-2:
  - S0→S1: `direct_question`으로 "외부 접촉은 있었다" 인정
  - S1→S2: `e-5` 제시 시 "회사를 살리려던 대비였다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-6` 제시 시 과거 방어적 은폐를 자백

### 핵심 모순 쌍
- A는 "상대 배신을 드러내려 했을 뿐"이라 말하지만 `e-7`은 현재 조작 의도를 보여준다
- B는 "실제 실행은 없었다"고 하지만 `e-5`는 과거 선행 배신을 입증한다
- A의 현재 조작과 B의 과거 배신이 서로를 완전히 지워주지 않는다

### 판결 딜레마
`지금 조작된 증거를 쓰는 행위`를 더 중하게 볼지, `예전에 실제로 공동 신뢰를 깨뜨린 행위`를 더 중하게 볼지 갈린다.  
논리형 재판관은 현재 증거 진정성을 엄격히 볼 수 있고, 직관형 재판관은 과거 배신의 무게를 더 크게 느낄 수 있다.

## workplace-05 — 원출처 없는 승진안

### 사건 훅
승진 브리핑 날, A는 B가 공동기획안을 자기 이름으로 올렸다고 폭로한다.  
그런데 원안을 더 파면, B만 문제가 아니고 A 역시 2년 전 외부 컨설턴트의 아이디어 흔적을 의도적으로 지운 채 `내가 원래 만든 기획`처럼 포장해 왔다는 사실이 드러난다.

### 양측 프로필
- A: 피해자 프레임을 선점한 실무자, 숨긴 것=`더 오래된 원출처를 알고도 지운 점`, archetype=`victim_cosplay`
- B: 현재 브리핑에서 공을 가로챈 상급자, 숨긴 것=`공동기획 표기 삭제와 승진용 독식`, archetype=`cold_logic`

### 쟁점 구조
- d-1: 승진 브리핑에서 실제 공동기획 흔적이 삭제됐는가
- d-2: 아이디어의 더 오래된 원출처는 어디인가
- d-3: A는 왜 원출처 문제를 지금까지 숨겼는가
- d-4: B는 왜 공동 작업을 단독 실적으로 포장했는가
- d-5: 현재의 승진 공정성과 과거의 창작 윤리 중 무엇을 더 무겁게 볼 것인가

### 증거 설계
- 초기 공개: e-1 승진 브리핑 슬라이드, e-2 A의 개인 노트, e-3 회의 참석자 메모
- 1차 해금(S1+): e-4 브리핑 수정 이력 PDF
  - requires: `e-1`
- 2차 해금(S2+): e-5 2년 전 보류 파일과 외부 컨설턴트 코멘트
  - requires: `e-2`
- 3차 해금(S2+): e-6 공동기획 메신저 기록
  - requires: `e-3`
- 최종 해금(S4+): e-7 A의 원출처 인지 메일
  - requires: `e-5`, `e-6`
- 증거 조합:
  - `e-1 + e-4` → B가 현재 표기를 지웠다는 사실이 드러난다
  - `e-2 + e-5` → A의 아이디어 순수성 주장에 균열이 생긴다
  - `e-6 + e-7` → A도 원출처 문제를 알고 선택적으로 침묵했음이 굳어진다

### 거짓말 상태 전이 설계
- B / d-4:
  - S0→S1: `direct_question`으로 "최종 정리 과정에서 이름이 빠졌다" 인정
  - S1→S2: `e-4` 제시 시 "리더 역할을 대표 표기로 쓴 것" 핑계
  - S2→S3: `responsibility_question`으로 A의 원출처 문제를 들며 책임 전가
  - S3→S5: `e-6` 제시 시 공동기획 삭제를 자백
- A / d-3:
  - S0→S1: `empathy_question`으로 "지금 그걸 꺼내면 내 기여도 사라질까 봐" 일부 인정
  - S1→S2: `e-5` 제시 시 "당시엔 참고자료 수준이라 생각했다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-7` 제시 시 원출처 인지와 침묵을 자백

### 핵심 모순 쌍
- B는 "대표 표기였을 뿐"이라 말하지만 `e-1 + e-4`는 실질 삭제를 보여준다
- A는 "완전한 피해자"라 말하지만 `e-5 + e-7`은 원출처 인지와 은닉을 보여준다
- 현재 승진 조작과 과거 창작 윤리 위반이 서로를 면책하지 않는다

### 판결 딜레마
`지금 벌어진 공로 독식`을 더 중하게 볼지, `더 오래된 원출처 은닉`을 더 중하게 볼지 갈린다.  
과정 중시 재판관은 현재 승진 절차 왜곡을, 원칙 중시 재판관은 창작 출처 은닉을 더 크게 볼 수 있다.

## workplace-12 — 항상 켜진 웹캠

### 사건 훅
팀장은 재택 근무자의 웹캠 상시 연결과 상태 스크린샷 제출을 요구한다.  
직원은 이를 과잉 감시라고 폭로하지만, 추궁이 깊어질수록 마우스 지글러와 편집된 상태 캡처, 실제 성과 공백이 함께 드러난다.

### 양측 프로필
- A: 상시 모니터링을 밀어붙인 관리자, 숨긴 것=`과잉 감시와 비공식 압박`, archetype=`confrontational`
- B: 감시 피해를 주장하는 직원, 숨긴 것=`근무 공백 은폐와 편집 증거 사용`, archetype=`affect_flattening`

### 쟁점 구조
- d-1: 웹캠 상시 요구와 상태 스크린샷 제출은 정당한 관리였는가
- d-2: B의 근무 공백은 실제였는가, 관리자의 과장인가
- d-3: 제출된 상태 캡처는 원본인가 편집본인가
- d-4: A는 왜 HR 절차 밖에서 개별 압박을 했는가
- d-5: 성과 관리 실패와 사생활 침해 중 무엇을 더 무겁게 볼 것인가

### 증거 설계
- 초기 공개: e-1 관리자 요구 메일, e-2 상태 스크린샷 묶음, e-3 주간 업무 산출물 요약
- 1차 해금(S1+): e-4 화상회의 녹화 일부
  - requires: `e-1`
- 2차 해금(S2+): e-5 마우스 지글러 설치 기록
  - requires: `e-2`
- 3차 해금(S2+): e-6 HR 상담 PDF
  - requires: `e-4`
- 최종 해금(S4+): e-7 편집 전 원본 상태 로그
  - requires: `e-2`, `e-5`
- 증거 조합:
  - `e-1 + e-6` → A의 관리가 공식 절차 밖 압박까지 갔음을 보여준다
  - `e-2 + e-7` → B의 제출 캡처가 편집됐음을 드러낸다
  - `e-3 + e-5` → 실제 성과 공백이 일부 있었다는 점이 강화된다

### 거짓말 상태 전이 설계
- A / d-4:
  - S0→S1: `direct_question`으로 "성과 확인 차원이었다" 인정
  - S1→S2: `e-6` 제시 시 "팀 일정이 급해서 그랬다" 핑계
  - S2→S3: `responsibility_question`으로 B의 근무 공백 탓으로 전가
  - S3→S5: `e-1 + e-6` 재제시 시 비공식 압박을 자백
- B / d-3:
  - S0→S1: `direct_question`으로 "보기 쉽게 정리한 캡처" 인정
  - S1→S2: `e-7` 제시 시 "핵심은 같았다" 핑계
  - S2→S4: `emotion_threshold`로 무감정이 무너짐
  - S4→S5: `e-5` 제시 시 근무 공백 은폐를 자백

### 핵심 모순 쌍
- A는 "정상적 관리"라 말하지만 `e-1 + e-6`은 절차 밖 감시 압박을 보여준다
- B는 "완전한 감시 피해자"라 말하지만 `e-2 + e-7`은 편집 증거를 보여준다
- B는 "업무는 다 했다"고 하지만 `e-3 + e-5`는 실제 공백을 시사한다

### 판결 딜레마
`사생활을 침범한 관리 방식`을 더 크게 볼지, `감시를 핑계로 성과 공백을 숨긴 행위`를 더 크게 볼지 갈린다.  
해결 중시 재판관은 관리 체계 개선과 재발 방지를 우선할 수 있고, 과정 중시 재판관은 감시 방식 자체를 더 엄격히 볼 수 있다.
