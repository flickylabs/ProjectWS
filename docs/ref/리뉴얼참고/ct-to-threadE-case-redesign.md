# 게임 이해 가이드

> 이 섹션을 먼저 읽고 게임을 충분히 이해한 뒤 작업을 시작하세요.

## 1. 게임 한 줄 요약

**솔로몬** — AI 캐릭터 둘이 갈등하고, 플레이어(재판관)가 심문·증거·중재로 진실을 밝히는 리플레이형 추리 게임.

## 2. 재판 루프 (8단계)

```
Phase 0: 사건 소개 — 배경/인물 파악. "이 사건이 뭔지" 이해
Phase 1: 초기 진술 — 양측 각자의 입장 듣기 (스크립트 자동 재생)
Phase 2: 반박 진술 — 상대 진술에 대한 반박 (스크립트 자동 재생)
Phase 3: 심문     — ★ 플레이어가 질문하여 진실 추궁 (핵심 플레이)
Phase 4: 증거 조사 — ★ 증거 제시/조사/감별 (핵심 플레이)
Phase 5: 재심문   — ★ 추가 추궁 + 특수 행동 (핵심 플레이)
Phase 6: 중재     — 결론 도출
Phase 7: 판결     — 책임 배분 + 해결안 선택
Result:  점수/칭호/후일담
```

**핵심 Phase는 3~5.** 플레이어의 질문 선택 → NPC 응답 → 거짓말 상태 전이 → 새 증거 해금 → 더 깊은 질문... 이 루프가 게임의 본체.

## 3. 거짓말 상태 기계 (LieState: S0→S5)

각 NPC는 쟁점(dispute)마다 독립적인 거짓말 상태를 가짐:

```
S0: 완전 부정   — "그런 적 없습니다"
S1: 일부 인정   — "그 부분은 맞지만..."
S2: 핑계        — "어쩔 수 없었습니다"
S3: 책임 전가   — "상대방이 먼저..."
S4: 감정적      — "제가 뭘 어떻게 하라는 겁니까!"
S5: 자백        — "사실은..."
```

플레이어의 질문과 증거 제시에 의해 전이됨. **S0에서 S5까지 몰아가는 과정이 게임의 쾌감.**

### 전이 트리거
- `direct_question` — 직접 질문만으로 전이 (초반 허들)
- `hard_evidence` — **특정 증거를 제시해야** 전이 (핵심 관문)
- `empathy_question` — 공감 접근으로 전이
- `emotion_threshold` — 감정이 폭발하여 전이
- `responsibility_question` — 책임 추궁으로 전이
- `voluntary_confession` — 신뢰를 쌓아 자발적 자백 유도

### 기획에서 중요한 것
**"이 증거를 들이밀어야 이 단계를 넘을 수 있다"** 라는 관문이 있어야 증거 제시에 의미가 생김. 아무 질문이나 해도 진행되면 재미없음.

## 4. Truth Throttle (진실 공개 곡선)

NPC는 LieState에 따라 공개할 수 있는 정보 수준이 다름:

| State | 금액 | 인물 | 기관 | 구체적 시각 |
|-------|------|------|------|------------|
| S0-S1 | "해당 금액" | "그 사람" | "그곳" | 허용 |
| S2    | "200만원대" | "김 씨" | 약칭만 | 허용 |
| S3+   | 구체적 | 실명 | 정식명칭 | 전부 |
| S5    | 전부 공개 | 전부 | 전부 | 전부 |

**기획에서 중요한 것**: 초반(S0~S1)에는 모호한 표현만 나오고, 플레이어가 추궁할수록 구체적 사실이 드러나는 구조. **플레이어가 직접 진실을 밝혀낸다는 느낌**이 핵심.

## 5. 심문 3종

플레이어가 선택하는 질문 유형:

| 유형 | 효과 | 재판관 톤 |
|------|------|----------|
| **사실 추궁** (fact_pursuit) | 모순 토큰↑, 논리적 압박 | 단호, 논리적 |
| **동기 탐색** (motive_search) | 누설 미터↑, 왜 그랬는지 | 설명 요구 |
| **공감 접근** (empathy_approach) | 신뢰 창구↑, 감정 이끌기 | 부드러움 |

**기획에서 중요한 것**: 같은 쟁점이라도 질문 유형에 따라 NPC 반응이 다름. 사실 추궁은 증거가 있을 때 강력하고, 공감 접근은 감정적 NPC에게 효과적. **사건마다 "이 NPC에게는 이 접근이 먹힌다"는 최적 경로가 있어야** 플레이어가 전략을 세우는 재미가 있음.

## 6. 증거 시스템

### 증거 8종 타입
bank(금융), chat(메시지), cctv(영상), contract(계약서), testimony(진술서), log(기록물), device(디지털), sns(SNS)

### 증거 발견의 재미
```
초기: 표면만 보이는 증거 3개 제공 (baseEvidenceIds)
      ↓ 심문으로 LieState 진행
중반: 조건 충족 시 새 증거 해금 (requires + requiredLieState)
      ↓ 새 증거로 더 깊은 심문
후반: 증거 조합 발동 → 결정적 사실 드러남 (evidenceCombinations)
```

### 해금 조건 예시
```
e-4 (사전 협조 DM):
  requires: ["e-1"]          ← e-1을 제시해야
  requiredLieState: "S2"      ← d-3 쟁점이 S2 이상이어야
  → 두 조건 모두 충족 시 해금
```

### 증거 조합
```
e-1(원본 비교) + e-2(CCTV) → 편집 의도가 드러남 (신뢰도 hard로 업그레이드)
```

**기획에서 중요한 것**: 증거가 **점진적으로 발견**되어야 함. 처음부터 다 보이면 탐색의 재미 없음. "이 증거를 열려면 저쪽을 먼저 파야 한다"는 체인이 게임의 뼈대.

## 7. 캐릭터 시스템

### Archetype 6종 (NPC의 거짓말 전략)
| 유형 | 특징 | 약점 |
|------|------|------|
| avoidant (회피형) | 질문을 피하고 돌려 말함 | 직접 추궁에 무너짐 |
| confrontational (대립형) | 공격적으로 반박 | 감정 폭발 후 실수 |
| victim_cosplay (피해자 행세) | 자신이 피해자라고 프레이밍 | 증거 앞에서 서사 붕괴 |
| cold_logic (냉정 논리) | 숫자와 기록으로 방어 | 감정적 접근에 균열 |
| affect_flattening (감정 무뎌짐) | 감정 반응 자체가 없음 | 끈질긴 공감에 결국 무너짐 |
| premature_summary (성급한 정리) | "이건 이미 정리된 일" 식 | 타임라인 모순으로 깨짐 |

### Verbal Tell 6종 (거짓말의 언어적 징후)
over_precision(과잉정밀), counter_question(역질문), timeline_padding(시간 늘리기), evidence_waving(증거 흔들기), motive_jump(동기 점프), selective_quote(선별 인용)

**기획에서 중요한 것**: A와 B의 archetype이 달라야 **플레이어가 각자에게 다른 전략**을 써야 함. "A는 추궁으로, B는 공감으로" 같은 차별화.

## 8. 재판관 성향 시스템 (3축)

플레이어(재판관)는 사건을 반복하며 성향이 쌓임:

```
탐구(inquiry)  ← → 판단(judgment)    : 질문을 많이 하는가 vs 빨리 결론 내는가
논리(logic)    ← → 직관(intuition)   : 증거 기반 vs 감정/동기 기반
과정(process)  ← → 해결(resolution)  : 절차적 정의 vs 실질적 해결
```

**기획에서 중요한 것**: 좋은 사건은 **재판관 성향에 따라 판결이 갈리는 딜레마**가 있어야 함.
- 논리적 재판관: "증거상 A가 더 책임 크다"
- 직관적 재판관: "하지만 B의 동기가 더 악질이다"
- 과정 중시: "절차를 어긴 쪽이 더 잘못"
- 해결 중시: "앞으로 재발 방지가 더 중요"

## 9. 무엇이 "재미있는 사건"인가

### 좋은 사건의 조건

1. **양측 모두 숨긴 게 있다** — 한쪽만 악당이면 재미없음. 둘 다 가담/은폐한 게 있어야 판결이 어려워짐
2. **증거가 점진적으로 발견된다** — 처음에는 A가 나빠 보이다가, 증거를 더 파면 B도 나쁜 게 드러남
3. **같은 증거를 다르게 해석할 수 있다** — "이 CCTV는 A의 잘못을 보여주는 건가, 아니면 B가 편집한 건가?"
4. **중반 뒤집기가 있다** — 플레이어가 "아, 이 사건은 이렇구나" 하고 생각했는데 새 증거가 판을 뒤집는 순간
5. **판결이 고민된다** — "현재의 잘못 vs 과거의 잘못", "작은 범죄를 저지른 피해자 vs 큰 범죄를 숨긴 가해자"

### 나쁜 사건의 특징

1. **한쪽이 명백히 나쁨** → 판결 고민 없음, 재미 없음
2. **오해 해소형** → 진실이 밝혀지면 끝, 갈등이 사라짐
3. **생활 민원 체급** → stakes가 낮아서 긴장감 없음 (6만원 수리비 분쟁 등)
4. **증거 없이 감정만** → 심문만 하면 끝, 증거 시스템 활용 안 됨
5. **제3자가 진범** → A vs B 대결이 아닌 외부인 탓, 당사자 공방이 약해짐

### 완성 케이스에서 배우는 좋은 패턴

**headline-01** (천만 조회의 저녁):
- 훅: 천만 조회 리뷰 영상, 누가 먼저 이용했나
- 양측 숨긴 것: A=무상 협조 DM, B=위생 위반+신상 보복
- 증거 체인: 원본 비교 → CCTV 해금 → DM 해금 → 타임라인 최종 해금
- 뒤집기: 처음에 B(사장)가 피해자처럼 보이지만, 보복성 신상 유포가 드러남
- 딜레마: 실제 위생 위반 vs 영향력 이용한 이익 추출

**spouse-11** (보험금의 설계도):
- 훅: 리모델링 누수 싸움처럼 보이지만, 실은 보험금 공모
- 양측 숨긴 것: 부부가 함께 보험 사기를 설계했다는 과거
- 뒤집기: "누가 먼저 배신했나" → "둘 다 공범이었다"
- 딜레마: 현재의 배신 vs 과거의 공동 가담

## 작업 요청

### Phase 1: 선별 — 재미를 살릴 수 있는 사건 골라내기

스레드 D의 분류(`thread-d-84-case-salvage-matrix.md`)를 기반으로, **실제로 리디자인하면 재미있는 게임이 될 수 있는 사건**을 골라주세요.

선별 기준:
- **갈등의 대칭성** — 양쪽 다 숨긴 게 있고, 누가 더 나쁜지 갈리는 구조
- **증거 해석전** — 같은 증거를 다르게 읽을 수 있는 장치
- **반전/뒤집기** — 중반 이후 판이 뒤집히는 순간이 있는 구조
- **생활 밀착성** — 플레이어가 "나도 이런 상황에 처할 수 있겠다"고 느끼는 소재
- **재판관 성향 분화** — 판결이 재판관 성향(논리/직관, 탐구/판단, 과정/해결)에 따라 갈리는 딜레마

**이미 완성된 5건과 겹치지 않는 것**만 선별합니다.

**drop으로 분류된 15건은 제외**합니다. keep/upgrade 중심으로, rebuild는 소재가 정말 좋은 경우에만 포함.

산출물: **선별 목록** (caseId + 선별 이유 + 기대 재미 요소)

### Phase 2: 기획 재설계 — 선별된 사건을 훨씬 재미있게

선별된 각 사건에 대해 **기획 문서를 재작성**합니다.

재설계 포인트:
1. **사건 훅 강화** — 첫 인상에서 "이건 뭐지?" 하고 끌리는 설정
2. **양측 숨긴 사실 재설계** — 단순 "한쪽이 나쁜" 구조 → "둘 다 가담한" 구조로
3. **증거 발견 순서** — 초반에 뭘 주고, 중반에 뭘 열고, 후반에 뭘 뒤집을지
4. **모순 쌍** — A가 이렇게 말했는데 B의 증거가 그걸 깬다, 같은 핵심 모순 설계
5. **감정 곡선** — 처음에 한쪽을 동정하다가 뒤집히는 순간, 양쪽 다 측은해지는 순간
6. **판결 딜레마** — "현재의 잘못 vs 과거의 잘못", "절차 vs 실체", "용서 vs 제재" 같은 축

산출물 형식: 사건별로 아래 구조의 기획서

```markdown
## [caseId] — 리디자인 제목

### 사건 훅
(2~3줄, 플레이어를 끌어당기는 설정)

### 양측 프로필
- A: 이름, 직업, 숨긴 것, 거짓말 전략
- B: 이름, 직업, 숨긴 것, 거짓말 전략

### 쟁점 구조 (3~5개)
- d-1: ...
- d-2: ...

### 증거 설계 (5~7개)
- 초기 제공: e-1, e-2, e-3
- 1차 해금 (S1+): e-4 (requires e-1)
```
