## 실행 규모

| 항목 | 수치 |
|------|------|
| 사건 수 | 84건 |
| **반복 횟수** | **3회** (LLM 비결정성 대응) |
| 턴/회 | 20턴 |
| 총 턴 | 84 × 3 × 20 = **5,040턴** |
| API 호출 | ~5,040회 (gpt-4o) |

3회 반복 이유:
- 1회만으로는 LLM 비결정성(같은 프롬프트에서 다른 결과)을 잡을 수 없음
- 3회 중 2회 이상 같은 문제가 나오면 **구조적 결함**, 1회만 나오면 **비결정적 변동**으로 분류
- 각 회차는 다른 시나리오 변주: 1회차 기본, 2회차 증거 우선, 3회차 공감 우선

## 10개 품질 축

### 축 1: 의미/내용 정합성
- 질문에 대한 답변이 맞는지 (질문 무시하고 딴소리 하지 않는지)
- 사건 데이터에 없는 사실 생성(hallucination) 여부
- 이전 턴과의 모순 (S 전이 없이 번복)

### 축 2: 맥락 파악
- 현재 쟁점에 맞는 답변인지
- 상대방이 말한 내용을 정확히 참조하는지
- 증거가 제시된 후 반응이 적절한지

### 축 3: 호칭
- 당사자→재판관에게 상대 언급: callTerms.toJudge ("제 아내가~")
- 당사자→상대에게 직접: callTerms.toPartner ("자기야~")
- 재판관→당사자: "OOO 씨" (이름+씨)
- **재판관이 "제 아내/남편" 사용 시 CRITICAL**
- 증인: addressA/addressB

### 축 4: 경칭/어법
- 재판관 대상: 합니다체 필수
- 당사자 간: 반말 유지
- emotional/confession만 해요체 예외
- 같은 턴 안에서 합니다체/해요체 혼용 금지
- 번역체 9패턴 + 추가 3패턴 ("제 불찰", "문의를 드린", "~하게 되었습니다")

### 축 5: 조사/문법
- 이/가, 을/를, 은/는, 과/와 받침 정합
- "이입니다", "이습니다" 이중 조사
- "만을" → "만"
- 아/야 접미사 (받침 없으면 "야")

### 축 6: 단계별 공개 가능 정보 (Truth Throttle)
- S0-S1: 구체적 금액/인물명/기관명 0건
- S2: 성씨까지만, 대략적 금액만
- S3: 구체적 정보 일부 허용
- S4-S5: 전체 구체적 진실
- S5 자백: 금액+인물+기관+목적 중 해당하는 것 포함 필수
- **직함+기관 조합 ("재가돌봄센터 상담팀장")은 S3+ 이전 CRITICAL**

### 축 7: 주체/객체 구분
- NPC가 자기 행위를 자기 시점으로 말하는지 (1인칭 "저"/"제가")
- NPC가 자기 이름을 3인칭으로 부르지 않는지
- blame stance에서 상대 행위 언급은 자연스러움 (구분)
- 증인이 당사자 시점의 호칭을 쓰면 FAIL

### 축 8: 상황 파악
- archetype에 맞는 말투 (avoidant=우회적, confrontational=날카롭게 등)
- tell 발현 빈도 15-25% (3-4턴에 1회)
- 감정 변화 곡선 자연스러움 (S0 무표정→S3 방어적→S5 무너짐)
- 비금전 사건에서 금전 표현 0건 (은유적 표현 포함)

### 축 9: 캐릭터 일관성
- 같은 NPC의 이전 발언과 모순 없음
- archetype 톤이 턴 간 일관
- tell이 자연스럽게 녹아있는지 (기계적 삽입 아닌지)

### 축 10: 구조적 완결성
- 20턴 안에 S5 도달 가능한지
- DossierCard 해금→사용 흐름 정상
- 증거 제시 후 lieState 변화 발생
- investigationStages 3단계 질문 자연스러움

## PASS/FAIL 기준

### 사건별 (3회 반복 기준)
- **CRITICAL**: 3회 중 2회 이상 발생 → **구조적 결함** → 반드시 수정
- **FAIL**: 3회 중 2회 이상 발생 → 수정 필요
- **WARN**: 3회 중 1회만 발생 → 비결정적 변동, 기록만
- **PASS**: 3회 모두 문제 없음

### 전체
- 84건 중 **CRITICAL 0건, FAIL 10건 이하** = Phase 1 품질 잠금 달성
