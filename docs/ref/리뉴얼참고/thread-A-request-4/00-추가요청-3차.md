## 요청 B: Phase3StructuredLog 런타임 수집기 설계

Phase 6 프롬프트 설계(phase6-result-prompt-v2-design.md)는 있지만, **게임 중 structuredLog를 실시간으로 쌓는 코드**가 없습니다.

### 필요한 설계

**1. 수집 대상과 수집 시점**

```
revealedAtoms
  - beat의 truthEnvelope.allowAtomIds가 실제 선택되었을 때
  - StateUnlockAtom이 해금되었을 때
  - evidence가 제시되어 onSuccess.revealAtom이 발동했을 때

disprovedFakeIssues
  - red_herring dispute의 misconceptionState가 M4(clarified)에 도달했을 때
  - d3.red_herring_disproved 같은 flag가 set되었을 때

resolvedLinks
  - linkEdge의 when 조건이 만족되어 effect가 발동했을 때

relationCoreRevealed
  - 쟁점의 core 층이 최초로 해금되었을 때 (depthLayer 'core' unlock)

playerStyleTags
  - 매 턴 질문 유형/angleTag/피로도/끼어들기 선택을 추적
  - 게임 종료 시 누적 패턴에서 태그 파생
```

**2. store에 수집 상태 추가**
- `phase3Log: Phase3StructuredLog` 필드
- 수집 함수: `recordRevealedAtom(atomId)`, `recordDisprovedFakeIssue(disputeId)`, `recordResolvedLink(linkId)`, `markRelationCoreRevealed()`, `addPlayerStyleHint(tag)`

**3. 수집 → Phase 6 브릿지**
- Phase 3 → Phase 6 전환 시점에 `Phase3PromptBridgeV2`로 변환하는 함수
- `resolvedRevealedFacts` = revealedAtoms를 factText로 resolve
- `contestedIssues` = 아직 S2 이하인 쟁점들
- `keyMoments` = gameEventLog에서 transition/evidence_hit/interjection/burst를 추출

**4. 기존 코드 통합 포인트**
- `useActionDispatch` — beat 선택 후 `recordRevealedAtom`
- `v3GameLoopLoader` — StateUnlockAtom 해금 시 `recordRevealedAtom`
- `v2DataLoader` — linkEdge 발동 시 `recordResolvedLink`
- `DossierCardPanel` — onSuccess.revealAtom 시 `recordRevealedAtom`

### 참고 파일
- `prev-04-phase6-result-prompt-v2-design.md` — Phase 6 프롬프트 설계
- `schema-02-dispute-v2.ts` — Phase3StructuredLog 타입 정의
- `ref-08-useActionDispatch-excerpt.ts` — 통합 지점

### 출력
`phase3-log-collector-design.ts` — 타입 + store 확장 + 수집 함수 + Phase6 브릿지 변환 함수
