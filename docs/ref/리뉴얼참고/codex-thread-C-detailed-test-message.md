## 테스트 5단계 (126개 항목)

상세 항목은 `ct-to-threadC-detailed-test-8cases.md`에 전부 정리되어 있습니다. 요약:

### Level 1: 기본 동작 + 데이터 정합성 (24항목)
- 데이터 정합성: 당사자 이름/쟁점 수/증거 수/caseId/baseEvidenceIds
- Phase 전환: Phase 0→1→2→3→4→5→6→7→Result 전체 전환 정상 여부
- Phase 1/2 자동 재생: 스크립트 로드, 대사 순차 재생, 화자 구분
- 증거 뷰어: viewerData/meta 렌더링, 타입별 뷰어 차이, 조사 단계
- ScriptedText 매칭: 6채널 커버리지, key 포맷 일치, variant 반환

### Level 2: 증거 해금 체인 (19항목)
- 초기 상태: 해금 3개 / 잠금 3개 정확히 일치
- 해금 과정: 선행 증거 제시→해금, LieState 조건 미충족→잠김 유지, 복합 조건
- 증거 조합: 두 증거 제시→조합 발동, 부분 제시→미발동, 중복 방지
- 조사 단계: 3단계 순차, revealKey 연결, surfaceName→name 전환

### Level 3: LieState 전이 (16항목)
- 기본: 초기 S0, 질문으로 S0→S1, 쟁점별 독립
- 증거 관문: **질문만으로는 관문 통과 불가**, 올바른 증거 제시→전이, 엉뚱한 증거→전이 안 됨
- 특수: 감정 임계, 신뢰 루트 자백, S5 도달
- 연동: 전이→NPC 대사 변화, 전이→증거 해금, 전이→미터 변화, 전이→감정 변화

### Level 4: 스크립트 상세 품질 (48항목, 최중요)
- **NPC 대사 (13항목)**: 호칭(callTerms 3종), 존칭/어체, Truth Throttle, 실명 조기 노출, Archetype 일관성, Verbal Tell, 감정 톤, 거짓말 단계별 정합성, 번역체 금지, 클리셰 금지, 메타 누출 금지, 금전 표현, 해요체 규칙
- **재판관 (5항목)**: 질문 유형별 톤, 중립성, NPC 호칭("OOO 씨"), 어체, 증거 기반 질문
- **시스템 메시지 (3항목)**: 정보 정확성, 스포일러 금지, 톤
- **증인 증언 (4항목)**: depth(vague/partial/full) 구분, 입장 일관성, 기관 증인 톤, 문장 수
- **증거 반응 + 특수 (5항목)**: 증거 인지 반응, DossierCard, 모순 추궁, 끼어들기, 비공개 진술
- **한국어 품질 (7항목)**: 조사(이/가, 을/를), 시제 일관성, 대명사 혼동, 문장 완결, 자연스러움, 반복 표현, 맥락 연결
- **variant (3항목)**: 5개 variant 간 차이, 일관성, 자연 전환
- **aftermath (2항목)**: 판결 결과 반영, 톤 적절성

### Level 5: 중재·판결·결과 (19항목)
- 중재: 옵션 맥락 적합, 선택의 영향
- 판결: 슬라이더, solutions 딕셔너리 파싱, 솔루션 카드 내용, 선택/확정
- 결과: 점수/칭호, 통찰/권위/지혜 분화, aftermath, 히스토리 저장, 재판관 드리프트, Home 복귀

- 유료 판매 기준이므로 **한 글자라도 어색하면 WARN 처리**
- 1차 5건 완료 후 CT에 보고. 2차 6건은 CT가 재생성 완료를 알려드리면 시작합니다
