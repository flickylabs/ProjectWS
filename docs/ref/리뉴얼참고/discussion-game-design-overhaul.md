# Solomon 게임 설계 근본 재검토 — GPT 논의 요청

## 배경

V2 atom 시스템 도입으로 **NPC 응답 품질**(자기모순, 반복, 호칭, archetype)은 크게 개선되었습니다. 하지만 실제 플레이 테스트를 반복한 결과, **게임으로서의 재미**가 아직 부족합니다.

현재 상태: "정상적인 대화에 근접했지만, 재미있는 심문 게임은 아직 아님"

이번 논의에서는 NPC 응답 품질이 아닌 **게임 메커닉/루프/피드백**에 집중합니다.

---

## 현재 플레이 루프의 문제

### 플레이어가 하는 것
```
인물 선택 → 액션 선택(사실추궁/동기탐색/공감접근) → 쟁점 선택 → NPC 응답 2문장 읽기 → 반복
```

### 빠져 있는 것
1. **"내 행동이 효과가 있었다"는 피드백** — 질문/증거 제시 후 lie state가 움직였는지, 방어가 약해졌는지 플레이어가 모름
2. **전략적 선택** — 어떤 액션을 골라도 비슷한 결과. "이걸 골라야 하는 이유"가 없음
3. **돌파 순간** — S2→S3 전이가 일어나도 "내가 해냈다!" 쾌감이 없음
4. **턴 간 변화/이벤트** — 모든 턴이 동일한 리듬. 긴장, 반전, 위기가 없음

---

## 논의 주제 5가지

### 1. 증거 시스템 재설계

**현재 문제:**
- 증거 6개 × 조사 6종 = 36개 선택지. 12~16턴에 비해 과다
- 증거를 제시해도 NPC 응답이 크게 달라지지 않음
- 조사 결과(방어선 봉쇄)가 플레이어에게 보이지 않음

**개발자 의견:**
- 증거를 3~4개로 줄이고, 각 증거의 게임 효과를 극대화하는 방향이 좋을 것 같음
- both(양측 관련) 증거는 A에게 제시 + B에게 제시 = 2회 사용 가능. 같은 증거라도 대상에 따라 다른 반응
- **증거 제시 시 선택할 수 있는 메시지를 각 증거에 특화** — 현재 "이 증거에 대해 입장을 밝혀 주십시오"라는 일반 질문 대신, 증거별로 "이 송금 내역의 수신인이 누구입니까?", "이 캡처의 잘린 부분은 무엇입니까?" 같은 구체적 질문 선택지 제공
- 증거 제시 → lie state 강제 전이 또는 방어 모드 축소가 **눈에 보여야** 함

**질문:**
- 증거 수를 줄이는 것이 맞는가? 줄인다면 어떤 기준으로?
- 증거 제시 시 "재판관 질문 선택지"를 증거별로 사전 정의하는 방식이 효과적인가?
- 증거 → lie state 전이를 자동으로 할 것인가, 아니면 "증거 + 적절한 질문"이 조합되어야 하는가?

---

### 2. 심문 하위 액션(사실추궁/동기탐색/공감접근) 재검토

**현재 문제:**
- 3가지 액션의 결과가 "약간 다른 톤의 회피"일 뿐, 게임 효과가 다르지 않음
- 플레이어가 "이 상황에서 이 액션을 골라야 하는 이유"를 모름
- Ace Attorney의 Press는 새 정보/진술 추가/모순 생성이라는 명확한 효과가 있음

**개발자 의견:**
- 하위 액션의 필요성 자체에 의구심이 있음. 현재로서는 변별력이 느껴지지 않음
- 유지한다면 각 액션이 **다른 게임 효과**를 가져야 함:
  - 사실추궁 → 모순 검출 확률 증가, timeline 압박
  - 동기탐색 → 감정 누설 확률 증가, 새 정보 발견
  - 공감접근 → 신뢰 상승, 자발적 고백 경로 개방
- 아니면 하위 액션을 제거하고 다른 구조로 대체?

**질문:**
- 하위 액션 3종을 유지할 것인가, 제거/통합할 것인가?
- 유지한다면 각 액션의 구체적 게임 효과(lie state 전이 확률, 모순 검출, 감정 변화 등)를 어떻게 설계할 것인가?
- Ace Attorney식 "Press + Present" 2종 구조가 더 나은가?

---

### 3. 돌파 순간 연출

**현재 문제:**
- lie state가 S1→S2→S3으로 전이되어도 플레이어가 모름
- 화면에 변화가 없고, NPC 응답 톤만 약간 달라질 뿐
- "내가 이 사람의 거짓말을 깼다!"는 쾌감이 전혀 없음

**개발자 의견:**
- 돌파 순간을 **강력하게 연출**할 필요가 있음
- lie state 전이 시: 화면 흔들림/금 가는 연출 + NPC 동요 표현 + "방어 균열" 알림
- S4/S5 전이 시: 특별 연출 (고백 씬?)
- EvidenceChallenge의 full collapse 시: "방어선 붕괴" 연출

**질문:**
- 어떤 시점에 어떤 수준의 연출이 적절한가?
- 연출이 게임 흐름을 방해하지 않으면서도 "쾌감"을 주려면?

---

### 4. 턴 간 이벤트 — 모순, 끼어들기, 변화

**현재 문제:**
- 모든 턴이 동일한 리듬: 질문 → 응답 → 질문 → 응답
- 기존에 있던 **모순 감지, 끼어들기** 등의 재미 요소가 현재 구조에서 작동하지 않음
- 단조로움을 깰 이벤트가 없음

**개발자 의견:**
- 턴당 피드백을 단순히 "정보 표시"로 강화하는 것이 아니라, **그 자체가 게임 요소**가 되어야 함
- 예시:
  - **모순 발생** → 플레이어가 "이 모순을 지적한다" 선택 → 지적 성공 시 lie state 강제 전이
  - **끼어들기** → 상대방이 끼어들려고 함 → "허용/차단" 선택 → 허용하면 새 정보, 차단하면 권위 유지
  - **감정 폭발** → NPC가 감정적으로 흔들림 → "더 압박/진정시키기" 선택 → 결과에 따라 경로 분기
  - **새 쟁점 발현** → 심문 중 숨겨진 쟁점이 드러남 → "추적할지 현재 쟁점 유지할지" 선택
- 이런 이벤트가 **예측 불가능하게** 발생하면 단조로움이 깨짐

**질문:**
- 기존 모순/끼어들기 시스템을 현재 V2 구조에서 어떻게 부활시킬 수 있는가?
- 턴 간 이벤트의 발생 빈도와 종류는?
- 이벤트 발생 시 플레이어 선택지는 어떻게 설계해야 하는가?

---

### 5. 품질 안정성 — 개연성, 일관성

**현재 문제:**
- V2 atom으로 자기모순/반복은 해결했지만, 여전히 LLM이 빈 말을 채우는 경우가 있음
- 같은 쟁점을 반복 심문하면 점점 구체성이 떨어짐 (새로운 정보가 없으니 같은 말 변주)
- 상태가 진행(S1→S3)되어도 응답의 "깊이"가 크게 달라지지 않음

**근본 원인:**
- 현재 ClaimPolicy atom은 "무엇을 말할 수 있는가"를 정의하지만, **"무엇을 새로 드러내는가"는 정의하지 않음**
- S1에서 S2로 전이되면 **새로운 사실**이 해금되어야 하는데, 현재는 같은 atom을 약간 다른 stance로 말할 뿐
- 즉, **상태 전이 = 새 정보 해금**이 되어야 게임이 진행되는 느낌이 남

**질문:**
- S1→S2→S3 각 전이 시 "이 상태에서만 드러나는 새 사실"을 atom으로 정의해야 하는가?
- BeatScript를 "상태 전이 순간"에 강제 삽입하여 일관된 품질을 보장하는 방식은?
- 반복 심문 시 "이미 말한 내용"을 추적하여 점점 더 깊은 답변을 강제하는 방법은?

---

## 요청 사항

위 5개 주제에 대해 **게임 설계 관점**에서 구체적인 개선안을 제시해 주세요.

특히:
1. **재미 향상** — 전략적 선택, 돌파 쾌감, 턴 간 이벤트
2. **품질 안정** — 개연성, 일관성, 상태 진행에 따른 정보 해금

두 축 모두를 동시에 잡을 수 있는 설계안이면 가장 좋습니다.

현재 기술 스택과 V2 구조(atom + slot + blueprint)를 기반으로 실현 가능한 수준이어야 합니다.
