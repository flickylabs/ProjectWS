## 후속 1: "연출이 에이전시보다 앞서있다" — 구체 진단

리뷰 Q7에서 "Phase 2 구현은 좋지만, 아직 연출이 에이전시보다 앞서있을 가능성이 크다"고 했습니다.

### 질문
1. 현재 구현된 연출 중 **구체적으로 어떤 것**이 "과하다"고 느끼는가? (돌파 연출, 교착 피드백, 효과 칩, Phase 전환, 모순 컷인 중)
2. "에이전시가 부족하다"의 핵심이 **선택지가 적은 것**인가, **선택의 결과가 안 보이는 것**인가, **선택이 의미없는 것**인가?
3. 연출을 줄이는 게 답인가, 아니면 연출은 유지하되 **연출 사이에 플레이어 행동을 끼워넣는** 방식이 답인가?
4. 구체적 개선 예시를 1~2개 들어줄 수 있는가? ("이 순간에 이런 선택지가 있었으면 에이전시가 살아난다" 형태로)

## 후속 2: 심문 3종 효과 분리 — 구현 방향

리뷰 Q8에서 "미구현 중 1순위는 심문 3종 효과 분리"라고 했습니다.

### 현재 구조
- `questionEffectEngine.ts`의 `computeEffectiveness()`가 subAction(fact/motive/empathy) × lieState × archetype 조합으로 strong/normal/weak 판정
- 판정 결과는 UI 칩으로 표시 (green/blue/red dot)
- 실제 게임 효과는 lie 전이 확률, leak/trust/contradiction 미터 변동량에 반영

### 질문
5. 현재 "효과성 칩 표시"만으로는 부족한가? **플레이어가 체감하는** 3종 차이를 만들려면 추가로 무엇이 필요한가?
6. 예를 들어 fact_pursuit는 "모순 토큰 직접 +1", empathy_approach는 "신뢰 게이지 직접 +15" 같은 **확정적 효과**를 줘야 하는가? 아니면 현재처럼 확률 기반이 맞는가?
7. 3종 선택이 "전략적으로 다른 결과"를 만들려면, 어떤 상황에서 어떤 선택이 **명확하게 더 나은** 결과를 내야 하는가? 구체적인 시나리오 2~3개를 들어줄 수 있는가?
8. archetype별로 "이 타입에는 이 접근이 효과적"이라는 걸 **플레이어가 자연스럽게 발견**할 수 있는 힌트 체계가 필요한가?

## 후속 3: 증거 3~4장 표면화 구조

리뷰 Q10에서 "보이는 증거 3~4개 축소, 내부 6개 유지"를 권했습니다.

### 질문
9. **어떤 기준으로** 3~4개를 선택하여 표면화하는가?
   - (a) 현재 쟁점 관련도 기반 자동 선별
   - (b) 플레이어가 직접 "활성 증거 덱" 구성
   - (c) Phase/턴 진행에 따라 순차 해금
   - (d) 기타
10. 나머지 2~3개는 완전히 숨기는가, 아니면 "잠김" 상태로 보이되 특정 조건(쟁점 진전 등) 충족 시 해금되는가?
11. 축소된 상태에서 **증거 조합(두 증거를 합쳐 더 강한 추궁)** 같은 메커니즘은 어떻게 작동하는가?
12. PC 프로토타입의 증거 상세 뷰어(8종 타입별 UI)와 이 축소 구조가 어떻게 연결되는가? 잠긴 증거도 뷰어로 볼 수 있어야 하는가?

## 후속 4: 주간 판례 챌린지 + 판결문 요약 — 구체화

리뷰 추가 아이디어에서 "주간 판례 챌린지(난이도 중, 임팩트 상)"과 "판결문 작성 요약(난이도 중, 임팩트 상)"을 제안했습니다.

### 질문
13. **주간 판례 챌린지**의 구체적 형태는?
    - 같은 사건을 모든 플레이어가 같은 조건(시드)으로 플레이?
    - 특수 제약 조건 (예: "증거 2개만 사용", "공감만으로 자백 유도")?
    - 순위 기준은? (점수, 턴 수, 칭호?)
    - 리더보드와 어떻게 연결?
14. **판결문 작성 요약**의 구체적 형태는?
    - 재판 완료 후 자동 생성? 플레이어가 직접 작성?
    - 어떤 내용이 포함? (책임 배분 근거, 핵심 증거, 결정적 순간)
    - 공유 기능? (Steam 스크린샷, SNS 공유)
    - 재판관 성향/칭호와 연동?

## 후속 5: 퍼크 → "새 선택지 개방형"

리뷰 Q2에서 "숫자 보정형보다 새 선택지 개방형으로 맞추는 게 좋다"고 했습니다.

### 현재 퍼크 분류
- **숫자 보정형**: penalty_buffer(권위 감소 35%↓), trust_start(초기 신뢰+8), fatigue_extend(교착 +1)
- **선택지 개방형**: evidence_preview(효과 예측 UI), free_summary(무료 분석), burst_warning(폭발 경고 UI)

### 질문
15. 구체적으로 어떤 퍼크를 어떻게 바꿔야 하는가? 예를 들어:
    - `penalty_buffer`(숫자 보정) → 어떤 "선택지 개방"으로?
    - `trust_start`(숫자 보정) → 어떤 형태로?
    - `fatigue_extend`(숫자 보정) → ?
16. "선택지 개방형"의 예시를 3~4개 더 제안할 수 있는가? (현재 없는 완전히 새로운 퍼크)
17. 퍼크가 12종이면 충분한가? 성장 단계별 해금 (regular→veteran→senior) 에 맞춰 더 필요한가?

## 후속 6: resolution 축 과결정 문제

리뷰에서 "resolution 축이 마지막 솔루션 선택에 꽤 크게 끌리므로, 심문 과정 성향보다 결과 카드가 성향을 과도하게 결정할 위험"이 있다고 지적했습니다.

### 현재 구조
- `judgeProfileEngine.ts`의 `deriveCaseProfile()`에서 resolution 축 계산
- 솔루션 선택의 orientation(principle/reconcile/hybrid) 태그가 해결 축에 큰 가중치
- 심문 과정에서는 질문 타입(fact=탐구↓, empathy=탐구↑)이 주로 탐구 축에 반영
- 판단 축은 증거 사용, 권위 행사, 판결 엄격도로 결정

### 질문
18. 심문 과정에서 해결 축 기여도를 높이는 방법은?
    - (a) 끼어들기 허용/제지 선택에 해결 축 가중치 부여
    - (b) 감정 폭발 대응(진정/방치)에 해결 축 반영
    - (c) 쟁점별 "관계 회복 질문" vs "사실 규명 질문" 분기
    - (d) 기타
19. 솔루션 선택의 해결 축 가중치를 **낮춰야** 하는가? 아니면 심문 과정 기여를 **높여서** 상대적으로 희석해야 하는가?
20. 3축이 각각 "언제, 무엇으로" 결정되는지 명확한 매핑을 제안할 수 있는가?
    - 탐구 축: 주로 ___에 의해 결정
    - 판단 축: 주로 ___에 의해 결정
    - 해결 축: 주로 ___에 의해 결정
