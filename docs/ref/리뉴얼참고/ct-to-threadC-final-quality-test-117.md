## Level 4 테스트 방법 (필수)

각 케이스에서 아래를 **반드시** 수행:

1. **Phase 3에서 최소 5턴 심문** — fact_pursuit 2회, motive_search 1회, empathy_approach 2회. 각 대사를 **소리 내어 읽는 수준으로** 확인
2. **증거 제시 최소 2회** — 서로 다른 증거를 대상에게 제시. NPC 반응이 **증거 내용을 구체적으로 인지**하는지 확인
3. **같은 조건 2회 반복 질문** — variant가 실질적으로 다른지 확인
4. **S0 → S2 이상 진행** — LieState가 바뀌면서 대사 내용/톤이 실제로 달라지는지 확인
5. **증인 증언 확인** — 증인이 **직접 말하는 형태**인지, "증인은 ~합니다" 같은 서술자 시점이 아닌지

**콘솔 로그만으로 Level 4를 판단하지 마세요. 반드시 텍스트를 직접 읽으세요.**

## 보고 형식

### PASS/WARN/FAIL 기준 (엄격 적용)

- **PASS**: 유료 판매 가능. 읽었을 때 자연스럽고 사건에 맞는 내용.
- **WARN**: 기능은 하지만 어색하거나 반복적. 수정하면 나아질 것.
- **FAIL**: 판매 불가. 기계적 문장, 범용 템플릿, JSON 필드 노출, 호칭 오류 등.

### 보고 형식

```

- 4-A 호칭(4): PASS / WARN / FAIL — (원문 인용)
- 4-A 어체(3): PASS / WARN / FAIL

- 4-A 캐릭터(4): PASS / WARN / FAIL
- 4-A 금지패턴(5): PASS / WARN / FAIL — (원문 인용)
- 4-B 재판관(6): PASS / WARN / FAIL
- 4-C 시스템(3): PASS / WARN / FAIL
- 4-D 증인(6): PASS / WARN / FAIL — (원문 인용)
- 4-E 증거반응(8): PASS / WARN / FAIL — (원문 인용)

(원문 전체 + 어떤 항목 위반인지)

```
