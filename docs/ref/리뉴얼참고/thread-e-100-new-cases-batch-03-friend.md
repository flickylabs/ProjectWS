# Thread-E 100 New Cases

## 배치 3: 친구 (10건)

### 3-01 1초 빠른 피니시칩 — friend (`friend-new-01`)

**훅**: 하프마라톤 브랜드 앰배서더 발표 하루 전, B의 기록이 기준선보다 정확히 1초 빠르다는 사실이 커뮤니티에 퍼진다. A는 "B가 페이서 칩으로 기록을 세탁했다"고 하고, B는 "A가 내 재활 진단서를 스폰서에 흘렸다"고 맞선다.  
처음엔 기록 조작 사건 같지만, 파고들수록 한쪽은 실제로 GPX를 이어 붙였고 다른 쪽은 부상 정보를 이용해 상대를 미리 탈락시키려 했다.

**A**: 서다빈(31, 브랜드 마케터) — 러닝크루 출석표와 B의 재활 정보를 편집해 스폰서에 전달한 것 — 거짓말 전략: `cold_logic`  
**B**: 민지후(30, 러닝 크루 운영자) — 페이서 친구의 칩과 GPX 구간을 섞어 기준 기록을 맞춘 것 — 거짓말 전략: `avoidant`

**쟁점** (5개):
- d-1: B의 공식 기록은 실제 완주 기록인가
- d-2: A는 왜 재활 진단과 출석표를 스폰서에 보냈는가
- d-3: 브랜드 앰배서더 선발은 기록 우선이었는가, 커뮤니티 평판도 포함이었는가
- d-4: 러닝크루 출석과 리더 기여는 누가 더 왜곡했는가
- d-5: 기록 조작과 의료정보 유출 중 무엇을 더 무겁게 볼 것인가

**증거 체인**:
- 초기: e-1 공식 기록 인증 화면, e-2 러닝크루 출석표 캡처, e-3 스폰서 shortlist 메일
- 1차 해금: e-4 GPX 원본 두 개와 merge 흔적 (requires e-1, S1+)
- 2차 해금: e-5 스포츠의학 재활 진단서 전달 메일 (requires e-3, S2+)
- 3차 해금: e-6 대회 당일 물품보관소·칩 수령 CCTV 확인서 (requires e-2, S2+)
- 최종: e-7 출석표 수정 이력과 페이서 DM (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → B 기록이 단일 러닝 로그가 아니라는 점이 드러난다
  - e-3+e-5 → A가 의료정보를 앰배서더 심사에 실제로 흘렸음이 열린다
  - e-2+e-7 → A 역시 출석과 기여표를 유리하게 손본 흔적이 확인된다

**중반 뒤집기**: 초반엔 B의 기록 조작이 결정타처럼 보이지만, e-5와 e-7이 열리면 A도 선발 경쟁에서 이기기 위해 개인정보와 운영 자료를 무기화했다는 사실이 드러난다.

**판결 딜레마**: `공정 경쟁 vs 사생활`, `결과 조작 vs 과정 훼손`, `개인 실적 vs 커뮤니티 신뢰`가 교차한다. B의 기록 조작이 핵심일 수 있지만, A의 정보 유출과 운영 데이터 가공도 별개로 제재될 수 있다.

**난이도**: hard

### 3-02 바우처의 로마자 철자 — friend (`friend-new-02`)

**훅**: 출국 당일 공항 카운터에서 A의 항공 바우처가 이미 사용 처리됐다는 안내가 뜬다. A는 "B가 내 여권 영문 이름 철자를 바꿔 몰래 써버렸다"고 하고, B는 "A가 내 여권 사본과 빚 독촉 캡처를 단톡과 항공사에 같이 퍼뜨렸다"고 맞선다.  
실은 한쪽은 영문 철자와 생년월일을 조합해 바우처를 먼저 태웠고, 다른 쪽은 공항에서 재결제한 직후 여권 사본과 채무 사정을 공개 압박 수단으로 써버렸다.

**A**: 김유나(29, 출판사 편집자) — 친구의 바우처 전용을 추궁하며 여권 사본과 빚 독촉 캡처를 여행 단톡·항공사 상담 메일에 함께 보낸 것 — 거짓말 전략: `avoidant`  
**B**: 장현수(29, 게임 QA 계약직) — A의 영문 이름 변형으로 바우처를 사용하고도 "가족 표기 실수"처럼 설명한 것 — 거짓말 전략: `confrontational`

**쟁점** (4개):
- d-1: 바우처는 누구의 항공권 구매에 실제로 쓰였는가
- d-2: B의 사용은 착오였는가, 의도적 차용이었는가
- d-3: A의 항공사 제보와 여권 사본 공개는 권리 구제였는가, 망신주기였는가
- d-4: 친구 사이의 편의 공유 관행과 개인정보 경계는 어디서 갈라지는가

**증거 체인**:
- 초기: e-1 바우처 발급 메일, e-2 공항 재결제 e-ticket, e-3 단톡 항의 캡처
- 1차 해금: e-4 항공사 영문명 수정·재발급 로그 (requires e-1, S1+)
- 2차 해금: e-5 항공사 상담 녹취와 여권 사본 첨부 메일, 여행 단톡 전달본 (requires e-3, S2+)
- 3차 해금: e-6 카드승인 시각과 수하물 구매내역 (requires e-2, S2+)
- 최종: e-7 여행 전 빚 상환 압박 DM과 바우처 분배 초안 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → B가 영문명 정보를 바꿔 사용했다는 점이 드러난다
  - e-3+e-5 → A가 항공사와 친구들에게 여권 사본과 채무 사정을 과도하게 함께 보냈음이 확인된다
  - e-6+e-7 → 두 사람 모두 여행비 정산과 채무 압박을 뒤에서 연결하고 있었다는 구조가 열린다

**중반 뒤집기**: 초반엔 B가 바우처를 가로챈 쪽이라 명확히 나빠 보이지만, e-5가 열리면 A도 공항에서 재결제한 직후 권리 구제를 넘어 개인정보와 빚 사정을 공개 압박 수단으로 썼다는 점이 드러난다.

**판결 딜레마**: `실질적 금전 피해 vs 개인정보 유출`, `편의 공유 관행 vs 명의 권리`, `용서 vs 선례 금지`가 충돌한다. B의 선사용이 먼저였더라도 A의 망신주기 방식이 정당화되지는 않는다.

**난이도**: easy

### 3-03 레퍼런스북의 그림자 — friend (`friend-new-03`)

**훅**: 독립서점 공동전시 하루 전, A는 "B가 내 콘셉트를 훔쳐 큐레이터에게 단독 전시처럼 팔았다"고 폭로한다. B는 "A는 외국 작가 레퍼런스를 AI로 짜깁기한 사람"이라고 맞받아친다.  
전시는 창작 크레딧 싸움처럼 보이지만, 실은 A는 원출처를 숨겼고 B는 큐레이터에게 단독 독점 굿즈를 약속하며 공동 크레딧을 지웠다.

**A**: 이나경(32, 프리랜서 일러스트레이터) — AI 레퍼런스와 해외 작품 참고 범위를 숨기고도 순수 창작처럼 밀어붙인 것 — 거짓말 전략: `premature_summary`  
**B**: 박시온(33, 편집 디자이너) — 큐레이터에게 단독 굿즈와 판매 수수료를 약속하며 공동 크레딧을 삭제한 것 — 거짓말 전략: `cold_logic`

**쟁점** (5개):
- d-1: 전시 콘셉트의 실제 원출처는 어디인가
- d-2: 큐레이터에게 전달된 기획안은 공동전시였는가, 단독전시였는가
- d-3: A의 AI 레퍼런스 사용은 참고 수준이었는가
- d-4: B의 크레딧 삭제는 운영 편의였는가, 의도적 독식이었는가
- d-5: 현재의 독식과 과거의 원출처 은닉 중 무엇을 더 무겁게 볼 것인가

**증거 체인**:
- 초기: e-1 전시 포스터 시안, e-2 큐레이터 회의 요약, e-3 A의 레퍼런스북 이미지 폴더
- 1차 해금: e-4 포스터·제안서 수정 이력 PDF (requires e-1, S1+)
- 2차 해금: e-5 큐레이터 DM과 단독 굿즈 조건표 (requires e-2, S2+)
- 3차 해금: e-6 AI 프롬프트 export와 해외 작가 원본 비교표 (requires e-3, S2+)
- 최종: e-7 공동 크레딧 합의 메모와 판매 수수료 정산 초안 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → B가 시안에서 공동 크레딧을 실제로 덜어냈다는 점이 드러난다
  - e-3+e-6 → A의 순수 창작 주장에 균열이 생긴다
  - e-2+e-7 → B가 운영 편의가 아니라 독식 구조를 설계했음이 확인된다

**중반 뒤집기**: 초반엔 B의 크레딧 삭제가 핵심 문제처럼 보이지만, e-6이 열리면 A 역시 원출처를 깨끗하게 설명하지 않았던 사실이 드러나면서 피해자 프레임이 흔들린다.

**판결 딜레마**: `현재의 공로 독식 vs 과거의 원출처 은닉`, `창작 윤리 vs 계약 윤리`, `공동 작업 신뢰 vs 개인 브랜드`가 맞부딪친다. 누구의 기여가 더 컸는지보다 어떤 위반을 더 중하게 볼지가 갈린다.

**난이도**: hard

### 3-04 임보캠의 새벽 4시 — friend (`friend-new-04`)

**훅**: 해외출장 중인 A가 친구 B에게 반려묘 임시보호를 맡긴 뒤, 새벽 4시 라이브캠 클립을 근거로 "고양이를 방치했다"는 글을 구조자 단톡에 올린다. B는 "A가 편집 클립으로 날 매장했다"고 하고, A는 "B가 병원비 영수증을 조작했다"고 한다.  
실제로는 한쪽이 가장 안 좋은 새벽 순간만 잘라 공개했고, 다른 쪽은 응급병원비를 부풀리고 후원사 샘플 사료 일부를 따로 빼돌렸다.

**A**: 정혜린(31, 공연기획 프리랜서) — 편집된 펫캠 클립으로 구조자 커뮤니티에 B를 먼저 낙인찍은 것 — 거짓말 전략: `victim_cosplay`  
**B**: 윤태오(30, 반려동물 용품 MD) — 응급병원비 일부를 부풀리고 후원 샘플 사료를 개인 판매한 것 — 거짓말 전략: `affect_flattening`

**쟁점** (5개):
- d-1: 고양이 상태 악화는 실제 방치였는가, 일시적 위기였는가
- d-2: A의 라이브캠 클립은 원본 맥락을 반영하는가
- d-3: B의 병원비와 사료 사용 내역은 정확한가
- d-4: 구조자 단톡 공개는 공익적 경고였는가, 여론전이었는가
- d-5: 동물 보호 명분과 친구 평판 파괴 중 어디에 선을 그을 것인가

**증거 체인**:
- 초기: e-1 새벽 4시 라이브캠 18초 클립, e-2 응급병원 영수증, e-3 구조자 단톡 캡처
- 1차 해금: e-4 펫캠 원본 타임라인 3시간 (requires e-1, S1+)
- 2차 해금: e-5 병원 EMR 요약과 결제 상세 (requires e-2, S2+)
- 3차 해금: e-6 후원사 샘플 재고표와 중고거래 내역 (requires e-3, S2+)
- 최종: e-7 급여 앱 기록과 산책·투약 체크 시트 원본 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → A가 가장 불리한 장면만 잘라 공개했다는 점이 드러난다
  - e-2+e-5 → B가 병원비 설명을 부풀린 부분이 확인된다
  - e-3+e-6 → B가 후원 샘플을 따로 빼돌린 구조가 열린다

**중반 뒤집기**: 초반엔 B의 방치가 사실인 듯 보이지만, e-4가 열리면 A가 편집으로 여론을 선점했다는 점이 드러난다. 이후 e-5와 e-6이 열리면서 B도 비용·후원물자를 정직하게 처리하지 않았음이 확인된다.

**판결 딜레마**: `동물 보호 vs 평판 제재`, `현재 공개 낙인 vs 숨겨진 비용 부풀리기`, `공익적 경고 vs 사적 보복`이 충돌한다. A의 경고가 완전히 허위는 아닐 수 있어도, 편집된 공개가 정당한지는 따로 판단해야 한다.

**난이도**: medium

### 3-05 정가표 없는 위탁부스 — friend (`friend-new-05`)

**훅**: 플리마켓 위탁부스에서 한정판 스니커즈와 빈티지 가방이 팔린 뒤 정산표가 맞지 않는다. A는 "B가 실제 판매가를 깎아 적고 수수료를 더 떼 갔다"고 하고, B는 "A가 수선 이력과 정품 감정 리스크를 숨겼다"고 한다.  
겉으로는 단순 판매 정산처럼 보이지만, 한쪽은 정가표와 현금 판매분을 나눠 적었고 다른 쪽은 하자 정보를 빼고 위탁을 맡겼다.

**A**: 문지유(30, 패션 콘텐츠 프리랜서) — 수선 이력과 감정 불확실성을 숨긴 채 위탁을 맡긴 것 — 거짓말 전략: `cold_logic`  
**B**: 하승민(31, 주말 플리마켓 셀러) — 실제 판매가와 현금 판매분을 다르게 기록하고 정산서를 편집한 것 — 거짓말 전략: `victim_cosplay`

**쟁점** (4개):
- d-1: 실제 판매가는 얼마였는가
- d-2: A는 하자·감정 리스크를 얼마나 숨겼는가
- d-3: B의 정산서 편집은 실수였는가, 의도적 차액 확보였는가
- d-4: 친구 사이 위탁 관행과 상인의 고지 의무는 어디서 갈라지는가

**증거 체인**:
- 초기: e-1 위탁 정산서, e-2 판매 부스 정가표 사진, e-3 구매자 컴플레인 DM
- 1차 해금: e-4 카드단말 매출·현금메모 비교표 (requires e-1, S1+)
- 2차 해금: e-5 수선 영수증과 감정 문의 답변 (requires e-3, S2+)
- 3차 해금: e-6 구매자와의 가격 협상 원본 채팅 (requires e-2, S2+)
- 최종: e-7 정산 시트 수정 이력과 현장 CCTV 스냅샷 (requires e-4+e-5, S3+)
- 핵심 조합:
  - e-1+e-4 → B가 판매금액 일부를 다르게 적었다는 점이 드러난다
  - e-3+e-5 → A가 하자·정품 리스크를 충분히 고지하지 않았음이 확인된다
