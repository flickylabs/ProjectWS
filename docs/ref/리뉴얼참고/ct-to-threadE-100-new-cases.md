## 목표

기존 84건 리디자인과 별개로, **완전히 새로운 사건 100건**의 기획안을 작성합니다.

이 게임의 핵심 재미는 **"둘 다 나쁜데 누가 더 나쁜지 판단하기 어려운"** 상황에서 플레이어가 심문과 증거로 진실을 파헤치는 것입니다. 100건 모두 이 원칙에 충실해야 합니다.

---

## 사건 기획 원칙

### 절대 규칙: "단순 악당 vs 피해자" 구조 금지

모든 사건에서 **양측 모두**:
- 숨기고 있는 사실이 있어야 함
- 자기에게 유리하게 프레이밍하고 있어야 함
- 상대에게 책임을 돌리고 있어야 함

"한쪽이 명백히 나쁜" 사건은 **재판할 이유가 없으므로 게임이 안 됩니다.**

### 자극성의 기준

"자극적"이란 **선정적이거나 폭력적**이라는 뜻이 아닙니다. 이 게임에서의 자극성:

1. **도덕적 불편함** — "이 사람 행동이 이해는 되는데 용납은 안 돼"
2. **반전의 충격** — "피해자라고 생각했는데 이 사람도 가담했어?"
3. **판단의 고통** — "둘 다 잘못했는데 책임을 어떻게 나누지?"
4. **현실 공감** — "나도 이런 상황에 처할 수 있겠다"
5. **증거의 배신** — "이 증거가 진짜라고 생각했는데 편집본이었어?"

### 다양한 시각의 판단축

같은 사건을 **다른 관점에서 보면 판결이 달라지는** 구조:

| 판단축 | A 방향 | B 방향 |
|--------|--------|--------|
| 현재 vs 과거 | 지금 저지른 잘못이 더 크다 | 과거의 원인 행위가 더 근본적이다 |
| 절차 vs 실체 | 절차를 어긴 쪽이 잘못이다 | 실질적 피해를 준 쪽이 잘못이다 |
| 의도 vs 결과 | 의도가 나빴으니 잘못이다 | 결과가 나빴으니 잘못이다 |
| 개인 vs 관계 | 개인의 권리를 침해했다 | 관계의 신뢰를 파괴했다 |
| 보호 vs 통제 | 보호하려 한 것이다 | 통제하려 한 것이다 |
| 공익 vs 사익 | 공공의 이익을 위해 한 것이다 | 결국 자기 이익을 챙긴 것이다 |
| 용서 vs 제재 | 사정을 참작해야 한다 | 선례를 남기면 안 된다 |
| 투명성 vs 사생활 | 알 권리가 있다 | 숨길 권리가 있다 |

**좋은 사건은 위 판단축 중 2~3개가 동시에 교차합니다.**

## 카테고리 배분 가이드

### 기존 7종 카테고리 (각 10건 = 70건)

| 카테고리 | 핵심 갈등 영역 | 피해야 할 함정 |
|----------|-------------|-------------|
| **부부 (spouse)** | 재정 통제, 사생활 침범, 신뢰 배신, 공동 재산, 양육권, 과거 상처 | 단순 외도/폭력 → 일방 악당 |
| **가족 (family)** | 상속, 간병 공로, 가업 승계, 세대 갈등, 비밀 유출 | 효도/불효 이분법 |
| **친구 (friend)** | 창작 크레딧, 금전 대차, 평판 조작, 비밀 유출, 공유 계정 | "친구니까 용서" 감정극 |
| **이웃 (neighbor)** | 소음, 보안 장비, 공용 공간, 커뮤니티 여론, 반려동물 | 민원 정도의 낮은 stakes |
| **동업 (partnership)** | 지분, 거래처, 기밀 유출, 정산, 브랜드 소유 | 숫자 맞추기 분쟁 |
| **세입자 (tenant)** | 보증금, 하자, 무단 출입, 계약 해석, 원상복구 | 집주인=악당 공식 |
| **직장 (workplace)** | 아이디어 탈취, 평가 조작, 내부고발, 감시, 계약 위반 | 갑=악 을=선 이분법 |

### 신규 카테고리 (각 10건 = 30건)

기존 7종으로는 커버하지 못하는 **현대적 갈등 영역**을 추가합니다:

| 카테고리 | 핵심 갈등 영역 | 사건 예시 방향 |
|----------|-------------|-------------|
| **온라인 (online)** | 커뮤니티 운영, 인플루언서, 리뷰 조작, 익명 폭로, 디지털 증거 위조, AI 생성물 저작권, 딥페이크 | "누가 먼저 올렸는가"보다 "왜 올렸는가"가 핵심 |
| **의료/교육 (professional)** | 의료 과실 은폐, 교사-학부모, 코치-선수, 전문가 윤리, 자격증/학위 | 전문 권한의 남용 vs 클라이언트의 과도한 기대 |
| **공공/제도 (civic)** | 보험 청구, 보조금 부정, 공익 제보, 자원봉사 갈등, 관리비/입주민 | 제도의 허점을 이용한 것 vs 제도가 사각지대를 만든 것 |

## 사건 1건의 기획 구조

각 사건은 아래 형식으로 작성합니다:

```markdown
### [번호] [제목] — [카테고리]

**훅**: (2줄, 플레이어를 끌어당기는 첫 인상)

**A**: 이름(나이, 직업) — 숨긴 것 — 거짓말 전략
**B**: 이름(나이, 직업) — 숨긴 것 — 거짓말 전략

**쟁점** (3~5개):
- d-1: ...
- d-2: ...
- d-3: ...

**증거 체인**:
- 초기: e-1, e-2, e-3 (표면만 보이는 증거)
- 1차 해금: e-4 (requires e-1, S1+)
- 2차 해금: e-5 (requires e-2, S2+)
- 최종: e-6 (requires e-3+e-5, S2+)
- 핵심 조합: e-1+e-4 → [뒤집기 포인트]

**중반 뒤집기**: (어느 시점에 판이 뒤집히는지)

**판결 딜레마**: (재판관 성향에 따라 어떻게 갈리는지)

**난이도**: easy / medium / hard
```

## 자극적인 사건 소재 방향 (영감)

아래는 **소재의 방향만** 제시합니다. 구체적 사건 설계는 스레드 E가 합니다.

### 디지털 시대 갈등
- 단톡방 캡처를 편집해서 증거로 제출 — 편집한 쪽 vs 원본을 숨긴 쪽
- AI로 만든 음성을 증거라고 제출 — 기술의 진위 vs 내용의 진위
- 배달 앱 리뷰 테러와 역리뷰 — 업주의 품질 vs 소비자의 권리
- 공유 계정 해킹과 데이터 인질 — 소유권 vs 접근권
- 실시간 방송 중 사고와 편집본 유포 — 사실 보도 vs 명예훼손

### 돈과 신뢰
- 크라우드펀딩 목표 달성 후 사양 변경 — 후원자 기대 vs 제작자 현실
- 가족 보증 후 연대채무 — 도운 쪽의 선의 vs 빌린 쪽의 은폐
- 공동 투자 실패 후 손절 타이밍 분쟁 — 한쪽은 버틸 가치, 한쪽은 손절 가치
- 퇴직금 정산 vs 경업금지 — 회사의 보호 vs 개인의 자유
- 중고거래 하자 담보 — 판매자의 고지 의무 vs 구매자의 확인 의무

### 관계와 권력
- 멘토-멘티 사이의 아이디어 차용 — 가르침의 범위 vs 창작의 경계
- 동아리/커뮤니티 리더의 사적 이용 — 공적 역할 vs 사적 이익
- 프리랜서 계약 범위 분쟁 — "추가 요청"의 범위 vs "당연히 포함"의 범위
- 공동 창업 후 한쪽만 미디어 노출 — 브랜드 기여 vs 성과 독식
- 학교 프로젝트 무임승차 — 결과물 크레딧 vs 실제 기여

### 공간과 생활
- 에어비앤비 호스트-게스트 분쟁 — 숙소 상태 vs 게스트 행동
- 층간소음 측정 앱의 증거 능력 — 디지털 측정 vs 실제 체감
- 공유 오피스 분쟁 — 공용 공간 사용 규칙 vs 관행
- 반려동물 사고 후 책임 — 견주 관리 의무 vs 피해자 과실
- 리모델링 하자와 시공사-발주자-세입자 3자 구도

### 사회적 이슈 (민감하되 균형 잡힌)
- 온라인 모금과 사용처 투명성 — 기부자의 알 권리 vs 수혜자의 사정
- 직장 내 차별 제보와 역차별 주장 — 구조적 문제 vs 개별 사정
- 환경 규제와 소상공인 생존 — 공익 vs 생계
- 의료 기록 공유와 보험 거부 — 건강 정보 보호 vs 보험사 정보 요구
- 학교 평가와 학부모 항의 — 교육 자율 vs 학부모 참여

## 품질 기준

### 필수 조건 (이것이 없으면 탈락)
1. 양측 모두 숨긴 사실이 있다
2. 증거 체인이 점진적 발견을 허용한다 (최소 6개 증거, 초기 3 + 해금 3)
3. 중반 뒤집기가 있다
4. 판결이 재판관 성향에 따라 갈린다 (판단축 2개 이상 교차)
5. 한국 사회에서 현실감 있는 소재다

### 우대 조건 (이것이 있으면 더 좋다)
- 양측 공모/공동 가담 후 서로 배신하는 구조
- 같은 증거를 양쪽이 반대로 해석하는 장면
- 증인이 양쪽 모두에게 불리한 사실을 알고 있는 구조
- 디지털 증거(채팅, SNS, CCTV, 앱 기록 등)가 핵심 역할
- 감정적으로 한쪽에 동조했다가 뒤집히는 순간

### 탈락 조건
- 한쪽이 명백히 나쁜 사건 (재판 불필요)
- 오해 해소형 (진실 밝혀지면 갈등 소멸)
- 범죄 수사물 (이 게임은 민사 분쟁 재판)
- 제3자가 진짜 범인인 사건 (A vs B 대결이 약해짐)
- 금액만 다투는 정산 분쟁 (증거 해석전이 없음)

## 기존 사건과의 차별화

이미 완성/진행 중인 사건들과 **소재가 겹치지 않도록** 주의하세요:

| 이미 있는 소재 | 피할 것 |
|-------------|---------|
| 리뷰어 vs 업주 (headline-01) | 유사 리뷰 분쟁 |
| 사생팬/매니저 (headline-02) | 유사 연예계 분쟁 |
| 리모델링 보험금 (spouse-11) | 유사 보험 사기 |
| 댓글 저격 + DM 위조 (friend-03) | 유사 SNS 위조 |
| 곰팡이 무단 출입 (tenant-02) | 유사 하자 분쟁 |
| 발명자 표기 (workplace-11) | 유사 특허 분쟁 |
| 지원금 부풀리기 (partnership-09) | 유사 보조금 분쟁 |
| 도어벨카메라 (neighbor-08) | 유사 CCTV 분쟁 |
| 보증금 허위 정리비 (tenant-09) | 유사 보증금 사기 |
| 미니홈피 과거 증거 (spouse-12) | 유사 과거 소환 |
| 부모 집 차액 (family-09) | 유사 부동산 공모 |

**같은 "종류"의 갈등이라도 설정과 증거 구조가 완전히 다르면 괜찮습니다.** 예: "보험 사기"는 이미 있지만, "보험 거부"는 다른 사건.
