# V3 배치 요청서 — GPT

## 프로젝트 배경

**Solomon**은 LLM 기반 법정 심문 게임입니다. 플레이어는 재판관으로서 양측 당사자(AI NPC)를 심문하여 진실을 밝힙니다.

### 아키텍처
- **엔진이 의미 결정**, LLM은 연기만. 매 턴 `ResponseBlueprint` 생성 → atom 선택 → LLM 1~3문장 응답.
- NPC는 lieState 상태 머신(S0 완전부정 → S5 시인)을 따름.
- **ClaimAtom**: 의미 단위 (사실+슬롯). 기존 publicClaim 문장을 원자로 분해한 것.
- **Slot**: 표현 변형 (exact: "2,800,000원", neutral: "해당 금액", judgeRef: "장모님")
- 질문: fact_pursuit(사실추궁), motive_search(동기탐색), empathy_approach(공감접근), evidence_present(증거제시)

### 이번 요청에서 만들 것
첨부된 사건 원본 JSON을 읽고, 아래 **4가지 데이터**를 한 번에 생성합니다.

## 요청 A: ClaimPolicy V2 (atom 기반)

각 파티 × 쟁점 × S0~S5 state마다 ClaimPolicy를 생성합니다.
핵심은 `claimAtoms` 배열 — publicClaim 문장을 원자로 분해.

### atom 분해 규칙
1. publicClaim의 각 문장을 1~2개 atom으로 분해
2. 숫자/날짜/인명은 반드시 slot으로 분리
3. 태그: act, timeline, motive, emotion, rule, evidence, context, identity, responsibility, relationship, counter, self_justification, harm, fear, shame, privacy, admission, denial, uncertainty, quote, threshold, beneficiary, institution, legacy_sentence (24종)
4. S0~S1: deny/hedge용 (핵심 회피), S2~S3: partial/blame (부분 인정+전가), S4~S5: emotional/confess (시인)
5. privateKnowledge → 고 state에서 해금되는 atom의 factText 소재
6. 각 atom에 stanceHints, usableInSubActions 부여

### 출력 구조
```json
{
  "caseId": "family-09",
  "claimPolicies": {
    "a": {
      "d-1": {
        "S0": {
          "disputeId": "d-1", "state": "S0",
          "publicClaim": ["원문 유지"],
          "privateKnowledge": ["원문 유지"],
          "suppressions": ["원문 유지"],
          "emotionalLeakRisk": "low",
          "tellPool": ["tell_id_1"],
          "claimAtoms": [
            { "id": "family09:a:d-1:act:0", "factText": "...", "tags": [...], "slots": {...}, "stanceHints": [...] }
          ]
        }
      }
    }
  }
}
```

### atom 예시 (spouse-01에서 발췌)
```json
{
  "id": "spouse01:a:d-1:act:0",
  "factText": "공동계좌에서 이체가 있었다는 사실",
  "tags": ["act", "timeline"],
  "slots": {
    "amount": { "exact": "2,800,000원", "rounded": "280만원", "neutral": "해당 금액" },
    "time": { "dateExact": "9월 20일", "period": "추석 연휴 직전", "neutral": "그 시점" }
  },
  "stanceHints": ["deny", "hedge"],
  "usableInSubActions": ["fact_pursuit", "evidence_present"]
}
```

## 요청 B: ExecutableVerbalTell + BeatScript

### Tell (파티당 3개 = 6개)
사건 원본의 `speechStyle`과 `archetype`을 반영한 말버릇 규칙.

```json
{
  "id": "tell_id",
  "appliesWhen": ["lying", "cornered", "emotional", "avoiding", "shame", "hurt", "defensive"],
  "lexicalHooks": ["어휘1", "어휘2"],
  "sentenceShape": "number_first|question_end|enumeration|echo_repeat|conditional|self_reference",
  "cadence": "every_turn|once_every_2_turns|max_once_per_turn|on_trigger_only",
  "originalPattern": "이 말버릇의 설명"
}
```

### Beat (30~40개)
쟁점별 deny/hedge/partial/blame/confession/evidence_hit × 해당 state.

```json
{
  "caseId": "family-09",
  "party": "a",
  "disputeId": "d-1",
  "beatType": "deny|hedge|partial|blame|confession|evidence_hit",
  "line": "1~3문장, 캐릭터 말투 반영",
  "behaviorHint": "행동/표정 연출 지시",
  "applicableStates": ["S0", "S1"],
  "afterEvidence": "e-1"
}
```

## 요청 C: V3 Game Loop

### DossierCard 3개
증거 2개를 묶은 카드. 각 카드에 A 대상 3질문 + B 대상 3질문.

```json
{
  "id": "dossier-1",
  "name": "카드 이름",
  "description": "1줄 설명",
  "evidenceIds": ["e-1", "e-2"],
  "relatedDisputes": ["d-1", "d-3"],
  "subjectParty": "both",
  "challenges": [
    {
      "targetParty": "a",
      "questions": [
        {
          "id": "dossier-1.a.q1",
          "text": "재판관 말투 질문",
          "attackVector": "identity|context|legality|authenticity",
          "onSuccess": { "blockVector": "identity", "revealAtom": "d1.unlock.s2.xxx", "lieAdvance": true }
        }
      ]
    }
  ]
}
```

### StateUnlockAtom
a(해당 쟁점) + b(해당 쟁점) × S2/S3/S4/S5 각 1~2개.
**revealAtom ID와 일치해야 함.**

```json
{
  "id": "d1.unlock.s2.xxx",
  "factText": "새로 드러나는 사실",
  "tags": ["context", "motive"],
  "unlockedAtState": "S2",
  "slots": { "person": { "fullName": "...", "judgeRef": "...", "neutral": "그 사람" } },
  "stanceHints": ["partial", "emotional"]
}
```

### 이벤트 텍스트
- 모순 3~4개, 끼어들기 3개, 감정폭발 양측 각 2개 (총 ~10개)

### TransitionBeat 20~24개
각 파티 × 쟁점 × S1→S2~S4→S5 (4단계).
d-5 같은 쌍방 쟁점은 양측 모두 필요.
