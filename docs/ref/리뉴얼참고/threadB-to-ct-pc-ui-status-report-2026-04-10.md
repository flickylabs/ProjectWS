# Thread B PC UI 통합 상태 보고

작성일: 2026-04-10  
작성 대상: CT  
범위: PC UI 리뉴얼, 플레이 UX 개편, 조합 스킬 UI, 증거 뷰어/액션 연결, Blocker 및 잔여 이슈 정리

## 1. 이번 턴까지 반영된 핵심 방향

이번 작업은 단순 스타일 수정이 아니라 아래 4개 축을 같이 정리하는 방향으로 진행했다.

1. Home / Session / 사건 선택 흐름을 모바일 잔재 없이 PC 전용 UX로 재구성
2. 플레이 화면을 좌/중앙/우 역할이 분명한 PC형 인터페이스로 재배치
3. 핫바, 액션, 조합 스킬, 증거 제시/열람 흐름을 실제 플레이 체인에 맞게 다시 연결
4. 증거 해금, Phase 제한, verdict stage progression, 일반 모드 신규 100건 매핑 같은 엔진 연동 이슈를 같이 정렬

## 2. Home / Session / 사건 선택 UI 반영 사항

### 2-1. Home 재설계

- Home는 텍스트 나열형 구조에서 PC 전용 허브형 구조로 재구성했다.
- 초기에는 로비형 대형 카드 구조로 갔고, 이후 사용자 피드백에 맞춰 더 심플한 구조로 다시 눌렀다.
- 현재는 중앙 타이틀 + 상단 아이콘 + 일반/시즌 모드 메인 카드 + 하단 정보 카드 위상으로 정리되어 있다.
- `일반 모드 >`, `시즌 모드 >` 형태로 CTA를 단순화했고 카드 전체가 버튼처럼 보이도록 조정했다.
- `판결 기록`, `설정` 버튼은 Home에서 제거했다.
- 하단은 `내 정보`, `리더보드` 중심의 단순 구조로 재정리했다.

### 2-2. Session 카드 개선

- 세션별 상징 이모지, 감성 문구, 진행도(`n/total`)를 반영했다.
- 세션 진행 완료 시 평균 점수를 노출하도록 구성했다.
- 세션 선택 화면은 한 줄 4개 배치 기준으로 폭을 줄였다.
- 카드 하단 부연 설명, `세션 입장` 같은 중복 CTA는 제거했다.

### 2-3. 사건 리스트 / Case Browser

- 기존 카드 그리드를 버리고 스테이지 맵 + 인스펙터 구조로 변경했다.
- 스테이지 번호 hover / click / pin 구조를 붙였다.
- 클릭 시 짧은 제목, 난이도, 점수, 요약, `상세 보기 >`만 보이게 정리했다.
- `상세 보기`를 누르면 한 번 더 중간 단계를 거치지 않고 바로 상세 패널이 보이도록 수정했다.
- 상세 패널은 좌측 설명, 우측 캐릭터/난이도/증거 메타 구조로 재정리했다.
- `재판 진행` 버튼은 더 강한 CTA로 변경했다.

### 2-4. 브리핑 화면

- `A vs B` 같은 임시형 제목 대신 사건 제목형 타이틀을 우선 사용하도록 수정했다.
- 우측 캐릭터 패널 폭과 높이를 재조정했다.
- `당사자 A`, `당사자 B` 라벨은 제거 방향으로 정리했다.
- `재판 진행` 버튼을 더 도드라지게 조정했다.

## 3. 플레이 화면 UX / 레이아웃 반영 사항

### 3-1. 전체 위상 재정리

- 좌측: 증거 / 중요 발언 노트 / 숨김형 사건 타임라인
- 중앙: 대사 로그 / 중앙 오버레이 / 대화 진행
- 우측: 현재 대상 / LieState / 대상 미터 / 조합 스킬

초기에 남아 있던 모바일 잔재와 중복 패널을 줄이고, PC 중심 구조로 다시 정리했다.

### 3-2. 상단 정보 구조 변경

- 좌측 상단 `솔로몬 법정 ~ Phase` 헤더는 중앙 정렬 구조로 옮기는 방향으로 조정했다.
- 좌측 상단에는 `나가기` 버튼을 배치했다.
- 우측 상단 아이콘은 돋보기 / 번개 / 저울 / Turn / 설정 순으로 재정렬했다.
- 우측 상단 아이콘들은 클릭 인터랙션을 붙였다.

### 3-3. 중앙 Dialogue / Phase 오버레이

- 모바일식 `초기 진술` 잔상, `DIALOGUE PHASE`류 잔재는 단계적으로 제거했다.
- 대화 시작 시 화면을 가리는 하단 고정 패널 대신 중앙 오버레이형 구조로 바꿨다.
- 일부 모바일 전환 선택 모달(`균열`, `궁지`, `증거를 낸다`류 카드)은 PC 중앙 패널로 치환했다.

### 3-4. 좌측 패널 재구성

- 쟁점 현황을 좌측 패널에서 빼고 상단 중앙 `쟁점 리본`으로 옮겼다.
- 그만큼 확보된 공간에 `증거`, `중요 발언 노트`를 위로 끌어올렸다.
- 중요 발언 노트는 고정 높이 내부 스크롤 구조로 변경했다.
- 중요 발언 노트는 drag reorder가 가능하도록 구성했다.
- 사건 타임라인은 기본 숨김 + 북마크 확장 구조로 바꿨다.
- 타임라인 확장 영역은 별도 오버레이 컬럼처럼 펼쳐지며 내부 스크롤 위치를 유지한다.

### 3-5. 우측 패널 재구성

- 대상 선택 토글과 캐릭터 프로필 프레임을 한 세트처럼 보이게 재설계했다.
- 선택된 캐릭터에 맞춰 파란/빨간 라인 강조가 들어가도록 구성했다.
- 상단 대상 프레임과 하단 좌우 캐릭터 선택 상태가 연동되도록 정리했다.
- 중복되던 `신뢰` 게이지는 제거하고, 상태 칩과 대상 미터 관계를 정리했다.
- 현재 기준 우측 대상 미터는 `누설`, `모순` 중심으로 정리된 상태다.

### 3-6. 쟁점 리본

- 쟁점 바를 중앙 상단 토스트형 리본으로 옮겼다.
- `| 쟁점 | 01 | 02 | 03 | ... |` 형태의 축약 상태와 hover / click 확장 구조를 붙였다.
- hover 시 임시 확장, click 시 고정 확장, 재클릭 시 축소되게 했다.
- 확장된 쟁점은 관련 단서와 연결 정보를 보여준다.
- 해당 쟁점과 관련된 액션이 발생하면 그 쟁점이 강조되도록 연결했다.

## 4. 핫바 / 액션 UX 반영 사항

### 4-1. 기본 방향 수정

초기 PC 구현은 `캐릭터 선택 -> 핫바 액션 클릭 -> 추가 패널 -> 실행` 형태가 섞여 있어서 흐름이 무거웠다.  
이후 사용자 피드백에 따라 아래 기준으로 다시 정리했다.

- 캐릭터 선택 + 세부 액션 선택
- 필요 없는 경우는 즉시 실행
- 증거, 질문 선택, 자유질문 입력처럼 실제 선택이 필요한 경우만 중앙 패널을 띄움

### 4-2. 현재 액션 흐름

- 캐릭터 선택 + 질문/특수 행동류:
  중앙 선택 패널에서 실제 쟁점/질문/실행 대상을 고른 뒤 진행
- 캐릭터 선택 + 증거:
  중앙 패널에서 해당 대상에게 relevant한 증거만 고르고 제시 방식 선택
- 증거 열람:
  별도 PC 증거 뷰어 오픈
- 자유질문:
  중앙 입력 패널

즉, “아무 액션이나 누르면 무조건 또 다른 패널이 뜨는 상태”는 줄였고, 의미 있는 선택이 필요한 경우에만 중앙 패널이 뜨도록 정리했다.

### 4-3. Phase 제한 / 리소스 체크

- Phase별 탭 잠금은 계속 유지했다.
- 증거 제시는 `Phase 1~3`에서 버튼은 보이지만 잠금 상태로 유지한다.
- `Phase 4~5`에서 해금되도록 맞췄다.
- 조사 토큰 / 스킬 포인트 / 관련 대상성 / 이미 제시한 증거 여부도 같이 체크한다.

## 5. 조합 스킬(Combination Lab) 반영 사항

### 5-1. 구조 변경

조합 기능은 다른 스레드의 확장 작업과 사용자 피드백을 반영해서 크게 손봤다.

- 우측 하단 고정 실험실 느낌에서
- `작고 확실히 강조된 조합 스킬 박스`로 재구성

### 5-2. 현재 UX

- 증거 / 중요 발언 노트 모두 `드래그앤드롭`, `Shift+클릭`으로 적재 가능
- `A`, `B` 2슬롯에 순차 적재
- 조합 영역 내부 어디에 드롭해도 `A -> B` 순으로 들어감
- 둘 다 채워지면 같은 영역 안에서 큰 `조합 시도` 버튼으로 상태가 전환됨
- 성공/실패 모두 중앙 PC 팝업으로 크게 피드백
- 조합 시도 후 슬롯은 리셋

### 5-3. 추천 조합 / 보조 UX

- `!` 버튼을 통해 추천 조합 힌트를 볼 수 있도록 했다.
- `!` 위치는 `조합 스킬` 타이틀과 같은 줄 상단으로 정리했다.
- 조합 카드 크기는 고정하고, 상태만 내부에서 바뀌도록 조정했다.
- `A`, `B` 슬롯은 평소 내용 전문이 아니라 “채워짐 여부”만 간단히 보여주고 hover로 내용을 확인하게 했다.

### 5-4. 요약 스킬

- 조합 영역과 별개로 `요약 스킬` 영역을 분리했다.
- `요약 실행`은 스킬 소비형 잠금 해제 -> 이후 반복 사용 구조를 목표로 붙였다.
- 현재 UI/스토어 경로는 구성했지만, 사건별 요약 품질은 generic한 편이다.

## 6. 증거 시스템 / 증거 뷰어 / 상세 뷰어 반영 사항

### 6-1. 초기 증거 해금 Blocker 수정

초기에는 게임 시작 시 모든 증거가 해금되어 보이는 Blocker가 있었다.  
이 부분은 엔진 및 store 경로를 정리해서 고쳤다.

- `baseEvidenceIds`가 있으면 그 증거만 초기 해금
- 없으면 기본 3개만 초기 해금
- 조건 없는 증거를 후속 unlock check에서 무조건 열어버리던 경로도 차단

### 6-2. 증거 제시 / 열람 / 조사 연결 정리

- 좌측 증거 카드 팝업에 `증거 열람 / 증거 제시 / 관련 쟁점 보기`를 정리했다.
- `증거 제시`는 현재 대상 NPC 기준 relevant evidence만 중앙 패널에 노출되게 연결했다.
- 이미 해당 대상에게 제시한 증거는 재제시 방지 로직을 넣었다.

### 6-3. PCEvidenceViewer 재연결

이전에는 `caseData.evidence`와 runtime `evidenceDefinitions` 참조가 어긋나서 열람이 제대로 안 뜨는 문제가 있었다.

현재는 아래를 정리했다.

- runtime `evidenceDefinitions`에서 실제 evidence node를 찾음
- `viewerData`가 있으면 runtime 기준 상세 뷰어로 사용
- 없으면 generic fallback viewer 사용

### 6-4. 타입별 상세 뷰어 매핑 수정

초기 상태는 아래 문제가 있었다.

- `viewerData.meta.type`가 `photo_video`, `document`, `platform_log` 등 원본 타입으로 남아 상세 뷰어 switch를 통과하지 못함
- `EvidenceSubViewers.tsx`가 깨진 상태라 의도한 PC 서브 뷰어가 안정적으로 쓰이지 않음

이후 아래를 반영했다.

- runtime viewer type을 상세 뷰어용 타입으로 정규화
- `bank`, `chat`, `contract`, `testimony`, `cctv`, `log`, `device`, `sns` 뷰어를 실제로 다시 구성
- `.ev-bank-table`, `.ev-chat-bubble`, `.ev-cctv-frame`, `.ev-device-shell`, `.ev-sns-post` 구조를 CSS와 함께 재반영

### 6-5. 중요한 제한 사항

타입별 상세 뷰어는 이제 사용되지만, “실제 이미지가 보이는 증거”는 아직 거의 없다.  
이유는 데이터에 실제 이미지 URL / 썸네일 / 미디어 첨부 필드가 거의 없기 때문이다.

즉:

- 상세 뷰어 셸은 매핑됨
- 하지만 비트맵 이미지 자체는 데이터가 없어서 안 뜨는 케이스가 많음

이건 아래 `Thread A 요청 필요 항목`에 정리했다.

## 7. Verdict / stage progression / 모드 매핑 반영 사항

### 7-1. VerdictScreen stageMap

- `headline`, `friend`, `online`, `professional`, `civic`가 누락되어 stage progression이 빠지던 문제를 수정했다.
- 단순 stageMap 추가뿐 아니라 caseId prefix 기준 보정도 같이 넣었다.

### 7-2. 일반 모드 10세션 매핑

- 기존 84건 + 신규 100건(DROP 2 제외) 기준으로 일반 모드 10세션에 매핑했다.
- 기존 7종 세션은 `기존 + new` 합산
- 신규 3종 세션은 `online-new-*`, `professional-new-*`, `civic-new-*` 전용
- 시즌 모드는 현재 비워둔 상태로 유지

### 7-3. 시즌 모드

- 한 시점에는 20건 allowlist 매핑을 했었으나, 이후 CT 지시에 따라 다시 비우는 방향으로 정렬했다.
- 지금 기준으로 시즌은 별도 선별 전까지 비워두는 상태를 기준으로 맞춰져 있다.

## 8. 기타 오류 수정

### 8-1. 모바일 잔재 제거

- 모바일 `TransitionChoiceModal`류가 PC 화면에 그대로 튀어나오던 문제를 PC 중앙 패널로 치환했다.
- `DIALOGUE PHASE` 보조 패널, `초기 진술` 잔상 등 모바일성 연출도 단계적으로 제거했다.

### 8-2. 텍스트 / 라벨 / 인코딩 정리

- 우측 패널 archetype / tell 태그의 영문 노출을 한국어 라벨로 교체했다.
- `과정밀 진술`, `농담 방패`, `공격형`, `냉정 논리` 등으로 정리
- 발언 노트 제목은 `쟁점명 + 화자` 규칙으로 바꿨다.

### 8-3. 우측 상단 아이콘 인터랙션

- 돋보기 / 번개 / 저울 / Turn / 설정 아이콘에 클릭 반응과 정보 표시를 붙였다.

## 9. 현재 남아 있는 문제 / 미완 항목

아래는 아직 완전히 끝나지 않았거나, 실제 브라우저 시각/플레이 기준 추가 검증이 필요한 항목이다.

### 9-1. 실제 플레이 전체 클릭 체인 전수 점검

- 개별 기능 연결은 계속 보강했지만, “모든 케이스 / 모든 Phase / 모든 액션 조합”에 대한 수동 플레이 검증은 아직 전수 완료 상태가 아니다.
- 특히 `질문 -> 응답 -> 균열/궁지 -> 추가 선택 -> 증거 제시 -> 후속 피드백` 흐름은 케이스별 smoke가 더 필요하다.

### 9-2. Verdict / Result 화면 전면 리디자인

- Verdict / Result는 일부 stage progression 문제는 해결했지만, 화면 디자인 자체는 Home/플레이 레벨로 아직 완성되지 않았다.
- CT가 지정한 우선순위 기준으로는 이 부분이 다음 큰 작업이다.

### 9-3. 캐릭터 SVG 확장

- 현재는 일부 PC face symbol과 fallback이 작동하지만, 남/여 + 연령대 + 표정까지 정교하게 다루는 최종 형태는 아니다.
- 정밀 매핑은 추가 메타가 있으면 더 좋아진다.

### 9-4. 일부 과거 파일의 인코딩/문자열 리스크

- 이번에 PC 라벨과 몇몇 핵심 파일은 정리했지만, 오래된 데이터/샘플/레거시 파일 중에는 여전히 문자열 깨짐 리스크가 있는 파일이 남아 있을 수 있다.
- 실제 브라우저에서 다시 문제로 보이는 구간은 추가로 걷어낼 필요가 있다.

## 10. Thread A 지원이 있으면 좋은 항목

이번 수정 대부분은 Thread B 단독으로 처리 가능했다.  
다만 아래 항목은 Thread A가 데이터 차원에서 보강해 주면 품질이 크게 올라간다.

### 10-1. 실제 이미지/미디어 증거 필드

현재 상세 뷰어는 타입별 셸로 매핑되어 있다.  
하지만 실제 비트맵 이미지가 보이려면 evidence `viewerData`에 아래 같은 필드가 필요하다.

- `imageUrl`
- `thumbnailUrl`
- `mediaUrl`
- `posterUrl`
- 또는 타입별 media attachment 필드

지금 데이터는 대부분 구조화 텍스트여서, 상세 뷰어가 떠도 “이미지”는 거의 안 나온다.

### 10-2. 캐릭터 face 메타

정확한 PC 얼굴 매핑을 위해 사건 당사자별:

- `pcFaceType`
- 또는 `gender`
- `ageBand`
- `baseExpression`

같은 필드가 있으면 좋다.

### 10-3. 사건 표시용 제목/티저

지금은 일부 사건에서 첫 쟁점명 fallback을 제목처럼 쓰는 경우가 있다.  
각 케이스에 아래가 있으면 더 안정적이다.

- `displayTitle`
- `teaserShort`
- `briefTitle`

### 10-4. 발언 노트 전용 제목

지금은 `쟁점명 + 화자` 규칙으로 처리했지만, 작가 의도 수준으로 더 정교하게 가려면:

- `noteTitle`
- `noteShortLabel`

같은 필드가 있으면 좋다.

### 10-5. 요약 스킬 전용 콘텐츠

현재 `요약 스킬`은 generic 구조로 가능하지만, 사건별 전용 요약 품질을 올리려면:

- `summarySections`
- `judgeMemo`
- `keyEvidenceSummary`

같은 필드가 있으면 좋다.

## 11. 현재 기준 검증 상태

최근 반영분 기준 반복 확인한 항목:

- `npx tsc -p tsconfig.app.json --noEmit` 통과
- `npm run build:pc` 통과

남아 있는 공통 경고:

- CSS `@import` 순서 경고 1건
- large chunk 경고

이 둘은 현재 기능 Blocker는 아니다.

## 12. CT 확인 요청 포인트

다음 라운드 작업 우선순위를 CT 기준으로 다시 정하면 좋다.

1. Verdict / Result 화면 전면 리디자인
2. 남은 플레이 클릭 체인 전수 점검
3. 캐릭터 SVG 확장
4. Thread A에 evidence media / face meta / displayTitle / summarySections 요청 여부 확정

특히 `실제 이미지가 보이는 증거`를 원하면, 이건 UI 쪽만으로는 끝나지 않고 Thread A 데이터 보강이 필요하다.
