# CT → 스레드 E: GPT Pro 5.4 검토용 100건 정리 요청

> 발신: CT
> 수신: 스레드 E (사건 기획)
> 일시: 2026-04-09

---

## 목적

100건 전체를 **GPT Pro 5.4에게 넘겨서 재미 검증 + 스토리 디벨롭**을 받으려 합니다. GPT가 각 사건을 빠르게 판단하고, 구체적 개선안을 낼 수 있도록 **GPT 친화적 포맷**으로 정리해 주세요.

## 산출물

**단일 파일** `thread-e-100-cases-for-gpt-review.md`

## 포맷 요구 사항

### 사건 1건의 구조

```markdown
---
## [N] [제목] | [카테고리] | [난이도]

**훅**: (2줄 — 첫 인상에서 끌리는 설정)

**A**: [이름](나이, 직업) — 숨긴 것: [핵심 1줄] — 전략: [archetype]
**B**: [이름](나이, 직업) — 숨긴 것: [핵심 1줄] — 전략: [archetype]

**쟁점**: d-1 [이름] / d-2 [이름] / d-3 [이름] / ...

**증거 체인**:
- 초기: e-1 [이름], e-2 [이름], e-3 [이름]
- 해금: e-4 [이름](조건: e-1 제시 + S1), e-5 [이름](조건: e-2 + S2), ...
- 최종: e-7 [이름](조건: e-3+e-5 + S2)
- 조합: e-1+e-4 → [뒤집기 효과] / e-2+e-5 → [효과] / e-3+e-6 → [효과]

**중반 뒤집기**: [어느 시점에 뭐가 드러나서 판이 뒤집히는지 1~2줄]

**판결 딜레마**: [재판관 성향에 따라 어떻게 갈리는지 1~2줄]

**판단축**: [현재 vs 과거] [절차 vs 실체] [보호 vs 통제] 등 해당하는 것 나열
---
```

### 핵심 원칙

1. **압축적으로** — GPT가 100건을 한 번에 읽어야 하므로, 사건당 **15줄 이내**
2. **구조적으로** — 모든 사건이 동일한 필드 순서. GPT가 비교/패턴 인식하기 쉽게
3. **판단 재료를 명시** — 훅, 숨긴 것, 뒤집기, 딜레마가 빠지면 재미 판단 불가
4. **번호를 연속으로** — 1~100번 연속 번호. 카테고리는 태그로만 표시

### 순서

배치 순서(부부→가족→...→공공)를 유지하되, 번호는 1~100 연속.

### 파일 크기

100건 × 15줄 = 약 1,500줄. GPT Pro 5.4의 컨텍스트에 충분히 들어가는 분량입니다.

### 기존 배치 파일과의 관계

배치 파일(batch-01~10)에는 상세 프로필/증거 해금 조건/LieState 전이 등이 있지만, GPT 검토용에는 **재미 판단에 필요한 핵심만** 넣습니다. 상세 데이터는 GPT 피드백 후 스레드 A가 생성 시 참조.
