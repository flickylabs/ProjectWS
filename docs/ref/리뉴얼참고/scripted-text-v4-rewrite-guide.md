> 발신: CT (Control Tower)
> 대상: GPT Pro 프롬프트 설계 + Thread R 재작성 기준
> 기준점: 유저가 직접 작성한 Phase1 spouse-v3-01.json

## 1. 핵심 원칙

### Phase1에서 유저가 확립한 기준

1. **캐릭터가 실제로 할 법한 말만 쓴다**
   - ✗ "'오피스텔 출입과 새벽 통화는 딴살림 흔적인가' 문제에 대해서는..."
   - ✓ "'남편의 딴집 살림'을 그냥 넘길 수는 없습니다"

2. **쟁점을 캐릭터의 믿음/관점으로 표현한다**
   - A는 "딴살림"이라고 확신하며 말함 (본인 관점)
   - B는 "그건 아니다"로 부정하되 구체적 설명은 안 함 (회피 관점)
   - 같은 사건을 양쪽이 완전히 다른 프레임으로 말함

3. **각 대사가 새로운 정보를 추가한다**
   - 같은 말을 톤만 바꿔서 반복하지 않음
   - 한 대사마다 플레이어가 "아, 이런 것도 있구나" 하는 순간이 있어야 함

4. **정보의 비대칭이 유지된다**
   - A는 B의 진실을 모름 → 자기가 믿는 대로 말함
   - B는 A의 비밀을 모름 → 자기 사정만 방어함
   - 플레이어는 양쪽을 들으며 퍼즐을 맞춤

5. **기획 용어/문서 용어 절대 금지**
   - "실행 책임", "셋째 갈래", "다음 쟁점", "상대 쪽 조치 책임"
   - 쟁점 제목을 따옴표로 그대로 인용하지 않음

## 3. 질문 유형별 고유 정보 설계

### 원칙: 같은 LieState에서도 질문 유형에 따라 다른 "조각"이 나온다

| 질문 유형 | 주는 정보 조각 | 톤 |
|----------|-------------|-----|
| **사실추궁** (fact_pursuit) | "무엇을" — 구체적 사실, 시간, 장소, 금액, 행동 | 단호, 논리적 |
| **동기탐색** (motive_search) | "왜" — 동기, 이유, 판단 근거, 심리 | 설명 요구, 탐색적 |
| **공감접근** (empathy_approach) | "어떤 마음으로" — 감정, 두려움, 내면 갈등, 후회 | 공감적, 부드러움 |

### 예시: B를 심문, d-1, S2

| 유형 | B의 대답 | 고유 정보 |
|------|---------|----------|
| 사실추궁 | "그곳에 드나든 건 맞습니다. 주로 저녁 7시쯤 가서 9시 전에 나왔습니다" | **방문 시간대 특정** |
| 동기탐색 | "말하면 더 복잡해질 것 같아서 안 했습니다. 집에서 꺼내기 어려운 사정이었습니다" | **"집에서 꺼내기 어렵다" = 가정 내 갈등 힌트** |
| 공감접근 | "돌아올 때마다 마음이 무거웠습니다. 아내 얼굴 보면서 거짓말하는 게 제일 힘들었습니다" | **죄책감 + 아내에 대한 감정** |

**3가지를 조합하면:** "저녁 7~9시에 가고, 가정 사정 때문에 말 못 하고, 아내에게 죄책감을 느꼈다"
→ 하나만 들으면 부분적, 셋 다 들으면 입체적 이해

### 게임플레이 보상: 질문 다양성 보너스

- 같은 쟁점에 3가지 유형을 모두 사용하면 → "종합 이해" 보너스 (판결 시 정확도 ↑)
- 한 가지만 반복하면 → 피로도 누적 + 같은 정보 반복

## 4. variant 설계 규칙

### 현재 문제: 5개 variant가 같은 말의 패러프레이즈

```
[현재 — BAD]
v1: "매일 저녁 같은 곳에 들어간다"
v2: "퇴근하면 어디론가 사라진다"
v3: "같은 곳을 들락날락한다"
v4: "매일 저녁 두 시간씩 어딜 다녀온다"
v5: "같은 장소, 같은 시간, 같은 번호"
→ 전부 "남편이 매일 어딘가에 간다"의 변형
```

### 변경: 5개 variant는 같은 핵심 사실의 "다른 측면"

```
[변경 — GOOD]
v1: 핵심 사실 (가장 직접적, 기본형)
v2: 핵심 사실 + 부가 디테일 A
v3: 핵심 사실 + 부가 디테일 B  
v4: 핵심 사실 + 감정/반응 결합
v5: 핵심 사실 + 다른 쟁점 연결 암시
```

예시 (A, d-1, S1, fact_pursuit):
```
v1: "차에서 영수증을 봤습니다. 생리대, 스타킹이 찍혀 있었습니다."
v2: "영수증에 생리대, 스타킹, 컵죽까지 있었습니다. 누군가 그 집에서 생활하고 있다는 뜻입니다."
v3: "같은 편의점에서 반복 결제였습니다. 오피스텔 바로 아래층 편의점이에요."
v4: "그 영수증을 보는 순간 손이 떨렸습니다. 생리대요. 제 것이 아닌 생리대."
v5: "영수증 날짜를 보니 적금이 줄기 시작한 시점과 겹칩니다."
```

→ v1은 기본, v2는 품목 상세, v3는 장소 연결, v4는 감정, v5는 d-2 연결
→ 랜덤 1개가 나와도 핵심 정보는 전달되지만, 다른 variant가 나오면 추가 힌트

## 5. 호칭/어체 규칙 (Phase1 기준 계승)

### 심문 대사 (대부분)
- 재판관에게 하는 말 = 합니다체
- "재판관님,"은 첫 발언에서만. 이후 생략 가능
- 상대를 언급할 때: A→"제 남편", B→"제 아내"

### 끼어들기/감정 폭발 시
- 상대에게 직접 = 반말
- spouse: "자기야," (구어체)
- "이준호!" / "박지연!" (격앙 시)

### 절대 금지
- "자기, ~~입니다" (호칭 반말 + 어체 합니다체 모순)
- 기획 용어 ("실행 책임", "셋째 갈래")
- 쟁점명 따옴표 그대로 인용

## 6. behaviorHint 규칙

Phase1에서 유저가 작성한 기준:
```
✓ "억울함을 앞세우며 자기 행동의 선긋기를 시도한다."
✓ "남편에게 직접 반말로 몰아붙이며 감정이 터진다."
✓ "재판관에게 공정한 분리 판단을 호소한다."

✗ "첫 문장에서는 자기 행동의 출발점과 선긋기를 함께 깐다." (프롬프트 메모)
✗ "a S0 fact_pursuit ?? 1" (코드 디버그)
```

behaviorHint = **캐릭터의 표정, 말투, 분위기** 묘사
- 짧게 (1~2문장)
- 캐릭터의 archetype 질감 반영
- A(victim_cosplay): 피해자 포지셔닝, 억울함, 과장
- B(avoidant): 최소 발화, 회피, 눈 마주침 회피

## 7. 증거 제시 응답 설계

### 원칙: 증거는 심문으로는 알 수 없는 "확정 사실"을 준다

| 채널 | 역할 |
|------|------|
| 심문 → ScriptedText | 캐릭터의 "말"로 얻는 정보 (주관적, 불완전) |
| 증거 제시 → evidence_present | 물증이 "확정"하는 정보 (객관적, 확실) |

예시:
- 심문에서 B가 "가족 쪽 일"이라고 함 → 누구? 뭐? 불분명
- e-4(형 문자) 제시 → "오늘도 부탁해" 메시지 확인 → 형이 돌봄을 요청했다는 확정

### evidence_present 대사 규칙
- 증거의 **구체적 내용**에 반응해야 함
- "그 자료에 대해서는..." 같은 범용 반응 금지
- 캐릭터가 그 증거를 보고 실제로 할 법한 반응

## 8. GPT Pro 프롬프트 설계 핵심

GPT Pro에 전달할 때 반드시 포함할 지시:

```
### 필수 지시

1. 각 LieState(S0~S5)에서 반드시 1개 이상의 NEW FACT를 포함하세요.
   NEW FACT = 이전 State에서는 알 수 없었던 구체적 사실.
   같은 사실을 다른 톤으로 반복하는 것은 NEW FACT가 아닙니다.

2. 질문 유형별로 다른 정보 조각을 주세요:
   - fact_pursuit: "무엇을" (사실, 시간, 장소, 금액)
   - motive_search: "왜" (동기, 이유, 판단 근거)
   - empathy_approach: "어떤 마음으로" (감정, 두려움, 내면)

3. 5개 variant는 같은 핵심 사실의 다른 측면입니다:
   v1: 기본형 (핵심 사실만)
   v2: 핵심 + 부가 디테일 A
   v3: 핵심 + 부가 디테일 B
   v4: 핵심 + 감정/반응
   v5: 핵심 + 다른 쟁점 연결 암시

4. 대사에서 다음을 절대 사용하지 마세요:
   - 쟁점 제목 원문 따옴표 인용
   - 기획 용어 (실행 책임, 갈래, 상대 쪽 조치)
   - Party 코드 (A, B)
   - 발표 문체 ("셋째 갈래인", "다음 쟁점은")

5. Truth Throttle을 준수하세요:
   S0~S1: 구체적 명사(형, 조카, 3000만원 등) 금지
   S2: 범위 수준("가족 쪽", "몇천만 원")
   S3+: 구체적 명사 허용

6. 호칭/어체:
   재판관에게: 합니다체 + "제 남편"/"제 아내"
   상대에게 직접(감정 폭발 시만): 반말 + "자기야"
   "재판관님,"은 첫 발언에서만

7. behaviorHint = 캐릭터의 표정/분위기 (1~2문장)
   프롬프트 요약이나 코드 태그 금지
```

### 프롬프트에 포함할 데이터

각 쟁점별로:
1. NEW FACT 매핑 테이블 (위의 State별 테이블)
2. 질문 유형별 고유 정보 (위의 3조각 설계)
3. 캐릭터 archetype + callTerms
4. Truth Throttle 단계별 허용 범위

## 9. 검증 체크리스트 (Thread QW용)

### 필수 PASS 항목
- [ ] 각 LieState에서 이전 State에 없던 NEW FACT가 있는가
- [ ] fact_pursuit / motive_search / empathy_approach가 다른 정보 조각을 주는가
- [ ] 5개 variant가 패러프레이즈가 아니라 다른 측면을 보여주는가
- [ ] Truth Throttle 준수 (S0~S1 구체명사 금지)
- [ ] 호칭/어체 규칙 준수
- [ ] 기획 용어/쟁점명 원문/Party 코드 0건
- [ ] behaviorHint가 캐릭터 분위기만 포함
- [ ] A의 대사에 B만 아는 사실이 없고, B의 대사에 A만 아는 사실이 없는가 (정보 비대칭)
