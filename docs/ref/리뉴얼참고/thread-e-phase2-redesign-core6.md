## workplace-11 — 특허 마감 전야

### 사건 훅
잠정출원 마감 12시간 전, 팀장이 연구원의 이름을 발명자 목록에서 뺐다는 고발이 터진다.  
그런데 추궁을 이어가면, 둘 다 출원 전 청구항과 실험 데이터를 외부 특허 브로커에게 넘기고 지분성 대가를 나눠 갖기로 했던 공모가 드러난다.

### 양측 프로필
- A: 권태준, 플랫폼IP전략팀 팀장, 숨긴 것=`발명자 삭제를 통해 회사 내 공을 독식하려 한 점`, archetype=`cold_logic`
- B: 노서린, 선임 알고리즘 연구원, 숨긴 것=`외부 브로커와 먼저 접촉해 데이터를 넘기고도 피해자처럼 행동한 점`, archetype=`victim_cosplay`

### 쟁점 구조
- d-1: 발명자 목록에서 노서린 이름이 빠진 이유는 무엇인가
- d-2: 외부 특허 브로커에게 전달된 자료는 누가 먼저 열었는가
- d-3: 유출된 것은 요약 수준인가, 실제 청구항과 실험 데이터인가
- d-4: 둘은 발명자 다툼을 방패로 공모를 숨기려 했는가
- d-5: 회사 기여와 개인 아이디어 권리를 어디까지 분리할 수 있는가

### 증거 설계
- 초기 공개: e-1 잠정출원 초안, e-2 연구노트 사본, e-3 특허 데모 일정표
- 1차 해금(S1+): e-4 DLP 경보 로그
  - requires: `e-1`
- 2차 해금(S2+): e-5 브로커 카페 미팅 영수증 + 출입기록
  - requires: `e-4`
- 3차 해금(S3+): e-6 외부 전송 ZIP 메타데이터
  - requires: `e-2`, `e-5`
- 최종 해금(S4+): e-7 수익 배분 합의 채팅
  - requires: `e-6`
- 증거 조합:
  - `e-1 + e-2` → B의 실질 기여가 실제였음이 드러나 A의 삭제가 고의였음이 강화된다
  - `e-4 + e-6` → 유출이 단순 조회가 아니라 실제 반출이었음이 드러난다
  - `e-5 + e-7` → 둘이 공모 후 서로 먼저 배신했다는 구조가 열린다

### 거짓말 상태 전이 설계
- A / d-1:
  - S0→S1: `direct_question`으로 "단순 실무 누락"까지는 인정
  - S1→S2: `e-1` 제시 시 "기여 기준이 모호했다"는 핑계
  - S2→S3: `e-4` 제시 시 "외부 유출 위험이 있어 뺐다"며 책임 전가
  - S3→S5: `e-7` 제시 시 공 독식과 공모 은폐를 자백
- B / d-2:
  - S0→S1: `empathy_question`으로 "억울해서 외부 자문을 본 적은 있다" 인정
  - S1→S2: `e-5` 제시 시 "내 권리를 지키려던 것"으로 방어
  - S2→S4: `responsibility_question`으로 감정 폭발
  - S4→S5: `e-6`, `e-7` 제시 시 브로커 전달과 수익 합의를 자백

### 핵심 모순 쌍
- A는 "유출 우려 때문에 이름을 뺐다"고 말하지만 `e-7`은 A도 공모 당사자였음을 보여준다
- B는 "권리 구제를 위한 상담"이라 말하지만 `e-6`은 실제 반출 범위를 보여준다
- 둘 다 "상대가 먼저 배신했다"고 하지만 `e-5 + e-7`은 선행 공모 자체를 입증한다

### 판결 딜레마
`상사의 인사 권력 남용`을 더 무겁게 볼지, `연구원의 외부 기밀 유출 공모`를 더 무겁게 볼지 갈린다.  
과정 중시 재판관은 현재 절차 위반과 삭제 조작을 더 크게 볼 수 있고, 해결 중시 재판관은 특허 회수와 조직 피해 복구를 우선할 수 있다.

## partnership-09 — 실적의 얼굴

### 사건 훅
정부 지원금이 확정되자 두 공동대표가 서로 "내가 이 성과의 얼굴"이라고 주장한다.  
하지만 더 깊이 파면, 지원금을 받게 만든 실증 매장 수와 절감률 자체가 둘이 함께 부풀린 허위 실적이었다는 사실이 열린다.

### 양측 프로필
- A: 조인호, 공동대표 겸 제품기술총괄, 숨긴 것=`허위 수치 설계와 사후 편집에 직접 관여한 점`, archetype=`confrontational`
- B: 손다예, 공동대표 겸 사업개발대외협력총괄, 숨긴 것=`공로 독식 홍보와 허위 제출 승인`, archetype=`victim_cosplay`

### 쟁점 구조
- d-1: 언론 홍보와 성과급 책정에서 누가 공로를 독식했는가
- d-2: 내부 평가 피드백 캡처는 편집본인가
- d-3: 실증 매장 수와 절감률은 실제로 얼마였는가
- d-4: 허위 실적 부풀리기를 누가 먼저 제안했는가
- d-5: 사후점검이 오면 누구 책임으로 정리할 것인가

### 증거 설계
- 초기 공개: e-1 기관 보도자료 초안, e-2 인센티브 배분안, e-3 실증 매장 리스트 요약표
- 1차 해금(S1+): e-4 원본 에너지 대시보드 export
  - requires: `e-3`
- 2차 해금(S2+): e-5 지원금 신청 부속표 원본
  - requires: `e-4`
- 3차 해금(S2+): e-6 편집 전 평가 피드백 메일
  - requires: `e-2`
- 최종 해금(S4+): e-7 "이번만 넘기자" 공동 승인 채팅
  - requires: `e-5`, `e-6`
- 증거 조합:
  - `e-2 + e-6` → A의 "캡처는 원본" 주장이 무너진다
  - `e-3 + e-4` → 성과 수치가 실제보다 부풀려졌음이 드러난다
  - `e-5 + e-7` → 단독 범행이 아니라 공동 승인 구조였음이 확정된다

### 거짓말 상태 전이 설계
- A / d-2:
  - S0→S1: `direct_question`으로 "보기 좋게 정리만 했다" 인정
  - S1→S2: `e-6` 제시 시 "핵심 뜻은 같았다"는 핑계
  - S2→S3: `responsibility_question`으로 "B가 외부 발표 때문에 서둘렀다" 책임 전가
  - S3→S5: `e-7` 제시 시 편집과 공동 허위 제출 인정
- B / d-4:
  - S0→S1: `empathy_question`으로 "기관 기대치가 너무 높았다" 일부 인정
  - S1→S2: `e-5` 제시 시 "투자와 직원 급여를 지키려 했다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-7` 제시 시 허위 승인과 공로 독식 계산을 자백

### 핵심 모순 쌍
- B는 "실적은 거의 실제였다"고 하지만 `e-3 + e-4`는 수치 부풀리기를 보여준다
- A는 "홍보만 문제"라 말하지만 `e-5`는 신청 단계부터 허위가 들어갔음을 드러낸다
- 둘 다 상대의 단독 책임을 말하지만 `e-7`이 공동 승인 구조를 입증한다

### 판결 딜레마
`공동 허위 제출` 자체를 가장 무겁게 볼지, 아니면 그 위에서 `누가 더 많이 챙기고 누구를 희생양으로 만들었는가`를 더 무겁게 볼지 갈린다.  
해결 중시 재판관은 사업 존속과 환수 최소화를 볼 수 있고, 과정 중시 재판관은 기관 기망을 이유로 강한 제재를 택할 수 있다.

## tenant-09 — 8천의 계약서

### 사건 훅
세입자는 "계약서에 8천만원이라 써 있는데 왜 6천만원만 돌려주느냐"고 몰아붙인다.  
하지만 송금 흐름을 열어보면, 계약서상 보증금 8천만원 중 실제로 집주인에게 간 돈은 6천만원뿐이고 나머지는 세입자와 중개사가 짜서 만든 허위 정리비 구조로 빠져 있었다.

### 양측 프로필
- A: 김세라, 디저트 카페 운영자, 숨긴 것=`중개사와 보증금을 부풀리고 허위 비용 구조를 만든 점`, archetype=`cold_logic`
- B: 박태윤, 근린상가 건물주, 숨긴 것=`최종 계약서 확인 소홀과 퇴거 후 자의적 반환 지연`, archetype=`victim_cosplay`

### 쟁점 구조
- d-1: 계약서상 8천만원 보증금은 실제로 누구에게 얼마가 갔는가
- d-2: 세라와 중개사는 보증금 부풀리기 구조를 사전에 설계했는가
- d-3: 태윤의 반환 지연과 공제 항목 미통지는 정당했는가
- d-4: 실제 미납 월세와 철거비는 어느 정도였는가
- d-5: 집주인의 최종 계약서 확인 책임은 어디까지인가

### 증거 설계
- 초기 공개: e-1 최종 임대차계약서, e-2 집주인 계좌 입금내역, e-3 퇴거 직후 점포 사진
- 1차 해금(S1+): e-4 중개사 정산표 원본
  - requires: `e-1`, `e-2`
- 2차 해금(S2+): e-5 허위 정리비 세금계산서
  - requires: `e-4`
- 3차 해금(S2+): e-6 미납 월세 및 간판 철거 견적서
  - requires: `e-3`
- 최종 해금(S4+): e-7 세라-중개사 공모 채팅
  - requires: `e-5`, `d-2` S3 이상
- 증거 조합:
  - `e-1 + e-2` → 계약 금액과 실수령액이 갈라진다
  - `e-4 + e-5` → 허위 정리비 구조가 세팅된 것이 드러난다
  - `e-3 + e-6` → 집주인의 자의적 공제만이 아니라 실제 일부 손해도 존재했음이 드러난다

### 거짓말 상태 전이 설계
- A / d-2:
  - S0→S1: `direct_question`으로 "중개사가 알아서 정리했다" 일부 인정
  - S1→S2: `e-4` 제시 시 "관행적인 정산이었다" 핑계
  - S2→S3: `e-5` 제시 시 중개사 책임으로 전가
  - S3→S5: `e-7` 제시 시 공모 사실 인정
- B / d-3:
  - S0→S1: `direct_question`으로 "정리 전이라 늦었다" 인정
  - S1→S2: `e-6` 제시 시 "손해를 확인하려면 어쩔 수 없었다" 핑계
  - S2→S4: `responsibility_question`으로 "사기당한 건 나도 마찬가지" 감정 폭발
  - S4→S5: `e-1 + e-2` 재제시 시 확인 소홀과 무항목 지연을 자백

### 핵심 모순 쌍
- A는 "계약서대로 냈다"고 하지만 `e-2`는 실제 집주인 수령액이 다름을 보여준다
- B는 "전액 못 준 건 전부 사기 때문"이라 말하지만 `e-6`은 실제 손해와 자의적 지연이 섞여 있음을 보여준다
- A는 중개사 단독을 주장하지만 `e-7`이 사전 설계를 드러낸다

### 판결 딜레마
`계약서 신뢰`를 더 중시할지, `실제 돈의 흐름과 사기 구조`를 더 중시할지 갈린다.  
논리형 재판관은 실수령 증거를 중시할 수 있고, 과정형 재판관은 집주인의 확인 소홀과 반환 절차 위반도 크게 볼 수 있다.

## spouse-12 — 2006년의 캡처

### 사건 훅
20주년 동창회 다음 날 새벽, 익명 계정이 2006년 장학금 소문을 다시 꺼내며 오래된 방명록 캡처와 문자 이미지를 올린다.  
배우자는 확인도 끝나기 전에 세 채널로 그 이미지를 퍼뜨리고, 당사자는 불리한 옛 기록을 백업에서 빼며 억울함만 강조한다. 포렌식 끝에 드러나는 건, 결정적 증거가 2026년에 합성된 위조본이라는 사실이다.

### 양측 프로필
- A: 문세아, 수제디저트 가게 대표, 숨긴 것=`2006년 실제로 선을 넘은 행위 일부와 현재 백업 선별 은닉`, archetype=`premature_summary`
- B: 최재우, 구청 홍보팀 주무관, 숨긴 것=`확인 없는 선제 전파와 도덕적 우위 선점 욕구`, archetype=`confrontational`

### 쟁점 구조
- d-1: 재우의 선제 전파는 어디까지 고의적이었는가
- d-2: 세아는 왜 옛 백업에서 일부 기록을 빼고 제출했는가
- d-3: 익명 계정 운영과 위조본 제작에는 누가 관여했는가
- d-4: 결정적 증거는 정말 2006년 원본인가
- d-5: 세아의 2006년 실제 행위 범위는 어디까지인가

### 증거 설계
- 초기 공개: e-1 익명 계정 업로드 캡처, e-2 동창회 참석자 명단, e-3 재우의 전달 로그 3종
- 1차 해금(S1+): e-4 세아 백업 폴더의 누락 흔적
  - requires: `e-3`
- 2차 해금(S2+): e-5 이미지 포렌식 보고서
  - requires: `e-1`
- 3차 해금(S2+): e-6 2006년 실제 장학금 회의 메모 + 동창 진술
  - requires: `e-4`
- 최종 해금(S4+): e-7 위조 의뢰 DM
  - requires: `e-5`, `d-3` S3 이상
- 증거 조합:
  - `e-1 + e-5` → 현재 돌고 있는 결정적 캡처가 위조본임이 확정된다
  - `e-4 + e-6` → A도 과거의 불리한 기록을 선별 은닉했음이 드러난다
  - `e-3 + e-7` → B의 선제 전파가 단순 실수가 아니라 도덕적 우위 선점과 결합됐음이 강화된다

### 거짓말 상태 전이 설계
- A / d-2:
  - S0→S1: `direct_question`으로 "너무 오래된 일이라 일부만 찾았다" 인정
  - S1→S2: `e-4` 제시 시 "쓸모없는 잡파일인 줄 알았다" 핑계
  - S2→S3: `responsibility_question`으로 "상대가 위조를 퍼뜨렸으니 나도 방어적이었다" 책임 전가
  - S3→S5: `e-6` 제시 시 과거 실제 행위와 현재 은닉을 자백
- B / d-1:
  - S0→S1: `direct_question`으로 "사실 확인 차원이었다" 일부 인정
  - S1→S2: `e-3` 제시 시 "배우자로서 알아야 할 사람들에게만 보냈다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-7` 제시 시 위조 의심을 알면서도 퍼뜨린 정황을 인정

### 핵심 모순 쌍
- A는 "전부 위조라 억울하다"고 하지만 `e-6`은 과거 실제 잘못 일부를 보여준다
- B는 "가족 보호 차원의 확인"이라고 하지만 `e-3`은 다중 채널 확산을 보여준다

### 판결 딜레마
`현재 위조 증거와 무검증 확산`을 더 무겁게 볼지, `과거 실제 상처와 현재 선별 은닉`을 더 무겁게 볼지 갈린다.  
직관형 재판관은 오래된 상처의 무게를 더 느낄 수 있고, 논리형 재판관은 현재 위조본의 파괴력을 더 엄격하게 볼 수 있다.

## family-09 — 계약 밖의 6,600

### 사건 훅
부모 재활비와 노후자금을 마련하려고 상가주택을 팔던 과정에서, 누나는 "동생이 부모 집값 차액 6,600만원을 혼자 숨겼다"고 폭발한다.  
하지만 매매 구조를 파고들면, 남매와 동네 중개사가 공식 계약 밖 보전금을 따로 빼돌려 나눠 갖기로 먼저 합의했고, 둘 다 부모에게는 세금과 명도비 정도로만 축소 설명했다는 사실이 드러난다.

### 양측 프로필
- A: 최민아, 피부관리실 실장, 숨긴 것=`부모 위임장 선서명 유도와 오프북 차액 합의 참여`, archetype=`victim_cosplay`
- B: 최도윤, 인테리어 현장소장, 숨긴 것=`차액 선수령 후 개인채무 상환`, archetype=`avoidant`

### 쟁점 구조
- d-1: 계약 외 6,600만원은 누가 먼저 숨기자고 했는가
- d-2: 민아의 부모 위임장 선서명 유도는 어디까지 계획적이었는가
- d-3: 도윤은 차액을 먼저 받아 개인채무를 갚았는가
- d-4: 부모는 계약 외 돈 구조를 실제로 알고 동의했는가
- d-5: 세금·명도비 핑계는 누가 얼마나 축소 설명했는가

### 증거 설계
- 초기 공개: e-1 공식 매매계약서, e-2 부모 위임장 사본, e-3 중개사 명도비 정산표
- 1차 해금(S1+): e-4 계약 외 보전금 송금 캡처
  - requires: `e-1`
- 2차 해금(S2+): e-5 도윤 개인채무 상환 내역
  - requires: `e-4`
- 3차 해금(S2+): e-6 부모 재활비 메모와 상담기록
  - requires: `d-4` S2 이상
- 최종 해금(S4+): e-7 남매-중개사 3자 채팅
  - requires: `e-2`, `e-5`
- 증거 조합:
  - `e-1 + e-4` → 공식 가격 밖 숨겨진 차액이 있었다는 사실이 열린다
  - `e-2 + e-7` → A도 구조 설계 초기부터 개입했음이 드러난다
  - `e-5 + e-6` → B의 개인 실리와 부모 명분이 동시에 엮여 있음을 보여준다

### 거짓말 상태 전이 설계
- A / d-2:
  - S0→S1: `empathy_question`으로 "부모 상태가 급해 서명만 먼저 받았다" 인정
  - S1→S2: `e-2` 제시 시 "어차피 같은 방향이었다" 핑계
  - S2→S3: `e-7` 제시 시 중개사와 동생에게 책임 전가
  - S3→S5: `responsibility_question`으로 선서명 유도와 차액 합의를 자백
- B / d-3:
  - S0→S1: `direct_question`으로 "돈을 잠깐 맡은 적은 있다" 인정
  - S1→S2: `e-5` 제시 시 "급한 채무를 막아야 거래가 안 깨졌다" 핑계
  - S2→S4: `emotion_threshold`로 감정 폭발
  - S4→S5: `e-7` 제시 시 선수령과 개인상환을 자백

### 핵심 모순 쌍
- A는 "동생 단독 은닉"이라 말하지만 `e-7`은 A도 초반 설계에 참여했음을 보여준다
- B는 "부모를 위해 잠깐 돌린 돈"이라 말하지만 `e-5`는 개인채무 상환으로 바로 연결된다
- 둘 다 "부모도 대충 아셨다"고 하지만 `e-6`은 구체 구조 인지는 없었음을 시사한다

### 판결 딜레마
`부모를 위한 현실적 돈 만들기`라는 명분을 참작할지, `가족 자산을 오프북으로 빼돌린 공모`를 더 엄격히 볼지 갈린다.  
해결 중시 재판관은 부모 회복과 자산 환원을 우선할 수 있고, 과정 중시 재판관은 위임과 설명 절차를 어긴 책임을 크게 볼 수 있다.

## neighbor-08 — 27초의 복도

### 사건 훅
복도 전단지 분실이 이어지자 민석은 도어벨카메라를 달고, 맞은편 하린은 자기 현관까지 찍히는 것 같아 불안을 느낀다.  
며칠 뒤 주민방에 "카메라가 집 안까지 따라왔다"는 27초 클립이 퍼지며 민석은 순식간에 스토커 취급을 받지만, 원본을 열어보면 그 클립은 프리뷰와 다른 시점 영상을 이어붙인 위조본이다. 다만 설치 초반 이틀간 촬영각이 도어매트 쪽으로 과하게 넓었던 것도 사실이고, 민석도 반격 과정에서 하린의 귀가 시간표를 올리며 선을 넘는다.

### 양측 프로필
- A: 황민석, 홈네트워크 보안업체 원격지원 엔지니어, 숨긴 것=`초기 촬영각 초과와 반격성 귀가 시간표 공개`, archetype=`cold_logic`
- B: 주하린, 필라테스 강사, 숨긴 것=`위조 클립 편집과 공개 낙인`, archetype=`victim_cosplay`

### 쟁점 구조
- d-1: 민석은 왜 설치 사실과 초기 촬영각 문제를 즉시 고지하지 않았는가
- d-2: 하린은 왜 원본 확인 전에 공개 낙인과 위조 클립 유포를 했는가
- d-3: 27초 클립은 원본인가 편집본인가
- d-4: 실제 카메라가 하린 집 내부를 추적했는가
- d-5: 둘은 왜 관리실 확인 절차를 건너뛰었는가

### 증거 설계
- 초기 공개: e-1 설치 프리뷰 캡처, e-2 주민방 27초 클립, e-3 관리실 카메라 설치 안내문
- 1차 해금(S1+): e-4 SD카드 원본 영상
  - requires: `e-2`
- 2차 해금(S2+): e-5 초기 이틀간 각도 조정 로그
  - requires: `e-1`
- 3차 해금(S2+): e-6 귀가 시간표 게시글 캡처
  - requires: `e-4`
- 최종 해금(S4+): e-7 하린의 편집 작업 기록
  - requires: `e-4`, `d-2` S3 이상
- 증거 조합:
  - `e-2 + e-4` → 27초 클립이 위조본임이 확정된다
  - `e-1 + e-5` → A의 초기 촬영각 초과가 실제였음이 드러난다
  - `e-4 + e-6` → A도 반격 과정에서 사생활 정보를 공개했음이 열린다

### 거짓말 상태 전이 설계
- A / d-1:
  - S0→S1: `direct_question`으로 "세부 각도는 바로 손봤다" 인정
  - S1→S2: `e-5` 제시 시 "택배 도난이 반복돼 급했다" 핑계
  - S2→S3: `e-6` 제시 시 "낙인을 막으려다 그랬다" 책임 전가
  - S3→S5: `responsibility_question`으로 귀가 시간표 공개를 자백

  - S1→S2: `e-4` 제시 시 "핵심은 같았다" 핑계
  - S2→S4: `emotion_threshold`로 불안과 분노 폭발
  - S4→S5: `e-7` 제시 시 직접 편집과 낙인 유포를 자백

### 판결 딜레마
`실제 사생활 불안`을 얼마나 참작할지와 `허위 낙인·편집 유포`를 얼마나 강하게 제재할지가 갈린다.  
직관형 재판관은 초기 불안을 참작할 수 있지만, 논리형 재판관은 원본 조작과 귀가 정보 공개를 각각 별개의 침해로 볼 가능성이 크다.
