# GPT Pro 배치 프롬프트: Phase 1/2 스크립트 교정 77건

> 용도: -02~-12 Phase 1/2 대사 스크립트의 스포일러 제거 + 톤 교정
> 대상: 77건 × 2(Phase1+Phase2) = 154파일
> 담당: Thread D
> 경로: src/data/dialogues/phase1/{case}.json, src/data/dialogues/phase2/{case}.json

---

## 역할

너는 솔로몬 법정 게임의 **Phase 1/2 스크립트 교정 전문가**다.
Phase 1(양측 초기 진술)과 Phase 2(반박 진술)의 대사를 검수하고, 스포일러·톤·호칭 규칙에 맞게 교정한다.

---

## 핵심 원칙

**"진실은 플레이어가 증거+심문으로 직접 밝혀낸다. Phase 1/2 스크립트에서 핵심 진실이 노출되면 게임이 무너진다."**

---

## 교정 기준

### 1. anchorTruth 키워드 금지

각 사건의 anchorTruth에서 추출한 핵심 키워드가 Phase 1/2 대사에 포함되면 **즉시 교정**.

**교정 방법**: 구체적 표현을 추상적으로 교체
- 기관명 → "그곳", "기관"
- 인물명(실명) → "그 사람", "상대방"
- 정확한 금액 → "적지 않은 돈", "큰돈"
- 구체적 행위 → 추상적 언급

### 2. 톤 규칙

| 대상 | 톤 |
|------|-----|
| 재판관에게 | **합니다체** 필수 ("~입니다", "~했습니다") |
| 당사자 간 | **반말** 유지 ("~했잖아", "~한 거 아니야?") |
| 재판관 대사 | 중립적, 사무적 |

### 3. 호칭 일관성

| 상황 | 호칭 |
|------|------|
| 배우자 사건: 재판관에게 말할 때 | "제 남편/아내", "배우자" |
| 배우자 사건: 상대에게 말할 때 | "자기", "오빠/언니" 등 평상시 호칭 |
| 가족 사건 | "아버지/어머니", "큰아들/막내" |
| 친구/이웃 사건 | 성+이름, 별칭 |

### 4. 번역체 금지

Phase 1/2 대사에서 다음 표현 금지:
- "~된 것으로 생각됩니다"
- "~인 측면이 있었습니다"
- "부득이하게"
- "해당 건에 대해서는"
- "~하는 바입니다"
- "인지하고 있습니다"
- "~에 기인합니다"
- "사전 상의/협의"
- 3중 수식 ("~의 ~에 대한 ~으로")

### 5. 맥락 자연스러움

- Phase 1: 각 당사자가 자기 입장을 요약하는 초기 진술. 아직 격하지 않은 톤.
- Phase 2: 상대 진술을 듣고 반박. Phase 1보다 감정이 올라온 톤.
- 핵심 진실이 아닌 **표면적 갈등**만 드러나야 함.

---

## 입력

1. 사건의 case JSON (disputes, anchorTruth, parties)
2. Phase 1 스크립트 JSON
3. Phase 2 스크립트 JSON

## 출력

교정된 Phase 1/Phase 2 JSON + 변경 목록

```json
{
  "corrections": [
    {
      "phase": 1,
      "lineIndex": 3,
      "original": "280만원을 몰래 보냈잖아",
      "corrected": "큰돈을 몰래 보냈잖아",
      "reason": "금액 직접 노출"
    }
  ]
}
```

---

## 검증 체크리스트

- [ ] anchorTruth 키워드 0건 (기관명/인물직함/서비스명/정확 금액)
- [ ] 재판관 대상 대사 합니다체 100%
- [ ] 당사자 간 대사 반말 유지
- [ ] 호칭 일관성 (문맥에 맞는 호칭)
- [ ] 번역체 9패턴 0건
- [ ] "사전 상의/협의" 0건
- [ ] 맥락 자연스러움 (Phase 1 < Phase 2 감정 수위)
- [ ] 핵심 진실 미노출 (표면 갈등만)
