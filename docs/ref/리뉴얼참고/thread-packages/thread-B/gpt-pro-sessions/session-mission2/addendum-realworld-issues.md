# [추가자료] 7건 실전 테스트에서 발견된 모순 추궁 관련 문제

---

## 1. 현재 모순 추궁 질문의 실제 출력

실전 테스트에서 모순이 감지되었을 때 LLM이 생성한 추궁 질문 예시:

```
실제 출력: "한지석 씨, 이전에는 '그 금액에 대해서는 아는 바가 없습니다'라고 하셨는데,
          방금은 '가족 쪽에 급한 일이 생겨서 어쩔 수 없었습니다'라고 하셨습니다.
          진술이 달라진 이유를 설명해 주십시오."
```

**문제점:**
- LLM 자유 생성이라 **톤이 불안정**: 때로 해요체가 섞임
- 매번 같은 구조: `"이전에는 'X'라고 하셨는데, 방금은 'Y'라고 하셨습니다"` → 변형 없음
- API 호출 지연 0.5~2초 (게임 흐름 끊김)
- 실패 시 기계적 폴백이 더 나쁨

## 2. 모순 감지 시 전달되는 실제 데이터

### previousClaim 예시 (실제 게임에서 기록된 진술)
```
"해당 금액에 대해서는 전혀 관여한 바 없습니다"
"특별한 이유는 없었습니다"
"그런 일이 있었는지 잘 모르겠습니다"
"업무적으로 확인한 것뿐입니다"
"그 사진들은 맞습니다. 하지만 전체적인 손상 범위를 단정짓기는 어렵습니다"
```

### currentClaim 예시 (입장 변화 설명 또는 실제 진술)
**입장 변화 설명 (transitionDesc):**
```
"이전 답변과 달리, 태도에 변화가 감지됩니다"
"이전 답변과 달리, 감정적으로 동요하고 있습니다"
"이전 답변과 크게 달라진 태도를 보이고 있습니다"
```

**실제 진술 요약:**
```
"가족 쪽에 급한 일이 생겨서 어쩔 수 없었습니다"
"상대방이 먼저 그렇게 진행한 겁니다"
"사정이 있었습니다"
```

→ 템플릿은 **두 경우 모두** 자연스러워야 함.

## 3. 품질 기준 — 금지 표현

| 금지 | 이유 |
|------|------|
| "거짓말을 하셨습니다" | 유죄 전제 |
| "왜 숨기셨습니까" | 은폐를 전제 |
| "해당 건에 대해서는" | 딱딱한 번역체 |
| "진실을 말씀해 주십시오" | 직접적 유죄 전제 |
| "방금 하신 말씀은" + "거짓입니다" | 과도한 단정 |

## 4. "맞습니다. 다만..." 패턴 주의

NPC가 모순 추궁에 대해 "맞습니다. 다만..." 으로 응답하는 패턴이 15회 이상 반복됨.
이것은 NPC 대사 문제이지만, **재판관 질문이 항상 같은 패턴이라서** NPC도 같은 구조로 방어하는 것.
→ 추궁 질문에 변형이 생기면 NPC 응답 패턴도 자연스럽게 다양해짐.
