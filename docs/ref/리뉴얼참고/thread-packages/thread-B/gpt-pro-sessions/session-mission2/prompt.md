## 1. 프로젝트 배경

### 솔로몬 게임이란
한국 민사 분쟁 재판 시뮬레이션 게임. 플레이어가 **재판관**이 되어 양측 NPC를 심문한다.
NPC는 처음에 거짓말(S0)을 하다가, 심문이 진행되면서 점점 진실에 가까워진다(S5).

### 모순이란
NPC의 lieState가 **부정(S0-S1)에서 인정(S2+)으로** 넘어갈 때, 이전 진술과 현재 진술이 **실질적으로 모순**된다.

예시:
```
3턴 전 (S0): "그 금액에 대해서는 아는 바가 없습니다"
현재 (S2):   "그 건은 가족 사정 때문이었습니다"
→ "모른다"에서 "가족 사정 때문"으로 입장이 바뀜 = 모순
```

### 재판관의 모순 추궁
모순이 감지되면 재판관이 **이전 진술과 현재 진술을 구체적으로 인용하여** 추궁한다.
이것이 게임에서 가장 극적인 순간 중 하나다.

## 2. 현재 문제

### 현재 코드
모순 추궁 질문을 **LLM에게 자유 생성**시킨다:
```
"이전 진술: '${previousClaim}'
 현재 진술: '${currentClaim}'
 → 두 진술의 차이점을 짚어 추궁하는 질문을 작성하세요"
```

**문제점:**
1. LLM 자유 생성이라 **톤/형식이 불안정** (때로 반말, 때로 과도한 경어)
2. **추가 API 호출** → 응답 지연 0.5~2초
3. 실패 시 **기계적 폴백**: `"이전에는 '${prev}…'라고 하셨는데, 방금은 '${curr}…'라고 하셨습니다."`

### 목표
**템플릿 기반 즉시 생성** — LLM 없이, previousClaim + currentClaim을 변수로 넣어 바로 생성.
- 톤이 안정적 (재판관 위엄 보장)
- 지연 없음 (즉시 표시)
- lieState별 강도 조절

## 3. 설계 요청

### 3-1. 템플릿 구조

```
"${name} 씨, [도입부] '${previousClaim}'[연결부] '${currentClaim}'[마무리]"
```

**도입부**: 모순을 지적하는 방식
**연결부**: 두 진술을 잇는 방식
**마무리**: 추궁 강도에 따른 질문/요구

### 3-2. 강도 3단계 × 5종 = 15종

| 강도 | 적용 상황 | 재판관 태도 |
|------|----------|----------|
| **soft** (S0-S1) | NPC가 아직 부정/동요 중 | 탐색적, "혹시 기억이 다른 것인지" |
| **mid** (S2) | NPC가 일부 인정한 직후 | 지적, "입장이 달라진 것 같은데" |
| **hard** (S3-S4) | NPC가 책임 전가/감정 호소 중 | 압박, "진술이 모순됩니다" |

각 강도별 **5종 변형** 필요. (같은 강도에서도 3번 반복하면 다른 질문이 나와야 함)

### 3-3. 변형 차별화 전략

**도입부 변형:**
- 시간 기반: "방금", "앞서", "잠시 전에는"
- 인용 기반: "분명히 ~라고 하셨습니다", "제가 기록한 바로는"
- 관찰 기반: "진술에 달라진 부분이 있습니다", "입장이 바뀐 것 같습니다"

**연결부 변형:**
- 대조형: "~라고 하셨는데, 지금은 ~라고 하십니다"
- 질문형: "~라고 하셨습니다. 그런데 왜 지금은 ~라고 하십니까?"
- 확인형: "~라고 하신 것과 ~라고 하신 것"

**마무리 변형:**
- soft: "어느 쪽이 맞는 것인지 확인하겠습니다", "설명해 주시겠습니까?"
- mid: "왜 달라진 것입니까?", "어느 쪽이 사실입니까?"
- hard: "분명히 답하십시오", "더 이상 모호하게 넘길 수 없습니다"

## 4. 품질 기준

**1. 재판관 위엄**: 격식체, ~십시오/~입니까 종결. 반말/해요체 금지.

**2. 날카로움**: 모순을 정확히 짚되, **유죄를 전제하지 않음**.
```
좋은 예: "진술이 달라졌습니다. 어느 쪽이 사실입니까?"
나쁜 예: "거짓말이 들통났습니다. 인정하십시오." (유죄 전제)
```

**3. 변수 호환성**: `${previousClaim}`과 `${currentClaim}`이 **20자~60자** 범위일 때 자연스러워야 함.
```
짧은 경우: "모른다" + "가족 사정이었다"
긴 경우: "해당 금액에 대해서는 전혀 관여한 바 없습니다" + "가족 쪽에 급한 일이 생겨서 어쩔 수 없었습니다"
```

**4. 15종 모두 구별 가능**: 비슷한 구조끼리 묶이면 FAIL.

## 6. 참고: 실제 모순 발생 예시

### 예시 1 (spouse-01: 비밀 송금)
```
이전(S0): "해당 금액에 대해서는 전혀 관여한 바 없습니다"
현재(S2): "가족 쪽에 급한 일이 생겨서 어쩔 수 없었습니다"

soft 예상: "${name} 씨, 앞서 '관여한 바 없다'고 하셨는데, 지금은 '어쩔 수 없었다'고 하십니다. 어느 쪽이 맞는 것인지 말씀해 주시겠습니까?"
```

### 예시 2 (workplace-01: 보고서 저작)
```
이전(S1): "업무적으로 확인한 것뿐입니다"
현재(S3): "상대방이 먼저 그렇게 진행한 겁니다"

hard 예상: "${name} 씨, 분명히 '확인한 것뿐'이라고 하셨습니다. 그런데 지금은 상대 탓을 하고 계십니다. 진술이 왜 이렇게 달라진 것입니까?"
```

### 예시 3 (neighbor-01: 소음)
```
이전(S0): "그런 일이 있었는지 잘 모르겠습니다"
현재(S2): "그때 사정이 있었습니다"

mid 예상: "${name} 씨, '모르겠다'고 하셨다가 지금은 '사정이 있었다'고 하십니다. 입장이 달라진 이유를 설명해 주십시오."
```
