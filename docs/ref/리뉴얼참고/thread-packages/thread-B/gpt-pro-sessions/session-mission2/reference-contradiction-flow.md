# 참고자료: 게임 내 모순 추궁 흐름

## 모순 감지 → 추궁 질문 → NPC 응답 흐름

```
1. NPC가 심문에 답변한다 (LLM 생성)
2. lieState가 S0/S1 → S2+ 로 전이된다
3. 시스템이 이전 진술과 현재 진술의 모순을 감지한다
4. 시스템 메시지: "⚡ {name}의 진술에서 이전과 다른 점이 발견되었다 — 탭하여 추궁"
5. 플레이어가 탭하면:
   a. 재판관이 모순 추궁 질문을 던진다 ← ★ 이것을 설계하는 것이 이번 작업
   b. NPC가 해명 답변을 한다 (LLM 생성)
   c. 추가 거짓말 전이 시도 (성공하면 lieState가 더 올라감)
   d. 감정 +12 (압박 효과)
```

## 변수로 전달되는 데이터

```typescript
previousClaim: string  // 이전 진술 요약 (20~60자)
currentClaim: string   // 현재 진술 요약 or 입장 변화 설명
name: string           // NPC 이름
lieState: LieState     // 현재 NPC 상태 (S0~S5)
```

## 현재 transitionDesc (입장 변화 설명)
```
S0→S2: "이전 답변과 달리, 태도에 변화가 감지됩니다"
S0→S3: "이전 답변과 달리, 감정적으로 동요하고 있습니다"
S0→S4: "이전 답변과 달리, 심리적 압박을 받고 있는 듯합니다"
S0→S5: "이전 답변과 크게 달라진 태도를 보이고 있습니다"
S1→S2: "이전 답변과 달리, 태도에 변화가 감지됩니다"
...
```

이 `currentClaim`은 실제 진술 요약이 올 수도 있고, 위의 태도 변화 설명이 올 수도 있다.
따라서 템플릿은 **두 가지 경우 모두** 자연스러워야 한다.
