# [추가자료] 7건 실전 테스트에서 발견된 증인 품질 문제

---

## 1. 증인 증언의 보고서 톤 문제

실전 테스트에서 LLM이 생성한 증인 증언 중 보고서 톤이 발견된 사례:

```
나쁜 예: "해당 건에 대해서는, 제가 관찰한 바에 의하면,
         해당 당사자가 해당 행위를 한 것으로 판단됩니다."

좋은 예: "그날... 저도 옆에 있었는데요. 갑자기 큰 소리가 나서
         나가봤더니 두 사람이 다투고 있었습니다."
```

### 원인
- few-shot 예시가 없어서 LLM이 기본 보고서 톤으로 회귀
- 증인 유형(가족/동료/기관/친구)별 톤 가이드가 약함
- "번역체 금지" 규칙은 있지만 **좋은 예시가 없으면 무력**

## 2. 금지 표현 목록 — few-shot에 "나쁜 예"로 포함

| 금지 표현 | 이유 | 대체 |
|----------|------|------|
| "해당 건에 대해서는" | 보고서 톤 | "그 일에 대해서는" |
| "~된 것으로 생각됩니다" | 관료적 | "~인 것 같습니다" |
| "여러 가지 상황이 얽혀" | 추상적 회피 | 구체적 상황 1가지 |
| "제가 관찰한 바에 의하면" | 관찰 보고서 | "제가 봤을 때는" |
| "해당 당사자" | 법률 문서 | "그 사람" / 이름 |
| "적지 않은" | 이미 클리셰 | "큰", "꽤 많은" |
| "그 무렵" | 4회 반복 클리셰 | "그때", "그날", "그 시점에" |

## 3. 증인 유형별 실전 품질

### family — 감정 표현이 부족
```
기대: "우리 아이가 그런 적이 있었나요? 글쎄요... (눈시울 붉어짐)"
실제: "네, 그런 일이 있었던 것 같습니다. 다만 자세한 건..."
→ 가족 증인인데 감정이 전혀 안 드러남
```

### institutional — 양호하나 단조로움
```
기대: "저희 기관 기록에 따르면 ~ 했습니다."
실제: 대체로 이 패턴으로 나오지만 변형이 없음
```

### 증인 품질 우선순위
1. family가 가장 부족 (감정 표현 필수)
2. friend는 편안한 톤이 안 나옴 (법정 긴장감이 과함)
3. colleague/institutional은 상대적으로 양호

## 4. hiddenAgenda 반영 현황

실전에서 hiddenAgenda가 증언에 반영된 비율: **매우 낮음**
- 프롬프트에 `"숨기고 싶은 것: ${hiddenAgenda}"`로 전달하지만
- LLM이 이를 "언급하지 않는 것"으로만 처리 (소극적)
- 기대: "말을 고르다 멈추는", "화제를 돌리는" 적극적 회피 행동

→ few-shot에 hiddenAgenda가 있을 때의 **적극적 회피 패턴**을 포함해야 함.
