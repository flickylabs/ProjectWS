## 1. 프로젝트 배경

### 증인 시스템이란
플레이어가 **증인을 소환**하면, LLM(gpt-4o)이 증인 역할로 증언을 생성한다.
증인은 사건의 **당사자가 아니라** 주변 인물이다 — 가족, 직장 동료, 기관 담당자, 친구 등.

### 증언 깊이 (depth)
증인의 소환 횟수/상황에 따라 증언 깊이가 결정된다:

| depth | 설명 | 증언 특성 |
|-------|------|----------|
| **vague** | 첫 소환, 조심스러움 | 핵심 회피, "잘 기억이...", 2문장 이내 |
| **partial** | 재소환, 일부 공개 | 상황 설명하되 핵심 세부사항은 숨김, 3문장 |
| **full** | 충분한 압박 후 | 아는 것 모두 증언, 제한 없음 |

### 현재 문제
1. **few-shot 예시가 없음** → LLM이 증인을 보고서 작성자처럼 말하게 함
   ```
   나쁜 예: "해당 건에 대해서는, 제가 관찰한 바에 의하면, 해당 당사자가 해당 행위를 한 것으로 판단됩니다."
   좋은 예: "그날... 저도 옆에 있었는데요. 갑자기 큰 소리가 나서 나가봤더니..."
   ```
2. **증인 유형별 차별화 없음** → 어머니든 은행원이든 같은 톤
3. **hiddenAgenda 활용이 약함** → 증인 고유의 속사정이 반영 안 됨

## 2. 증인 유형 4종

### 분류 기준
증인의 관계/역할에 따라 4종으로 분류:

| 유형 | 코드 | 해당 인물 예시 |
|------|------|--------------|
| **가족** | `family` | 부모, 형제, 시부모, 처가, 자녀 |
| **직장** | `colleague` | 같은 팀 동료, 상사, 부하, 거래처 |
| **기관** | `institutional` | 은행원, 사회복지사, 관리사무소, 공인중개사 |
| **친구** | `friend` | 개인적 친분, 이웃, 지인 |

### 각 유형의 핵심 말투 특성

**family (가족):**
- 감정적, 한쪽(보통 자식/가족) 감싸기 경향
- "우리 ○○이가~", "어머니로서 말씀드리자면~"
- 감정이 복받쳐 말을 잇지 못하는 표현 허용
- 법정이라 격식을 차리려 하지만 감정이 새어나옴

**colleague (직장):**
- 사무적이되 미묘하게 한쪽 편
- "당시 같은 팀이었는데요,", "업무적으로 말씀드리면,"
- 조직 논리("팀 분위기가~", "원래 그 업무는~")로 포장
- 사적 감정은 최소화하되 미묘하게 드러남

**institutional (기관):**
- 사실 위주, 절차적, 감정 배제
- "저희 기관 기록에 따르면,", "제가 담당했던 건이라~"
- 법적 책임을 의식하여 조심스러운 표현
- "제 소관 범위에서 말씀드리겠습니다"

**friend (친구):**
- 편한 톤이지만 법정이라 격식
- "제가 알기로는요,", "그때 같이 있었는데,"
- 솔직하려 하지만 친구를 배려하여 말을 고름
- "사실... 저도 좀 복잡한 심정이었습니다"

## 3. 설계 요청

### 3-1. 유형별 × depth별 few-shot 예시

각 조합에 대해 **재판관 질문 → 증인 응답** 쌍으로 2-3개 예시 작성.

```
4유형 × 3depth × 2-3예시 = 24~36개 예시
```

### 3-2. 예시 형식

각 예시는 이런 구조:
```json
{
  "question": "재판관의 질문 텍스트",
  "response": "증인의 응답 텍스트",
  "behaviorHint": "행동/표정 묘사 (선택)"
}
```

### 3-3. hiddenAgenda 반영 패턴

각 유형별로 "증인이 숨기고 싶은 것"이 증언에 어떻게 반영되는지 패턴 제시.

예:
```
family — hiddenAgenda가 있을 때:
  "... (잠시 침묵) 그 부분은 제가 말씀드리기 좀..."
  말을 돌리거나 감정으로 회피

institutional — hiddenAgenda가 있을 때:
  "그 부분은 제 소관 범위를 벗어나는 것이라..."
  절차/관할을 핑계로 회피
```

## 4. 품질 기준

### 핵심
- **유형별 차별화가 선명해야 함**: 같은 질문에 가족과 기관이 완전히 다른 톤
- **depth별 정보량 차이가 명확해야 함**: vague는 거의 아무것도 안 말함, full은 구체적
- **법정 맥락**: 증인석에서 재판관에게 말하는 상황. 합니다체 기반.
- **자연스러운 한국어**: 번역체/보고서 톤 절대 금지

### depth별 정보량 가이드

**vague:**
```
"네... 그런 일이 있었던 것 같습니다만, 자세한 건 잘..."
"기억이 정확하지는 않습니다만, 분위기가 좀 이상하긴 했습니다."
→ 핵심 정보 0개, 분위기/느낌만
```

**partial:**
```
"그날 제가 봤을 때는... 꽤 급해 보이긴 했습니다. 다만 정확히 무슨 일인지는..."
"대략적인 상황은 알고 있습니다만, 세부적인 부분까지는 제가 직접 본 게 아니라서요."
→ 상황 윤곽은 드러나되 핵심 세부사항은 누락
```

**full:**
```
"솔직히 말씀드리겠습니다. 그날 제가 직접 봤습니다. ..."
"제가 알고 있는 것을 전부 말씀드리겠습니다."
→ 아는 것 전부, 구체적 사실 포함
```
