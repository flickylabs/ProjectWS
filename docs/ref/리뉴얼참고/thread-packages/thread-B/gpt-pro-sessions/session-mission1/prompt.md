## 1. 프로젝트 배경

### 솔로몬 게임이란
한국 민사 분쟁 재판 시뮬레이션 게임. 플레이어가 **재판관**이 되어 양측 NPC(당사자 A, B)를 심문하고, 증거를 수집하여 진실을 밝힌다.
- 한국어 전용 게임
- NPC 대사는 LLM(gpt-4o)이 실시간 생성
- **재판관 질문**은 엔진이 템플릿에서 선택하여 생성 (LLM 불필요)

### 재판관의 역할
재판관은 **4가지 질문 유형**으로 NPC를 추궁한다:

| 유형 | 코드명 | 목적 | 예시 |
|------|--------|------|------|
| 사실추궁 | `fact_pursuit` | 사실 여부 확인, 구체적 행위 추궁 | "~에 대해 사실 여부를 확인하겠습니다" |
| 동기탐색 | `motive_search` | 왜 그랬는지, 어떤 판단이었는지 | "왜 그런 선택을 하셨습니까?" |
| 공감접근 | `empathy_approach` | 심정, 감정, 어려움 탐색 | "당시 어떤 심정이셨습니까?" |
| 증거제시 | `evidence_present` | 증거를 보여주며 입장 요구 | "이 증거에 대해 입장을 밝히십시오" |

### NPC의 lieState (거짓말 상태)
NPC는 쟁점(dispute)별로 `lieState`를 가진다. 심문이 진행될수록 올라간다:

| 상태 | NPC 태도 | 재판관이 느끼는 것 |
|------|---------|-----------------|
| **S0** | 완강한 부정 "그런 적 없습니다" | 방어적, 뭔가 숨기는 느낌 |
| **S1** | 동요 "글쎄요, 정확히는..." | 흔들리기 시작, 빈틈이 보임 |
| **S2** | 일부 인정 "그건 맞지만..." | 부분적으로 인정했으나 핵심은 아직 |
| **S3** | 책임 전가 "내 잘못이 아니라..." | 인정하면서도 남 탓 |
| **S4** | 감정 호소 "사정이 있었습니다..." | 감정에 호소, 동정 유발 |
| **S5** | 전면 인정 | 진실 고백 |

### interrogationDepth (심문 깊이)
같은 쟁점에 같은 질문 유형을 반복하면 depth가 올라간다:
- depth 0: 첫 질문 → 탐색적, 부드러운 톤
- depth 1: 재질문 → 조금 더 구체적 요구
- depth 2: 압박 → 날카로운 지적
- depth 3+: 최종 압박 → "여러 차례 확인했습니다. 사실대로 말하십시오"

## 2. 현재 문제

### 현재 코드 (judgeQuestionEngine.ts 참조)
- fact_pursuit: depth별 **각 1종** = 4종
- motive_search: depth별 **각 1종** = 4종
- empathy_approach: depth별 **각 1종** = 4종
- evidence_present: **1종 고정** ("입장을 밝혀 주십시오")
- **lieState**: 파라미터로 받지만 **사용하지 않음** (dead parameter)
- **총 13종** → 같은 상황에서 항상 같은 질문이 반복됨

### 구체적 문제점
1. 플레이어가 fact_pursuit depth 0을 3번 쓰면 3번 모두 "사실 여부를 확인하겠습니다"가 나옴
2. NPC가 S0(완강 부정)일 때와 S4(감정 호소)일 때 재판관 톤이 동일
3. 증거를 제시하는 상황이 "입장을 밝혀 주십시오" 하나뿐 → 극적 긴장감 없음

## 3. 설계 목표

### 질문 풀 구조

```
fact_pursuit:      depth 4단계 × 변형 3종 × 톤 2벌(soft/hard) = 24종
motive_search:     depth 4단계 × 변형 3종 × 톤 2벌 = 24종
empathy_approach:  depth 4단계 × 변형 3종 × 톤 2벌 = 24종
evidence_present:  depth 3단계 × 변형 2종 × 톤 2벌 = 12종

총 ~84종
```

### 톤 분화 (lieState 기반)

| NPC lieState | 재판관 톤 | 코드 | 특징 |
|-------------|----------|------|------|
| S0-S1 | 부드럽고 탐색적 | `soft` | "~말씀해 주시겠습니까?", "~설명해 주십시오" |
| S2 | 구체화 요구 | 상황에 따라 soft/hard | depth 0-1은 soft, depth 2+는 hard |
| S3-S4 | 압박적, 날카로운 | `hard` | "~분명히 말씀하십시오", "~왜 달라진 것입니까?" |
| S5 | 확인적, 정리 | `soft` (확인 톤) | "~마지막으로 확인하겠습니다" |

## 4. 품질 기준

### 반드시 지켜야 할 것

**1. 재판관 위엄**
한국 법정 재판관의 격식. 합니다체/십시오체 사용. 해요체·반말 절대 금지.
```
좋은 예: "~에 대해 사실 여부를 확인하겠습니다. 말씀해 주십시오."
나쁜 예: "~에 대해 얘기해 주세요." (해요체)
나쁜 예: "~에 대해 말해 봐라." (반말)
```

**2. 스포일러 금지**
질문이 진실을 전제하면 안 됨. NPC가 아직 부정 중인데 질문이 유죄를 전제하면 게임이 망가진다.
```
좋은 예: "왜 진술이 달라졌습니까?" (중립적)
나쁜 예: "왜 거짓말을 했습니까?" (유죄 전제)
나쁜 예: "횡령한 금액이 얼마입니까?" (진실 노출)
```

**3. 자연스러운 한국어**
번역체 금지. 한국인 판사가 실제로 쓸 법한 표현.
```
좋은 예: "당시 어떤 판단으로 그렇게 하셨습니까?"
나쁜 예: "당신은 그 행동에 대해 어떻게 생각하십니까?" (번역체)
나쁜 예: "그것에 관련하여 당신의 설명을 요청합니다." (보고서 톤)
```

**4. 변형 간 차별화**
같은 depth의 3종 변형이 **접근 방식** 자체가 달라야 함.
```
변형 A (직접): "~을 설명해 주십시오."
변형 B (우회): "~에 대해 당시 상황부터 말씀해 보시겠습니까?"
변형 C (반문): "~이 아직 분명하지 않습니다. 왜 그렇습니까?"

FAIL: 변형 A와 B가 종결어미만 다른 경우
"~말씀해 주십시오" vs "~말씀해 주시겠습니까?" → 이것은 1종임
```

**5. 84건 범용**
이 질문 풀은 84건의 서로 다른 사건에 적용된다. 특정 사건에 종속되면 안 됨.

## 5. 템플릿 변수

모든 질문에 사용 가능한 변수:
- `${name}`: NPC 이름 (예: "한지석", "박서윤")
- `${subject}`: 쟁점 소재 (예: "해당 사안", "금전 거래", "휴대폰 열람")
  - 스포일러 제거된 중립 표현
  - "외도 상대인가" → "해당 사안", "비밀 송금 280만원" → "송금"

`${subject}`에 들어올 수 있는 예시:
```
금전 거래, 해당 사안, 휴대폰 열람, 송금, 야간 접촉,
월세 지연, 벽지 손상, 수전 누수, 보증금 반환,
보고서 작성, 인사정보 열람, 수정이력, 평가 개입,
보증금 환급, 정산, 택시비, 공개 비난, 누수 원인
```
