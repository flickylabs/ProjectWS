# [추가자료] 7건 실전 테스트에서 발견된 재판관 질문 문제

> 이 문서는 실제 7건 플레이테스트 결과에서 추출한 재판관 질문 관련 문제점이다.
> 질문 풀을 설계할 때 이 문제를 반드시 해결해야 한다.

---

## 1. evidence_present 질문 반복 (7건 모두 동일)

현재 evidence_present가 1종 고정이라 모든 사건에서 동일한 질문이 나옴:

```
spouse-01:    "한지석 씨, 방금 제시된 증거에 대해 입장을 밝혀 주십시오."
workplace-01: "윤태성 씨, 방금 제시된 증거에 대해 입장을 밝혀 주십시오."
family-01:    "윤서아 씨, 방금 제시된 증거에 대해 입장을 밝혀 주십시오."
...
```

→ evidence_present를 depth 3단계 × 변형 2종 × 톤 2벌 = 12종으로 확대해야 함.

**특히 evidence_present는 증거 유형에 따라 질문이 달라져야 의미 있음:**
- 거래 내역서: "이 거래 내역에 대해~"
- 메신저 캡처: "이 메시지 기록에 대해~"
- 수정 이력: "이 문서 기록에 대해~"
→ 하지만 증거 유형은 런타임에 결정되므로, **범용 표현**을 써야 함.
→ "이 증거", "방금 제시된 자료", "이 기록" 등으로 변형.

## 2. lieState 톤 변화 없음

7건 모두에서 확인됨:
- S0(완강 부정) 상태인 NPC에게 fact_pursuit → "말씀해 주십시오" (부드러운)
- S4(감정 호소) 상태인 NPC에게 fact_pursuit → "말씀해 주십시오" (동일)

→ S0-S1에서는 탐색적 톤, S3-S4에서는 압박적 톤이 필요.

## 3. 범사건 클리셰 — 질문 풀에서 사용 금지

아래 표현들이 NPC 대사에서 반복 발견됨. **재판관 질문에서도** 비슷한 클리셰가 나오면 안 됨:

| 금지 표현 | 대체 표현 |
|----------|----------|
| "미리 말씀드리지 못한" | 쓰지 마라 (NPC 대사 클리셰) |
| "적지 않은 돈" | "상당한 금액" 도 이미 클리셰화됨 |
| "그 무렵" | 4회 반복 → 다른 시간 표현 사용 |
| "해당 ~" | 딱딱함 유발 → "그 ~", "이번 ~" 교체 사용 |
| "입장을 밝혀 주십시오" | 이미 클리셰 → 변형 필수 |

**재판관 질문에서 주의할 추가 클리셰:**
- "~에 대해 사실 여부를 확인하겠습니다" — 현재 fact depth 0 유일 질문
- "좀 더 구체적으로 말씀해 주십시오" — 범용 요청이라 날카로움 없음
- "정확히 무슨 일이 있었는지" — 반복 시 기계적

## 4. NPC 응답의 "맞습니다. 다만..." 패턴

7건 통합 리포트에서 15회 이상 발견:
```
"~은 맞습니다. 다만 ~"
"~은 사실입니다. 하지만 ~"
```

이것은 NPC 대사 문제이지만, **재판관 질문이 항상 같아서** NPC도 같은 패턴으로 응답하는 측면이 있음.
→ 재판관 질문에 변형이 생기면 NPC 응답도 자연스럽게 다양해질 것.

## 5. S5 자백 품질 참고 (양호)

재판관의 S5 단계 질문은 "마지막으로 확인하겠습니다" 톤으로 부드럽게.
S5에서 나온 좋은 자백 예시:

```
한지석 S5: "네, 그 280만원은 공동계좌에서 9월 20일에 송금했습니다.
          재가돌봄센터 상담팀장 최민정에게 보낸 돈입니다."
→ 구체적 이름/금액/날짜 포함 = 좋은 S5

partnership-01 S5: "9월 3일, 법인 준비금 1,800만원, 법무사 신탁계좌"
→ 최우수 자백 (핵심 사실 3개 압축)
```

→ S5 단계 재판관 질문은 **확인/정리 톤**으로 설계하되 과하게 압박하지 않기.
