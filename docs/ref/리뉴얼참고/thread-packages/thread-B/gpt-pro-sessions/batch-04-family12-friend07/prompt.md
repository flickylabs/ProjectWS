## 프롬프트 (복사용)

```
너는 솔로몬 법정 게임의 품질 검수 전문가다.
첨부 파일 14개를 두 가지 기준으로 동시에 검수해.

■ 파일 구분
- evidence-*.json (7개): 사건 JSON → evidence 배열의 surfaceName/surfaceDescription 검수
- dossier-*.json (7개): v3-game-loop-data → dossierCards 배열의 질문 검수

위반을 전부 찾아서 교정안을 제시해라.

━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 작업 A: Evidence 안전성 검수
━━━━━━━━━━━━━━━━━━━━━━━━━━

[핵심 원칙]
"증거의 surfaceName과 surfaceDescription은 플레이어가 처음 보는 텍스트다.
핵심 진실이 여기서 노출되면 게임이 무너진다."

[surfaceName 금지]
- 기관명, 서비스명, 직함, 인물 실명
- 허용: 증거 종류 ("이체 내역", "통화 기록", "영수증", "캡처 이미지")

[surfaceDescription 금지]
- 핵심 진실 직접 노출
- 기관명/서비스명/인물 직함
- 번역체 (했던 것이다, 한 바 있다, 인 것으로 보인다, 것으로 확인된다, 할 수 있을 것이다, 라고 할 수 있다, 에 해당한다, 인 것으로 판단된다, 인 것으로 사료된다)
- "특정 X" 패턴

[anchorTruth 키워드 대조 — 매우 중요]
각 사건의 anchorTruth에서 핵심 키워드를 추출하고,
그 키워드가 surfaceName/surfaceDescription에 포함되면 반드시 교정.

[name/description 필드도 참고 확인]

━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 작업 B: DossierCard 질문 검수
━━━━━━━━━━━━━━━━━━━━━━━━━━

[규칙]
1. q1 = hint: 방향만 제시. 구체적 정보 금지. requiredLieState 없거나 S1.
   - "금지": 구체 금액, 구체 시각, 특정 행위 전체 나열, anchorTruth 핵심어 직접 지적
   - "허용": 사실 여부 질문, 모호한 방향 제시, "설명하십시오" 형태
2. q2 = partial: 핵심 일부 노출. requiredLieState = S2.
3. q3 = full: 결정적 질문. requiredLieState = S3.
4. 번역체 금지 (위와 동일 9패턴)
5. "특정 X" 금지
6. "당신" 금지 (재판관은 "이름 + 씨"로 호칭)
7. 자연스러운 한국어
8. 클리셰 금지: "미리 말씀드리지 못한", "적지 않은 돈", "해당 ~", "입장을 밝혀 주십시오"
9. lockedHint 텍스트도 검수 대상

━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 각 사건의 anchorTruth
━━━━━━━━━━━━━━━━━━━━━━━━━━
```
