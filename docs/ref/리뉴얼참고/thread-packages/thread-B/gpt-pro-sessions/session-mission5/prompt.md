## 1. 프로젝트 배경

### 솔로몬 게임이란
한국 민사 분쟁 재판 시뮬레이션 게임. 플레이어가 재판관이 되어 양측 NPC를 심문하고, **증거를 수집/조사**하여 진실을 밝힌다.

### 증거 시스템
각 사건에는 3~5개의 증거가 있다. 각 증거에는:
- **surfaceDescription**: 증거 탭에서 플레이어에게 보이는 설명 (1~2문장)
- **investigationStages**: 플레이어가 증거를 조사할 때 선택하는 질문 목록 (3~6단계)
  - 각 단계에 `question.text` — 재판관이 NPC에게 던지는 질문
  - `attackVector` — 어떤 약점을 공격하는지 (authenticity/context/legality/identity)

### 플레이어가 보는 화면
```
[증거 탭]
📄 계좌 거래 기록
   "은행에서 발급된 계좌 거래 기록이다. 이체 금액과 수취인 정보가 포함되어 있다."
                                              ↑ surfaceDescription

[증거 조사 시]
🔍 조사 옵션:
   ① "공동계좌에서 상당한 금액이 이체된 사실을 인정하십니까?"  ← question.text
   ② "이 이체를 본인이 직접 실행했다는 점에 대해 설명하십시오."
```

## 2. 교정 대상

### 2-1. surfaceDescription (증거 설명)
플레이어가 증거 탭에서 처음 보는 텍스트. **Progressive Truth Throttle이 적용되어야 함.**
- 증거 자체가 어떤 진실을 밝히는지 미리 알려주면 안 됨
- 중립적, 사실적 표현 필요
- 자연스러운 한국어

### 2-2. investigationStages[].question.text (조사 질문)
재판관이 NPC에게 증거를 들이밀며 하는 질문.
- 재판관 위엄 유지 (격식체)
- 스포일러 금지 (질문이 답을 전제하면 안 됨)
- 단계별 깊이 차이 (stage 0 = 탐색적, stage 5 = 압박적)

## 3. 품질 기준

### 공통 기준 (모든 텍스트)
| 항목 | 기준 |
|------|------|
| **의미** | 질문과 설명이 논리적으로 정확 |
| **맥락** | 증거의 실제 내용과 일치 |
| **스포일러** | 진실을 전제하거나 노출하지 않음 |
| **경어** | 합니다체 통일 |
| **어법** | 번역체 0건, 자연스러운 한국어 |

### surfaceDescription 전용 기준
```
좋은 예: "휴대폰에서 캡처된 메신저 대화 화면이다. 대화 일부만 보여 전체 맥락을 파악하려면 조사가 필요하다."
나쁜 예: "오세린이 새벽에 남편 폰에서 캡처한 메시지로, 외도 의심의 근거가 된다."
         → 인물명 + 행위 + 판단이 모두 노출됨
```

### question.text 전용 기준
```
좋은 예: "이 대화가 전체 원본이 아니라 일부 캡처인 점을 알고 계셨습니까?"
나쁜 예: "왜 대화를 잘라서 증거를 조작했습니까?" → 조작을 전제
```

### 금지 표현 (클리셰)
| 금지 | 이유 | 대체 |
|------|------|------|
| "해당 건에 대해서는" | 번역체/딱딱 | "이 건에 대해서는" / "그 부분은" |
| "~된 것으로 판단됩니다" | 관료적 | "~인 것으로 보입니다" |
| "적지 않은" | 클리셰 | "상당한", "큰" |
| "입장을 밝혀 주십시오" | 1종 고정 클리셰 | 다양한 변형 사용 |

## 4. 작업

### 4-1. 7건의 증거 텍스트를 순회하며 교정
첨부 파일 `evidence-texts-7cases.md`에 7건의 모든 증거 텍스트가 정리되어 있다.

각 텍스트에 대해:
1. 스포일러가 있는지 확인 → 있으면 중립 표현으로 교체
2. 번역체/보고서 톤이 있는지 확인 → 자연스러운 한국어로 교체
3. 재판관 위엄이 유지되는지 확인 (question.text)
4. 단계별 깊이 차이가 있는지 확인 (stage 0 = 부드러움 → stage 5 = 압박)

### 4-2. 교정이 불필요하면 "PASS"로 표시
모든 텍스트가 이미 품질 기준을 충족하면 교정 불필요.
하지만 실전 테스트에서 "딱딱함", "반복감", "스포일러"가 발견되었으므로 주의 깊게 검토.
