# Progressive Truth Throttle — v2-atoms 교정 규칙

## 핵심 원칙
NPC의 lieState에 따라 atom이 제공하는 정보의 구체성이 단계적으로 증가해야 한다.
S0-S1에서 진실이 조기 노출되면 게임의 추리/심문 긴장감이 파괴된다.

## 상태별 허용 범위

### S0-S1 (부정/동요)
**factText에 절대 금지:**
- 인물 실명 (최민정, 윤도현, 박서윤 등) → "그 사람", "상대방"
- 기관명 (재가돌봄센터, HR, 관리사무소, 법무사 등) → "그 곳", "해당 기관"
- 직함/역할 (상담팀장, 보고서 오너, 공인중개사 등) → "담당자", "관계자"
- 서비스명 (재가돌봄, 장기요양, 인스타그램 스토리, 정산 앱, 슬랙 등) → "그 서비스", "해당 플랫폼"
- 구체적 금액 (2,800,000원, 1,800만원, 92,000원 등) → "그 돈", "해당 금액"
- 구체적 날짜/시간 (9월 20일, 14시 03분, 7월 12일 등) → "그때", "그 무렵"
- 구체적 수량/기간 (5일, 31시간, 두 차례 등) → "며칠", "여러 차례"

**slots에서:**
- `fullName` 필드: 반드시 neutral과 동일한 값으로 설정 (예: "그 사람")
- `exact` 필드 (금액/시간/기간): 반드시 neutral과 동일한 값으로 설정
- `role` 필드: "관계자", "담당자" 수준으로 제한
- `institution.name` / `institution.exact`: "해당 기관" 수준으로 제한
- `evidence.fullName` / `evidence.exact`: "해당 자료" 수준으로 제한

### S2 (일부 인정)
**factText에 허용:**
- "가족 일", "개인 사정", "업무 관련" 수준의 일반 표현
- "그 금액을 보낸 건 맞다", "늦은 건 인정한다" 수준의 행위 인정

**factText에 금지:**
- 인물 실명, 기관명, 직함은 여전히 금지
- 구체적 금액/날짜는 여전히 금지

**slots에서:**
- `fullName`: "그 사람" 유지, `judgeRef`는 성씨만 허용 ("최 씨", "윤 씨")
- `exact` (금액): `rounded`와 동일하게 ("약 280만원" → "상당한 금액")
- `institution`: "그 기관" 또는 일반 분류 ("의료 관련", "법률 관련")

### S3 (책임 전가)
- 행위 인정 + 상대 탓 가능
- `judgeRef` 허용 ("박 대리", "윤 팀장")
- 구체적 진실은 아직 제한

### S4-S5 (감정 호소/인정)
- 전체 진실 허용
- `fullName`, `exact`, `role` 모두 원본 유지

## 주의사항
- suppressions에 숨기라고 명시된 정보가 같은 상태의 atom slots에 노출되면 안 된다
- factText는 NPC가 실제로 말하는 텍스트이므로 가장 엄격하게 관리
- slots는 LLM이 선택할 수 있는 후보이므로 S0-S1에서는 neutral 값만 남겨야 한다
