## 1. 프로젝트 배경

### 솔로몬 게임이란
민사 분쟁을 다루는 재판 시뮬레이션 게임. 플레이어가 재판관이 되어 양측 NPC를 심문하고, 증거를 수집하여 진실을 밝힌다.

### NPC 심문의 핵심 메커닉: lieState
각 NPC는 쟁점(dispute)별로 `lieState`(S0~S5)를 가진다.
- **S0**: 완강한 부정 — "그런 적 없다"
- **S1**: 동요 — "글쎄요, 정확히는..."
- **S2**: 일부 인정 — "그건 맞지만..."
- **S3**: 책임 전가 — "내 잘못이 아니라 상대가..."
- **S4**: 감정 호소 — "사정이 있었다..."
- **S5**: 전면 인정

플레이어가 증거를 제시하고 질문을 반복하면 lieState가 올라가면서 NPC가 점점 진실에 가까운 말을 한다.

### v2-atoms 구조
각 사건의 v2-atoms JSON은 이런 구조:
```
{
  "caseId": "workplace-01",
  "claimPolicies": {
    "a": {                          // Side A (partyA)
      "d-1": {                      // 쟁점 1
        "S0": {                     // lieState S0
          "publicClaim": [...],     // ← 이 NPC가 공개적으로 주장하는 것
          "privateKnowledge": [...],// ← 이 NPC만 아는 숨겨진 정보
          "suppressions": [...],    // ← 이 상태에서 적극적으로 숨기는 것
          "claimAtoms": [...]       // ← 개별 진술 단위 (factText + slots)
        },
        "S1": { ... },
        ...
      }
    }
  }
}
```

### Progressive Truth Throttle 규칙
NPC가 **아직 부정/회피 중인 상태(S0~S2)**에서 핵심 진실이 노출되면 게임의 추리 긴장감이 파괴된다.

| 상태 | NPC가 말할 수 있는 것 | 금지 |
|------|----------------------|------|
| **S0-S1** | 모호한 부정/회피 | 기관명, 인물 실명, 직함, 서비스명, 구체적 금액/날짜/시간 |
| **S2** | "개인 사정"/"업무 관련" 수준 인정 | 기관명, 인물 실명 (성씨만 허용), 구체적 금액 |
| **S3+** | 점진적 진실 허용 | 제한 완화 |

### 왜 narrative 필드도 교정해야 하는가
`publicClaim`, `privateKnowledge`, `suppressions`는 **LLM에게 NPC 역할을 부여하는 시스템 프롬프트**에 직접 삽입된다. 예를 들어:

```
당신은 윤태성(팀장)입니다. 현재 심문 상태: S0 (부정 중)
[공개 주장] {publicClaim 배열 내용}
[내면 지식] {privateKnowledge 배열 내용}
[숨기는 것] {suppressions 배열 내용}
이 맥락에서 재판관의 질문에 대답하세요.
```

여기에 "박서윤이 23시 48분에 최종 수정했다"같은 구체적 진실이 들어있으면, LLM이 S0 상태에서도 그 정보를 활용해 대사를 생성할 수 있다. 따라서 `claimAtoms`뿐 아니라 이 narrative 필드들도 같은 수준으로 교정되어야 한다.

## 4. 공통 작업 지침

각 파일에 대해:

1. S0, S1, S2 상태의 `publicClaim`, `privateKnowledge`, `suppressions` 배열을 순회한다.
2. 아래 사건별 교정표에 따라 금지어를 교체한다.
3. **문맥을 유지**하면서 자연스러운 한국어가 되도록 한다. 단순 치환이 어색하면 문장을 다듬어도 된다.
4. 교정 후에도 해당 필드의 **의미와 역할**이 보존되어야 한다:
   - `publicClaim`: NPC가 재판관 앞에서 공개적으로 주장하는 내용
   - `privateKnowledge`: NPC가 속으로 알고 있지만 드러내지 않는 내용
   - `suppressions`: NPC가 이 상태에서 적극적으로 숨기려는 사실

**절대 건드리지 말 것:**
- `claimAtoms` 배열 전체 (1차 교정 완료)
- `tellPool` (있는 경우)
- S3, S4, S5 상태의 모든 필드
- JSON 구조, 키 이름, 배열 순서
