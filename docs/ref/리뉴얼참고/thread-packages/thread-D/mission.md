# 스레드 D — Phase 1/2 스크립트 교정

> 담당: Phase 1(초기 진술)과 Phase 2(반박) 사전 스크립트의 스포일러 제거 + 품질 교정
> 주 도구: GPT Pro (텍스트 교정) + 자체 검증
> 산출물: 교정된 7건의 Phase 1/2 JSON 파일

---

## 1. 배경

Phase 1과 Phase 2는 게임 시작 시 NPC가 자동으로 진행하는 대화다.
플레이어가 아직 아무 조사/심문도 하지 않은 시점이므로, **여기서 진실이 노출되면 게임이 시작도 전에 끝난다.**

### 핵심 원칙
> Phase 1/2 대사에는 **진실이 절대 포함되면 안 된다.**
> 양쪽 당사자가 자기 입장만 말하되, 구체적 사실(이름, 금액, 기관명)은 숨긴다.

---

## 2. 이번에 바뀐 규칙

### 진실 노출 수준
Phase 1/2는 게임 내에서 S0-S1 수준에 해당한다. Truth Throttle 규칙:
- 기관명, 인물 직함, 서비스명 금지
- 정확한 금액/날짜 금지 ("상당한 금액", "그 무렵"으로)
- "사실은~", "실은~" 고백 금지
- 상대의 구체적 행위를 단정하는 표현 금지

### 증거 이름 노출 금지
Phase 1/2에서 증거의 실제 이름이 언급되면 안 된다.
증거는 아직 surfaceName 상태이므로, 구체적 증거명을 말하는 건 논리적으로도 맞지 않다.

### 존대 규칙
- 재판관에게: 합니다체 ("~습니다", "~입니다")
- 해요체 ("~에요", "~해요") 금지
- 상대 언급: 재판관에게 3인칭 ("제 남편이~", "상대방이~")
- 상대에게 직접 반말 금지

---

## 3. 작업

### 대상 파일
```
Phase 1: src/data/dialogues/phase1/{case}.json  (7건)
Phase 2: src/data/dialogues/phase2/{case}.json  (7건)

case: spouse-01, family-01, friend-01, neighbor-01, partnership-01, tenant-01, workplace-01
```

### 각 파일에 대해

1. **스포일러 스캔**: 진실이 노출되는 대사 식별
   - 구체적 이름 (최민정, 오미숙, 정우성 등)
   - 구체적 금액 (280만원, 150만원 등)
   - 기관명 (재가돌봄센터 등)
   - 서비스명/직함 (상담팀장 등)
   - 사건의 핵심 사실을 직접 언급하는 표현

2. **교정**: 스포일러 부분을 모호한 표현으로 교체
   - "280만원" → "상당한 금액" 또는 "적지 않은 돈"
   - "최민정" → "어떤 사람" 또는 아예 삭제
   - "재가돌봄센터" → "어떤 곳" 또는 맥락에 맞는 모호한 표현
   - "간병 예약금" → "개인적인 용도"

3. **존대 검증**: 해요체 사용 여부 확인 → 합니다체로 교정

4. **자연스러움 검증**: 교정 후에도 대사가 자연스러운지 확인

---

## 4. 교정 예시 (spouse-01 기준)

### Before
"그날 280만원을 최민정이라는 사람에게 보냈다는 걸 알게 됐어요."

### After
"그날 상당한 금액이 공동계좌에서 빠져나갔다는 걸 알게 되었습니다."

### Before
"재가돌봄센터 간병 예약금이었다고 하지만..."

### After
"나름의 사정이 있었다고 하지만..."

## 5. 산출물

교정된 JSON 파일 7건 × 2 (Phase 1 + Phase 2) = **14개 파일**

각 파일에 교정 내역을 주석이나 별도 문서로 기록:
- 원래 텍스트
- 교정 텍스트
- 교정 이유 (스포일러/존대/어법)

---

## 6. 검증 기준

- [ ] Phase 1/2 어디에도 사건의 핵심 진실이 드러나지 않음
- [ ] 구체적 이름/금액/기관명 없음
- [ ] 합니다체 통일 (해요체 0건)
- [ ] 상대 언급 시 3인칭 사용
- [ ] 교정 후에도 대사가 캐릭터 성격에 맞고 자연스러움

---

## 7. 직접 수행 가이드

### 추천 방식: 스캔 에이전트 → 교정 에이전트 → 검증 에이전트

#### 1단계: 스포일러 스캔 (에이전트 2개 병렬)
Phase 1과 Phase 2를 나눠서 병렬 스캔한다.

- 에이전트 A: Phase 1 스크립트 7건 스캔
- 에이전트 B: Phase 2 스크립트 7건 스캔

에이전트 프롬프트:
```
다음 Phase {1/2} 스크립트 JSON 파일들을 읽고 스포일러를 찾아라.

스포일러 기준:
- 사건의 핵심 진실을 직접 언급하는 표현 (예: "간병 예약금", "돌봄센터", "월세 대납")
- 구체적 인물명이 진실과 연결되는 문맥 (예: "최민정이 실은 상담팀장")
- 정확한 금액 (예: "280만원", "150만원")
- 기관명/서비스명 (예: "재가돌봄센터")
- 증거의 실제 이름 (surfaceName이 아닌 원래 name)

파일 목록:
src/data/dialogues/phase{1/2}/spouse-01.json
src/data/dialogues/phase{1/2}/family-01.json
src/data/dialogues/phase{1/2}/friend-01.json
src/data/dialogues/phase{1/2}/neighbor-01.json
src/data/dialogues/phase{1/2}/partnership-01.json
src/data/dialogues/phase{1/2}/tenant-01.json
src/data/dialogues/phase{1/2}/workplace-01.json

각 파일에서 스포일러가 있는 대사를 정확히 인용하고,
어떤 진실이 노출되는지, 어떻게 교정해야 하는지 제안하라.
```

#### 2단계: 교정 (GPT Pro 추천)
스포일러가 많을 경우 GPT Pro에 교정을 맡긴다.

```
gpt-session-D1/
  spoiler-report.md       ← 1단계에서 발견된 스포일러 목록
  phase1-spouse-01.json   ← 원본 스크립트
  rules.md                ← 교정 규칙 (섹션 3 복사)
  prompt.md               ← "스포일러 부분을 규칙에 따라 교정하라"
```

직접 교정할 경우:
- 각 스포일러 대사를 개별 Edit로 수정
- 한 번에 여러 곳을 바꾸지 말고 하나씩 (실수 방지)

#### 3단계: 교정 검증 (별도 에이전트)
```
교정된 Phase 1/2 스크립트를 검증하라.

[검증 기준]
- 스포일러가 완전히 제거되었는가? (기관명/인물명/금액/서비스명 0건)
- 교정 후에도 대사가 자연스러운가? (어색한 표현, 문맥 단절 없는지)
- 합니다체가 통일되었는가? (해요체 0건)
- 캐릭터 성격(archetype)에 맞는 말투가 유지되는가?
- Phase 1(초기 진술)의 정보량이 적절한가? (너무 많이 말해도, 너무 적게 말해도 안 됨)

교정 전 원본과 비교하여 변경된 부분만 나열하라.
```

### 존대 검증 자동화
해요체가 남아있는지 기계적으로 확인:
```
grep -n "에요\|해요\|거예요\|했어요\|있어요\|같아요\|됐어요\|없어요" src/data/dialogues/phase1/*.json src/data/dialogues/phase2/*.json
```

### 주의
- Phase 1/2는 **게임의 첫인상**이다. 교정으로 대사가 너무 비어버리면 안 된다.
- 스포일러를 제거하되, 양쪽 당사자의 **입장과 감정은 충분히 전달**되어야 한다.
- 핵심: "무엇이 있었는지"는 숨기되, "어떤 기분인지"는 보여주는 것.
