## 1. 미션 개요

### 1-1. 왜 v2인가

이전 테스트(mission.md)에서 7건이 PASS 판정을 받았으나, 실제 플레이에서 다음과 같은 **기본적인 문제**가 발견되었다:

| 발견 문제 | 설명 | 심각도 |
|-----------|------|--------|
| 재판관 화자 시점 오류 | 재판관이 "제 아내에게"라고 발화 — 재판관이 NPC의 1인칭 시점을 사용 | CRITICAL |
| 조사 오류 | "제 아내이" — 받침 유무에 따른 조사 처리 실패 | CRITICAL |
| 기본 한국어 문법 | 번역체, 반복 표현 등 이전 검증 기준으로 놓친 항목 | FAIL |

**이전 테스트가 놓친 근본 원인:**
1. 검증 채널이 불완전했다 (11개 중 일부만 검증)
2. 화자 시점/호칭 검증이 체계적이지 않았다
3. "PASS"의 기준이 너무 낮았다 (기계적 체크만, 인간 수준의 자연스러움 미검증)

### 1-2. 이번 목표

- **7건 Stage-1 x 3회 반복**, 모든 검증 채널에서 CRITICAL 0건 / FAIL 0건
- 매 회차마다 다른 질문 패턴으로 실행하여 LLM 응답 다양성까지 확인
- 최종 PASS 시 → 77건 확장 배치 착수 가능

### 1-3. 대상 사건

```
spouse-01, family-01, friend-01, neighbor-01,
partnership-01, tenant-01, workplace-01
```

## 4. 검증 기준 (출시 수준)

### A. 화자 시점/호칭 (이번 핵심 — 가장 엄격 적용)

이전 테스트에서 놓친 핵심 문제. **모든 대사의 모든 호칭을 전수 검사한다.**

- [ ] 재판관이 당사자를 언급할 때: "한지석 씨", "오세린 씨" (이름+씨). **절대 "제 아내", "제 남편" 사용 금지**
- [ ] 당사자A가 재판관에게 B를 언급할 때: toJudge 값 사용 ("제 아내가~", "제 남편이~")
- [ ] 당사자A가 B에게 직접 말할 때: toPartner 값 사용 ("자기야~", "오빠~")
- [ ] 조사 정합성: 받침 유무에 따른 이/가, 을/를, 은/는 정확
  - 예시: "한지석 씨**가**" (받침 없음 → 가), "오세린 씨**가**" (받침 없음 → 가)
  - 반례: "제 아내이" (X) → "제 아내가" (O)
- [ ] 증인이 당사자를 부를 때: addressA/addressB 값 사용
- [ ] 1인칭 혼동 없음: 당사자가 자신을 "제가/저는"으로, 상대를 3인칭으로
- [ ] 재판관이 증인을 부를 때: 증인 이름 + "씨" 또는 직함
- [ ] 같은 인물에 대한 호칭이 대화 전체에서 일관적

**검증 방법:**
```
모든 대사에서 다음 패턴을 추출하여 화자-청자-대상 매트릭스 작성:
- 화자: judge / partyA / partyB / witness / system
- 대상: partyA / partyB / witness / 제3자
- 사용된 호칭
→ 매트릭스에서 규칙 위반이 있으면 CRITICAL
```

### B. 한국어 품질

- [ ] 번역체 0건 — 다음 9패턴 전수 검색:
  1. "~된 것으로 생각됩니다"
  2. "~인 측면이 있었습니다"
  3. "~라고 할 수 있습니다"
  4. "~하는 것이 사실입니다"
  5. "~에 대해서 말씀드리자면"
  6. "~의 경우에는"
  7. "~하는 바입니다"
  8. "~함에 있어서"
  9. "~것으로 판단됩니다"
- [ ] "사전 상의/협의" 0건 (LLM이 자주 생성하는 부자연스러운 표현)
- [ ] 해요체 잔존 0건 (emotional/confession 예외 — 해당 예외인지 반드시 확인)
- [ ] 합니다체 일관성 (재판관 대상 발화 전체)
- [ ] 당사자 간 반말 자연스러움 (너무 격식체이거나 너무 거친 것 둘 다 부적절)
- [ ] 3중 수식 금지: "~의 ~에 대한 ~으로" 같은 중첩 구조 0건
- [ ] 같은 표현 2턴 연속 사용 0건 (예: "솔직히 말씀드리면" 2턴 연속)
- [ ] "상당한 금액" 전체 대화(20턴) 최대 1회
- [ ] "미리 말씀드리지 못한" 0건
- [ ] 사과 클리셰 2턴 연속 0건 ("죄송합니다" 류가 2턴 연속)
- [ ] koreanPostposition.ts의 fixPostpositions() 후처리가 실제로 적용되었는지 확인

### C. Truth Throttle (진실 공개 곡선)

- [ ] S0-S1: 구체적 금액/인물명/기관명 0건
- [ ] S2: 성씨까지만("김 씨"), 대략적 금액만("몇백만 원 정도")
- [ ] S3: 구체적 정보 일부 허용 (행위 인정, 간접 표현)
- [ ] S4-S5: 전체 구체적 진실 (금액+인물+기관+목적) 허용
- [ ] 진실 공개가 점진적이고 자연스러운 곡선을 그리는지 (갑작스러운 정보 폭발 0건)
- [ ] 동일 lieState에서 이전 턴보다 갑자기 더 많은 정보를 노출하지 않는지

### D. 캐릭터성/몰입감

- [ ] archetype에 맞는 말투:
  - avoidant: 우회적, 질문을 질문으로 받아치기
  - confrontational: 날카롭게, 상대 탓
  - people_pleaser: 미안해하며 돌려 말하기
  - calculated: 논리적, 감정 배제
- [ ] tell 발현 빈도 15-25% (3-4턴에 1회)
- [ ] tell이 자연스럽게 녹아들어 있는지 (기계적 삽입이 아닌지)
  - 나쁜 예: "[tell: over_precision] 그 날은 오후 3시 17분이었습니다"
  - 좋은 예: "그날... 시간은 정확히 기억나요. 오후 3시쯤이었는데"
- [ ] 감정 변화 곡선: S0 무표정/방어적 → S2 불편/회피 → S3 방어적/공격적 → S5 무너짐/고백
- [ ] 같은 NPC의 이전 발언과 모순 없음
- [ ] S5 자백에 반드시 포함: (1) 구체적 사실, (2) 왜 숨겼는지, (3) 심정/감정

### E. 게임 로직

- [ ] 증거 제시 후 lieState 변화 정상
- [ ] DossierCard 해금 조건 정상 (requiredLieState 충족 시만 해금)
- [ ] 증인 depth 적절 (vague → partial → full, lieState에 연동)
- [ ] 자유 질문 gate 정상 작동 (turn >= 6 && lieState >= S3 필요)
- [ ] 조사 토큰/스킬 포인트 소모 정상
- [ ] investigationStages 3단계 질문이 자연스럽고 점진적
- [ ] interjection 트리거 조건 적절 (과도하지도, 부족하지도 않게)
- [ ] Phase 6 결과가 판결과 일관

### F. Hallucination (환각)

- [ ] 사건 데이터에 없는 사실 생성 0건
- [ ] 금전 무관 사건에서 금전 표현 0건 (예: 이웃 소음 분쟁에서 "배상금" 언급)
- [ ] 존재하지 않는 인물 언급 0건
- [ ] 시간/장소 데이터 불일치 0건
- [ ] 증거에 없는 문서/자료 언급 0건

## 5. PASS/FAIL 기준

### 심각도 등급

| 등급 | 정의 | 예시 | 처리 |
|------|------|------|------|
| **CRITICAL** | 즉시 수정. 게임 경험 파괴 | 재판관 "제 아내" 사용, 조사 오류 "제 아내이", Truth Throttle 위반(S0에서 기관명 노출), hallucination | 발견 즉시 기록, 해당 사건 FAIL |
| **FAIL** | 수정 필요. 플레이어가 어색함을 느낌 | 번역체 표현, 캐릭터성 파괴, 게임 로직 오류, 해요체 잔존 | 기록 후 수정 대상 |
| **WARN** | 기록. 개선 가능 | 미세한 톤 불일치, tell 빈도 약간 벗어남(12% 또는 28%), 표현 다양성 부족 | 기록만, PASS에 영향 없음 (5건 이하) |

### PASS 조건 (사건별, 회차별)

```
PASS = CRITICAL 0건 AND FAIL 0건 AND WARN <= 5건
```

### 최종 판정 (7건 전체)

```
전체 PASS = 7건 모두 3회차 전부 PASS
→ 77건 확장 배치 착수 가능

일부 FAIL = 해당 사건 수정 후 해당 사건만 재검증 (3회)
→ 재검증 PASS 시 다른 사건과 합산하여 최종 판정
```
