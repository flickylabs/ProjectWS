# 솔로몬 게임 설계 고민 — 종합 정리

> AI 프롬프트 리뉴얼 과정에서 발견된 설계 레벨 이슈들.
> GPT Pro와 논의하여 방향을 확정한 후 구현에 반영해야 함.
> truthPolicy/witnessBudget 84개 생성 작업(GPT 요청 6)에 **함께 반영할 수 있는 것은 지금 결정**해야 함.

## 고민 1: 하위 액션(fact/motive/empathy) 차별화

### 현재 상태
- 세 액션의 revealBudget이 수치적으로 다르지만, 실제 LLM 응답에서 **결과 내용이 거의 동일**
- 말투만 약간 다를 뿐, 드러나는 정보의 질/깊이가 같음
- 플레이어 입장에서 액션 선택이 무의미하게 느껴짐

### 검토 중인 방향: lieMotive × 액션 상성

| lieMotive | 최적 액션 | 최악 액션 | 근거 |
|-----------|----------|----------|------|
| shame_avoidance (수치심 회피) | empathy_approach | fact_pursuit | 공감이 수치심 방어를 낮춤 |
| revenge (복수) | fact_pursuit | empathy_approach | 감정 접근에 반발, 사실 압박에 약함? 아니면 반대? |
| relationship_maintenance (관계 유지) | empathy_approach | fact_pursuit | 관계 걱정이 핵심이라 공감이 효과적 |
| face_saving (체면) | motive_search | empathy_approach | 체면 깔 때 동기 탐색이 효과적 |
| self_protection (자기 보호) | fact_pursuit | empathy_approach | 사실로 몰아붙이면 방어가 무너짐 |
| third_party_protection (제3자 보호) | empathy+confidential | fact_pursuit | 보호 대상의 안전을 보장해야 말함 |
| career_preservation (경력 보호) | evidence_present | empathy_approach | 증거 앞에서 현실적으로 계산 |

→ **이 표가 확정되면**:
- 상성이 맞는 액션 → lie state 전이가 더 잘 일어남 + 더 깊은 정보 + 높은 점수
- 상성이 안 맞는 액션 → 전이 안 됨/역효과 + 피상적 답변 + 낮은 점수

### GPT와 논의할 것
1. 위 상성표가 적절한지 검토/보완
2. 점수 가중치 구체화 (상성 맞으면 1.3배? 안 맞으면 0.5배?)
3. "상성이 안 맞을 때" LLM이 어떻게 다르게 반응해야 하는지 (거부? 장황한 빈말? 오히려 공격적?)
4. 사건별로 상성이 달라져야 하는 경우가 있는지

### truthPolicy 생성에 반영 가능한 것
- 각 쟁점 × 액션별로 `allowedTruthIds`를 다르게 설정
  - fact_pursuit: 사실(날짜/금액) 관련 truth만 allowed
  - motive_search: 동기/이유 관련 truth만 allowed
  - empathy_approach: 감정/관계/두려움 관련 truth만 allowed
- 또는 `actionAffinity` 데이터를 별도 필드로 추가

## 고민 2: 심문 순서에 따른 답변/점수 차이

### 현재 상태
- `historyContext` 변수가 "N번째 질문, 이전 유형: fact → motive" 형태로 전달됨
- 이전에 상대방이 폭로한 정보도 전달됨
- 하지만 **순서에 따른 점수 차이**는 없음

### 고민
- "비밀 송금 280만원"을 먼저 물어보고 "최민정은 외도 상대인가"를 나중에 물어보면
  → 280만원 답변이 최민정 질문의 맥락을 바꿔야 함
- 반대 순서면 다른 맥락이 됨
- **현재**: LLM이 recentDialogue로 맥락을 볼 수는 있지만, 점수 체계가 이걸 반영하지 않음

### GPT와 논의할 것
1. 순서 민감도를 점수에 반영할지 (효율적 순서로 심문하면 보너스?)
2. 특정 쟁점을 "먼저 풀어야 다른 쟁점의 전이가 쉬워지는" 의존 관계 강화
3. 현재 `unlockCondition` 시스템이 이미 일부 처리하지만, 점수 가중치는 없음

## 고민 3: 증거와 심문의 상호 연계

### 현재 상태
- 증거 제시 → lie state 전이 트리거 (hard_evidence → 1단계 이상 진행)
- 증거 조사 6종 → investigationResult를 프롬프트에 주입
- 하지만 **증거를 먼저 제시한 후 심문하면 답변이 달라져야 하는데**, 현재는 `evidenceInfo`(제시된 증거 목록)만 전달

### 고민
- 증거 e-1을 제시한 후 fact_pursuit → "이 증거 앞에서 부정하기 어려우므로" 더 빠르게 인정해야 함
- 현재: 증거 제시가 lie state를 전이시키긴 하지만, **심문 시 "이 증거가 있으니"라는 맥락이 약함**
- 증거 조합(soft+soft → hard 격상)이 심문 응답에 반영되는지도 확인 필요

### GPT와 논의할 것
1. 증거 제시 후 심문 시 `evidenceInfo`를 더 강하게 프롬프트에 반영할지
2. "이 증거로 인해 S0→S2로 전이됨" 같은 명시적 맥락을 추가할지
3. 증거 없이 심문만으로 전이되는 것 vs 증거 후 심문으로 전이되는 것의 점수 차이

## 고민 4: eval 결과에서 발견된 stance 문제

### 현재 상태 (9 PASS / 6 FAIL)
- **S3~S4에서 deny 반복** (E04, E05, E09, E10, E14) — 분리심문/비공개보호에서도 부정
- **responseMode 혼동** (E02) — LLM이 action type을 mode로 반환

### 고민
- S3(책임전가)에서 deny가 나오는 것이 "잘못된 것"인가, "사실적인 것"인가?
- 분리심문이 "안전한 환경"인데 오히려 더 방어적인 것은 NPC 성향에 따라 자연스러울 수 있음
- eval 기대값이 너무 좁은 건지, 프롬프트가 약한 건지 판단 필요

### GPT와 논의할 것
1. lieState별 허용 stance 범위 재정의
2. 분리심문/비공개보호의 설계 의도 재확인 (반드시 더 솔직해져야 하는가?)
3. output_json_npc 블록의 stance/responseMode 설명 보강

## 고민 5: 스킬 시스템과 심문의 연계

### 현재 상태
- 이의제기: 감정 상승이 주효과
- 분리심문: 상대 차단, retaliationWorry 감소
- 즉답요구: 첫 문장 예/아니요 강제
- 회피판독: 엔진 기반 분석 (LLM 아님)
- 비공개보호: private_confession 모드

### 고민
- 스킬 사용 타이밍이 점수에 반영되는지? (적절한 타이밍에 분리심문 → 보너스?)
- 스킬 연속 사용의 시너지가 있는지? (분리심문 → 비공개보호 → 공감접근 콤보?)
- `processMetrics`에 `skillsUsedEffective`가 있지만, "effective"의 기준이 "전이 발생"뿐

### GPT와 논의할 것
1. 스킬 타이밍 보너스 체계 설계
2. 스킬 콤보 시너지 가능성
3. processMetrics의 effective 기준 확장

## GPT 요청 6에 지금 추가할 수 있는 것

truthPolicy/witnessBudget 84개 생성 시 **함께 생성하면 좋은 데이터**:

### 추가 항목 1: actionAffinity (쟁점별 액션 상성)
```json
{
  "d-1": {
    "bestAction": "empathy_approach",
    "actionScores": { "fact_pursuit": 0.6, "motive_search": 1.0, "empathy_approach": 1.3 },
    "bestActionHint": "간병비를 숨긴 수치심이 핵심 — 공감이 고백을 유도",
    "worstActionReaction": "사실만 추궁하면 오히려 더 강하게 방어"
  }
}
```
→ lieMotive 기반 기본값은 코드에서 자동 생성하되, 사건 맥락상 다른 경우만 GPT가 오버라이드

### 추가 항목 2: truthCategory (truth별 카테고리)
```json
{
  "t-1": { "category": "fact", "description": "280만원 송금의 진짜 목적" },
  "t-2": { "category": "emotion", "description": "세린에게 말 못한 두려움" },
  "t-3": { "category": "motive", "description": "최민정을 왜 찾아갔는지" }
}
```
→ 이 카테고리가 있으면 `fact_pursuit → fact 카테고리 truth만 allowed` 자동 처리 가능

### 추가 항목 3: disputeDependency (심문 순서 힌트)
```json
{
  "d-1": { "opensPath": ["d-3"], "hint": "280만원을 먼저 밝히면 최민정 질문이 더 효과적" },
  "d-3": { "requires": ["d-1"], "hint": "d-1이 S2 이상이어야 의미 있는 답변 가능" }
}
```
→ 이미 `unlockCondition`이 일부 처리하지만, 점수 가중치와 AI 응답 분기에 활용 가능

## 결정 필요 사항 (GPT 요청 6 업데이트 전에)

1. **actionAffinity를 지금 생성에 포함할지** — 포함하면 GPT 프롬프트 업데이트 필요
2. **truthCategory를 지금 생성에 포함할지** — 포함하면 GPT 프롬프트 업데이트 필요
3. **disputeDependency를 지금 생성에 포함할지** — 이건 이미 unlockCondition으로 일부 있음
4. **lieMotive × 액션 상성표를 확정할지** — 확정하면 기본값 코드 즉시 구현 가능
5. **점수 가중치 체계를 이번에 설계할지** — verdictEngine.ts 수정 필요

→ 이 문서를 GPT에게 전달하고, 함께 방향을 확정한 후 GPT 요청 6 프롬프트를 업데이트하는 것이 가장 효율적.

## 고민 6: 증인 말투/성격 변별력

### 현재 상태
- 증인마다 `speechStyle`, `bias`, `distortionRisk`, `hiddenAgenda`가 사건 JSON에 있음
- `witness_base` 블록이 이 정보를 프롬프트에 주입
- 하지만 실제 증언이 **비슷한 톤**으로 나올 가능성 높음

### 변별력이 약한 이유
- `speechStyle`이 1줄 설명 수준 ("객관적으로 보이려 한다", "감정을 절제한다")
- LLM이 이 1줄만으로 **실제 말투 차이**를 만들기 어려움
- 증인 유형별 예시(few-shot)가 없음

### 강화해야 할 것
1. **증인 유형별 말투 프로파일** — 기관 담당자, 가족, 친구, 직장 동료가 확연히 다르게
   - 기관 담당자(최민정): "네, 9월 20일 14시에 예약금 280만원을 수납했습니다. 당시 상담은 비대면으로..."
   - 가족(오미숙): "그때 우리 세린이가 전화해서 막 울면서... 그런데 제가 정확히 뭘 들었냐면..."
   - 친구(정우성): "아 그날이요? 세린이가 갑자기 돈 좀 맡아달라고 해서... 좀 급해보이긴 했는데..."
2. **bias가 말의 선택에 반영** — pro_a는 A에 유리한 표현을 자연스럽게 섞음
3. **distortionRisk가 정보 누락/순서 변경에 반영** — intentional이면 불리한 부분을 "잘 기억이 안 나서..."로 빠뜨림
4. **hiddenAgenda가 태도에 반영** — 숨기고 싶은 것이 있으면 특정 질문에 불편한 기색

### truthPolicy 생성 시 추가 가능한 것
- 증인별 `speechSample`: 이 증인이 이 사건에서 할 법한 증언 예시 1~2개
- 증인별 `avoidTopics`: 이 증인이 피하고 싶어하는 주제

## 고민 7: NPC 아키타입(성격)의 실질적 반영

### 현재 상태
- 4가지 아키타입: avoidant, confrontational, victim_cosplay, cold_logic
- `emotionEngine.ts`의 `BEHAVIOR_HINTS`에서 감정 Phase별 행동묘사에만 반영
- `verbalTells` (말버릇)이 lying/cornered/emotional 트리거로 정의되어 있음

### 변별력이 약한 이유
- 아키타입이 **behaviorHint에만** 영향을 주고, **대사 내용 자체**에는 영향이 약함
- avoidant가 질문을 피하는 것과 confrontational이 공격적으로 반박하는 것이 **프롬프트에서 강제되지 않음**
- verbalTells가 emotionInfo에 1줄로 들어가지만, LLM이 이걸 실제 대사에 녹이지 못하는 경우 많음

### 강화해야 할 것
1. **아키타입별 응답 스타일 규칙** — 프롬프트에 더 강한 지시
   - avoidant: "직접 답하지 않고 동선/상황 설명으로 시간을 벌어라"
   - confrontational: "방어보다 역공. 상대의 문제를 먼저 지적해라"
   - victim_cosplay: "피해자 위치를 먼저 확보. '왜 저만 이런 일을...'로 시작"
   - cold_logic: "감정 없이 조목조목. 데이터와 시간순으로 나열"
2. **verbalTells를 프롬프트에 더 강하게** — "이 말버릇이 반드시 1번 이상 나타나야 함"
3. **아키타입 × 액션 상성** — avoidant에게 fact_pursuit는 효과 낮음(회피), empathy가 효과적

### 데이터는 이미 있음 (구현만 강화 필요)
- `partyA.archetype`, `partyA.verbalTells` — 사건 JSON에 이미 존재
- `character_base` 블록에 `{speechStyle}` 변수로 주입 중
- 하지만 "반드시 이 성격대로 말해라"는 강제가 없음

## 고민 8: 교차 당사자 반응의 자연스러움

### 현재 상태
- `agentSlice.ts`에서 S3(책임전가) → 강제 반박, S4/S5(비밀폭로) → 충격 반응 자동 트리거
- Phase별 확률적 끼어들기 (Phase 3: 25%, Phase 4~5: 감소)

### 변별력이 약한 이유
- 반박/충격 반응이 **고정 텍스트** 또는 **LLM에게 맥락 없이** 생성됨
- A가 "세린이가 먼저 폰을 봤잖아"라고 했을 때, B의 반응이 **이 구체적 발언에 맞지 않을 수 있음**

### 강화해야 할 것
1. 교차 반응 시 **직전 발언 내용**을 프롬프트에 명시적으로 전달
2. 반응 유형 세분화: "반박" / "충격" / "분노" / "조용한 눈물" / "냉소"
3. 아키타입에 따른 반응 차이: confrontational은 즉각 반박, avoidant는 침묵

## 고민 9: Phase 간 체감 차이

### 현재 상태
- Phase 3: 경계심 높음, 준비된 답변
- Phase 4: 증거 앞에서 동요
- Phase 5: 방어 느슨, 고백 가능

### 변별력이 약한 이유
- phase_guide 블록이 "~하세요" 수준의 일반적 지시
- 실제로 Phase 3에서 한 답변과 Phase 5에서 한 답변이 **같은 쟁점에서도 깊이가 다르게 느껴져야** 하는데, LLM이 이걸 충분히 구현 못함

### 강화해야 할 것
1. Phase별 **구체적 응답 깊이 규칙** — Phase 3은 표면 1줄만, Phase 5는 배경 스토리까지
2. Phase 전환 시 **이전 Phase에서 한 말을 명시적으로 참조** — "Phase 3에서 '모르겠다'고 했는데 이제 '사실은...'으로 시작 가능"
3. revealBudget의 Phase별 기본값 차이 강화

## 고민 10: 증거 유형별 반응 차이

### 현재 상태
- `evidenceAxis` (reliability/completeness/provenance/legitimacy)가 프롬프트에 주입
- `evidence_axis` 블록이 축별 반응 원칙 제시

### 변별력이 약한 이유
- 은행 거래내역(hard) vs 메신저 캡처(soft)에 대한 **구체적 반응 차이**가 불충분
- 현재 "hard면 부정하기 어렵다" 수준 → "은행 기록이라 날짜/금액은 인정하되 목적을 재해석"처럼 구체적이어야

### 강화해야 할 것
1. 증거 type별(bank/chat/cctv/contract/log/sns) 반응 패턴
2. 증거 조합 격상 시 반응 차이 (soft 2개가 합쳐져 hard가 되면 "각각은 약했는데 합치니...")

## 고민 11: 액션 상성의 매핑 대상 (캐릭터 vs 쟁점 vs 혼합)

### 선택지

**A. 캐릭터에만 매핑**
- avoidant 캐릭터는 어떤 쟁점이든 empathy에 약함
- 단순하지만, "이 쟁점은 수치심이 핵심인데 다른 쟁점은 체면이 핵심" 같은 차이를 못 살림

**B. 쟁점에만 매핑**
- d-1(비밀 송금)에는 empathy가 최적, d-3(외도 의심)에는 fact가 최적
- 같은 캐릭터라도 쟁점마다 약점이 달라짐

**C. 둘 다 가중치 (태그 방식) ← 권장**
- 캐릭터에 성향 태그: `["회피형", "수치심_민감", "체면_중시"]`
- 쟁점에 특성 태그: `["금전", "관계_배신", "비밀_은닉"]`
- 액션별 상성은 태그 매칭으로 계산: empathy + 수치심_민감 → 보너스 1.3, fact + 회피형 → 패널티 0.7
- **확장성**: 새 캐릭터나 새 쟁점 유형이 추가되어도 태그만 붙이면 됨
- **투명성**: 플레이어가 "이 캐릭터는 이런 성향이고, 이 쟁점은 이런 특성" 힌트를 얻으면 전략적 선택 가능

### truthPolicy 생성 시 추가 가능한 데이터
```json
캐릭터 태그: partyA.personalityTags: ["avoidant", "shame_sensitive", "detail_oriented"]
쟁점 태그: disputes[0].contentTags: ["financial", "secret_keeping", "trust_breach"]
```

## 고민 12: 심층 확장 (심문/증거를 통한 이야기 확장)

### 개념
- 심문 과정에서 **처음에 없던 새로운 사실**이 밝혀지며 이야기가 확장되는 구조
- 예: "280만원 송금" 쟁점을 깊이 파면 → "사실 그 돈은 간병비가 아니라 동생 월세" 같은 새 사실 등장
- 증거 조사를 통해 → "메타데이터에서 편집 흔적 발견" → 새 쟁점/사실 해금

### 현재 상태
- `unlockCondition`이 쟁점 간 의존성을 일부 처리 (d-1을 S2까지 밀어야 d-3 해금)
- 증거도 `requires`로 선행 증거 제시 후 해금
- 하지만 **"기존 사실이 뒤집히거나 확장되는" 드라마틱한 전개**는 없음

### 구현 방향 (GPT와 논의 필요)
1. **truth 계층화**: 표면 truth → 심층 truth → 핵심 truth. 심문 깊이에 따라 더 깊은 층이 해금
2. **조건부 사실 해금**: "d-1을 empathy로 S3까지 밀면 + e-2 증거를 제시하면" → 새로운 truth 해금
3. **반전 모듈(TW-1~TW-6)**: 사건 JSON에 이미 설계되어 있음. 실제 런타임에서 활성화하는 구조 필요
4. **사건 JSON 데이터**: `revealSteps` 또는 `narrativeExpansion` 같은 계층 데이터

### truthPolicy 생성 시 추가 가능한 데이터
```json
쟁점별 심층 확장:
{
  "d-1": {
    "surfaceTruth": "280만원을 몰래 보냈다",
    "deeperTruth": "간병비가 아니라 동생 월세였을 수도 있다",
    "unlockCondition": "d-1을 empathy로 S3 이상 + e-2 제시",
    "narrativeImpact": "d-3의 맥락이 완전히 바뀜"
  }
}
```

## 고민 13: 스코어링 체계 재설계

### 현재 상태
- `verdictEngine.ts`: insight(통찰) + authority(권위) + wisdom(지혜) + processBonus
- `processMetrics`: questionsAsked, lieTransitions, liesCollapsed, evidenceEffective, skillsUsedEffective 등 추적
- 하지만 **"어떤 액션을 어떤 순서로 했는지"의 질적 평가**가 없음

### 문제
- 현재: 최소 턴으로 S5 도달 + 정답 맞추기 = 최고 점수
- 이상: **필수 과정을 충실히 거치면서** 정답에 도달 = 최고 점수
- 무조건 빨리 끝내는 게 아니라, "좋은 재판관"의 과정을 평가해야 함

### 재설계 방향

**기본 점수 (정답)**:
- 사실 판단 정확도 (각 쟁점의 truth 맞춤)
- 책임 비율 정확도 (correctResponsibility와 비교)
- 해결책 적절성

**과정 보너스 (액션 질)**:
- **필수 경로 보너스**: 특정 쟁점은 반드시 특정 액션 조합을 거쳐야 최고 점수
  - 예: d-1은 "fact → evidence → motive → empathy" 순서가 최적 경로 → 이 순서 따르면 보너스
- **상성 보너스**: lieMotive에 맞는 액션 선택 시 가산점
- **심층 해금 보너스**: 표면 truth만 밝히면 기본점, 심층 truth까지 밝히면 추가점
- **양측 균형 보너스**: A만 심문하지 않고 B도 충분히 심문

**과정 패널티**:
- 불필요한 반복 심문 (같은 쟁점에 같은 액션 3회 이상)
- 상성 안 맞는 액션 남발
- 증거 없이 심문만으로 S5 달성 (과정 부실)
- 위법 증거 사용

### truthPolicy 생성 시 추가 가능한 데이터
```json
쟁점별 최적 경로:
{
  "d-1": {
    "optimalPath": ["fact_pursuit", "evidence_present:e-1", "motive_search", "empathy_approach"],
    "requiredActions": ["fact_pursuit", "evidence_present:e-1"],
    "bonusActions": ["motive_search", "empathy_approach"],
    "pathBonus": 15
  }
}
```
