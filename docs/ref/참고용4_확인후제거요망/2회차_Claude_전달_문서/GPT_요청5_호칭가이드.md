# 요청 5: 호칭(callTerms) + 어법 가이드 수립

## 1. 문제 개요

사건 JSON의 `callTerms` 데이터와 AI 프롬프트 코드(`llmSpeechGuide.ts`)에 호칭 부적절 이슈가 있습니다.
Phase 1 스크립트(사전 생성 대사)와 Phase 3~5 AI 생성 대사 **양쪽 모두** 같은 `callTerms`를 참조하므로 동시에 영향받습니다.

GPT Pro에게 요청하는 것:
1. **84개 사건 전체에 적용할 호칭 규칙표** 수립
2. **문제 사건 37건의 구체적 수정값** 제시
3. **AI 프롬프트 코드 수정 방향** (llmSpeechGuide.ts)

Claude Code가 하는 것:
- 규칙표/수정값 기반으로 사건 JSON callTerms 일괄 수정
- Phase 1/2 스크립트 대사 내 호칭 치환
- llmSpeechGuide.ts 코드 수정

## 2. 현재 시스템 구조

### callTerms JSON 구조 (각 사건의 partyA/partyB 안에 있음)
```json
{
  "callTerms": {
    "toPartner": "자기",        // 상대를 직접 부를 때
    "toJudge": "제 아내",       // 재판관에게 상대를 언급할 때
    "angry": "오세린 씨!"       // 감정 폭발 시
  }
}
```

### AI 프롬프트에서 callTerms를 사용하는 코드 (llmSpeechGuide.ts)

```typescript
// 1. 호칭 결정: callTerms.toPartner를 우선 사용, 없으면 관계별 폴백
getHonorifics(duo) → { aCallsB, bCallsA }

// 2. 재판관에게 상대 언급: callTerms.toJudge를 사용, 없으면 "상대방"
getJudgeReference(duo, party) → string

// 3. 격분 시: callTerms.angry를 사용, 없으면 "이름 씨!"
getAngryCall(duo, party) → string

// 4. 반말/존댓말 판별: 관계 유형으로 일괄 결정
isInformalRelation(relType) → spouse/family/friend = 반말, 나머지 = 존댓말
```

### 말투 가이드 생성 (buildSpeechGuide)
- `isInformalRelation === true`이면:
  - "상대에게: 반말 (~야, ~잖아, ~거야, ~했어)"
  - "재판관에게: 존댓말"
- `isInformalRelation === false`이면:
  - "상대에게: 존댓말 기본. 감정 격해지면 반말 전환 가능."

**문제: family를 일괄 반말로 처리** → 자녀가 부모에게 반말하는 가이드가 AI에 전달됨.

## 5. 제약 조건

- Phase 1/2 스크립트의 **대사 텍스트 안에서도 호칭이 직접 사용**됨. callTerms만 바꾸면 JSON은 맞지만 대사가 안 맞음.
  → Claude Code가 대사 내 호칭도 일괄 치환할 예정이므로, 수정값만 정확히 주면 됨.
- `naming_rules` 블록은 `{callForm}`, `{judgeRef}`, `{angryCall}` 변수 기반이므로 callTerms JSON만 수정하면 AI 프롬프트에 자동 반영됨.
- 단, `isInformalRelation()`의 반말/존댓말 판별은 코드 수정이 필요함.
