# 솔로몬 게임 및 AI 시스템 요약

> 이 문서는 데이터 생성 작업을 위한 **게임 이해용**입니다.
> 작업 규칙과 출력 형식은 `GPT_요청7_사건데이터_생성_프롬프트.md`를 참고하세요.

---

## 게임 컨셉
- **솔로몬**: 두 AI 당사자의 갈등을 플레이어(재판관)가 심문·증거·스킬로 진실을 파헤치고 판결하는 게임
- 84개 사건 (7관계 × 12), 504개 증거, 253명 증인, 420개 쟁점
- 타깃: 20~30대, 스토리/시뮬레이션 선호

## Phase 흐름
```
Phase 0 사건개요 → Phase 1~2 초기진술/반박(사전 스크립트, NO LLM)
→ Phase 3 심문(AI) → Phase 4 증거심리(AI) → Phase 5 최종심문(AI)
→ Phase 6 중재안(선택) → Phase 7 판결 → 결과
```

## 액션 체계
| 카테고리 | 액션 |
|----------|------|
| 심문 | 사실추궁(fact_pursuit) / 동기탐색(motive_search) / 공감접근(empathy_approach) / 자유질문(AI) |
| 증거 | 증거제시 / 증거조사(6종) / 조합 격상 |
| 스킬(토글) | 회피판독 / 비공개보호 |
| 스킬(액티브) | 이의제기 / 분리심문 / 즉답요구 |
| 증인 | 증인소환 + AI증언 |

## 거짓말 상태머신 (쟁점 단위)
```
S0(숨김) → S1(주장) → S2(충돌) → S3(붕괴) → S4/S5(재프레이밍/인정)
경로: 압박 루트(증거+사실추궁) vs 신뢰 루트(공감+비공개보호)
```

## 거짓말 동기 (lieMotive, 7종)
- self_protection / face_saving / shame_avoidance / relationship_maintenance / revenge / third_party_protection / career_preservation
- 각 동기마다 잘 먹히는 액션과 안 먹히는 액션이 다름 (상성 시스템)

## NPC 성격 아키타입 (4종)
- avoidant(회피형) / confrontational(대결형) / victim_cosplay(피해자연기) / cold_logic(냉정논리)

## 점수 체계 (3축)
- 🔍 통찰(insight): 사실 적중 + 책임 비율 + 과정 보너스
- ⚖️ 권위(authority): 절차 통제 + 증거 활용
- 💡 지혜(wisdom): 판단 적절성 + 해결책

## 리소스
- 🔍 조사 토큰 6개 / ⚡ 스킬 포인트 5 / ⚖️ 법정 통제력 3

## 증거 시스템
- Hard(기관 발급, 강력) / Soft(캡처/목격, 약함) / 함정(위법, 사용 시 패널티)
- 증거 조합: Soft 2개 이상 → Hard 격상
- 증거 조사 6종: 원본확인, 메타데이터, 맥락복원, 출처검증, 편집확인, 취득경위

## 증인 시스템
- 사건당 3~4명, 각각 편향(bias)/왜곡위험(distortionRisk) 보유
- 직접 목격 여부, 숨은 의도(hiddenAgenda)가 증언에 반영
- 편향 유형: pro_a, pro_b, neutral, self_interest

## 관계 유형 (7가지)
1. 부부(spouse) — 감정 밀도 최고
2. 이웃(neighbor) — 공간 공유 갈등
3. 직장(boss_employee) — 권력 비대칭
4. 동업(partnership) — 지분/계약
5. 가족(family) — 세대 갈등, 상속
6. 임대(tenant_landlord) — 계약/비용
7. 친구(friend) — 신뢰/빚

## 사건 JSON 핵심 구조
```
caseId, meta(relationshipType, familyRelation, anchorTruth)
duo(partyA, partyB — name, age, archetype, speechStyle, verbalTells, callTerms)
    socialGraph(증인 목록)
context(description)
disputes(id, name, truth, truthDescription, correctResponsibility)
evidence(id, name, type, reliability, proves, investigationResults)
truthTable(fact, isTrue, quadrant: both_know/a_only/b_only)
lieConfigA/B(disputeId, lieMotive, initialState, collapseViaTrust, transitions)
```

## 이번 데이터 생성의 목적
사건 JSON에는 위 데이터가 이미 있지만, AI가 더 정교하게 동작하기 위한 **보강 데이터**가 필요합니다:
- truthPolicy: 거짓말 상태별 진실 공개 범위
- witnessBudget: 증인별 발언 범위
- personalityTags/contentTags: 상성 계산용 태그
- actionAffinity: 쟁점별 액션 상성표
- optimalPath: 최적 심문 경로
- narrativeExpansion: 심층 맥락 확장
- witnessSpeechSample: 증인 예시 증언
