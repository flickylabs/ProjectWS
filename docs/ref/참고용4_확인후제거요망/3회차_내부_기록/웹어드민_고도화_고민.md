# WebAdmin 고도화 — 실제 운영을 위한 기능 확장 고민

> 현재 WebAdmin: Dashboard, Content(공지/우편), AI System(Agents/Blocks/DataFields/Prompts), Eval Test, Case Meta, Users(Players/History), Settings
> 게임 출시 후 **실제 운영**에 필요한 기능을 정리합니다.

---

## 1. 경제/수익 관리

### 1-1. 광고 관리
**현재**: 하드코딩 (`MAX_AD_INVEST=5`, `MAX_AD_SKILL=2`)
**필요한 것**:
- 광고 유형별 설정 (보상형 광고, 배너, 인터스티셜)
- 일일 광고 시청 횟수 제한 조정
- 광고 시청당 보상량 조정
- 광고 수익 대시보드 (일별/주별/월별)
- 광고 네트워크 설정 (AdMob, Unity Ads 등)
- 특정 시간대/이벤트 기간 광고 보상 배율 조정

### 1-2. 매출/결제 관리
**현재**: `purchaseInvest`, `purchaseSkill` 함수만 있고 실제 결제 연동 없음
**필요한 것**:
- 인앱 상품 목록 관리 (가격, 수량, 설명)
- 결제 내역 조회 (플레이어별, 상품별, 기간별)
- 매출 대시보드 (DAU, ARPU, ARPPU, LTV)
- 환불 처리
- 프로모션/할인 관리 (기간 한정, 첫 구매 할인)

### 1-3. 리소스 밸런스 조정
**현재**: 상수 하드코딩 (`FREE_CAP=10`, `PAID_CAP=99`, `RECHARGE_INTERVAL=1시간`)
**필요한 것**:
- WebAdmin에서 실시간 조정 가능
  - 무료 캡 / 유료 캡
  - 자동 충전 간격
  - 일일 광고 최대 횟수
  - 광고 시청당 보상량
  - 게임 내 초기 리소스 (조사토큰 6, 스킬포인트 5, 법정통제력 3)
- 변경 이력 추적
- A/B 테스트 지원 (그룹별 다른 밸런스)

## 2. 콘텐츠 운영

### 2-1. 시즌 관리
**현재**: DB에 `season_id` 필드 있지만 관리 UI 없음
**필요한 것**:
- 시즌 CRUD (이름, 시작/종료일, 활성 여부)
- 시즌별 리더보드 초기화
- 시즌 보상 설정 (순위별 보상)
- 현재 시즌 대시보드

### 2-2. 이벤트 관리
**현재**: 없음
**필요한 것**:
- 기간 한정 이벤트 생성 (특정 사건 개방, 보너스 보상, 특수 조건)
- 이벤트 배너 관리
- 이벤트 참여/달성 현황 대시보드
- 데일리 챌린지 설정 (매일 다른 사건 + 특수 조건)

### 2-3. 사건 콘텐츠 관리
**현재**: Case Meta 페이지에서 태그/상성/경로 편집 가능
**추가로 필요한 것**:
- 사건별 개방 조건 관리 (스테이지 잠금/해제)
- 사건별 난이도 조정
- 사건 비활성화 (버그 발견 시 긴급 숨김)
- 사건별 플레이 통계 (클리어율, 평균 점수, 평균 플레이 시간)

---

## 3. 유저 관리

### 3-1. 플레이어 상세
**현재**: Players 페이지에 기본 목록만
**필요한 것**:
- 플레이어 개별 상세 (플레이 이력, 보유 리소스, 구매 이력, 접속 패턴)
- 플레이어 검색/필터 (이름, ID, 리전, 활동 상태)
- 개별 보상 지급 (우편으로 리소스/아이템 전달)
- 제재 관리 (경고, 정지, 차단)
- VIP/고과금 유저 태그

### 3-2. 유저 행동 분석
**현재**: Game History에 기본 기록만
**필요한 것**:
- 코호트 분석 (가입일 기준 리텐션)
- 퍼널 분석 (사건 시작 → Phase 진행 → 판결 → 재플레이)
- 이탈 지점 분석 (어느 Phase에서 가장 많이 이탈?)
- 사건별 / 관계유형별 클리어율 비교
- AI 응답 만족도 (같은 쟁점 반복 심문 = 불만족 신호?)

## 4. AI 운영

### 4-1. AI 품질 모니터링
**현재**: Eval Test (15 preset + custom)
**추가로 필요한 것**:
- 실시간 AI 응답 로그 (최근 N건)
- TruthGuard 누출 감지 로그
- stance/responseMode 분포 대시보드
- 평균 응답 시간/토큰 사용량
- 프롬프트 버전별 품질 비교 (A/B 테스트)

### 4-2. LLM 비용 관리
**현재**: 없음
**필요한 것**:
- 일별/월별 API 호출 횟수 + 토큰 사용량
- 예상 비용 대시보드
- 모델별 비용 비교 (gpt-4o-mini vs gpt-4o)
- 비용 알림 (일별 한도 초과 시)
- 긴급 차단 (비용 폭주 시 LLM 호출 중단 → 폴백 모드)

## 5. 시스템 운영

### 5-1. 서버 상태
**현재**: Health Check만 (`/api/health`)
**필요한 것**:
- DB 크기/테이블별 레코드 수
- API 응답 시간 모니터링
- 에러 로그 뷰어
- 서버 재시작/캐시 클리어 버튼

### 5-2. 설정 관리
**현재**: Settings 페이지에 기본 설정
**추가로 필요한 것**:
- 게임 밸런스 상수 편집 (리소스, 점수, 상성 계수)
- 기능 플래그 (LLM 모드 on/off, 특정 Phase 스킵, 디버그 모드)
- 점검 모드 (전체 서비스 중단 + 점검 공지)
- 백업/복원
