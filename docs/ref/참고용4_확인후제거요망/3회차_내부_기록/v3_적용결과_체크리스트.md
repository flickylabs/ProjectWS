### allowedTruthIds / forbiddenTruthIds

현재 빈 배열 `[]`로 설정됨. **A1 작업(truthIds 매핑표)이 완료되면 반영 예정.**
actionContract 블록의 프롬프트는 이미 "allowedTruthIds에 없는 진실은 말하지 않는다" 규칙을 포함하고 있으므로, 매핑표가 연결되면 즉시 효과를 발휘함.

## 8. 검토 요청 사항

다른 에이전트에게 다음을 확인해 주세요:

1. **actionContract JSON 형식이 블록 프롬프트(`action_contract`)와 정합하는지**
   - 특히 `goal` 필드의 문구가 적절한지
   - `revealBudget` 계산 로직 (lieState 기반)이 합리적인지

2. **User Message 템플릿이 System Prompt(블록 조합)와 중복/충돌 없이 보완하는지**
   - System Prompt: 캐릭터 + 상태 + 규칙
   - User Message: 이번 턴의 구체적 액션 + 재판관 질문

3. **evidence_investigate 6종의 User Message가 `investigation_result_guide` 블록과 잘 연결되는지**

4. **responseMode 결정 로직이 적절한지**
   - fact_pursuit → answer_only
   - motive_search → answer_then_counter
   - empathy_approach → private_confession
   - evidence_present/investigate → evidence_rebuttal
   - confidential_protection → private_confession
   - separation → answer_only

5. **쟁점 순서 민감도가 충분한지**
   - `historyContext`가 "N번째 질문, 이전 유형: fact → motive" 형태로 전달됨
   - `recentDialogue`가 현재 쟁점 관련 대화만 필터링됨
   - 이 두 변수만으로 순서 민감도가 충분한지, 추가 컨텍스트가 필요한지
