1. 게임 개요

🐣 컨셉

AI 펫 '펩비(Pebby)'가 나의 하루를 함께 챙기는 AI가 탑재된 생활관리·정서지원 모바일 앱. 사용자가 펩비를 돌보고, 펩비가 사용자를 챙기는 '쌍방향 돌봄' 구조.

2. 스탯 시스템

2.1 스탯 정의

모든 스탯은 정수 0~5 범위이며, 초기값은 4/4/4/4입니다.

스탯배지의미범위회복 수단
HPHP체력 / 배고픔0~5음식(FOOD), 링겔(EMERGENCY)
APAP활력 / 수면0~5핫식스(EMERGENCY)
CPCP청결0~5세면용품(CARE), 주사(EMERGENCY)
MPMP정신 / 기분0~5장난감(TOY), 엄마펫(EMERGENCY)
핵심 공식
  • TOTAL = HP + AP + CP + MP   (범위: 0~20)
  • minStat = min(HP, AP, CP, MP)
  • 초기 스탯: 4 4 4 4   (TOTAL = 16)
  • UI 스케일링: 도메인 0~5 → UI 표시 0~100 (×20 변환)

3. 시간 감쇠

감쇠 규칙
  • 활성 시간대: 매일 08:00 ~ 22:00 (KST)
  • 야간: 22:00 ~ 08:00은 감소 없음 (펫도 잔다)
  • 감쇠 주기: 활성 시간 누적 4시간(240분)마다 1단계
  • 감쇠 효과: HP, AP, CP, MP 모두 -1 (최소 0 clamp)
  • 구현: 백그라운드 타이머 없음. 앱 포그라운드 복귀 시 last_decay_at 기준 계산

3.1 감쇠 알고리즘

calcActiveMinutes(last, now, tz) → 08:00~22:00 사이 활성 분 계산
calcDecaySteps(activeMinutes)     → floor(activeMinutes / 240)
applyDecay(stats, steps)          → 모든 스탯 -= steps, 0 clamp

3.2 감쇠 타임라인 예시

KST 시간이벤트HP/AP/CP/MPTOTAL
08:00활성 시간 시작4/4/4/416
12:004시간 → 1단계 감쇠3/3/3/312
16:008시간 → 2단계 감쇠2/2/2/28
20:0012시간 → 3단계 감쇠1/1/1/14
22:00활성 시간 종료1/1/1/1 (이후 감소 없음)4
⚠️
하루 최대 3단계 감쇠 가능 (14시간 활성 / 4시간 = 3.5 → 3스텝). 아무것도 안 하면 4/4/4/4 → 1/1/1/1 → RUNAWAY 상태에 진입.

4. 상태 판정 (State Machine)

4.1 우선순위 결정표

상태는 위에서 아래로 우선 순위 적용. 먼저 매칭되면 하위 조건은 무시.

#상태조건결과UX
1 DEAD minStat == 0 AND TOTAL ≤ 5 사망 → 무료 리셋만 가능 "너무 힘들어서 쓰러졌어..."
2 EMERGENCY TOTAL 6..9 AND minStat ≤ 1 응급 → Health 강제 모달 "지금 당장 치료가 필요해!"
3 RUNAWAY minStat == 1 AND TOTAL ≤ 5 가출 → 엄마펫 호출 필요 "상처받아서 집을 나가버렸어..."

4.2 컨디션 (디스플레이 전용)

위 3개 상태에 해당하지 않을 때, TOTAL에 따라 컨디션 표시 (기능적 패널티 없음):

TOTAL 범위컨디션톤
6~9VERY_SICK_SIGN"컨디션이 많이 안 좋아 보여..."
10~12BAD_MOOD살짝 풀이 죽은 얼굴
13~15NORMAL평온한 얼굴
16~20HAPPY기분 좋은 얼굴

4.3 Health 시스템 (강제 모달)

모달 트리거 시점

  • 앱 런치 / 포그라운드 복귀 (감쇠 적용 후)
  • 아이템 사용 후
  • ToDo 완료 / Diary 저장 후
  • 채팅 화면 진입 시 (EMERGENCY/RUNAWAY/DEAD인 경우)

모달 정책

  • EMERGENCY / RUNAWAY / DEAD = 닫기 불가
  • 버튼: ① "필수 아이템 사용" ② "상점으로" ③ "무료 리셋" (DEAD만)

응급 아이템 매핑

부족 스탯조건필수 아이템효과
HPHP ≤ 1링겔HP = max(HP, 2)
APAP ≤ 1핫식스AP = max(AP, 2)
CPCP ≤ 1주사CP = max(CP, 2)
MPMP ≤ 1엄마펫 호출RUNAWAY 해제 + MP = max(MP, 2)
💀
DEAD 리셋 규칙: MVP에서는 무료 리셋만. 리셋 시 스탯 → 4/4/4/4, 코인·인벤토리는 유지.

9. 세계관

핵심 전제
"내가 강가에서 데려온 조약돌이, 내 손끝에서 첫 숨을 쉬었다."

모든 돌은 본래부터 살아 있었으며, 단지 깊이 잠들어 있었을 뿐. 외부의 온기(사용자의 관심)가 닿아야 깨어날 수 있다.

9.1 온기 시스템 (Worldview ↔ 게임 매핑)

활동세계관 해석게임 효과
밥/간식"안쪽을 데우는 것" (따뜻한 것 = 지속, 차가운 것 = 순간 자극)HP 회복
목욕"물로 돌아가는 것" (원래 기억) → 매끈함 회복, 표면 데움CP 회복
마사지"굳은 곳을 녹이는 것" → 차가운 곳이 녹아 온기 흐름CP/MP 회복
낮잠"온기를 보존하는 가벼운 잠" (깊은 잠과 다름)AP 회복
놀기"유일하게 스스로 열을 만드는 것" (예외적 자가발열)MP 회복
산책"세상의 온기를 직접 만나는 것"복합 회복
사용자 부재온기가 빠져나감감쇠

9.2 펩비 (Pebby) 캐릭터

  • 종류: 깨어난 조약돌
  • 외형: 손바닥 크기 둥근 돌, 매끄러운 표면, 작은 눈, 짧은 팔다리
  • 출생: 강가에서 사용자가 주워 와 체온으로 깨어남
  • 핵심 이미지: 주머니 속 돌처럼 체온을 나누며 조용히 곁을 지키는 존재
  • 고유 특성: 경청하는 돌, 일관성, 느린 시간, 작은 감각, 온기에 민감

10. 날씨 인텔리전스

10.1 설계 원칙

  1. Time-Aware: 시간대별 Attention Window
  2. Threshold-Driven: 기준치 미만은 표시하지 않음
  3. Priority-Ranked: 최대 3개만 표시
  4. Rule-First, AI-Second: 핵심 경고는 로컬 규칙으로 즉시 → AI는 보충 코멘트만

10.2 Attention Window

    • JSON Schema textarea 20행, 모노스페이스

RoutingTab (라우팅 설정, 704줄)

4개 카드, 각각 독립 Save 버튼:

  1. Provider Routing: Primary 드롭다운 + ⇄ Swap 버튼 + Secondary 드롭다운. "(env default)" / "(no fallback)" 옵션.
  2. Model Routing: Base Model(ModelSelect) + 언어별(ko/en/ja/zh) 모델 각각 설정. ModelSelect = 드롭다운(카탈로그 + "Custom..." 텍스트 입력 모드 전환).
  3. Context Flags: 2컬럼 그리드. 12개 KNOWN_FLAGS — boolean은 체크박스, number(rag_k)는 숫자 입력. 플래그: include_owner_name, include_pet_name, include_personality, include_affection, include_relationship_stage, include_mbti_axes, include_timeband, include_worldview, include_speech_preset, include_memory_summary, include_history, rag_enabled, rag_k
  4. Params JSON: textarea 8행, {"temperature":0.7, "max_tokens":300} 형식. JSON 검증.

Effective Config 카드 (하단, 읽기전용 회색): 현재 적용 중인 Provider/Secondary/Model/Last Updated 표시. "(env default)" = 서버 .env 값 사용.
