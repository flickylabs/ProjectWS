웹 UX 구조

짧게 플레이하고, 즉시 이해하고, 공유하기 쉬운 화면 구성이 필요하다.

페이지 구조

좌측

현재 캐릭터 정보 / 오늘의 목표 / 스트레스 / 시간 압박 / 보유 개입권 / 사건 로그 요약

우측

장면 텍스트 / 인물 초상 또는 아이콘 / 대화 카드 / 선택지 버튼 / 직후 독백 / 후속 결과

장면 UI 가이드

  • 한 화면에 너무 많은 정보를 쌓지 않는다. 장면당 핵심 선택 1회, 마무리 선택 1회 정도가 적당하다.
  • NPC의 감정은 숫자보다 표정, 말줄임, 손떨림, 속도감 있는 문장으로 전달한다.
  • 이전 스테이지의 "과거의 나"가 등장할 때는 별도 배지나 로그 태그로 표시하면 이해가 쉬워진다.

결과 화면

이번 런 리포트

누가 임계치까지 갔는지, 어디서 고리가 끊겼는지, 어떤 사건이 가장 크게 번졌는지 요약한다.

속사정 해금

이번에 플레이한 인물들의 숨겨진 맥락을 열어줘, 첫인상과 실제 체감의 차이를 복습하게 만든다.

원본 vs 평행 로그

과거 개변을 사용했다면, 어떤 세계선이 어떻게 달라졌는지 비교 카드로 보여준다.

다음 런 유도

이번에 NPC였던 인물 중 파장이 큰 2~3명을 추천하여 다음 런에서 플레이해 보게 한다.

✓
모바일 대응 원칙
사이드바는 모바일에서 슬라이드 메뉴로 접고, 장면 텍스트와 선택지는 한 손으로 누르기 쉬운 카드 형태로 정리한다.

AI 활용 전략

AI는 세계의 법칙을 정하는 역할이 아니라, 이미 정해진 법칙을 사람처럼 보이게 만드는 역할로 쓰는 편이 안전하고 완성도가 높다.

LLM을 쓰는 5가지 영역

1. 속말 잔상

경계/배려 선택 직후, 이전 캐릭터의 내면 독백을 LLM이 캐릭터 성향에 맞게 한 줄 생성. 스트레스 > 60일 때만 발동.

2. 기억 왜곡 / 목격자 버전

같은 공유 사건을 다른 캐릭터 시점으로 재구성할 때, 공개층 사실은 유지하면서 체감층의 기억을 캐릭터 성향에 맞게 왜곡.

3. 라디오 / 알림 / 사연

장면 전환 시 배경 라디오 멘트, 핸드폰 알림 텍스트, 커뮤니티 사연 글 등 분위기 텍스트를 현재 세계선 상태에 맞게 생성.

4. 런 종료 한 줄

결과 화면에서 "이번 런을 한 줄로 요약하면?" 형태의 서술형 피드백. 감정 DNA 데이터를 입력으로 받아 생성.

5. 숨겨진 연결선

캐릭터 간 이면 관계가 해금될 때, 그 관계를 설명하는 짧은 에피소드 텍스트를 양쪽 캐릭터 플레이 데이터를 바탕으로 생성.

LLM을 쓰지 않는 5가지 영역

영역이유대안
선택지 구조분출/경계/배려 3축은 게임 밸런스의 뼈대. LLM이 만들면 밸런스 붕괴.룰 기반으로 선택지 구조 고정, 텍스트 표현만 AI 보조
수치 판정스트레스, 불씨, 잠금치 계산은 deterministic해야 함.JSON 상태 머신으로 관리
다음 캐릭터 선정스테이지 순서는 가중치 랜덤이지만 공정해야 함.가중치 알고리즘으로 고정
아이템 효과아이템 발동 결과가 예측 불가능하면 전략성 상실.효과 테이블로 고정
공개층 로그같은 사건의 사실 기록은 시점에 관계없이 동일해야 함.이벤트 로그 DB에 고정 기록
⚠
핵심 원칙
AI에게 선택지 구조나 승패 판정까지 맡기면 밸런싱이 무너진다. 이 게임은 시스템이 뼈대, AI가 표정과 살이어야 한다.

수익화 설계

무료로 핵심 재미를 충분히 경험하게 하되, 더 깊이 파고들고 싶은 순간에 자연스럽게 유료 전환이 발생하는 구조.

무료 / 유료 경계선

구분내용제한
무료 플레이매일 런 2회, 기본 캐릭터 6명, 기본 아이템 3종핵심 루프 체험에 충분
광고 보상추가 런 1회 / 아이템 획득 / 상세 리포트 열람보상형 광고 시청 후 지급
IAP — 소모품개입권 패키지소진 시 재구매
IAP — 콘텐츠캐릭터 팩 (4명 묶음), 프리미엄 아이템1회 구매, 영구 해금
IAP — 구독무제한 런 패스 (월정액) + 광고 제거 + 일일 프리미엄 아이템 1개월간/연간 구독

결제와 게임 메시지의 균형

  • Pay-to-Win 방지: 유료 아이템은 "다른 시점을 더 볼 수 있는" 도구이지, "더 좋은 결과를 만드는" 도구가 아니다.
  • 공명 순간은 무료: 게임의 핵심 경험은 무료 범위에서 반드시 체험 가능해야 한다.
  • 광고는 개입권: 광고를 "추가 기회를 얻는 행위"로 프레이밍.
  • 구독은 편의: 무제한 런 + 광고 제거.

MVP 범위

최초 버전은 "복잡한 구조를 실제로 재미있게 느끼는가"를 검증하는 데 집중해야 한다.

추천 MVP 스코프

항목정식 목표MVP 권장
등장인물20명12명
플레이 인물 수10명6명
공유 사건 템플릿10종 이상5~6종
아이템20종8~10종
배경 시간대여러 반나절 패턴하나의 반나절만
세계선 개변평행우주 시스템 완비현재 되감기 + 평행 메모 정도만

MVP에서 꼭 검증해야 할 것

  • 같은 사건을 다른 인물로 다시 겪을 때 실제로 신선하고 아픈가
  • 풀 핑퐁 1회 + 후속층 플레이 구조가 지루하지 않은가
  • 배려가 손해 같지만 의미 있게 느껴지는가
  • 광고/아이템이 메시지를 해치지 않고 오히려 전략성을 높이는가
  • 공명 순간이 실제로 전율을 주는가
  • 한 런 후 재도전 의사가 발생하는가
💡
MVP 성공 기준
플레이어가 한 런 뒤에 "이번에는 다른 인물로 저 사건을 다시 보고 싶다"라고 느끼면 구조는 성공이다.
