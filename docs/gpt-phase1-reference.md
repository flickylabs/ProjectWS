# Phase 1 대사 생성 가이드 (GPT용)

## 개요
갈등 해결 시뮬레이션 게임 "솔로몬"의 Phase 1(초기 진술) 대사를 생성합니다.
사건별로 **8~10개**의 대사를 JSON 배열로 출력합니다.

---

## 입력 정보 (사건별로 제공)

```json
{
  "caseId": "case-spouse-01",
  "relationshipType": "spouse",
  "partyA": {
    "name": "한지석", "age": 39, "occupation": "물류센터 야간 운영팀장",
    "archetype": "avoidant",
    "speechStyle": "곤란한 질문에는 숫자부터 꺼내며 감정을 뒤로 미룬다...",
    "fear": "아내에게 무능하다고 보일까 봐 두렵다",
    "verbalTells": [
      {"trigger": "lying", "pattern": "불편한 질문이 올수록 숫자가 정밀해진다..."},
      {"trigger": "cornered", "pattern": "의도를 추궁당하면 답하는 대신 되묻는다..."}
    ]
  },
  "partyB": { ... },
  "context": { "description": "추석이 일주일 앞으로..." },
  "disputes": [
    {"id": "d-1", "name": "지석의 비밀 송금 280만원"},
    {"id": "d-2", "name": "세린의 새벽 휴대폰 열람"},
    ...
  ]
}
```

---

## 대사 규칙

### 1. 대화 흐름 (8~10개)
```
① system: 재판 시작 선언 + 배경 요약 한 줄 (1개)
② a: 재판관에게 자기 입장 진술 — 존댓말 (1~2개)
③ b: 참다 못해 끼어들어 반박 (1개)
④ a↔b: 번갈아 서로 따지며 감정 격화 (4~5개)
⑤ system: "진정하세요" / "정리해서 말씀하세요" 등 개입 (0~1개)
⑥ a 또는 b: 자기 입장 정리 마무리 (1개)
```

### 2. 말투 규칙 (매우 중요)

**원칙: 한 대사 = 한 대상. 재판관에게 하는 말과 상대에게 하는 말을 한 문장 안에 섞지 마세요.**

**관계별 기본 규칙:**
| 관계 유형 | 상대에게 | 재판관에게 |
|-----------|---------|-----------|
| spouse (부부) | 반말 + "자기"로 호칭 | 존댓말 |
| family (가족) | 반말 + 관계 호칭 (형/동생/엄마 등) | 존댓말 |
| friend (친구) | 반말 + 이름 호칭 | 존댓말 |
| neighbor (이웃) | 존댓말 (~씨). 격해지면 반말 가능 | 존댓말 |
| boss_employee (직장) | 존댓말. 격해지면 반말 가능 | 존댓말 |
| partnership (동업) | 존댓말. 격해지면 반말 가능 | 존댓말 |
| tenant_landlord (임대) | 존댓말. 격해지면 반말 가능 | 존댓말 |

**부부 호칭:**
- A ↔ B: 서로 **"자기"** (고정, 다른 호칭 사용 금지)

**금지 표현:**
- "~냐?" 금지 → "~야?" 사용
- "~했느냐?" 금지 → "~한 거야?" 사용
- 한 대사 안에서 반말/존댓말 혼합 금지

**나쁜 예:**
```
"자기, 그건 말씀드리면 간병비였습니다."
→ 상대 호칭(자기)인데 존댓말(~습니다) 혼합
```

**좋은 예:**
```
"자기, 다시 말하지만 오해야. 그 송금은 정말 급해서 그랬어."
"재판관님, 그 돈은 간병비였습니다. 제 아내가 오해하고 있습니다."
```

### 3. 이름 플레이스홀더 (매우 중요)

대사에 캐릭터 이름을 직접 쓰지 마세요. 아래 플레이스홀더를 사용합니다:

| 문법 | 설명 | 예시 (A=한지석, B=오세린) |
|------|------|------------------------|
| `{A}` | A 이름 | 한지석 |
| `{B}` | B 이름 | 오세린 |
| `{A:이}` | A + 이/가 | 한지석이 |
| `{A:은}` | A + 은/는 | 한지석은 |
| `{A:을}` | A + 을/를 | 한지석을 |
| `{A:과}` | A + 과/와 | 한지석과 |
| `{A:로}` | A + 으로/로 | 한지석으로 |
| `{B:이}` | B + 이/가 | 오세린이 |
| `{B:은}` | B + 은/는 | 오세린은 |

- 시스템은 런타임에 실제 이름 + 올바른 조사로 자동 치환합니다.
- 재판관에게 말할 때도 상대 이름은 플레이스홀더로: "재판관님, {B:이} 오해하고 있습니다."
- "자기" 같은 호칭은 플레이스홀더가 아니라 그대로 씁니다.

### 4. 각 대사 형식
- 2~3문장으로 완결 (중간에 끊지 마세요)
- text 끝에 **(행동 묘사)**를 괄호로 포함
- behaviorHint 필드에 행동 묘사만 따로 기입
- relatedDisputes에 관련 쟁점 ID 배열 기입

### 4. 캐릭터 반영

**archetype별 행동 패턴:**
| archetype | 특징 | 대사 방향 |
|-----------|------|-----------|
| avoidant | 회피, 돌려 말하기, 화제 전환 | 직답을 피하고, "그건 순서가 있다"며 미룸 |
| confrontational | 정면 돌파, 공격적 | 직접 따지고, 증거를 들이밀며 추궁 |
| people_pleaser | 눈치, 양쪽 맞추려 함 | "제가 잘못한 건 맞지만..." 식으로 시작 |
| narcissistic | 자기 중심, 자기 정당화 | "내가 아니었으면 어쩔 뻔했어" 식 |
| passive_aggressive | 겉은 순응, 속은 공격 | "네, 제가 다 잘못했죠" (비꼬는 톤) |
| emotional | 감정 폭발, 눈물, 호소 | 감정이 앞서서 논리가 흐트러짐 |

**반영 요소:**
- speechStyle의 특징이 대사에 자연스럽게 드러나야 함
- verbalTells의 pattern을 1~2번 자연스럽게 포함
- fear에 연결되는 감정선이 초반엔 억눌려 있다가 후반에 드러남
- 초반엔 절제, 후반으로 갈수록 감정 노출

### 5. 쟁점 분배
- 모든 쟁점이 최소 1번은 언급되어야 함
- 핵심 쟁점(d-1, d-2)은 2~3번 반복 가능
- 후반 쟁점(d-4, d-5)은 1번이면 충분
- system 대사의 relatedDisputes는 빈 배열 []

---

## 출력 형식

**JSON 배열만 출력하세요. 다른 텍스트(설명, 코멘트) 없이.**

```json
[
  {
    "speaker": "system",
    "text": "재판을 시작하겠습니다. (배경 요약)",
    "relatedDisputes": [],
    "behaviorHint": null
  },
  {
    "speaker": "a",
    "text": "재판관님, 제 아내가 오해하고 있습니다. 대사 내용 (행동 묘사)",
    "relatedDisputes": ["d-1"],
    "behaviorHint": "행동 묘사"
  }
]
```

**⚠️ 필수 체크:**
- speaker는 반드시 소문자 **"a"**, **"b"**, **"system"** 중 하나. 캐릭터 이름을 넣지 마세요.
- 캐릭터 대사에 이름을 직접 쓰지 마세요. 관계 호칭을 쓰세요 ("제 남편이", "제 아내가" 등).
- system 대사와 behaviorHint에서만 **`{A}`, `{B}`, `{A:이}`** 등 플레이스홀더를 사용하세요.
- 쟁점 ID는 입력 데이터의 disputes 배열에 있는 ID만 사용하세요.
- 총 대사 수 8~10개. 너무 길면 게임이 느려집니다.

### 이 예시의 호칭/플레이스홀더 사용 패턴:

**캐릭터 대사 (speaker "a", "b") 안에서:**
- 상대에게 직접 말할 때: **"자기"** (부부), 관계 호칭 (가족/친구 등)
- 재판관에게 상대를 언급할 때: **관계 호칭** 사용 (이름 아님)

| 관계 | A가 B를 (재판관에게) | B가 A를 (재판관에게) |
|------|-------------------|-------------------|
| spouse | "제 아내가", "제 아내의" | "제 남편이", "제 남편의" |
| family | "제 형이", "제 동생이" 등 | 관계에 맞게 |
| friend | "제 친구가", "걔가" | 관계에 맞게 |
| neighbor | "옆집 분이", "저 사람이" | 관계에 맞게 |
| workplace | "제 상사가", "제 부하가" | 관계에 맞게 |
| partnership | "제 동업자가" | 관계에 맞게 |
| tenant | "세입자가", "집주인이" | 관계에 맞게 |

예시:
- ✕ "재판관님, 한지석이 공동통장에서..." (이름 직접 사용 — 부자연스러움)
- ○ "재판관님, 제 남편이 공동통장에서..." (관계 호칭 — 자연스러움)

**system 대사 · behaviorHint 안에서:**
- 플레이스홀더 `{A}`, `{B}`, `{A:이}` 등 사용 (시스템이 이름으로 치환)

### 이 예시가 좋은 이유:
1. **흐름**: system→a진술→b반박→a↔b 격화→system 개입→b 정리 — 기승전결
2. **호칭**: 서로 "자기"로 통일, 재판관에게는 존댓말
3. **archetype 반영**: A(회피형)가 화제를 전환하고(d-3→d-2), B(대면형)가 끌어당김
4. **쟁점 커버**: d-1(3회), d-2(2회), d-3(4회) — 핵심 쟁점 반복, d-4/d-5는 Phase 3에서 발굴
5. **감정 곡선**: 차분→끼어들기→따지기→소리 높이기→정리 — 자연스러운 에스컬레이션
