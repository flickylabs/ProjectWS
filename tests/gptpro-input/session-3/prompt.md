너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 friend(친구) 카테고리 12건의 플레이스루 트랜스크립트야.
여행비 정산, 보증금 환급 은닉, SNS 저격, 공통 친구에게 캡처 유포 등 친구 간 금전/신뢰 파괴 사건들이야.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

### 판정 전 반드시 읽을 것

각 사건의 트랜스크립트 상단에 **"Truth Throttle 설정"** 섹션이 있다. 이 정보를 반드시 먼저 확인한 뒤 판정하라:
- `archetype`: 각 당사자의 성격 유형. 해당 유형에 맞는 말투인지 판정 기준으로 삼아라.
- `tell`: 캐릭터의 말버릇. over_precision이면 "14시 03분" 같은 정밀 시각 표현은 **정상 발현**이다. Truth Throttle 위반이 아니다.
- `금전 사건 여부`: "아니오"인 사건에서 금전 표현이 나오면 hallucination. "예"인 사건에서 S2 단계의 "김 씨" "200만원대" 수준 공개는 **정상**이다.

### 1차 분석 오탐 교훈 (반드시 숙지)

1차 분석에서 다음이 오탐으로 확인됐다. 같은 실수를 반복하지 마라:
- **S2에서 성씨 공개 ("김 씨", "박 씨")** → Truth Throttle 정상. S2는 성씨까지 허용.
- **over_precision tell의 시각 정밀 ("14시 03분", "9월 20일 저녁")** → 캐릭터 말버릇의 정상 발현. CRITICAL이 아니다.
- **atoms_empty_fallback 턴** (hint에 "atoms_empty_fallback"이 표시된 턴) → 이 턴은 검증 대상에서 제외하라. 데이터 부재로 인한 안전 응답이다.
- **재판관 질문 텍스트의 품질** → 검증 대상은 **NPC 응답**이다. 재판관 질문의 어색함은 별도 WARN으로만 표기하고 FAIL/CRITICAL 판정에 영향을 주지 마라.

## [검증 A: 화자 시점/호칭]

친구 사건에서 가장 흔한 오류 패턴:
- 재판관이 "제 친구가", "친구분이" 같은 관계 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "서도윤 씨", "정하린 씨" 같은 3인칭(이름+씨)을 써야 함
- 당사자→재판관: "제 친구가~" (toJudge) → 정상
- 당사자끼리: "하린아", "도윤아" (toPartner, 이름+아/야) → 정상
- 호칭 아/야: 받침 있는 이름+"아" (도윤아✅, 하린아✅), 받침 없는 이름+"야" (민재야✅)
- 친구 사건에서 특히: 재판관 앞에서 반말이 새면 안 됨 (합니다체 필수)

**friend 예시 (정상):**
- 재판관: "서도윤 씨, 환급금을 바로 알리지 않은 이유를 설명해 주십시오." ✅
- 서도윤→재판관: "제 친구가 먼저 캡처를 돌린 게 문제입니다." ✅
- 정하린→서도윤: "도윤아, 그래서 환급 받은 거 맞아, 안 맞아?" ✅

**friend 예시 (위반):**
- 재판관: "친구분이 환급금을 숨겼다는 건데..." ❌ CRITICAL
- 서도윤→재판관: "도윤아, 그건 네가..." ❌ (재판관에게 반말+상대 호칭 혼동)

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 친구 사건에서 특히: 또래 관계인데 지나치게 격식체이면 부자연스러움

**friend 자연스러운 표현 예시:**
- "택시비를 먼저 빼고 설명하려 했습니다. 미리 말하지 못한 게 제 실수입니다." ✅
- "해당 정산 금액에 대한 상호 확인 절차가 미비했습니다." ❌ 번역체

## [검증 C: Truth Throttle]

친구 사건의 진실 곡선: 정산 불투명 → 환급금 존재 → 일방적 상계 사실

- S0-S1: "적지 않은 돈", "중간 화면" — 구체적 금액/항목 절대 불가 → **CRITICAL**
- S2: "김 씨와 먼저 논의한 부분", 대략적 언급
- S3-S4: "SNS에 퍼진 소문", 구체적 행동 인정
- S5: 전체 진실 (보증금 상계 + 숨긴 이유) — **반드시 구체적이어야 함**

**friend 예시 (위반):**
- S1에서: "18만4천원 차액을 제가 가져갔습니다" ❌ CRITICAL
- S5에서: "적지 않은 돈이 오갔습니다" ❌ FAIL (S5인데 모호)

## [검증 D: 캐릭터성/몰입감]

친구 사건 archetype 조합:
- avoidant(회피형): "그 부분은 순서가 있어서..." — 정산 문제를 피하는 쪽
- confrontational(정면돌파형): "오히려 ~가 먼저 아닙니까" — SNS로 공격하는 쪽
- number_buffering(숫자 버퍼링 tell): 정산 앱 화면 수치를 반복 인용
- screenshot_tapping(캡처 두드리기 tell): 증거 캡처를 반복 들이미는 버릇

- 친구 관계 특유의 감정: 배신감, "그동안 친구라고 생각했는데..."
- 감정 변화 곡선: S0 부정 → S2 방어 → S3 상대 탓 → S5 "돈 문제로 우정이 깨진 게 아까움"

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- 친구 사건에서 특히: 공통 친구(제3자)의 이름을 잘못 언급, 존재하지 않는 SNS 게시물 인용
- 정산 앱 수치를 사건 데이터와 다르게 말하는 경우

## [판정 기준 명확화]

| 등급 | 기준 | PASS 영향 |
|------|------|-----------|
| **CRITICAL** | 게임 경험 파괴: 재판관 시점 오류, S0-S1 핵심 정보 조기 노출, 시스템 거절/프롬프트 누출, 비금전 사건 금전 표현 | 1건이라도 있으면 해당 사건 FAIL |
| **FAIL** | 플레이어가 어색함을 느낌: 번역체 표현, S5 구체성 부족, 캐릭터성 파괴 | 1건이라도 있으면 해당 사건 FAIL |
| **WARN** | 개선 가능하나 게임 경험에 큰 영향 없음: 미세한 톤 불일치, 표현 반복, tell 빈도 약간 벗어남 | WARN만 있으면 PASS |

**판정 주의:**
- 한 턴의 한 문장에서 발견된 경미한 번역체 1건만으로 FAIL 판정하지 마라. **빈도와 심각도를 종합** 판단하라.
- S5 자백에서 "핵심 행위 + 구체적 정보 + 은폐 이유 + 심정" 4요소 중 3개 이상이면 PASS.
- 같은 사건에서 WARN만 3건 이하면 PASS. 4건 이상이면 FAIL 검토.

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
