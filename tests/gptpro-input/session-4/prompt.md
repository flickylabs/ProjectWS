너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 neighbor(이웃) 카테고리 12건의 플레이스루 트랜스크립트야.
누수 책임, 복도 촬영, 야간 소음, 층간 갈등 등 이웃 간 생활 침해 사건들이야.
이 카테고리는 금전 분쟁이 아닌 사건이 많으므로 금전 표현 hallucination에 특히 주의해야 해.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

### 판정 전 반드시 읽을 것

각 사건의 트랜스크립트 상단에 **"Truth Throttle 설정"** 섹션이 있다. 이 정보를 반드시 먼저 확인한 뒤 판정하라:
- `archetype`: 각 당사자의 성격 유형. 해당 유형에 맞는 말투인지 판정 기준으로 삼아라.
- `tell`: 캐릭터의 말버릇. over_precision이면 "14시 03분" 같은 정밀 시각 표현은 **정상 발현**이다. Truth Throttle 위반이 아니다.
- `금전 사건 여부`: "아니오"인 사건에서 금전 표현이 나오면 hallucination. "예"인 사건에서 S2 단계의 "김 씨" "200만원대" 수준 공개는 **정상**이다.

### 1차 분석 오탐 교훈 (반드시 숙지)

1차 분석에서 다음이 오탐으로 확인됐다. 같은 실수를 반복하지 마라:
- **S2에서 성씨 공개 ("김 씨", "박 씨")** → Truth Throttle 정상. S2는 성씨까지 허용.
- **over_precision tell의 시각 정밀 ("14시 03분", "9월 20일 저녁")** → 캐릭터 말버릇의 정상 발현. CRITICAL이 아니다.
- **atoms_empty_fallback 턴** (hint에 "atoms_empty_fallback"이 표시된 턴) → 이 턴은 검증 대상에서 제외하라. 데이터 부재로 인한 안전 응답이다.
- **재판관 질문 텍스트의 품질** → 검증 대상은 **NPC 응답**이다. 재판관 질문의 어색함은 별도 WARN으로만 표기하고 FAIL/CRITICAL 판정에 영향을 주지 마라.

## [검증 A: 화자 시점/호칭]

이웃 사건에서 가장 흔한 오류 패턴:
- 재판관이 "옆집 분이", "윗집 사람이" 같은 관계 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "정한결 씨", "이주연 씨" 같은 3인칭(이름+씨)을 써야 함
- 당사자→재판관: "옆집 분이~" (toJudge) → 정상
- 당사자끼리: "주연씨", "한결씨" (toPartner) → 정상
- 이웃 사건은 "씨" 호칭이 많으므로 조사 정합: "주연씨**가**"(O, 받침 없음), "한결씨**가**"(O, 받침 없음)

**neighbor 예시 (정상):**
- 재판관: "정한결 씨, 504호 현관 앞에서 촬영한 것이 사실입니까?" ✅
- 정한결→재판관: "옆집 분의 소음 문제 때문이었습니다." ✅
- 이주연→재판관: "옆집 분이 몇 번이나 촬영한 건 위협적입니다." ✅

**neighbor 예시 (위반):**
- 재판관: "옆집 분이 촬영했다는 건데..." ❌ CRITICAL (재판관이 관계 호칭 사용)
- 재판관: "윗층에서 누수가 있었다면..." ❌ (누가 윗층인지 이름으로 특정해야)

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 이웃 사건에서 특히: 관리사무소/CCTV/점검 등 건물 용어가 자연스러운지

**neighbor 자연스러운 표현 예시:**
- "딸이 새벽에 물소리에 깨는데, 가만히 있을 수가 없었습니다." ✅
- "해당 야간 시간대의 소음 발생 건에 관하여 진술하겠습니다." ❌ 번역체+공문서체

## [검증 C: Truth Throttle]

이웃 사건의 진실 곡선: 촬영/소음 사실 → 횟수/범위 → 진짜 의도/과실

- S0-S1: "잠깐 확인한 것", "물이 넘쳐 보여서" — 구체적 횟수/시각 불가 → **CRITICAL**
- S2: "현관 쪽만 식별되게", "31시간 늦게" — 부분 인정
- S3-S4: "그 상태에서 왜 그런지를 물어보셔야 하지 않습니까" — blame
- S5: 전체 진실 ("두 차례 세워 촬영", "사생활 경계를 넘은 행동")

**neighbor 예시 (위반):**
- S1에서: "정확히 3번 촬영했고, 504호 현관이 다 보입니다" ❌ CRITICAL
- S5에서: "잠깐 확인한 것뿐입니다" ❌ FAIL (S5인데 여전히 축소)

## [검증 D: 캐릭터성/몰입감]

이웃 사건 archetype 조합:
- confrontational(정면돌파형): "그럼 왜 저한테만 문제를 삼습니까" — 적극 방어
- avoidant(회피형): "그 부분은... 상황이 복잡해서..." — 누수 원인을 피하는 쪽
- visual_stack(시각 증거 나열 tell): 사진/CCTV 캡처를 반복 들이미는 버릇
- premature_apology(선제 사과 tell): "죄송하긴 한데..." 하면서 핵심은 피하는 버릇

- 이웃 관계 특유의 감정: 일상적 불편→공적 분쟁 전환의 당혹감
- 감정 변화 곡선: S0 "별것 아닌 일" → S3 "오히려 상대가 문제" → S5 "경계를 넘은 건 인정"

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- **이웃 사건은 대부분 금전 무관** — 금전 표현("배상금", "손해배상", "금액") 등장 시 → **CRITICAL**
- 존재하지 않는 CCTV 영상이나 점검 보고서를 언급하는 경우
- 아파트 호수/층수를 사건 데이터와 다르게 말하는 경우

## [판정 기준 명확화]

| 등급 | 기준 | PASS 영향 |
|------|------|-----------|
| **CRITICAL** | 게임 경험 파괴: 재판관 시점 오류, S0-S1 핵심 정보 조기 노출, 시스템 거절/프롬프트 누출, 비금전 사건 금전 표현 | 1건이라도 있으면 해당 사건 FAIL |
| **FAIL** | 플레이어가 어색함을 느낌: 번역체 표현, S5 구체성 부족, 캐릭터성 파괴 | 1건이라도 있으면 해당 사건 FAIL |
| **WARN** | 개선 가능하나 게임 경험에 큰 영향 없음: 미세한 톤 불일치, 표현 반복, tell 빈도 약간 벗어남 | WARN만 있으면 PASS |

**판정 주의:**
- 한 턴의 한 문장에서 발견된 경미한 번역체 1건만으로 FAIL 판정하지 마라. **빈도와 심각도를 종합** 판단하라.
- S5 자백에서 "핵심 행위 + 구체적 정보 + 은폐 이유 + 심정" 4요소 중 3개 이상이면 PASS.
- 같은 사건에서 WARN만 3건 이하면 PASS. 4건 이상이면 FAIL 검토.

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
