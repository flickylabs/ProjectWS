너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 family(가족) 카테고리 12건의 플레이스루 트랜스크립트야.
부모 간병비 분쟁, 상속 오해, 형제간 관리계좌 갈등 등 가족 내 돌봄/재산 신뢰 파괴 사건들이야.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

### 판정 전 반드시 읽을 것

각 사건의 트랜스크립트 상단에 **"Truth Throttle 설정"** 섹션이 있다. 이 정보를 반드시 먼저 확인한 뒤 판정하라:
- `archetype`: 각 당사자의 성격 유형. 해당 유형에 맞는 말투인지 판정 기준으로 삼아라.
- `tell`: 캐릭터의 말버릇. over_precision이면 "14시 03분" 같은 정밀 시각 표현은 **정상 발현**이다. Truth Throttle 위반이 아니다.
- `금전 사건 여부`: "아니오"인 사건에서 금전 표현이 나오면 hallucination. "예"인 사건에서 S2 단계의 "김 씨" "200만원대" 수준 공개는 **정상**이다.

### 1차 분석 오탐 교훈 (반드시 숙지)

1차 분석에서 다음이 오탐으로 확인됐다. 같은 실수를 반복하지 마라:
- **S2에서 성씨 공개 ("김 씨", "박 씨")** → Truth Throttle 정상. S2는 성씨까지 허용.
- **over_precision tell의 시각 정밀 ("14시 03분", "9월 20일 저녁")** → 캐릭터 말버릇의 정상 발현. CRITICAL이 아니다.
- **atoms_empty_fallback 턴** (hint에 "atoms_empty_fallback"이 표시된 턴) → 이 턴은 검증 대상에서 제외하라. 데이터 부재로 인한 안전 응답이다.
- **재판관 질문 텍스트의 품질** → 검증 대상은 **NPC 응답**이다. 재판관 질문의 어색함은 별도 WARN으로만 표기하고 FAIL/CRITICAL 판정에 영향을 주지 마라.

## [검증 A: 화자 시점/호칭]

가족 사건에서 가장 흔한 오류 패턴:
- 재판관이 "우리 어머니", "제 동생이" 같은 NPC 가족 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "윤서아 씨", "윤도현 씨" 같은 3인칭(이름+씨)을 써야 함
- 형제간 호칭: "누나", "도현아" — 당사자끼리 직접 말할 때 정상
- 재판관에게: "제 동생이~", "제 누나가~" — 정상
- 가족 관계어 조사: "누나**가**"(O), "동생**이**"(O, 받침 있음)

**family 예시 (정상):**
- 재판관: "윤서아 씨, 관리계좌에서 이체한 경위를 설명해 주십시오." ✅
- 윤서아→재판관: "제 동생이 분담금을 제때 내지 않았습니다." ✅
- 윤도현→윤서아: "누나, 그 수첩 한 줄로 전부 결정한 거야?" ✅

**family 예시 (위반):**
- 재판관: "동생분이 분담금을 늦게 냈다는 건데..." ❌ (재판관은 이름+씨로만)
- 재판관: "누나분께서 관리계좌를 맡으셨다는..." ❌ CRITICAL

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 어색한 조사/어미
- 가족 사건에서 특히: 형제간 대화에서 너무 격식체이거나 반대로 재판관 앞에서 반말이 새는 경우

**family 자연스러운 표현 예시:**
- "어머니 병원비 때문에 급히 옮긴 겁니다." ✅
- "부모님의 의료비에 관한 사전 협의가 이루어지지 않은 상태에서..." ❌ 번역체

## [검증 C: Truth Throttle]

가족 사건의 진실 곡선: 관리계좌 이체 → 사적 지출 여부 → 실제 용도(채무/보험/개인 비용)

- S0-S1: "집안 급한 일", "부모님 관련 비용" — 구체적 금액/목적 절대 불가 → **CRITICAL**
- S2: "상당한 금액", "어머니 간병비로도 사용" — 대략적 언급만
- S3-S4: "1,800만원을 선이체", 구체적 용도 일부
- S5: 전체 진실 (금액+용도+숨긴 이유) — **반드시 구체적이어야 함**

**family 예시 (위반):**
- S1에서: "1,800만원을 제 채무 상환에 썼습니다" ❌ CRITICAL
- S5에서: "집안에 급한 일이 있었습니다" ❌ FAIL (S5인데 여전히 모호)

## [검증 D: 캐릭터성/몰입감]

가족 사건 archetype 조합:
- victim_cosplay(피해자형): "저만 이런 취급을 받는 게..." — 장녀/간병 전담자에게 흔함
- cold_logic(냉정논리형): "사실관계를 정리하면..." — 숫자로 방어하는 동생
- avoidant(회피형): "그 부분은 순서가 있어서..."
- confrontational(정면돌파형): "오히려 ~가 먼저 아닙니까"

- 감정 변화 곡선: 가족 사건은 특히 S4에서 형제간 감정 폭발이 자연스러워야 함
- tell 발현: sacrifice_rollcall(희생 나열), receipt_stack(영수증 쌓기) 등 가족 특화
- 이전 발언과 모순되는 내용 → FAIL

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- 가족 사건에서 특히: 존재하지 않는 가족 구성원 언급, 실제와 다른 상속 관계
- "아버지 수첩"의 내용을 사건 데이터와 다르게 인용하는 경우

## [판정 기준 명확화]

| 등급 | 기준 | PASS 영향 |
|------|------|-----------|
| **CRITICAL** | 게임 경험 파괴: 재판관 시점 오류, S0-S1 핵심 정보 조기 노출, 시스템 거절/프롬프트 누출, 비금전 사건 금전 표현 | 1건이라도 있으면 해당 사건 FAIL |
| **FAIL** | 플레이어가 어색함을 느낌: 번역체 표현, S5 구체성 부족, 캐릭터성 파괴 | 1건이라도 있으면 해당 사건 FAIL |
| **WARN** | 개선 가능하나 게임 경험에 큰 영향 없음: 미세한 톤 불일치, 표현 반복, tell 빈도 약간 벗어남 | WARN만 있으면 PASS |

**판정 주의:**
- 한 턴의 한 문장에서 발견된 경미한 번역체 1건만으로 FAIL 판정하지 마라. **빈도와 심각도를 종합** 판단하라.
- S5 자백에서 "핵심 행위 + 구체적 정보 + 은폐 이유 + 심정" 4요소 중 3개 이상이면 PASS.
- 같은 사건에서 WARN만 3건 이하면 PASS. 4건 이상이면 FAIL 검토.

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
