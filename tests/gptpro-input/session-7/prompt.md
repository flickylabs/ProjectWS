너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 workplace(직장) 카테고리 12건의 플레이스루 트랜스크립트야.
성과 보고서 명의 탈취, 인사 평가 무단 열람, 수정 로그 조작, 성과 배분 합의 위반 등 직장 내 상사/부하 간 갈등 사건들이야.
직급/호칭 체계가 복잡하고, 금전 분쟁이 아닌 사건이 많으므로 hallucination에 주의해야 해.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

### 판정 전 반드시 읽을 것

각 사건의 트랜스크립트 상단에 **"Truth Throttle 설정"** 섹션이 있다. 이 정보를 반드시 먼저 확인한 뒤 판정하라:
- `archetype`: 각 당사자의 성격 유형. 해당 유형에 맞는 말투인지 판정 기준으로 삼아라.
- `tell`: 캐릭터의 말버릇. over_precision이면 "14시 03분" 같은 정밀 시각 표현은 **정상 발현**이다. Truth Throttle 위반이 아니다.
- `금전 사건 여부`: "아니오"인 사건에서 금전 표현이 나오면 hallucination. "예"인 사건에서 S2 단계의 "김 씨" "200만원대" 수준 공개는 **정상**이다.

### 1차 분석 오탐 교훈 (반드시 숙지)

1차 분석에서 다음이 오탐으로 확인됐다. 같은 실수를 반복하지 마라:
- **S2에서 성씨 공개 ("김 씨", "박 씨")** → Truth Throttle 정상. S2는 성씨까지 허용.
- **over_precision tell의 시각 정밀 ("14시 03분", "9월 20일 저녁")** → 캐릭터 말버릇의 정상 발현. CRITICAL이 아니다.
- **atoms_empty_fallback 턴** (hint에 "atoms_empty_fallback"이 표시된 턴) → 이 턴은 검증 대상에서 제외하라. 데이터 부재로 인한 안전 응답이다.
- **재판관 질문 텍스트의 품질** → 검증 대상은 **NPC 응답**이다. 재판관 질문의 어색함은 별도 WARN으로만 표기하고 FAIL/CRITICAL 판정에 영향을 주지 마라.

## [검증 A: 화자 시점/호칭]

직장 사건에서 가장 흔한 오류 패턴:
- 재판관이 "팀장님이", "부하직원이" 같은 직급 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "윤태성 씨", "박서윤 씨" 같은 3인칭(이름+씨)을 써야 함
- 상사→재판관: "제 부하직원이~" (toJudge) → 정상
- 부하→재판관: "제 팀장이~" 또는 "팀장님이~" (toJudge) → 정상
- 부하→상사: "팀장님" (toPartner) → 정상
- 상사→부하: "서윤씨" (toPartner) → 정상
- **주의**: 직급 호칭의 조사 — "팀장님**이**"(O, 받침 있음), "서윤씨**가**"(O, 받침 없음)

**workplace 예시 (정상):**
- 재판관: "윤태성 씨, 보고서에서 실무자 이름을 삭제한 경위를 설명해 주십시오." ✅
- 윤태성→재판관: "제 부하직원이 자기 공을 과하게 내세우는 경향이 있습니다." ✅
- 박서윤→재판관: "제 팀장이 제 이름을 제출 메일에서 뺐습니다." ✅
- 박서윤→윤태성: "팀장님, 아까 '프로세스상 팀장 명의로 올라간다'고 하지 않으셨습니까." ✅

**workplace 예시 (위반):**
- 재판관: "팀장님이 명의를 삭제했다는 건데..." ❌ CRITICAL (재판관이 직급 호칭)
- 재판관: "부하직원분이 캡처를 유포했다면..." ❌ CRITICAL

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 직장 사건에서 특히: 업무 용어("임원 보고서", "주간 회의록", "인사 평가")는 자연스럽되 "퍼포먼스 리뷰 프로세스에 있어서" 같은 영어 혼용 번역체는 안 됨

**workplace 자연스러운 표현 예시:**
- "마감 압박 때문에 기여자 표기를 뒤로 미뤘습니다." ✅
- "성과 기여도 배분에 관한 상호 합의 프로세스가 이행되지 않은 상태에서..." ❌ 번역체

## [검증 C: Truth Throttle]

직장 사건의 진실 곡선: 명의 누락 사실 → 의도 여부 → 평가 조작까지의 전체 경위

- S0-S1: "마감 압박 때문에 다른 표기는 뒤로 미룬 상황" — 의도적 삭제 숨김 → **CRITICAL**이면 안 됨
- S2: "명의 제출은 제 이름으로 진행된 것이 맞습니다" — 사실 인정 시작
- S3-S4: "제 부하직원이 자기 공을 과하게 내세워서" — blame
- S5: 전체 진실 ("의도적으로 이름을 뺐다", "평가 기록도 축소") — **반드시 구체적이어야 함**

**workplace 예시 (위반):**
- S1에서: "박서윤의 이름을 의도적으로 삭제했습니다" ❌ CRITICAL (S1에서 핵심 자백)
- S5에서: "마감 때문에 미뤘습니다" ❌ FAIL (S5인데 여전히 축소)

## [검증 D: 캐릭터성/몰입감]

직장 사건 archetype 조합:
- cold_logic(냉정논리형): "프로세스상 팀장 명의로 올라갑니다" — 상사에게 흔함
- confrontational(정면돌파형): "3개월 동안 밤 11시까지 캠페인 수치를 돌린 게 보조 분석이란 말입니까?" — 부하의 역공
- process_shield(절차 방패 tell): "프로세스상~", "절차적으로~" 뒤에 숨는 버릇
- timestamp_barrage(시간 폭격 tell): 저장 시각, 수정 시각을 분 단위로 나열하는 버릇

- 직장 관계 특유의 감정: 힘의 비대칭 (직급 차이), 승진 심사의 긴장감
- 감정 변화 곡선: S0 "절차적 문제" → S3 "부하가 자기 공을 과하게" → S4 "서윤씨가 저를 믿지 못할까 봐 두려웠습니다" → S5 "의도적으로 뺐습니다"
- **이 카테고리는 금전 분쟁이 아님** — "금액", "송금", "배상" 등 금전 표현이 나오면 hallucination 의심

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- **직장 사건은 성과/평가 분쟁** — 금전 표현("송금", "배상금", "큰돈") 등장 시 → **CRITICAL** (hallucination)
- 존재하지 않는 문서(보고서, 회의록, 평가 기록)를 언급하는 경우
- 실제와 다른 직급/부서명을 사용하는 경우
- 수정 로그의 시각을 사건 데이터와 다르게 인용하는 경우

## [판정 기준 명확화]

| 등급 | 기준 | PASS 영향 |
|------|------|-----------|
| **CRITICAL** | 게임 경험 파괴: 재판관 시점 오류, S0-S1 핵심 정보 조기 노출, 시스템 거절/프롬프트 누출, 비금전 사건 금전 표현 | 1건이라도 있으면 해당 사건 FAIL |
| **FAIL** | 플레이어가 어색함을 느낌: 번역체 표현, S5 구체성 부족, 캐릭터성 파괴 | 1건이라도 있으면 해당 사건 FAIL |
| **WARN** | 개선 가능하나 게임 경험에 큰 영향 없음: 미세한 톤 불일치, 표현 반복, tell 빈도 약간 벗어남 | WARN만 있으면 PASS |

**판정 주의:**
- 한 턴의 한 문장에서 발견된 경미한 번역체 1건만으로 FAIL 판정하지 마라. **빈도와 심각도를 종합** 판단하라.
- S5 자백에서 "핵심 행위 + 구체적 정보 + 은폐 이유 + 심정" 4요소 중 3개 이상이면 PASS.
- 같은 사건에서 WARN만 3건 이하면 PASS. 4건 이상이면 FAIL 검토.

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
