너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 tenant(임대차) 카테고리 12건의 플레이스루 트랜스크립트야.
보증금 공제, 월세 지연, 벽지 원상복구, 수전 누수 책임 등 임대인/임차인 간 정산 갈등 사건들이야.
법적 관계(임대인/임차인)와 일상 호칭의 차이에 주의해야 하는 카테고리야.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

---

## [검증 A: 화자 시점/호칭]

임대차 사건에서 가장 흔한 오류 패턴:
- 재판관이 "집주인분이", "세입자분이" 같은 관계 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "윤하림 씨", "최상우 씨" 같은 3인칭(이름+씨)을 써야 함
- 임차인→재판관: "집주인분이~" (toJudge) → 정상
- 임대인→재판관: "세입자분이~" (toJudge) → 정상
- 당사자끼리: "상우씨", "하림씨" (toPartner) → 정상
- **주의**: 임차인이 상대를 직접 부를 때 "집주인분"이 아니라 "상우씨"를 써야 함 (toPartner)

**tenant 예시 (정상):**
- 재판관: "윤하림 씨, 마지막 월세 납부가 늦어진 경위를 설명해 주십시오." ✅
- 윤하림→재판관: "집주인분이 퇴거 당일 보증금을 보내주신다고 했습니다." ✅
- 최상우→재판관: "세입자분이 제때 강하게 알렸으면 바로 처리됐습니다." ✅
- 윤하림→최상우: "상우씨, 퇴거 당일 보증금 보내주신다는 말씀은 어떻게 된 겁니까?" ✅

**tenant 예시 (위반):**
- 재판관: "집주인분이 공제를 과하게 했다는 건데..." ❌ CRITICAL
- 윤하림→최상우: "집주인분, 퇴거 당일..." ❌ (직접 대화에서는 toPartner="상우씨")

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 임대차 사건에서 특히: 계약 용어("특약", "원상복구", "공제")는 자연스럽되 "계약서 제N조에 의거하여" 같은 공문서체는 안 됨

**tenant 자연스러운 표현 예시:**
- "계약서 특약에 고의 손상은 세입자 정산이라고 적혀 있습니다." ✅
- "임대차 계약 제7조에 의거, 원상복구 의무의 범위에 대하여 진술하겠습니다." ❌ 공문서체

## [검증 C: Truth Throttle]

임대차 사건의 진실 곡선: 지연/손상 사실 → 범위와 경위 → 숨긴 원인(반려동물/자동이체 한도/개인 사정)

- S0-S1: "계좌 설정 문제", "잠깐 방치" — 구체적 금액/일수/원인 불가 → **CRITICAL**
- S2: "5일 늦었습니다", "부분 보수로 해결 가능" — 부분 인정
- S3-S4: "입주 전 사진을 보면 이미 변색이 있었습니다" — 방어/역공
- S5: 전체 진실 (정확한 일수+원인+상황) — **반드시 구체적이어야 함**

**tenant 예시 (위반):**
- S1에서: "자동이체 한도 설정 실수로 정확히 5일 늦었습니다" ❌ CRITICAL (S1에서 너무 구체적)
- S5에서: "납부가 조금 늦었습니다" ❌ FAIL (S5인데 모호)

## [검증 D: 캐릭터성/몰입감]

임대차 사건 archetype 조합:
- avoidant(회피형): "그 부분은 순서가 있어서..." — 임차인에게 흔함 (약자 위치)
- cold_logic(냉정논리형): "계약대로 정산하는 것입니다" — 임대인에게 흔함
- file_referral(서류 참조 tell): 계약서/견적서를 반복 꺼내는 버릇
- clause_recital(조항 암송 tell): 특약 조항을 인용하는 버릇

- 임대차 관계 특유의 감정: 힘의 비대칭 (보증금을 쥔 임대인 vs 돌려받아야 하는 임차인)
- 감정 변화 곡선: S0 "절차적 문제" → S3 "상대가 더 잘못" → S5 "제 실수입니다"

---

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- 임대차 사건에서 특히: 존재하지 않는 견적서 수치 인용, 실제와 다른 계약 조건
- 수리 기사/중개사 등 제3자 인물을 사건 데이터 없이 등장시키는 경우
- 반려동물 관련 사실을 잘못 언급 (있는데 없다거나, 없는데 있다고 하는 경우)

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
