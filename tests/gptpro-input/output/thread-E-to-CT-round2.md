## 남은 병목 3가지

### 병목 1: tenant v2-atoms S0-S1 — 데이터 문제 (CRITICAL 20건)

tenant 12건 중 11건이 S1에서 핵심 원인/경위를 조기 노출하여 FAIL.
이건 코드가 아니라 **v2-atoms의 S0-S1 factText/slots에 구체적 값이 남아있는 데이터 문제**입니다.

예시:
- "은행/계좌 설정 문제" → S1에서 원인을 바로 공개
- "이미 합의됐다" → d-1 핵심을 S1에서 공개
- "장비 가동 시점과 검침 증가 시점 일치" → 핵심 연결고리 조기 노출

**필요 작업**: tenant 12건 v2-atoms S0-S1 전수 중립화 (CT 또는 Thread B)

### 병목 2: 시스템 거절 잔존 (~10건)

LLM이 atoms를 받았는데도 "죄송합니다" 또는 프롬프트 누출 응답을 생성하는 케이스.
2가지 원인:
- **monetaryGuard를 LLM이 과잉 적용**: "금전 표현 금지"를 사건 내 발화에까지 적용하여 거부
- **atoms는 있지만 내용이 빈약**: claimAtoms 1개에 factText만 있고 slots가 없는 케이스

**Thread E가 수정 가능**: monetaryGuard 조건부 완화 + fallback 확대

### 병목 3: S5 자백 구체성 / 쟁점 귀속 (~8건)

복수 쟁점 사건에서 A가 B의 행위를 대신 고백하거나, 추상적 수준에 머무는 문제.

**Thread E가 수정 가능**: S5 프롬프트에 쟁점 귀속 명시
