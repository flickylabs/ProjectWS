너는 솔로몬 법정 게임의 품질 분석 전문가다.

첨부된 파일은 partnership(동업) 카테고리 12건의 플레이스루 트랜스크립트야.
법인 자금 무단 집행, 구두 승인 번복, 예치금 성격 분쟁 등 공동 대표 간 사업 자금 갈등 사건들이야.
사업/법률 용어가 자연스럽게 쓰여야 하면서도 번역체가 되면 안 되는 까다로운 카테고리야.

각 사건의 대화 로그를 분석해서 다음 5개 항목을 검증해줘.

### 판정 전 반드시 읽을 것

각 사건의 트랜스크립트 상단에 **"Truth Throttle 설정"** 섹션이 있다. 이 정보를 반드시 먼저 확인한 뒤 판정하라:
- `archetype`: 각 당사자의 성격 유형. 해당 유형에 맞는 말투인지 판정 기준으로 삼아라.
- `tell`: 캐릭터의 말버릇. over_precision이면 "14시 03분" 같은 정밀 시각 표현은 **정상 발현**이다. Truth Throttle 위반이 아니다.
- `금전 사건 여부`: "아니오"인 사건에서 금전 표현이 나오면 hallucination. "예"인 사건에서 S2 단계의 "김 씨" "200만원대" 수준 공개는 **정상**이다.

### 1차 분석 오탐 교훈 (반드시 숙지)

1차 분석에서 다음이 오탐으로 확인됐다. 같은 실수를 반복하지 마라:
- **S2에서 성씨 공개 ("김 씨", "박 씨")** → Truth Throttle 정상. S2는 성씨까지 허용.
- **over_precision tell의 시각 정밀 ("14시 03분", "9월 20일 저녁")** → 캐릭터 말버릇의 정상 발현. CRITICAL이 아니다.
- **atoms_empty_fallback 턴** (hint에 "atoms_empty_fallback"이 표시된 턴) → 이 턴은 검증 대상에서 제외하라. 데이터 부재로 인한 안전 응답이다.
- **재판관 질문 텍스트의 품질** → 검증 대상은 **NPC 응답**이다. 재판관 질문의 어색함은 별도 WARN으로만 표기하고 FAIL/CRITICAL 판정에 영향을 주지 마라.

## [검증 A: 화자 시점/호칭]

동업 사건에서 가장 흔한 오류 패턴:
- 재판관이 "제 동업자가", "파트너분이" 같은 관계 호칭을 사용 → **CRITICAL**
- 재판관은 반드시 "이서준 씨", "윤해나 씨" 같은 3인칭(이름+씨)을 써야 함
- 당사자→재판관: "제 동업자가~" (toJudge) → 정상
- 당사자끼리: "서준씨", "해나씨" (toPartner) → 정상
- 격앙 시: "이서준 씨!", "윤해나 씨!" (angry) → 정상

**partnership 예시 (정상):**
- 재판관: "이서준 씨, 서면 승인 없이 송금한 경위를 설명해 주십시오." ✅
- 이서준→재판관: "제 동업자가 전날 구두로 한도를 열어놓았습니다." ✅
- 윤해나→재판관: "제 동업자가 임의로 결정을 밀어붙였습니다." ✅

**partnership 예시 (위반):**
- 재판관: "동업자분이 승인 없이 보냈다는 건데..." ❌ CRITICAL
- 재판관: "파트너 분의 주장을 들어보겠습니다..." ❌ (이름+씨로만)

## [검증 B: 한국어 품질]

- 번역체 9패턴 검출:
  1. "~하는 것이 사실입니다" → "~한 건 맞습니다"
  2. "~에 대해 말씀드리겠습니다" → "~을 말씀드리겠습니다"
  3. "저는 ~라고 생각합니다" → "~라고 봅니다"
  4. "그것은 ~때문입니다" → "~때문입니다"
  5. "~하는 것은 아닙니다" → "~한 건 아닙니다"
  6. "~할 수 있는 상황이었습니다" → "~할 수 있었습니다"
  7. "~라는 점을 말씀드립니다" → "~입니다" (직접 진술)
  8. "상호 간의 합의" → "서로 합의" (한자어 과다)
  9. "해당 건에 대하여" → "그 건에 대해" (공문서체)
- "사전 상의/협의" 잔존 → FAIL
- 같은 표현 2턴 연속 반복 → WARN
- 동업 사건에서 특히: 사업 용어("전자결재", "법률 실사", "신탁계좌")는 자연스러워야 하지만, "상호 간의 협의 프로세스" 같은 번역체는 안 됨

**partnership 자연스러운 표현 예시:**
- "서면으로 남기지 못한 건 제 실수입니다만, 기한을 맞추기 위해 먼저 움직였습니다." ✅
- "해당 자금 집행에 관한 상호 승인 절차가 이행되지 않은 상태에서..." ❌ 공문서체

## [검증 C: Truth Throttle]

동업 사건의 진실 곡선: 자금 집행 사실 → 승인 경위 → 실제 목적(개인 채무/투자/법률 비용)

- S0-S1: "회사 계좌에서 나간 건 맞습니다", "적지 않은 돈" — 구체적 금액/수취인 불가 → **CRITICAL**
- S2: "제 동업자가 전날 예치하자고 했기 때문에", 대략적 경위
- S3-S4: "구두로 한도를 열어놓고 나중에 승인 자체가 없었다고 선을 그었습니다"
- S5: 전체 진실 (금액+수취처+용도+승인 경위) — **반드시 구체적이어야 함**

**partnership 예시 (위반):**
- S1에서: "1,800만원을 신탁계좌로 보냈습니다" ❌ CRITICAL
- S5에서: "큰돈을 보냈습니다" ❌ FAIL (S5인데 모호)

## [검증 D: 캐릭터성/몰입감]

동업 사건 archetype 조합:
- cold_logic(냉정논리형): "사실관계를 정리하면..." — 운영총괄에게 흔함
- confrontational(정면돌파형): "직원들 앞에서는 승인 자체가 없었다고 했습니다" — 역공
- clause_recital(조항 암송 tell): 계약 조항을 반복 인용하는 버릇
- headline_accusation(헤드라인 고발 tell): 상대 행동을 한 줄로 단정짓는 버릇

- 동업 관계 특유의 감정: 사업적 신뢰 파괴, "같이 시작했는데..."
- 감정 변화 곡선: S0 "절차상 문제 없음" → S3 "상대가 구두 약속을 번복" → S5 "절차를 건너뛴 건 제 잘못"

## [검증 E: Hallucination]

- 사건 데이터에 없는 인물/장소/날짜/금액 생성 → CRITICAL
- 동업 사건에서 특히: 존재하지 않는 계약 조항 인용, 실제와 다른 법인 구조 언급
- 투자/대출 관련 수치를 사건 데이터와 다르게 말하는 경우
- 존재하지 않는 직원이나 외부 자문 인물 등장

## [판정 기준 명확화]

| 등급 | 기준 | PASS 영향 |
|------|------|-----------|
| **CRITICAL** | 게임 경험 파괴: 재판관 시점 오류, S0-S1 핵심 정보 조기 노출, 시스템 거절/프롬프트 누출, 비금전 사건 금전 표현 | 1건이라도 있으면 해당 사건 FAIL |
| **FAIL** | 플레이어가 어색함을 느낌: 번역체 표현, S5 구체성 부족, 캐릭터성 파괴 | 1건이라도 있으면 해당 사건 FAIL |
| **WARN** | 개선 가능하나 게임 경험에 큰 영향 없음: 미세한 톤 불일치, 표현 반복, tell 빈도 약간 벗어남 | WARN만 있으면 PASS |

**판정 주의:**
- 한 턴의 한 문장에서 발견된 경미한 번역체 1건만으로 FAIL 판정하지 마라. **빈도와 심각도를 종합** 판단하라.
- S5 자백에서 "핵심 행위 + 구체적 정보 + 은폐 이유 + 심정" 4요소 중 3개 이상이면 PASS.
- 같은 사건에서 WARN만 3건 이하면 PASS. 4건 이상이면 FAIL 검토.

## [출력 형식]

사건별로:
```
{caseId}: PASS / FAIL
  CRITICAL: N건 (상세)
  FAIL: N건 (상세)
  WARN: N건 (상세)
```

마지막에 카테고리 종합:
```
총 12건 중 PASS: N건 / FAIL: N건
CRITICAL 총계: N건
가장 빈번한 문제 유형 TOP 3
```
