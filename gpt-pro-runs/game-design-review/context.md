# 솔로몬 법정 — 게임 기획 리뷰 요청

## 게임 한 줄 요약

AI 캐릭터 둘이 갈등하고, 플레이어(재판관)가 심문·증거·조합으로 진실을 밝히는 리플레이형 추리 게임.

## 핵심 메커니즘

### 거짓말 상태 기계 (LieState: S0→S5)
각 NPC는 쟁점(dispute)마다 독립적인 거짓말 상태를 가짐:
- S0: 완전 부정 — "그런 적 없습니다"
- S1: 일부 인정 — "그 부분은 맞지만..."
- S2: 핑계 — "어쩔 수 없었습니다"
- S3: 책임 전가 — "상대방이 먼저..."
- S4: 감정적 — "제가 뭘 어떻게 하라는 겁니까!"
- S5: 자백 — "사실은..."

플레이어의 질문과 증거 제시에 의해 전이됨. S0에서 S5까지 몰아가는 과정이 게임의 쾌감.

### Truth Throttle (진실 공개 곡선)
NPC는 LieState에 따라 공개할 수 있는 정보 수준이 다름:
- S0-S1: "해당 금액", "그 사람", "그곳" (모호)
- S2: "200만원대", "김 씨" (부분 구체화)
- S3+: 구체적 금액, 실명, 정식명칭
- S5: 전부 공개

### 심문 3종
| 유형 | 효과 | 재판관 톤 |
|------|------|----------|
| 사실 추궁 (fact_pursuit) | 모순 토큰↑ | 단호, 논리적 |
| 동기 탐색 (motive_search) | 누설 미터↑ | 설명 요구 |
| 공감 접근 (empathy_approach) | 신뢰 창구↑ | 감정 이끌기 |

### 증거 시스템
- 8종 타입: bank, chat, cctv, contract, testimony, log, device, sns
- investigationStages: 3단계 질문 (stage 0→1→2, 점진적 핵심 접근)
- evidenceCombinations: 증거 2개 조합 → 새 사실 발견
- DossierCard: 특정 조건 해금 시 열리는 결정적 질문

### 캐릭터 시스템
- Archetype 6종: avoidant(회피형), confrontational(대립형), victim_cosplay(피해자 행세), cold_logic(냉정 논리), affect_flattening(감정 무뎌짐), premature_summary(성급한 정리)
- Verbal Tell 6종: 거짓말의 언어적 징후

### 재판관 성향 시스템 (3축)
- 탐구(inquiry) ↔ 판단(judgment)
- 논리(logic) ↔ 직관(intuition)
- 과정(process) ↔ 해결(resolution)

## 현재 Phase 구조 (8단계)

```
Phase 0: 사건 소개 → 배경/인물 파악
Phase 1: 초기 진술 → 양측 입장 청취 (스크립트 자동 재생)
Phase 2: 반박 진술 → 상대 진술에 대한 반박 (스크립트 자동 재생)
Phase 3: 심문 → 플레이어가 질문하여 진실 추궁 (핵심 플레이)
Phase 4: 증거 조사 → 증거 제시/조사/감별
Phase 5: 재심문 → 추가 추궁 + 특수 행동
Phase 6: 중재 → 결론 도출
Phase 7: 판결 → 책임 배분 + 해결안 선택
Result: 점수/칭호/후일담
```

## 현재 PC UI 구성

- 3-column 레이아웃 (좌패널 / 중앙 대화 / 우패널)
- 좌패널: 증거 카드 + 핵심 발언 노트(제목만/hover 확장) + 확장→타임라인
- 중앙: 대화 로그 + 쟁점 탭(01~05 S0~S5)
- 우패널 상단: 심문 대상 토글 + 대상 미터값
- 우패널 하단: 조합 스킬 + 요약 스킬
- 하단 핫바: 6슬롯×3페이지(심문/증거/특수) + 캐릭터 A/B 카드

## 확정된 v2 스토리 예시: spouse "새벽 통화기록"

**훅**: 남편 카드 명세서에 여성 생리대, 스타킹이 찍혀 있고, 매일 같은 오피스텔에 2시간씩 다녀온다.

**반전 구조**:
1차: 오피스텔에는 친형+중2 조카딸. 형은 빚 갚으려 일하고 조카가 혼자. 삼촌이 돌봄.
2차: 공동 적금 3,000만 원을 형 빚에 보냄.
3차: 아내는 딴살림 확신 → 이혼 대비로 2,000만 원 빼두려다 투자 사기에 당함.
4차: 부부 합산 5,000만 원 증발. 숨긴 돌봄은 B 먼저, 비밀 송금은 A 먼저.

**핵심**: 소재는 단순(카드 명세서, 오피스텔), 파면 팔수록 양쪽의 숨겨진 배신이 꼬리를 물고 나옴.

## 현재 고민 사항

### 1. Phase 구조 재편

현재 Phase 3(심문)/4(증거)/5(재심문)의 변별력이 약함. 
통합하여 "자유 심리" 단일 Phase로 만들면:
- 플레이어가 아무 때나 심문/증거 제시/조합/특수 행동 선택 가능
- Phase가 아니라 행동으로 게임이 진행
- "충분히 파냈다"고 판단하면 스스로 중재로 넘어감

```
제안:
Phase 0: 사건 소개
Phase 1: 초기 진술 (스크립트)
Phase 2: 반박 진술 (스크립트)
Phase 3: 자유 심리 ★★★ (심문+증거+조합+특수 통합)
Phase 4: 중재 + 판결
Result: 점수/칭호/후일담
```

### 2. 증거 해금 방식

현재: 일부 증거가 잠겨있다가 조건 충족 시 해금.
문제: "왜 재판관이 증거를 못 보나?" 부자연스러움.

제안: 증거는 처음부터 존재하되, 깊이가 단계적으로 열림.
```
Level 0: 존재만 보임 (제목 + 유형 아이콘)
Level 1: 표면 열람 (investigationStage 0)
Level 2: 심층 열람 (investigationStage 1~2) — 심문/조합으로 해금
Level 3: 연결 확정 (evidenceCombination 완성)
```

### 3. 조합 스킬 고도화

현재: 증거 2개 → 새 노트 생성 (제한적).

제안: 조합 결과 유형 대폭 확장:
- 신규 쟁점 생성
- 추가 증거 생성
- 추가 증언 생성
- 쟁점 확장(변경)
- 증거 업그레이드
- 증언 모순 발견
- 숨겨진 동기 발견
- 타임라인 정정
- 감정 폭발 트리거
- 제3자 연결

### 4. 심문 3종의 변별력

fact_pursuit / motive_search / empathy_approach가 실제 플레이에서 체감 차이가 약할 수 있음.
각 심문 유형이 만드는 결과가 더 극적으로 다르면 좋겠음.

## 질문

1. 위의 Phase 재편(8단계 → 5단계) 방향에 대한 의견은? 더 줄이거나 다르게 구성할 수 있는가?
2. 증거를 "잠금/해금"이 아니라 "깊이"로 바꾸는 접근에 대한 의견은? 더 나은 대안이 있는가?
3. 조합 스킬의 결과 유형을 더 확장할 수 있는가? 플레이어의 "히든 발견" 쾌감을 극대화하는 다른 메커니즘은?
4. 심문 3종(사실추궁/동기탐색/공감접근)의 변별력을 높이는 방법은?
5. "재판관"이라는 역할이 가진 고유한 게임 메커니즘 가능성은? (예: 재판관만 할 수 있는 특수 행동, 재판관 성향이 NPC 반응에 영향을 주는 방식 등)
6. 리플레이 가치를 높이는 구조적 장치는? (같은 사건을 다시 해도 다른 경험이 되는 방법)
7. 전반적으로 이 게임의 핵심 재미를 극대화하기 위해 추가/변경/제거해야 할 요소는?

자유롭게 의견을 주되, 구체적인 메커니즘 제안까지 포함해주면 좋겠습니다.
