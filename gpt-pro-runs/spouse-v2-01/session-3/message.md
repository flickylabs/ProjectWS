너는 솔로몬 법정 게임의 Thread W용 scripted text 작성 모델이다.
지금부터 만드는 것은 구조 채우기가 아니라 실제 플레이에서 자연스럽게 들리는 한국어 대사다.

이번 사건은 spouse 카테고리의 "새벽 통화기록"이다.
기존 spouse v1 사건이나 다른 케이스 문장을 참고하거나 모사하지 마라.
현재 첨부된 이 사건 자료만 사용하라.

사건의 필수 곡선:
- 딴살림 의심
- 형/조카 돌봄 1차 반전
- 시댁 불화가 숨김의 이유로 열림
- 공동 적금 3,000만 원 독단 해지로 B가 다시 무거워짐
- A의 투자 사기 2,000만 원이 열림
- 최종적으로는 부부 합산 5,000만 원 증발 사건으로 프레임이 바뀜

화자 정보:
- A 박지연: 36세, 학원 데스크 직원, victim_cosplay
  - 상대 직접 호칭은 "준호 씨", 감정이 올라가면 "이준호 씨", 재판관에게 상대를 말할 때는 "제 남편"
- B 이준호: 38세, 가전매장 직원, avoidant
  - 상대 직접 호칭은 "지연 씨", 감정이 올라가면 "박지연 씨", 재판관에게 상대를 말할 때는 "제 아내"

호칭 규칙:
- 재판관에게 말하는 문장은 반드시 합니다체
- 재판관에게 상대를 말할 때는 A는 "제 남편", B는 "제 아내"를 쓴다
- 상대에게 직접 말할 때만 해당 호칭 사용

Truth Throttle:
- early에서는 "그 번호", "그 집", "그 돈" 등 감춘 표현
- mid부터 형·조카 돌봄, 오피스텔 실제 용도 일부 열림
- late에서는 실제 숫자와 구체적 사정까지 열림

---

이번 세션 범위는 아래 두 채널이다.
- evidence_present
- dossier

현재 증거 범위:
- e-1 카드 명세서
- e-2 오피스텔 출입 기록
- e-3 새벽 통화기록
- e-4 형과의 문자 + 조카 학교 알림 문자
- e-5 공동 적금 해지 내역과 계좌이체 기록
- e-6 A와 지인 박미라의 카톡 + 송금 영수증
- e-7 통화·출입·계좌이체 대조표

생성 범위:
- evidence_present
  - parties: a, b
  - evidenceIds: e-1 ~ e-7
  - investigationStages: stage_0, stage_1, stage_2
  - lieBands: early, mid, late
  - key당 variants 5개
- dossier
  - dc-1: "새벽 통화 상대가 형이라면, 왜 아내에게 한마디도 못 했습니까?" (대상: b, 해금: d-1 S2+, e-4)
  - dc-2: "공동 적금 3,000만 원은 부부 돈인데, 형에게 줄 권리가 정말 당신 혼자에게 있었습니까?" (대상: b, 해금: d-2 S3+, e-5)
  - dc-3: "남편이 먼저 숨겼다고 해도, 먼저 2,000만 원을 움직인 건 당신이지요?" (대상: a, 해금: d-3 S3+, e-6 또는 c-3)
  - cardBands: unlock, pressure, collapse
  - key당 variants 3개

evidence reaction 핵심:
- e-1/e-2/e-3: 세 개가 겹치면 외도 오해가 가장 강해야 한다
- e-4: 같은 번호가 친형이고, 방문이 형·조카 돌봄이었다는 1차 반전
- e-5: B의 선의가 공동 적금 3,000만 원 독단 처분이라는 책임으로 뒤집힘
- e-6: A의 자기방어가 2,000만 원 비밀 송금과 투자 사기로 열림
- e-7: "숨긴 돌봄은 B 먼저 / 비밀 송금 실행은 A 먼저"라는 순서 차이

investigationStages 규칙:
- stage 0: 진위 확인 — 짧고 즉각적인 반응
- stage 1: 목적/대상이 좁혀짐 — 방어·축소·흔들림
- stage 2: 숨긴 것, 배우자에게 뭐라 말했는지, 관계 파장까지 — 압박 반응

evidenceCombinations 규칙:
- c-1 (e-1+e-3): 외도 오해 강화 함정
- c-2 (e-4+e-5): "돌봄이었다"가 "3,000만 원을 보낼 권리가 있었냐"로 전환
- c-3 (e-5+e-6): 사건 프레임이 "부부 돈 5,000만 원 증발"로 완전 전환

DossierCard 규칙:
- dc-1: "왜 말 못했는가"를 직접 찌르는 카드
- dc-2: 선의 프레임을 무너뜨리고 공동재산 동의권을 정면으로 묻는 카드
- dc-3: A의 자기방어를 "누가 먼저 실행했는가"로 바꾸는 카드

금지:
- 번역체 9패턴
- 설계 문서/JSON 복붙
- DossierCard가 기본 증거 질문의 반복이 되면 안 됨 — 한 단계 더 강제로 state를 밀어올리는 질문

Validator Stage3 하드 규칙:
- `evidence_present`와 `dossier`의 모든 variant는 첫머리를 반드시 `재판장님,`으로 시작하라.
- judge-facing 문장은 전부 합니다체로 끝내라. `-지요`, `-니까요`, `-거든요`, `-게요`, `-서요` 종결은 금지다.
- `evidence_present`와 `dossier`의 문장 수는 `stanceHint` 기준으로 맞춰라.
  - `deny` / `hedge`: 1~2문장
  - `partial` / `blame` / `emotional` / `confess`: 2~3문장
- 길이가 애매하면 2문장을 기본값으로 잡고, 3문장을 넘길 때는 `partial/blame/emotional/confess`에서만 허용하라.
- `evidence_present`의 모든 variant는 첫 문장 안에 해당 증거 키워드를 직접 넣어라.
  - `e-1`: `카드` 또는 `명세서` 또는 `결제`
  - `e-2`: `오피스텔` 또는 `출입` 또는 `기록`
  - `e-3`: `새벽` 또는 `통화` 또는 `기록`
  - `e-4`: `문자` 또는 `형` 또는 `조카`
  - `e-5`: `적금` 또는 `해지` 또는 `계좌이체` 또는 `3,000만 원`
  - `e-6`: `카톡` 또는 `송금` 또는 `영수증` 또는 `2,000만 원`
  - `e-7`: `통화` 또는 `출입` 또는 `계좌이체` 또는 `대조표`
- `e-1`, `e-2`, `e-3`, `e-4`, `dc-1`은 모두 `d-1` 계열이다. 여기서는 `돈`, `적금`, `송금`, `이체`, `계좌`, `원`, `만원` 같은 금전어를 넣지 마라.
- `dc-2`와 `dc-3`은 금전 쟁점을 다루되, 질문의 핵심은 금액 그 자체보다 권한과 실행 순서가 되게 하라.

출력 형식:
```json
{
  "evidence_present": { "entries": [] },
  "dossier": { "entries": [] }
}
```

JSON 바깥 텍스트 없이 반환하라.
