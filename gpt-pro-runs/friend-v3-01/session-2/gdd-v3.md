# GDD v3: 게임 구조 업데이트 (2026-04-11)

> 상태: CT 확정
> 기반: GPT Pro 기획 리뷰 + CT/유저 협의
> 적용 대상: 모든 스레드 (S/W/G/Q)

---

## 핵심 정체성

> **"관계 재판 게임"** — 범인을 찾는 게 아니라, 양쪽의 숨겨진 서사를 무너뜨리고 더 정확한 서사를 공식기록으로 세우는 게임.

---

## 1. Phase 구조 (5단계)

### Phase 0: 사건 접수
- 사건 개요, 관계도, 현재 제출된 증거의 "존재 목록"만 표시
- 플레이어 선택: 누가 먼저 말할지, 어느 쟁점부터 볼지
- 이 선택이 이후 발언 순서와 방어태세에 영향

### Phase 1: 개시 진술
- 기존 Phase 1(초기 진술) + Phase 2(반박 진술)을 통합
- 자동 재생 유지
- **의심 태그** 찍기 가능 (3종):
  - 정확성 의심
  - 감정 과잉
  - 빠진 맥락
- 태그는 Phase 2 자유 심리에서 "추천 공격점"으로 활용

### Phase 2: 자유 심리 ★★★ (핵심)
- 기존 Phase 3/4/5를 완전 통합
- 턴 기반이 아니라 행동 기반
- 5종 행동:
  1. **질문**: 사실추궁 / 동기탐색 / 공감접근
  2. **증거**: 열람 / 원본요구 / 제시
  3. **조합**: 두 단서를 연결하여 Lead 생성
  4. **재판관 행동**: 원본 제출 명령 / 정확히 답변하십시오 / 분리심문 / 잠정 인정 / 발언 제지 / 선처 창구 / 민감정보 봉인 해제
  5. **공식기록**: "이 사실을 잠정 인정"으로 등록

- **드라마 비트** (Phase가 아니라 조건부 이벤트):
  - 표면 균열: 첫 핵심 모순 발견
  - 재맥락화: 숨은 제3자나 동기 등장
  - 상호 붕괴: 한쪽 무너지자 반대편도 무너짐
  - 최종 압박: DossierCard나 동시 대질

- **평의 진입 조건**:
  - 핵심 쟁점 2개 이상 잠정 인정, 또는
  - 1개 이상 Dossier 성공 + 1개 이상 상호모순 해소, 또는
  - 플레이어 조기 강행 가능 (정확도/공정성 리스크)

### Phase 3: 평의
- 각 쟁점마다 4칸 채우기:
  1. 무슨 일이 있었나
  2. 왜 숨겼나
  3. 누구에게 어떤 피해가 갔나
  4. 어떤 복구가 가능한가
- 근거 약한 칸은 노란색, 충돌하는 칸은 빨간색
- 공식기록과 가설을 분리 표시

### Phase 4: 판결/합의
- 책임 배분 + 해결안 선택:
  - 금전 책임
  - 사과 의무
  - 정보 공개 범위
  - 재발 방지 약속
  - 관계 회복 시도 여부

### Result
- 점수/칭호/후일담 (기존과 동일)

## 2. 증거 시스템 (2축)

### 축 1: 열람 깊이
| 깊이 | 설명 |
|------|------|
| Stub | 존재만 보임 (제목 + 유형 아이콘) |
| Excerpt | 제출된 일부만 열람 |
| Original | 원본/전체본/메타 확보 |
| Context | 앞뒤 맥락까지 복원 |
| Established | 재판관이 공식기록에 채택 |

### 축 2: 신뢰 상태
| 상태 | 설명 |
|------|------|
| 제출됨 | 당사자가 제출한 상태 |
| 검증 중 | 원본 대조/맥락 확인 중 |
| 인증됨 | 신뢰할 수 있다고 판단 |
| 이의 제기됨 | 상대방이 이의 |
| 조작/오독 판정 | 위조/편집/발췌 확인 |

### 증거 타입별 깊이 차이
| 타입 | Stub | Excerpt | Original | Context |
|------|------|---------|----------|---------|
| bank | 거래 존재 | 정확 금액/시각 | 수취인 실체 | 사용 목적 |
| chat | 캡처 | 전체 대화 | 삭제 전후 맥락 | 발신/수신 기기 대조 |
| cctv | 특정 시각 | 전후 10분 | 다른 각도 대조 | 출입 동선 정정 |
| contract | 표면 조항 | 특약 | 서명/수정 흔적 | 체결 배경 |
| sns | 게시물 | 수정/삭제 로그 | 맥락 댓글 | 지목 가능성 분석 |

### 민감정보 봉인/해제
- 미성년자, 질병, 성적 민감정보는 초기 마스킹
- 플레이어가 봉인 해제 가능 → 공정성/인도성 점수 영향

## 3. 조합 시스템 (Lead → Dossier 2단계)

### 1차 조합: Lead 생성
- 입력: 증거+증거, 증거+진술, 진술+진술, 증거+타임라인 공백, 노트+노트
- 출력 8종 Lead:

| Lead 종류 | 설명 |
|-----------|------|
| Contradiction | 모순 압박용 |
| Beneficiary | 숨은 수혜자/동기 |
| Timeline | 시간축 정정 |
| Authenticity | 조작/발췌/오독 의심 |
| Context | 앞뒤 맥락 필요 |
| Third-party | 새 인물/기관 연결 |
| Emotion | 감정 폭발/수치심 접근점 |
| Reframe | 쟁점 자체 재정의 |

- **Lead는 "정답"이 아니라 "리드"** — 플레이어가 해석을 선택:
  - 예: 오피스텔+생활용품 → "같은 장소/반복 패턴" Lead
  - 플레이어 해석: 외도 / 돌봄 / 심부름 중 택
  - 해석에 따라 생성되는 Dossier가 달라짐

### 2차 조합: Lead → DossierCard 생성
- Lead + 진술/증거 → DossierCard
- DossierCard 성공 → state unlock / revealAtom / lie advance

### 히든 발견 쾌감 장치
1. **부재의 조합**: 있어야 할 게 없는 것 (삭제 로그, 빠진 원본)
2. **동일인 통합**: "그 사람"/"김 씨"/"삼촌"이 같은 인물
3. **시간대 공명**: 서로 다른 자료의 같은 시간 공백
4. **거울형 거짓말**: A와 B가 대칭적으로 숨기는 구조
5. **오해 정정 = 더 큰 비밀의 입구**: 외도 아님 → 왜 숨겼는가 → 돈은?

## 4. 심문 시스템 (3종 차별화)

### 사실추궁 (fact_pursuit)
- **캐내는 것**: 정확한 시간, 금액, 장소, 실명, 인과관계
- **성공 효과**: 모호어 금지 (다음 답변에서 "그 사람"→실명), 모순 토큰
- **리스크**: 신뢰 하락, 방어적 셧다운

### 동기탐색 (motive_search)
- **캐내는 것**: 왜 숨겼는가, 누가 이익 보는가, 왜 지금 말하는가
- **성공 효과**: 숨은 쟁점 생성, 제3자 Lead, 자기정당화 균열
- **리스크**: 변명 서사 정교화

### 공감접근 (empathy_approach)
- **캐내는 것**: 수치심, 두려움, 상실감, 관계 보존 욕구
- **성공 효과**: 오프더레코드 고백, 자발적 원본 제출
- **리스크**: 감정만 넘침, 조종형 NPC에게 먹힘

### Archetype별 약점
| Archetype | 약한 심문 유형 |
|-----------|--------------|
| avoidant | 공감접근 |
| cold_logic | 사실추궁 |
| victim_cosplay | 동기탐색 |
| premature_summary | 사실추궁 + 타임라인 |
| confrontational | 분리심문 후 사실추궁 |
| affect_flattening | 구체적 피해 환기형 동기탐색 |

## 5. 재판관 전용 행동 (Authority 시스템)

| 행동 | 효과 | 리소스 |
|------|------|--------|
| 원본 제출 명령 | 증거 깊이 상승 | 제한적 |
| 정확히 답변하십시오 | 다음 답변 모호어 금지 | 제한적 |
| 분리 심문 | A/B 따로 심문, 진술 차이 극대화 | 제한적 |
| 잠정 인정 | 사실을 기록으로 확정, 뒤집기 비용 증가 | 자유 |
| 발언 제지 / 기록 제외 | 프레임 싸움 차단 | 자유 (신뢰 리스크) |
| 선처 창구 | 자진 인정 시 책임 배분 반영 | 자유 |
| 민감정보 봉인 해제 | 마스킹된 정보 공개 | 공정성/인도성 영향 |

## 7. 리플레이 구조

### 고정 (70%)
- 핵심 배신 구조, 테마, 앵커 진실

### 가변 (30%)
- 누가 먼저 흔들리는가
- 어떤 자료가 발췌본으로 나오는가
- 어떤 verbal tell이 강하게 나타나는가
- 어느 쟁점이 먼저 떠오르는가
- 재판관 성향/로드아웃
- 시작 complaint 관점 (A 먼저 vs B 먼저)

## 8. UI 변경 사항

### 추가
- **가설 보드**: 플레이어가 "지금 내가 믿는 이야기"를 직접 작성
- **공식기록 시스템**: 판결에 쓸 사실을 따로 등록 (가설과 분리)
- **숨은 쟁점 등장 연출**: 처음에 2~3개만 보이고 나머지는 발견 시 탭 생성

### 변경
- 핫바: 맥락 중심 UI로 재설계 (지금 누구/어느 쟁점/방금 무슨 말)
- 미터: 숫자 대신 반응으로 체감 (NPC 표정/말버릇/톤 변화)
- 증거: "잠금/해금" 아이콘 제거 → 깊이 단계 표시

### 축소
- 동시 활성 쟁점: 처음엔 2~3개만 → 나머지는 숨은 쟁점
- 숫자 미터: 핵심만 표시, 나머지는 NPC 반응으로 체감
